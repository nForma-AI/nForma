## E2E Flows — Complete

### Flow 1: Provider Health & Slot Filtering

```
User initiates quorum dispatch
  ↓
qgsd-prompt.js receives prompt (UserPromptSubmit hook)
  ↓
triggerHealthProbe() [DISP-01]
  └─→ spawns check-provider-health.cjs with 3s timeout
  └─→ caches results in ~/.claude/qgsd-provider-cache.json
  ↓
getDownProviderSlots() [FAIL-02]
  └─→ reads provider cache
  └─→ matches providers.json slots to DOWN providers
  └─→ skips all DOWN provider slots (EventualConsensus preserved)
  ↓
getAvailableSlots() [DISP-02]
  └─→ reads scoreboard availability section
  └─→ excludes future-timestamp slots (cooling down)
  ↓
sortBySuccessRate() [DISP-03, OBS-03]
  └─→ reads flakiness_score from scoreboard (primary sort)
  └─→ uses success_rate as tiebreaker (secondary sort)
  └─→ reorders slots by reliability
  ↓
Build DISPATCH_LIST → emit Task(subagent_type="qgsd-quorum-slot-worker")
```

```
  └─→ returns data for chaining
  ↓

```

## Recommendations

### No Critical Issues Found

All integration checks PASS. System is ready for deployment.

### Minor Notes (Not Issues)

1. **Telemetry Fail-Open:** recordTelemetry() never blocks dispatch (correct), but errors are only logged to stderr. Consider adding a telemetry error counter to observability dashboard in future phases if needed.

2. **Provider Cache TTL:** getDownProviderSlots() respects 180s (healthy) / 300s (unhealthy) TTL from check-provider-health.cjs. If probes fail to update, stale cache may persist. System is fail-open (continues dispatch), but document this assumption.

3. **Flakiness Window:** computeFlakiness() uses trailing 10-round window. If quorum < 10 rounds, window is smaller. This is correct (fail-open), but document that flakiness_score may not be statistically significant in early runs.

4. **Agent Spec Compliance:** qgsd-quorum-slot-worker.md is thin (41 lines). Verify in production that worker agents follow the "one Bash call" instruction and emit output verbatim without modification. Early testing showed mixed compliance (some workers explored first).
