  <action>
Create `bin/check-bundled-sdks.cjs` — a pure Node.js static analysis linter (no external dependencies). Use the implementation skeleton from the research document as a starting point but with these refinements:

**Shebang and strict mode:**
```javascript
#!/usr/bin/env node
'use strict';
```

**Forbidden SDK list:**
```javascript
const FORBIDDEN_SDKS = [
  '@anthropic-ai/sdk',
  'anthropic',
  'openai',
  '@google/generative-ai',
  'google-generativeai',
  'cohere-ai',
  '@azure/openai',
];
```

**Import pattern builder — `buildImportPatterns()`:**
For each SDK name, build regex patterns:
1. `require\s*\(\s*['"]<sdk>['"]\s*\)` — CommonJS require
2. `require\.resolve\s*\(\s*['"]<sdk>['"]\s*\)` — require.resolve (used by update-scoreboard.cjs pattern)
3. `import\s+.*\s+from\s+['"]<sdk>['"]` — ESM import

Return an array of `{ sdk, patterns: RegExp[] }` objects. Escape special chars in SDK names (@ and /).

**Scope filtering — `isInScope(filePath)`:**
Include: files under `bin/` or `hooks/` with `.js`, `.cjs`, or `.mjs` extensions.
Exclude: files containing `.test.` in name, files under `node_modules/`, `docs/`, `examples/`, `.formal/`.

**Line-level scanning — `scanFile(filePath)`:**
Read file, split into lines, test each line against all patterns. But add a **comment/string filter**: skip lines where the SDK name appears ONLY inside a single-line comment (`//`) or in a string that is NOT part of a `require()` or `import` call. Implementation: for each regex match, check if the matched portion is preceded by `//` on the same line — if so, skip it. This handles the most common false-positive pattern.

Return array of violation objects: `{ file, line, content, sdk, violation: 'SDK_IMPORT' }`.

**File walker — `getAllSourceFiles(dir)`:**
Recursive directory walk starting from project root (`.`). Skip `node_modules`, `.git`, `dist`, `.formal` directories. Collect `.js`, `.cjs`, `.mjs` files.

**Main function:**
1. Walk project tree, filter in-scope files.
2. Scan each file for violations.
3. Report results:
   - Print header: `[LINT] Architecture: SDK Bundling Check`
   - Print file count: `Scanned N files`
   - Print violation count: `Found N violations`
   - For each violation: `  ERROR: <file>:<line>: <content trimmed>`
4. Exit code: 0 if no violations, 1 if any violations.

**Export for testing:**
```javascript
module.exports = { FORBIDDEN_SDKS, buildImportPatterns, isInScope, scanFile, getAllSourceFiles };
```

Also check `if (require.main === module) { main(); }` so it runs as CLI but is importable for tests.

**Key constraints:**
- Pure Node.js: `fs`, `path` only. No npm dependencies.
- Fail-open on file read errors (log warning, continue scanning).
- No baseline/grandfathering logic needed — Plan 01 removes all existing violations. If any violation is found, it is a new regression and should fail.
- Do NOT scan `package.json` — that is a separate check (and already has zero SDK deps).
