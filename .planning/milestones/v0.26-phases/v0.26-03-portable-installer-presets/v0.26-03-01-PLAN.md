  <name>Task 2: Fix cloneSlotFlow to copy qgsd.json agent_config metadata alongside mcpServers config</name>
  <files>bin/qgsd.cjs, bin/manage-agents.test.cjs</files>

The current `cloneSlotFlow()` in qgsd.cjs (lines 788-809) deep-clones the mcpServers entry but does NOT copy qgsd.json agent_config metadata (timeout_ms, update_policy, key_status). This means cloned slots lose their source's policy settings.

Fix by adding qgsd.json metadata copy after the mcpServers write:

1. In `cloneSlotFlow()`, after `writeClaudeJson(data);` (line 806), add:
```javascript
// Copy qgsd.json agent_config metadata from source to cloned slot
try {
  const qgsd = readQgsdJson();
  const sourceConfig = (qgsd.agent_config || {})[source.value];
  if (sourceConfig) {
    if (!qgsd.agent_config) qgsd.agent_config = {};
    qgsd.agent_config[newName] = JSON.parse(JSON.stringify(sourceConfig));
    // Clear key_status from clone (needs fresh probe)
    if (qgsd.agent_config[newName].key_status) {
      delete qgsd.agent_config[newName].key_status;
    }
    writeQgsdJson(qgsd);
  }
} catch (_) { /* qgsd.json might not exist yet -- non-fatal */ }
```

Key design decisions:
- Deep-clone sourceConfig to prevent reference sharing
- Delete `key_status` from clone because the cloned slot has no proven key yet (needs fresh probe)
- Wrap in try/catch because qgsd.json might not exist (fresh install)
- Copy timeout_ms and update_policy but NOT key_status (stale for new slot)

2. Ensure `readQgsdJson` and `writeQgsdJson` are already imported at the top of qgsd.cjs (they should be, from the line 67 destructured import of manage-agents-core.cjs). Verify by checking the existing imports near line 60-70.

3. Add 2 tests to manage-agents.test.cjs verifying the metadata copy logic:
- Test: After cloning, qgsd.json has agent_config[newName] with timeout_ms and update_policy copied from source
- Test: After cloning, qgsd.json agent_config[newName] does NOT have key_status (deleted by design)

Since cloneSlotFlow is a TUI function that can't be unit-tested directly, test the LOGIC pattern: write qgsd.json with source metadata -> deep-clone it -> delete key_status -> verify result structure. This proves the transformation is correct even though the TUI wrapper can't be tested.
