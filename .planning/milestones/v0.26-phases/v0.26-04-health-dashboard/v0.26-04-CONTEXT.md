<decisions>
## Implementation Decisions

### Dashboard display format
- Keep existing table format from v0.10-04: provider | model | health status columns
- Reuse `buildDashboardLines()` and `formatTimestamp()` pure functions already in `manage-agents-core.cjs`
- No visual redesign needed — existing layout meets DASH-01

### Refresh and staleness UX
- Keypress refresh on space/r with "Last updated: HH:MM:SS" timestamp at bottom (DASH-02)
- 60-second staleness threshold with yellow warning indicator
- Readline mode-switch architecture: inquirer fully exited before raw stdin loop starts

### Exit behavior
- Q or Escape exits dashboard, returns to main menu (DASH-03)
- `setRawMode(false)` + `removeAllListeners('keypress')` before re-entering `mainMenu()`
- No character-swallowing after exit

### Test architecture
- Pure function unit tests for `buildDashboardLines` (8 cases) and `formatTimestamp` (3 cases) — following established `_pure` export pattern
- Integration smoke test for `liveDashboard` raw mode loop
- Non-TTY fallback test: static one-time health print when `process.stdout.isTTY` is false

### Claude's Discretion
- Test fixture data structure and mock slot configurations
- Integration test mocking strategy for stdin/stdout
- Whether to add staleness warning rendering tests (stretch goal)

</decisions>
