## Summary

Wired `writeKeyStatus()` into the health probe flow via a new `probeAndPersistKey()` helper so that key validity status persists to `qgsd.json` after every probe where the provider responded.

## What Was Built

- **`probeAndPersistKey(slotName, baseUrl, apiKey)`** -- new async helper that encapsulates the probe-classify-write chain. Calls `probeProviderUrl`, then conditionally calls `classifyProbeResult` and `writeKeyStatus` only when the provider actually responded (statusCode present). Timeout/network errors are not persisted to avoid overwriting valid cached status.
- **Enhanced `probeAllSlots()`** -- now calls `probeAndPersistKey` instead of raw `probeProviderUrl`, so every dashboard refresh and health check persists key validity.
- **5 new tests** covering: successful probe persists 'ok', 401 persists 'invalid', timeout does NOT overwrite, multiple slots persist independently, persistence guard condition verified.
