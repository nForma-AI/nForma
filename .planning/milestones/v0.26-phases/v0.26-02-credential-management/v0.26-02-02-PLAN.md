Add a new async function `validateRotatedKeys` near batchRotateFlow (around line 1220, before the batchRotateFlow function):

```javascript
/**
 * Fire-and-forget post-rotation validation.
 * Probes each rotated slot and persists key_status to qgsd.json.
 * Does NOT block the caller -- called with .catch(() => {}).
 * Uses sequential for...of to avoid keychain concurrency (same pattern as rotation loop).
 * Reuses probeAndPersistKey from manage-agents-core.cjs (DRY -- do not duplicate probe/classify/write logic).
 */
async function validateRotatedKeys(rotatedSlots) {
  const { probeAndPersistKey, deriveKeytarAccount } = require('./manage-agents-core.cjs')._pure;
  const data = require('./manage-agents-core.cjs').readClaudeJson();
  const servers = require('./manage-agents-core.cjs').getGlobalMcpServers(data);
  for (const slotName of rotatedSlots) {
    const cfg = servers[slotName] || {};
    const env = cfg.env || {};
    if (!env.ANTHROPIC_BASE_URL) continue;
    let apiKey = env.ANTHROPIC_API_KEY || '';
    // If keytar is available, try to read the key from secure storage
    // Follow the SAME pattern as probeAllSlots (manage-agents-core.cjs line 620-627)
    if (secretsLib) {
      try {
        const account = deriveKeytarAccount(slotName);
        const k = await secretsLib.get('qgsd', account);
        if (k) apiKey = k;
      } catch (_) {}
    }
    await probeAndPersistKey(slotName, env.ANTHROPIC_BASE_URL, apiKey);
  }
}
```

IMPORTANT NOTES for the implementor:
- This function calls `probeAndPersistKey(slotName, baseUrl, apiKey)` from Plan 01's `_pure` exports. Do NOT manually call `probeProviderUrl`, `classifyProbeResult`, or `writeKeyStatus` -- that logic is encapsulated in `probeAndPersistKey` and duplicating it would violate DRY and break the key_link contract.
- `probeAndPersistKey` takes exactly 3 parameters: `(slotName, baseUrl, apiKey)`. It handles the probe, classification, persistence guard (skip on timeout), and `writeKeyStatus` call internally. It uses the default `QGSD_JSON_PATH` for persistence.
- Look at the actual batchRotateFlow implementation to understand how `secretsLib` is accessed in scope. The function above is a pattern -- adapt variable references to match the actual module scope (secretsLib is likely a module-level var or passed through).
- The key for each slot may come from keytar (via secretsLib.get) or from env.ANTHROPIC_API_KEY. Follow the SAME pattern used in probeAllSlots (manage-agents-core.cjs line 620-627) for reading the API key.
- Use sequential `for...of` (NOT `Promise.all`) to avoid keychain concurrency errors per the success criteria pattern from ROADMAP v0.10-03 SC-1.
- `getGlobalMcpServers(data)` takes a `data` parameter (the parsed claude.json content) -- it is NOT a zero-arg function. Call `readClaudeJson()` first to get the data, then pass it to `getGlobalMcpServers(data)`.
