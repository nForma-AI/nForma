Add a new async helper function `probeAndPersistKey(slotName, baseUrl, apiKey)` near `probeAllSlots()` (around line 610):

```javascript
async function probeAndPersistKey(slotName, baseUrl, apiKey) {
  const probe = await probeProviderUrl(baseUrl, apiKey);
  // Only persist status if provider actually responded (statusCode present).
  // Do NOT persist for timeout/network errors (statusCode is null) -- this is
  // a provider/network issue, not a key validity issue (see invariants.md).
  if (probe.healthy || probe.statusCode) {
    const status = classifyProbeResult(probe);
    writeKeyStatus(slotName, status);
  }
  return probe;
}
```

IMPORTANT SIGNATURE NOTE: `probeAndPersistKey` takes exactly 3 parameters: `(slotName, baseUrl, apiKey)`. It does NOT take a `filePath` parameter. The `writeKeyStatus(slotName, status)` call omits the third `filePath` arg, which causes `writeKeyStatus` to use its default `QGSD_JSON_PATH` (see manage-agents-core.cjs line 190-199 where `writeKeyStatus(slotName, status, filePath)` passes `filePath` to `readQgsdJson(filePath)` which defaults to `QGSD_JSON_PATH` when filePath is undefined). This is the correct production behavior -- always write to the real qgsd.json. Tests that need a custom path should test `writeKeyStatus` directly with an explicit filePath, not through `probeAndPersistKey`.

Then modify `probeAllSlots()` (line 613-632) to call `probeAndPersistKey` instead of raw `probeProviderUrl`:
- Replace `const probe = await probeProviderUrl(env.ANTHROPIC_BASE_URL, apiKey);` (line 628) with `const probe = await probeAndPersistKey(slotName, env.ANTHROPIC_BASE_URL, apiKey);`
- Keep the early-return for slots with no ANTHROPIC_BASE_URL (no persistence needed for subprocess slots)
- The function signature stays the same: `probeAllSlots(mcpServers, slots, secretsLib)`

Add `probeAndPersistKey` to the `_pure` export namespace (after `probeAllSlots` in the exports block around line 779).

IMPORTANT: Do NOT write status for 'unreachable' probes where statusCode is null. The guard `if (probe.healthy || probe.statusCode)` handles this -- when a probe times out, both `healthy` is false AND `statusCode` is null, so the guard prevents the write. This matches the formal invariant that operations must reach completion (IdleReachable) and avoids persisting stale "unreachable" state.
