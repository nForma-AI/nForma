2. Update `tuneTimeoutsFlow()` (around line 1807-1841) to validate input before applying:

   Current code (line 1829-1831):
   ```js
   if (trimmed && parseInt(trimmed, 10) !== currentMs) {
     const updated = applyTimeoutUpdate(providersData, providerSlot, parseInt(trimmed, 10));
     Object.assign(providersData, updated);
     changed = true;
   }
   ```

   Replace with:
   ```js
   if (trimmed) {
     const result = validateTimeout(trimmed);
     if (!result.valid) {
       toast(result.error, true);
       continue;
     }
     if (result.ms !== null && result.ms !== currentMs) {
       const updated = applyTimeoutUpdate(providersData, providerSlot, result.ms);
       Object.assign(providersData, updated);
       changed = true;
     }
   }
   ```

   This ensures:
   - Negative values show a toast like "Timeout must be a positive number (got: -500)" and the slot is re-prompted (continue in the for loop)
   - Zero shows the same error
   - Non-numeric input shows the error
   - Blank input still means "keep current" (the outer `if (trimmed)` guard handles this)
