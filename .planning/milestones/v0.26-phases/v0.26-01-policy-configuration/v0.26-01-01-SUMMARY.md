# Phase v0.26-01 Plan 01: Input validation for quorum timeout and update policy Summary

**One-liner:** Timeout and policy value validation pure functions with comprehensive edge-case tests and clear error messages.

## Objective

Add input validation for quorum timeout and update policy values with clear error messages. This satisfies SC-4 (invalid policy values rejected with clear error message) and strengthens SC-1/SC-2 by ensuring the existing TUI flows properly reject bad input at both the pure function layer and TUI presentation layer.

## Tasks Completed

### Task 1: Add validation pure functions to manage-agents-core.cjs

**Status:** Complete ✔

**Implementation:**
- Added `validateTimeout(value)` pure function:
  - Returns `{ valid: true, ms: <integer> }` for positive integers
  - Returns `{ valid: true, ms: null }` for null/undefined/empty (keep current)
  - Returns `{ valid: false, error: "..." }` for negative, zero, or NaN values
  - Error message includes the invalid input for debugging

- Added `validateUpdatePolicy(policy)` pure function:
  - Returns `{ valid: true, policy: <policy> }` for "auto", "prompt", "skip"
  - Returns `{ valid: false, error: "..." }` for invalid strings
  - Error message lists all valid options
  - Derives valid set from `POLICY_MENU_CHOICES` constant (lines 46-50)

- Exported both functions in `_pure` namespace (line 725-726) alongside existing helpers
- Placed functions near logical grouping with `buildTimeoutChoices/buildPolicyChoices` (lines 510-542)

**Verification:**

## Technical Decisions

1. **Input Handling Strategy:** Null/undefined/empty string input treated as "no change" (returns ms: null) rather than error. This allows the TUI to distinguish between "user didn't provide input" and "user provided invalid input".

2. **Policy Validation Source:** Valid policies derived from `POLICY_MENU_CHOICES` constant (lines 46-50) rather than hardcoded list, ensuring single source of truth.

3. **Error Message Format:** Error messages include the invalid value received and all valid options, enabling clear debugging in TUI and logs.

4. **Export Strategy:** Both functions exported via `_pure` namespace following pattern established by `buildTimeoutChoices`, `applyTimeoutUpdate`, `buildPolicyChoices` (similar transformation functions).
