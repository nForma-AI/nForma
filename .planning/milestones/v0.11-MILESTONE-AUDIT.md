## Tech Debt (3 items — no blockers)

### Issue B — `<voteCode>` mapping undefined *(Medium)*

**File:** `agents/qgsd-quorum-orchestrator.md` line 389
**Detail:** Vote file template uses `result: '<voteCode>'` with no specification of how synthesizer verdicts (`APPROVE`, `REJECT`, `FLAG`, `CONSENSUS`) map to scoreboard result codes (`TP`, `TN`, `FP`, `FN`, `TP+`, `UNAVAIL`). The pre-PAR orchestrator had this mapping inline; the wave-barrier rewrite does not. An executing orchestrator must infer the mapping from institutional knowledge.
**Impact:** Scoreboard win/loss scores may be incorrectly recorded during parallel quorum runs, degrading scoreboard reliability over time.
**Fix:** Add a 6-row mapping table to the orchestrator's Scoreboard Update section: `APPROVE → TP`, `REJECT → FN` (or `FP` depending on convention), etc.

### Issue A — Missing "Escalate (Mode B)" section header *(Low)*

**File:** `agents/qgsd-quorum-orchestrator.md` line 557
**Detail:** `DELIBERATION NEEDED after Round 2 → proceed to **Escalate (Mode B)**` references a named section that doesn't exist. Only `### Escalate — no consensus after 4 rounds` (Mode A) exists.
**Impact:** Non-blocking — orchestrator agents fall through to the Mode A section which is mode-agnostic. Risk is low: Mode B escalation only triggers on genuinely ambiguous FLAG/APPROVE splits after two full rounds.
**Fix:** Add `### Escalate (Mode B)` header above or alias the existing Mode A section, or add a note: "(use Mode A Escalate section — content applies to both modes)".

### Issue C — CROSS_POLLINATION_BUNDLE extraction boundary ambiguity *(Low)*

**File:** `agents/qgsd-quorum-orchestrator.md` line 295 vs `agents/qgsd-quorum-synthesizer.md` line 121-123
**Detail:** Synthesizer says paste "from `Prior positions:` onward, do NOT include the `CROSS_POLLINATION_BUNDLE:` marker." Orchestrator says extract "the `CROSS_POLLINATION_BUNDLE:` block as `$DELIBERATION_BUNDLE`." Reading only the orchestrator, it's ambiguous whether to include the marker line.
**Impact:** Non-blocking — synthesizer doc resolves it; correctly excludes the marker when both are read together.
**Fix:** Update orchestrator line 295 to: "extract everything after `CROSS_POLLINATION_BUNDLE:` (starting at `Prior positions:`) as `$DELIBERATION_BUNDLE`."
