# Phase v0.36-02 Plan 02: Content-Hash Classification Cache Keys Summary

**SHA-256 content hashes replace line-number-based classification cache keys so doc edits at the same line correctly invalidate cached Haiku verdicts**

## Accomplishments
- Replaced dtoc and dtor classification cache keys with 16-char SHA-256 content hashes
- Added automatic cleanup of old-format keys (containing ':' or '/') during reclassification
- Added 8 unit tests (TC-CLASS-1 through TC-CLASS-8) covering stability, differentiation, and backward compatibility

## Decisions Made
- Used 16-char truncated SHA-256 hex (collision probability negligible for expected item counts of hundreds)
- Old-format keys cleaned up lazily during classifyWithHaiku() runs rather than via a one-time migration script
- Archive keys (in archived-solve-items.json) intentionally left unchanged -- they use a different format and serve a different purpose
