## PERF-02: Aggregate Cache Invalidation

### Current State
- **Cache**: `bin/nf-solve.cjs` line 2195 — `let _aggregateCache = null;`
- **Getter**: `getAggregateGates()` (lines 2196-2208) — memoizes aggregate data from compute-per-model-gates.cjs
- **Writer**: `sweepPerModelGates()` (lines 2331-2371) — calls `compute-per-model-gates.cjs --aggregate --write-per-model --json`
- **Problem**: After `sweepPerModelGates()` writes per-model gate files, `_aggregateCache` still holds pre-write data

### Execution Order (lines 2580-2587)
```
l1_to_l2 → sweepL1toL2()     // reads via getAggregateGates() — may populate cache
l2_to_l3 → sweepL2toL3()     // reads via getAggregateGates() — uses cache
l3_to_tc → sweepL3toTC()     // reads via getAggregateGates() — uses cache
per_model_gates → sweepPerModelGates()  // WRITES new files, but cache is stale
```

### Fix
After `sweepPerModelGates()` completes (line 2587), add `_aggregateCache = null;`.
This ensures any subsequent reads in the same session reflect fresh data.
