## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] spawnSync buffer truncation on 200KB nf-solve output**
- **Found during:** Task 1
- **Issue:** `spawnSync` with default/1MB maxBuffer truncated the ~210KB JSON output at 64KB boundary, producing invalid JSON
- **Fix:** Used file-based stdio redirect (`fs.openSync(tmpFile, 'w')` as stdout fd) to bypass buffer limits
- **Files modified:** test/solve-focus-completeness.test.cjs

**2. [Rule 3 - Blocking] nf-solve takes 7+ minutes in non-fast mode**
- **Found during:** Task 1
- **Issue:** `nf-solve.cjs --report-only` without `--fast` takes ~7 minutes, making test impractical even with 300s timeout
- **Fix:** Used `--fast` mode (1.3s) and added source-pattern verification for the 7 layers skipped in fast mode (f_to_c, t_to_c, l1_to_l3, l3_to_tc, per_model_gates, formal_lint, hazard_model)
- **Files modified:** test/solve-focus-completeness.test.cjs

**3. [Rule 1 - Bug] dtoc golden set items produced duplicate itemKey values**
- **Found during:** Task 2
- **Issue:** `itemKey('dtoc', item)` hashes `item.reason` first, and most items shared `reason: "file not found"`, causing 25 items to collapse to 4 unique keys
- **Fix:** Made each dtoc item's `reason` field unique by including the file path (e.g., "bin/old-solver.cjs: file not found in project root")
- **Files modified:** test/classification-golden-set/dtoc.json

## Decisions Made

1. **Fast mode + source verification**: Used `--fast` for integration test speed (1.3s vs 7min) with static source analysis to confirm skipped layers have the `scoped: focusSet ? false : undefined` pattern
2. **Unique reason fields**: Made dtoc `reason` fields include the file path to ensure unique content-hash keys from `itemKey()`
3. **File-based stdout**: Used `fs.openSync()` as stdio fd to avoid spawnSync 64KB buffer truncation on large JSON output
