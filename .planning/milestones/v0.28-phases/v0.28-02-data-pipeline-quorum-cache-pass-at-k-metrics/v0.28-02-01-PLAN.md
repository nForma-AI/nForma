  <action>
Create `bin/quorum-cache.cjs` as a shared module exporting these functions:

1. `computeCacheKey(prompt, context, slots, configQuorumActive, gitHead)` — takes prompt string, context string, slots array (each with `.slot` property), quorum_active array from config, and git HEAD string. Computes SHA-256 of `[prompt, context, sortedSlotNames, configHash, gitHead].join('\x00')` where configHash is SHA-256 of `JSON.stringify(configQuorumActive || [])` truncated to 16 hex chars. Returns full hex digest.

2. `getGitHead()` — runs `git rev-parse HEAD` via `child_process.spawnSync` with 3s timeout, returns trimmed stdout or empty string on failure. Fail-open.

3. `readCache(cacheKey, cacheDir)` — reads `<cacheDir>/<cacheKey>.json`, returns parsed entry if: version === 1, TTL not expired (`Date.now() - new Date(entry.created).getTime() <= entry.ttl_ms`), and entry has `completed` field. Returns null on any error or invalid state. Fail-open.

4. `writeCache(cacheKey, entry, cacheDir)` — creates cacheDir with `{ recursive: true }`, writes entry as pretty-printed JSON. Fail-open (try/catch, no throw).

5. `isCacheValid(entry, currentGitHead, currentQuorumActive)` — returns false if: entry.git_head !== currentGitHead, or JSON.stringify(entry.quorum_active) !== JSON.stringify(currentQuorumActive), or TTL expired. Returns true otherwise.

Use only `node:crypto`, `node:fs`, `node:path`, `node:child_process`. No external dependencies.

Create `bin/quorum-cache.test.cjs` using `node:test` and `node:assert/strict` (matching existing test patterns in verify-quorum-health.test.cjs).

Tests to include:

1. **computeCacheKey determinism** — same inputs produce same hash; different inputs produce different hash.
2. **computeCacheKey slot ordering** — slots in different order produce same key (sorted internally).
3. **computeCacheKey null-byte separator** — changing which field a value belongs to changes the key (prevents "ab\x00c" === "a\x00bc" collisions).
4. **getGitHead** — returns non-empty string (runs in a git repo).
5. **writeCache + readCache roundtrip** — write an entry to a temp dir, read it back, verify fields match.
6. **readCache TTL expired** — write entry with `ttl_ms: 1`, wait 2ms, readCache returns null.
7. **readCache missing completed field** — entry without `completed` returns null (pending cache miss).
8. **readCache wrong version** — entry with version 2 returns null.
9. **readCache missing file** — returns null without throwing.
10. **isCacheValid** — returns false when git_head differs, when quorum_active differs, when TTL expired. Returns true when all match.

Use `os.tmpdir()` + `fs.mkdtempSync` for temp directories. Clean up after each test.
