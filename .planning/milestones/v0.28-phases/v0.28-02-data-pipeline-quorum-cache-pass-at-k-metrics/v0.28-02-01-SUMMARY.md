## Decisions Made
- Null-byte separator (\x00) between cache key fields prevents cross-field collisions (e.g., "ab"+"\x00"+"c" differs from "a"+"\x00"+"bc")
- Cache validity checks both git HEAD and quorum_active composition, ensuring stale results are never served after code changes or slot reconfiguration
- Config hash truncated to 16 hex chars (64 bits) for key component -- sufficient for change detection without bloating key payload
