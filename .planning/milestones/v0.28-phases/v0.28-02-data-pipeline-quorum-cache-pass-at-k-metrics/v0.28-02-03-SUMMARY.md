## Decisions Made
- countDeliberationRounds counts distinct assistant messages containing slot-worker dispatches, not individual slot calls -- multiple parallel dispatches in one message count as a single round
- Cache-hit events receive pass_at_k: 0 as a sentinel value to distinguish them from live quorum decisions in rate calculations
- computePassAtKRates excludes both cache hits (k=0) and pre-PASSK legacy events (no pass_at_k field) from rate calculations
- Pass@k reporting uses try/catch with fallback path resolution to maintain resilience in test environments where planning-paths.cjs is unavailable
