  <name>Task 1: Add cache check and write to nf-prompt.js</name>
  <files>

  </files>
  <action>
In `hooks/nf-prompt.js`, after the quorum dispatch detection (where `cappedSlots` is built and the prompt/context are available) but BEFORE building the slot dispatch instructions:

1. **Require** `bin/quorum-cache.cjs` (use relative path from hooks/ directory: `../bin/quorum-cache.cjs`). Wrap in try/catch — if require fails, skip all cache logic (fail-open).

2. **Compute cache key**: Call `computeCacheKey(prompt, contextYaml, cappedSlots, config.quorum_active, getGitHead())`. The `prompt` is the user's input prompt text. The `contextYaml` is the assembled context string. `cappedSlots` is the slot array already computed.

3. **Check cache**: Call `readCache(cacheKey, cacheDir)` where `cacheDir` is `path.join(process.cwd(), '.planning', '.quorum-cache')`. Also call `isCacheValid(entry, getGitHead(), config.quorum_active)`.

4. **On cache hit** (readCache returns non-null AND isCacheValid returns true):
   - Append a `cache_hit` conformance event to conformance-events.jsonl with fields: `ts`, `phase: 'DECIDING'`, `action: 'cache_hit'`, `cache_key: cacheKey.slice(0, 12)`, `slots_available: entry.slot_count`, `vote_result: entry.vote_result`, `outcome: 'APPROVE'`, `schema_version`.
   - Build instructions string: `<!-- NF_CACHE_HIT -->\n<!-- NF_CACHE_KEY:${cacheKey} -->\nQUORUM CACHE HIT: Identical dispatch was completed ${timeAgo} ago.\nCached result: ${entry.vote_result} of ${entry.slot_count} slots approved.\nDecision: ${entry.outcome}\nSkip all slot-worker Task dispatches. Use this cached quorum result.\nInclude <!-- GSD_DECISION --> in your final output.`
   - Output via `process.stdout.write(JSON.stringify({ hookSpecificOutput: { hookEventName: 'UserPromptSubmit', additionalContext: instructions } }))` and `process.exit(0)`.

5. **On cache miss**: After building the normal dispatch instructions, write a pending cache entry via `writeCache(cacheKey, { version: 1, key: cacheKey, created: new Date().toISOString(), ttl_ms: config.cache_ttl_ms || 3600000, git_head: getGitHead(), quorum_active: (config.quorum_active || []).slice(), slot_count: cappedSlots.length }, cacheDir)`. Also embed `<!-- NF_CACHE_KEY:${cacheKey} -->` in the instructions string so nf-stop.js can extract it for backfill.

6. **timeAgo helper**: Simple inline function — compute ms difference, format as "Xm ago" or "Xh ago".

All cache operations wrapped in try/catch with fail-open semantics. A cache error MUST NEVER prevent normal quorum dispatch.

After editing, copy to dist: `cp hooks/nf-prompt.js hooks/dist/nf-prompt.js`

  <action>
In `hooks/nf-stop.js`:

1. **Require** `bin/quorum-cache.cjs` (relative: `../bin/quorum-cache.cjs`). Wrap in try/catch — if require fails, set a `cacheModule = null` flag and skip all cache logic.

2. **Add `hasCacheHitMarker(currentTurnLines)` function**: Iterate through currentTurnLines, parse each JSON entry, check if `entry.type === 'user'` and the content (string or array of text blocks) includes `'<!-- NF_CACHE_HIT -->'`. Return true if found.

3. **Add `extractCacheKey(currentTurnLines)` function**: Same iteration pattern, look for `<!-- NF_CACHE_KEY:([a-f0-9]+) -->` regex match. Return the captured hex string or null.

4. **Cache hit bypass**: In `main()`, after the existing GUARD 6 (solo mode check), add a new guard: if `hasCacheHitMarker(currentTurnLines)` is true, log a `quorum_complete` conformance event with `cache_hit: true`, `outcome: 'APPROVE'`, `slots_available: 0`, `vote_result: 0`, then `process.exit(0)`. This allows the cached quorum to pass without slot evidence.

5. **Cache backfill on successful quorum**: After the existing quorum approval logic (where `successCount >= maxSize` leads to exit(0)), extract the cache key via `extractCacheKey(currentTurnLines)`. If a key is found and cacheModule is available, read the existing pending entry from `.planning/.quorum-cache/<key>.json`, update it with `vote_result: successCount`, `outcome: 'APPROVE'`, `completed: new Date().toISOString()`, and write it back. All wrapped in try/catch fail-open.

CRITICAL: The EventualConsensus invariant must be preserved. The cache hit bypass ONLY fires when nf-prompt.js has already verified the cache entry is valid (completed, within TTL, matching git HEAD and quorum_active). nf-stop.js trusts the marker because nf-prompt.js is the gatekeeper.
