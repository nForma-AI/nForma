  <action>
For each of the 8 hooks listed below, add a profile guard. The guard MUST come AFTER stdin is fully consumed but BEFORE any expensive operations.

**CRITICAL PATTERN -- Stdin Deadlock Prevention:** Each hook reads stdin differently. The profile guard must go INSIDE the stdin handler, not before it. Hooks that use `process.stdin.on('data') + process.stdin.on('end')` need the guard inside the 'end' callback. Read each hook's actual stdin pattern before placing the guard.

**Import change for all 8 hooks:** Update the require from config-loader to also import shouldRunHook:
```javascript
const { loadConfig, shouldRunHook } = require('./config-loader');
```

**Profile guard pattern (adapt to each hook's stdin pattern):**
```javascript
const profile = config.hook_profile || 'standard';
if (!shouldRunHook('HOOK_BASENAME', profile)) {
  process.stdout.write(JSON.stringify({}) + '\n');
  process.exit(0);
}
```

Replace 'HOOK_BASENAME' with the hook's basename (e.g., 'nf-prompt', 'nf-stop', 'gsd-context-monitor'). Match the stdout format to what the hook already outputs on its pass-through path (some use `{}`, some use `{}\n`, some use no output -- check each hook's existing early exit to replicate exactly).

**Hooks to guard (6 simple guards):**
1. `hooks/nf-session-start.js` -- basename: 'nf-session-start'
2. `hooks/nf-statusline.js` -- basename: 'nf-statusline'
3. `hooks/nf-token-collector.js` -- basename: 'nf-token-collector'
4. `hooks/nf-slot-correlator.js` -- basename: 'nf-slot-correlator'
5. `hooks/nf-spec-regen.js` -- basename: 'nf-spec-regen'
6. `hooks/gsd-context-monitor.js` -- basename: 'gsd-context-monitor'

**Hooks with profile guard + strict mode logic (2 complex):**

7. `hooks/nf-prompt.js` -- basename: 'nf-prompt':
   - Add profile guard (after stdin end, after loadConfig)
   - Add strict mode quorum expansion: when `profile === 'strict'`, treat ALL `/nf:` commands as quorum commands (not just the `quorum_commands` list). Find where the hook checks if the current command is in `quorum_commands` and add a condition: if strict, skip the quorum_commands check and always inject quorum instructions for any `/(nf|q?gsd):\w+` command.

8. `hooks/nf-stop.js` -- basename: 'nf-stop':
   - Add profile guard (after stdin end, after loadConfig)
   - Add strict mode command pattern override: when `profile === 'strict'`, the `buildCommandPattern()` result (or wherever the command regex is constructed) should be replaced with a broader regex that matches ANY `/(nf|q?gsd):\w[\w-]*` command, not just the quorum_commands list. This makes the stop hook enforce quorum on ALL nf commands in strict mode.

**DO NOT modify:**
- `hooks/nf-circuit-breaker.js` -- always active, no guard needed (MonitoringReachable invariant)
- `hooks/nf-precompact.js` -- always active, no guard needed (session continuity)
