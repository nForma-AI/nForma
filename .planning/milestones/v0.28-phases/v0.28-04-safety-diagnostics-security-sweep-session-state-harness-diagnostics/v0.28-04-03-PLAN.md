  <action>
Create `bin/harness-diagnostic.cjs` with the following exports:

1. `generateReport(cwd, options)` -- options: `{ config }` (optional, loads via loadConfig if not provided). Returns a structured report object:

   ```javascript
   {
     timestamp: '...',
     // Section 1: Per-slot availability from scoreboard
     slot_availability: [
       { slot: 'codex-1', total_rounds: 42, successes: 38, failures: 4, success_rate: 0.905, status: 'healthy' },
       // status: 'healthy' (>=80%), 'degraded' (>=50%), 'critical' (<50%), 'unknown' (no data)
     ],
     // Section 2: Pass@k from conformance events
     pass_at_k: { total: 0, pass_at_1: 0, pass_at_3: 0, avg_k: 0 },
     // Section 3: Token spend from token-usage.jsonl
     token_spend: {
       total_records: 0,
       total_input: 0,
       total_output: 0,
       by_slot: {},
       note: 'Token values are currently unreliable (all zeros in Claude Code subagent transcripts)',
     },
     // Section 4: Stall events from stall-detector
     stall_events: [
       { slot: '...', consecutiveTimeouts: 0, lastSeen: null },
     ],
     // Section 5: Circuit breaker status
     circuit_breaker: {
       active: false,
       disabled: false,
       last_triggered: null,
     },
     // Section 6: Actionable recommendations
     recommendations: [],
   }
   ```

   Implementation details:

   a. **Slot availability** -- read scoreboard via planning-paths.cjs `resolveWithFallback(cwd, 'quorum-scoreboard')`. Parse `scoreboard.slots` (keyed by "slotName:modelId"). For each unique slot name (extracted via `key.split(':')[0]`), aggregate tp and fn values across all model entries. Compute `success_rate = tp / (tp + fn)`. Status thresholds: healthy >=0.8, degraded >=0.5, critical <0.5, unknown if no data.

   b. **Pass@k** -- reuse `computePassAtKRates()` from `verify-quorum-health.cjs`. Resolve conformance-events path via planning-paths.cjs.

   c. **Token spend** -- read token-usage.jsonl via planning-paths.cjs. Sum input_tokens and output_tokens per slot. Include note about unreliable values.

   d. **Stall events** -- reuse `detectStalledSlots()` from `stall-detector.cjs`. Pass config loaded via loadConfig().

   e. **Circuit breaker** -- read `.claude/circuit-breaker-state.json` from cwd. Extract `active`, `disabled`, `last_triggered` fields.

   f. **Recommendations** -- generate based on findings:
      - Any slot with success_rate < 0.5: "Slot {name} has {rate}% success rate -- consider removing from quorum_active or checking provider health"
      - Any slot with consecutiveTimeouts >= 2: "Slot {name} has {N} consecutive timeouts -- run `node bin/check-mcp-health.cjs`"
      - pass@1 < 0.7: "pass@1 rate at {N}% -- quorum requires multiple rounds frequently. Consider adjusting maxDeliberation."
      - Circuit breaker active: "Circuit breaker is ACTIVE -- oscillation detected. Follow resolution procedure."
      - No data available: "No quorum data available yet -- run quorum rounds to populate diagnostic data."

   Every section is independent -- wrapped in try/catch. If a data source is missing, the section reports "no data available" defaults. Never crashes.

2. `formatTerminalReport(report)` -- formats the report object into a terminal-friendly string:

   ```
   nForma Harness Diagnostic Report
   =================================
   Generated: 2026-03-06T12:00:00Z

   ## Slot Availability
     slot           rounds   success   rate     status
     -----------------------------------------------
     codex-1        42       38        90.5%    healthy
     gemini-1       42       35        83.3%    healthy

   ## Pass@k Consensus Efficiency
     pass@1:    85.0%
     pass@3:    97.0%
     avg rounds: 1.2
     total events: 42

   ## Token Spend
     Note: Token values unreliable (all zeros)
     total records: 1124

   ## Stall Events
     No stalled slots detected.

   ## Circuit Breaker
     Status: inactive

   ## Recommendations
     - No issues detected. Harness is healthy.
   ```

3. CLI mode: when run as `node bin/harness-diagnostic.cjs [--json] [--cwd /path]`:
   - Runs generateReport(cwd).
   - With `--json`: outputs JSON report to stdout.
   - Without `--json`: outputs formatted terminal report to stdout.
   - Exit code 0 always (diagnostic, never blocks).
   - Supports `require.main === module` guard so it can be required for testing without auto-execution.

All data source reads are wrapped in try/catch with fail-open behavior. Use planning-paths.cjs for all path resolution. Require config-loader.js for loadConfig(). Require verify-quorum-health.cjs for computePassAtKRates(). Require stall-detector.cjs for detectStalledSlots().

Use only node:fs, node:path, node:child_process. No external dependencies.
