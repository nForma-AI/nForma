# Phase v0.28-04: Safety & Diagnostics -- Security Sweep + Session State + Harness Diagnostics - Context

**Gathered:** 2026-03-06
**Status:** Ready for planning

<domain>
## Phase Boundary

Three capabilities added to the harness: (1) a security scanner that runs at verify-phase boundary and reports secrets/debug artifacts in VERIFICATION.md, (2) a session state reminder injected into additionalContext on new sessions when work is in progress, and (3) a unified diagnostic tool (`bin/harness-diagnostic.cjs` / `/nf:health`) that reports harness health with per-slot availability, pass@k trend, token spend, stall events, and actionable recommendations.

Stop hook BLOCK/PASS behavior must be preserved across all new features (LivenessProperty2 and LivenessProperty3 invariants).

</domain>

<decisions>
## Implementation Decisions

### Security Sweep Scope
- Use standard regex patterns for API keys: AWS (`AKIA`), GitHub (`ghp_`/`gho_`/`github_pat_`), Stripe (`sk_live_`/`rk_live_`), generic (`sk-`, `api_key=`, `apiKey=`, `secret=`)
- Scan for `.env` file contents committed to repo, `debugger` statements, `console.log` with sensitive variable names
- Informational only — false positives are acceptable since the sweep is advisory
- Runs as standalone `bin/security-sweep.cjs`, invoked at verify-phase boundary
- Zero latency impact on normal tool calls — only runs during verification

### Session State Reminder
- Terse 2-3 line injection: phase number, current plan name, last activity timestamp
- Distinguish new session from compaction by checking for precompact hook marker in transcript context (compaction events carry precompact marker; fresh sessions do not)
- Fires only when STATE.md shows an in-progress phase — silent otherwise
- Injected via additionalContext in the prompt hook, same pattern as existing context injection

### Diagnostic Report Structure
- Terminal table format matching existing nForma UI patterns (status symbols from ui-brand.md)
- Sections: per-slot availability, pass@k trend, cumulative token spend, stall events, actionable recommendations
- `--json` flag for programmatic use
- Recommendations are concrete: "Slot X has 3 consecutive timeouts — run `node bin/check-mcp-health.cjs`" not vague "check health"

### Stop Hook Regression Safety
- Add regression tests that verify BLOCK/PASS behavior is unchanged when budget, stall, and compact features are active
- Existing LivenessProperty2 and LivenessProperty3 invariants must be preserved
- New features are informational only — they inject additionalContext but never modify stop hook decision logic

### Claude's Discretion
- Exact regex patterns for security sweep (as long as they cover the categories above)
- Internal data structures for the diagnostic report
- How to parse STATE.md for the session reminder (string matching vs structured parsing)
- Test fixture design for regression tests

</decisions>
