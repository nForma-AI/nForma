### Structural Gaps (2/13)

**Gap 1 — FAN-04b: qgsd-stop.js ceiling bypassed (HIGH)**

`hooks/qgsd-stop.js` line 478-481:
```javascript
if (wasSlotWorkerUsed(currentTurnLines)) {
  process.exit(0);  // exits before ceiling check for ALL Task-dispatch quorum
}
// Lines 486-490 NEVER REACHED in production quorum path
```
`--n N` emitted in `additionalContext` by `qgsd-prompt.js` is not accessible to `qgsd-stop.js` (different I/O channels). Stop hook reads `--n N` only from user's original prompt text. For envelope-driven fan-out without explicit `--n` flag, the ceiling check is bypassed.
- Minimum quorum (Claude + ≥1 external) still enforced via `wasSlotWorkerUsed()`.
- Exact-count verification per FAN-04 is absent.

**Gap 2 — ENV-03: CONTEXT_YAML implicit coupling (MEDIUM)**

`quorum.md:194` reads `${CONTEXT_YAML:-}` but `plan-phase.md §8.5` sets `ENVELOPE_PATH` (different name). In the hook path (`qgsd-prompt.js input.context_yaml`): correctly wired. In the inline workflow path: depends on Claude carrying `ENVELOPE_PATH` as `CONTEXT_YAML` — not code-guaranteed. Fail-open to 'medium' preserves safety; envelope risk_level may be silently lost.
