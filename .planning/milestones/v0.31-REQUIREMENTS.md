# Requirements: nForma v0.31 — Ruflo-Inspired Hardening

**Defined:** 2026-03-08
**Core Value:** Planning decisions are multi-model verified by structural enforcement, not instruction-following

## Baseline Requirements

*Included from nForma baseline defaults (profile: cli). Cross-cutting quality gates.*

### UX Heuristics
- [x] **UX-01**: Every user-initiated action produces immediate feedback and completion feedback
- [x] **UX-02**: Destructive actions require explicit confirmation or provide undo
- [x] **UX-03**: Error messages are human-readable, explain what went wrong, and suggest recovery

### Security
- [x] **SEC-01**: Pre-commit hook runs secret scanning to block commits containing credentials
- [x] **SEC-02**: CI pipeline runs deep secret scanning across full repo history on every PR
- [x] **SEC-03**: All external input is validated and sanitized at system boundaries before processing
- [x] **SEC-04**: Dependencies are scanned for known vulnerabilities in CI

### Reliability
- [x] **REL-01**: Graceful degradation on dependency failure
- [x] **REL-02**: Structured error handling with fail-open pattern

### Observability
- [x] **OBS-01**: Key operations produce structured log output on stderr

### CI/CD
- [x] **CI-01**: Test suite runs on every commit
- [x] **CI-02**: Lint and type checks run in CI
- [x] **CI-03**: Build artifacts are reproducible

## Milestone v0.31 Requirements

### Quality Hardening

- [x] **PRIO-01**: User can configure hook execution priority in nf.json so safety hooks (circuit-breaker) always run before enhancement hooks (prompt injection), with deterministic ordering guaranteed
- [x] **BRKR-01**: When circuit breaker fires, the oscillation trigger pattern (file set, alternation count, time window) is persisted to `.planning/formal/evidence/` so future sessions can preemptively avoid the same signature
- [x] **LTCY-01**: User can set per-slot latency budgets in providers.json; slots exceeding their budget are cut off mid-dispatch instead of blocking the quorum pipeline indefinitely
- [x] **EXEC-01**: Quorum slot workers performing review-only tasks (verification, code review) run with restricted tool access (Read/Grep/Glob only), reducing blast radius of misbehaving agents

### Governance & DX

- [x] **SHARD-01**: `.claude/rules/` files are tagged with relevance patterns (tool names, file globs, keywords) so only matching rules load per turn, measurably reducing per-turn token consumption
- [x] **ADAPT-01**: Config values are normalized through a bidirectional adapter (e.g., profile names, boolean strings, nested vs flat keys) so internal types never leak to config files and vice versa
- [x] **VALID-01**: Hook stdin JSON is validated against lightweight schemas with structured error reporting; malformed input produces a diagnostic message on stderr before fail-open exit
- [x] **ADR-01**: Quorum debates in `.planning/quorum/debates/` follow a consistent template (Context, Question, Positions, Decision, Consequences) that downstream tools can parse

## Out of Scope

| Feature | Reason |
|---------|--------|
| Pluggable consensus algorithms (Raft, Byzantine, CRDT) | Over-engineering for CLI plugin; fixed quorum voting sufficient |
| Semantic vector search over memory | ruflo uses HNSW; file-based grep over JSONL sufficient for nForma scale |
| Agent learning daemon (continuous background) | Batch at session boundaries is cheaper and sufficient |
| Rule optimizer A/B testing | Requires many sessions to converge; manual curation faster for now |
| Full MCP protocol 2025-11-25 support | nForma's MCP is thin passthrough; full spec adds unused complexity |
| WASM governance kernel | Deterministic policy engine is overkill for hook-based architecture |
