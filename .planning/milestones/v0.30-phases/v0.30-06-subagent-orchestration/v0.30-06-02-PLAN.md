Extend bin/quorum-slot-dispatch.cjs to enrich quorum prompts with domain-specific retrieved context before dispatching to the external CLI.

**Design note (ORCH-01 architectural rationale):** Pre-dispatch enrichment with 2 iterative retrieval rounds approximates on-demand retrieval without requiring slot workers to have direct tool access. This is architecturally necessary because slot workers dispatch to external CLIs (codex, gemini, opencode, copilot) that lack Read/Grep/Glob tools -- they cannot perform retrieval themselves. By enriching the prompt BEFORE dispatch, we satisfy ORCH-01's intent ("iterative retrieval") within Claude Code's subagent constraints. Add this as a code comment above enrichPromptWithRetrieval.

**New function -- enrichPromptWithRetrieval(prompt, question, artifactPath, cwd):**
- Use dual-path require for context-retriever.cjs: `try { return require(path.join(__dirname, 'context-retriever.cjs')); } catch { return null; }`. If module not found, return prompt unchanged (fail-open).
- Use dual-path require for context-stack.cjs similarly.
- Wrap entire function body in try/catch -- on ANY error, return the original prompt unchanged (fail-open: retrieval errors never prevent quorum dispatch).

Implementation:
- Retrieve the context-retriever module via dual-path require. If not found, return prompt unchanged.
- Retrieve the context-stack module via dual-path require (optional).
- Set charBudget from retriever.TOKEN_BUDGET_CHARS.
- Loop up to retriever.MAX_ROUNDS (2). Each round: call analyzeContextNeeds(question, artifactPath, existingContext), then fetchContext(cwd, needs, remainingBudget). Break if no needs or no results.
- After retrieval rounds, if contextStack module available, call formatInjection(cwd, 'current') and append if within budget.
- If any additional context accumulated, append to prompt with `=== RETRIEVED CONTEXT ===` markers.
- Return enriched prompt, or original prompt on any error.

**Wire into main():**
- After the prompt is built (after buildModeAPrompt or buildModeBPrompt call, around line 834) and before the child process spawn (line 840):
- Check config kill switch: read `.claude/nf.json` in cwd, check `context_retrieval_enabled !== false`. Fail-open: if config read fails, retrieval is ON.
- If enabled, call `prompt = enrichPromptWithRetrieval(prompt, question, artifactPath, cwd);`

**Add to module.exports:** Add `enrichPromptWithRetrieval` to the exports object at line 913.

- Fails open on errors (pass invalid cwd, verify returns original prompt without throwing).

```javascript
// Context stack injection (ORCH-02)
// Hook-level cap is 800 chars, intentionally tighter than the module-level
// INJECTION_CAP_CHARS (2000 chars), because additionalContext in hooks
// contends with other injections (quorum instructions, circuit breaker
// recovery prompt, thinking budget). Keeping this small avoids crowding
// out higher-priority hook content.
if (contextStack) {
  try {
    let currentPhase = null;
    const phaseMatch = prompt.match(/v[\d.]+-\d+/);
    if (phaseMatch) currentPhase = phaseMatch[0];
    const stackInjection = contextStack.formatInjection(cwd, currentPhase || 'unknown');
    if (stackInjection && stackInjection.length <= 800) {
      instructions += '\n\n' + stackInjection;
    }
  } catch (_) { /* fail-open */ }
}
```
