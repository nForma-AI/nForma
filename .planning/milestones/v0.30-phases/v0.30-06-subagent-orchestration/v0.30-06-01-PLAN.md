Create bin/context-retriever.cjs as a CommonJS module with 'use strict'. Zero new npm dependencies. This module implements the three specialized retrieval agents (ORCH-03) as domain-specific functions, and the iterative retrieval protocol (ORCH-01).

**Constants:**
- `TOKEN_BUDGET_CHARS = 32000` (8k tokens * 4 chars/token)

**Domain configurations (ORCH-03 -- three specialized retrievers):**
```javascript
const DOMAIN_CONFIG = {
  test: {
    files: ['.planning/formal/unit-test-coverage.json'],
    searchPatterns: [/\.(test|spec)\.(cjs|js|ts)$/],
    searchDirs: ['bin/', 'hooks/dist/'],
    maxFilesToScan: 20,
  },
  architecture: {
    files: [
      '.planning/STATE.md',
      '.planning/memory/decisions.jsonl',
    ],
    searchDirs: ['.planning/'],
    maxFilesToScan: 10,
  },
  formal: {
    files: [
      '.planning/formal/traceability-matrix.json',
      '.planning/formal/solve-state.json',
    ],
    searchPatterns: [/\.(tla|als|pm)$/],
    // Use explicit subdirectories instead of '.planning/formal/' alone,
    // because readdirSync is non-recursive and formal models live in
    // tla/, alloy/, prism/ subdirs -- scanning the parent would miss them.
    searchDirs: ['.planning/formal/tla/', '.planning/formal/alloy/', '.planning/formal/prism/'],
    maxFilesToScan: 15,
  },
};
```

**analyzeContextNeeds(question, artifactPath, existingContext):**
- Returns array of `{ domain, query }` objects.
- Uses keyword detection on `question.toLowerCase()`:
  - test/coverage/verify/spec/assertion -> `{ domain: 'test', query: question }`
  - architecture/design/pattern/decision/convention -> `{ domain: 'architecture', query: question }`
  - formal/invariant/spec/tla/alloy/prism/model -> `{ domain: 'formal', query: question }`
- Also analyze `artifactPath` segments using PATH_CATEGORY_MAP-style mapping:
  - Path contains 'test' -> test domain
  - Path contains 'formal'/'tla'/'alloy' -> formal domain
  - Path contains 'planning'/'state' -> architecture domain
- Deduplicate domains (no duplicate domain entries in result).
- If `existingContext` already contains content for a domain (check for domain marker like `--- test ---`), skip that domain in second round.
- Return empty array if no domains detected (signals no retrieval needed).

**fetchContext(cwd, needs, charBudget):**
- For each need in `needs`, look up `DOMAIN_CONFIG[need.domain]`.
- Skip unknown domains (fail-open).
- **Fast path -- known files:** For each `config.files` entry, read with `fs.readFileSync`. Truncate to remaining budget. Prefix with `\n--- ${relPath} ---\n`.
- **Search path (if budget remains):** For each `config.searchDirs`, list files with `fs.readdirSync` (NOT recursive). Filter by `config.searchPatterns`. Read up to `config.maxFilesToScan` matching files, appending to result within budget.
- Track `remaining = charBudget - result.length`. Stop when remaining <= 0.
- Wrap everything in try/catch per file -- fail-open: skip unreadable files.
- Return concatenated string, or empty string if nothing found.

**CLI interface (if run as main):**
- `analyze --question <text> [--artifact <path>]` -- prints JSON array of needs.
- `fetch --cwd <dir> --domain <name> [--budget <chars>]` -- fetches and prints context.
- Fail-open on CLI errors: stderr + exit 0.

**Module exports:** `{ analyzeContextNeeds, fetchContext, DOMAIN_CONFIG, TOKEN_BUDGET_CHARS, MAX_ROUNDS }`

Create bin/context-stack.cjs as a CommonJS module with 'use strict'. Zero new npm dependencies. Follows JSONL patterns from bin/memory-store.cjs.

**Constants:**
- `STACK_FILE = 'context-stack.jsonl'`
- `MAX_ENTRIES_PER_PHASE = 10`
- `INJECTION_CAP_CHARS = 2000`

**getStackPath(cwd):**
- Returns path to `.planning/memory/context-stack.jsonl`.
- Creates `.planning/memory/` directory with `mkdirSync({ recursive: true })`.

**append(cwd, { phase, type, content, tags }):**
- Validates `type` is one of: `architecture_decision`, `test_result`, `api_contract`, `constraint`.
- If invalid type, log to stderr and use 'constraint' as fallback (fail-open).
- Enforces MAX_ENTRIES_PER_PHASE: count existing entries for this phase. If >= 10, skip append and return `{ skipped: true, reason: 'phase_cap_reached' }`.
- Writes JSONL line with `{ phase, type, content, tags: tags || [], ts: new Date().toISOString() }`.
- Returns the written entry.

**queryRecentPhases(cwd, currentPhase, maxPhases = 3):**
- Reads all JSONL entries from context-stack.jsonl.
- Gets unique phases, filters out `currentPhase`, sorts by recency (order of appearance).
- Returns entries from the last `maxPhases` phases.
- Returns `[]` if file missing or no entries. Skips malformed lines (fail-open).

**queryByType(cwd, type, limit = 10):**
- Reads all entries, filters by `type`, returns newest-first up to `limit`.

**formatInjection(cwd, currentPhase):**
- Calls `queryRecentPhases(cwd, currentPhase)`.
- If no entries, returns null.
- Formats as markdown:
  ```
  ## Context Stack (accumulated from prior phases)

  - [v0.30-05] architecture_decision: Used node:test runner for inline checks
  - [v0.30-04] constraint: Weekly decay 0.05/week with 0.1 floor
  ```
- Enforces `INJECTION_CAP_CHARS` -- stops adding lines when approaching cap.
- Returns formatted string or null.

**prune(cwd, keepPhases = 5):**
- Reads all entries, keeps only entries from the most recent `keepPhases` phases.
- Rewrites file. Returns `{ removed, remaining }`.

**CLI interface (if run as main):**
- `append --phase <p> --type <t> --content <c> [--tags <csv>]` -- appends entry.
- `query --current-phase <p> [--max-phases <n>]` -- queries recent phases.
- `format --current-phase <p>` -- prints formatted injection.
- `prune [--keep-phases <n>]` -- prunes old entries.
- Fail-open on CLI errors: stderr + exit 0.

**Module exports:** `{ append, queryRecentPhases, queryByType, formatInjection, getStackPath, prune, STACK_FILE, MAX_ENTRIES_PER_PHASE, INJECTION_CAP_CHARS }`
