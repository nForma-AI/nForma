**In bin/quorum-slot-dispatch.cjs:**
1. First check if config-loader.js is already required at top of file (`grep -n 'config-loader' bin/quorum-slot-dispatch.cjs`). If not, add a dual-path require:
   ```javascript
   function loadNfConfig(cwd) {
     const candidates = [
       path.join(os.homedir(), '.claude', 'hooks', 'config-loader.js'),
       path.join(__dirname, '..', 'hooks', 'dist', 'config-loader.js'),
       path.join(__dirname, '..', 'hooks', 'config-loader.js'),
     ];
     for (const p of candidates) {
       try {
         const { loadConfig } = require(p);
         return loadConfig(cwd);
       } catch (_) {}
     }
     return {}; // fail-open: empty config uses defaults
   }
   ```
   NOTE: The three candidate paths ensure consistency with the install sync pipeline (hooks/ -> hooks/dist/ -> ~/.claude/hooks/). The `hooks/dist/` path is included as a second candidate because during development, hooks/dist/ is the authoritative installed copy that bin/install.js reads from.

2. Replace the direct config read block that reads `.claude/nf.json` directly with:
   ```javascript
   const nfConfig = loadNfConfig(cwd);
   const retrievalEnabled = nfConfig.context_retrieval_enabled !== false;
   ```

This eliminates the convention violation where only project-level config was read, missing global config and DEFAULT_CONFIG defaults.
