  <action>
Create `bin/memory-store.cjs` following the `bin/execution-progress.cjs` pattern (CommonJS, 'use strict', fail-open, CLI + module.exports).

Core functions (all synchronous, all take `cwd` as first arg):

1. `getMemoryPath(cwd, category)` -- returns path to JSONL file under `.planning/memory/`. Creates directory with `mkdirSync({ recursive: true })`. Categories: 'decisions', 'errors', 'quorum'. Map: `{ decisions: 'decisions.jsonl', errors: 'errors.jsonl', quorum: 'quorum-decisions.jsonl' }`.

2. `appendEntry(cwd, category, entry)` -- appends a JSON line with `ts: new Date().toISOString()` added to entry. Uses `fs.appendFileSync`. Returns the entry with ts added.

3. `readLastN(cwd, category, n = 5)` -- reads file, splits by newline, takes last N lines, parses each as JSON. Returns `[]` if file missing. Silently skips malformed lines.

4. `queryByField(cwd, category, field, keyword, limit = 5)` -- searches from end of file (newest first). Case-insensitive substring match on the specified field AND on the `tags` array (joined with space). Returns up to `limit` matches.

5. `isDuplicate(cwd, category, field, value, checkLast = 10)` -- reads last N entries, checks if any entry's field value contains the needle or vice versa (case-insensitive). Returns boolean.

6. `countEntries(cwd, category)` -- counts non-empty lines in the JSONL file. Returns 0 if file missing.

7. `generateSessionReminder(cwd)` -- reads last 3 decisions, counts errors and quorum entries. Returns formatted string (see research Pattern 3) or null if no entries. Hard cap at 800 characters.

8. `formatMemoryInjection(cwd)` -- reads last 3 decisions and last 2 errors. Returns formatted string (see research Pattern 4) or null if no entries. Hard cap at 1200 characters.

9. `pruneOlderThan(cwd, category, days = 90)` -- reads all entries, filters out those older than `days` days based on `ts` field, writes back remaining entries. Returns `{ removed, remaining }`.

Dedup behavior: `append-decision` and `append-error` CLI commands check `isDuplicate` before appending (field: 'summary' for decisions, 'symptom' for errors). Skip and print existing entry if duplicate found.

CLI interface (when `require.main === module`):
- `append-decision --summary "..." --detail "..." --rejected "A,B" --tags "x,y" --source "quorum" --phase "v0.30-03"`
- `append-error --symptom "..." --root-cause "..." --fix "..." --files "a.js,b.js" --tags "x,y"`
- `append-quorum --question "..." --chosen "..." --rationale "..." --rejected "A,B" --consensus "APPROVE" --rounds 1 --debate-file "path"`
- `query-decisions --last N` (default 5)
- `query-errors --symptom "keyword" --limit N` (default 5)
- `query-quorum --last N` (default 5)
- `session-reminder` -- outputs generateSessionReminder result
- `prune --days N` (default 90)

All CLI output goes to stdout as JSON. Errors go to stderr. Exit 0 on all errors (fail-open).

Do NOT use planning-paths.cjs for path resolution in this module -- use direct path.join with MEMORY_DIR constant (keeps it self-contained like execution-progress.cjs). The planning-paths registration in Task 2 is for external consumers that use the resolve() API.
