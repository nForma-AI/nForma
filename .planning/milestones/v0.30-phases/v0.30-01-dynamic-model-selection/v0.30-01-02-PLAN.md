Create `bin/token-dashboard.cjs` as a CommonJS module with `'use strict'` and `#!/usr/bin/env node` shebang. Zero new dependencies -- only Node.js built-ins (fs, path, readline).

**JSONL record format** (from nf-token-collector.js):
```json
{ "ts": "ISO8601", "session_id": "...", "agent_id": "...", "slot": "codex-1",
  "input_tokens": N, "output_tokens": N, "cache_creation_input_tokens": N, "cache_read_input_tokens": N }
```

**Cost table** (hardcoded, $/1M tokens, approximate):
```javascript
const COST_PER_M = {
  // Anthropic models (via Claude Code)
  'claude':    { input: 15.00, output: 75.00 },
  'sonnet':    { input: 3.00,  output: 15.00 },
  'haiku':     { input: 0.25,  output: 1.25 },
  // Third-party (via ccr/MCP)
  'codex':     { input: 2.50,  output: 10.00 },
  'gemini':    { input: 1.25,  output: 5.00 },
  'opencode':  { input: 2.00,  output: 8.00 },
  'copilot':   { input: 0.00,  output: 0.00, subscription: true },  // subscription-based, no per-token cost
  'default':   { input: 3.00,  output: 15.00 },  // fallback
};
```
Mark costs as "estimated" in output header.

**IMPORTANT: Copilot display handling.** For copilot slots, display "subscription" in the Est. Cost column instead of "$0.00" to avoid user confusion about zero-cost display. Check `COST_PER_M[slotFamily].subscription === true` and render accordingly.

**Exported functions:**

1. `parseTokenUsage(jsonlPath)` -- reads file line by line, skips empty/malformed lines (fail-open), returns array of record objects. If file doesn't exist, returns [].

2. `aggregateBySlot(records)` -- groups by slot family (strip trailing -N), sums input_tokens, output_tokens, cache_read_input_tokens. Returns Map of slot -> { input, output, cacheHit, recordCount }.

3. `aggregateBySession(records)` -- groups by session_id, sums tokens, returns Map of session -> { input, output, ts (earliest), recordCount }.

4. `estimateCost(slotFamily, inputTokens, outputTokens)` -- looks up COST_PER_M by slot family, returns dollar amount. Cache read tokens charged at 10% of input rate. For subscription-based slots (copilot), returns `{ cost: 0, isSubscription: true }` so callers can display "subscription" instead of "$0.00".

5. `formatDashboard(records, options)` -- formats the full dashboard output as a string. Options: `{ last: N }` to limit to last N sessions. Output format:
```
Token Usage Dashboard (estimated costs)
═══════════════════════════════════════════════════════
Slot          Input       Output      Cache Hit%   Est. Cost
──────────────────────────────────────────────────────
codex         145,200     32,100      62%          $0.42
gemini         98,300     28,400      71%          $0.31
copilot        12,400      5,200      45%          subscription
...
──────────────────────────────────────────────────────
TOTAL         453,900    105,700      61%          $1.62*

* Total excludes subscription-based slots
```

**CLI entrypoint** (when run directly via `node bin/token-dashboard.cjs`):
- Parse argv for `--last N` (default 5 sessions), `--jsonl PATH` (default `.planning/token-usage.jsonl`), `--json` (output JSON instead of table)
- Call `parseTokenUsage`, `formatDashboard`, write to stdout
- Exit 0 even if JSONL is empty or missing (print "No token usage data found.")

**Tests** in `bin/token-dashboard.test.cjs`:
- parseTokenUsage with empty file returns []
- parseTokenUsage with malformed lines skips them, returns valid records
- parseTokenUsage with valid JSONL returns correct record count
- aggregateBySlot groups 'codex-1' and 'codex-2' under 'codex'
- aggregateBySession groups by session_id correctly
- estimateCost('codex', 1000000, 500000) returns expected dollar amount
- estimateCost('copilot', 1000000, 500000) returns { cost: 0, isSubscription: true }
- estimateCost('unknown-slot', ...) uses default rates
- formatDashboard with empty records returns "No token usage data found."
- formatDashboard with sample records contains expected column headers
- formatDashboard with copilot records shows "subscription" not "$0.00"

Use `node:test` (built-in test runner) and `node:assert` for tests, consistent with existing test patterns in the project.
