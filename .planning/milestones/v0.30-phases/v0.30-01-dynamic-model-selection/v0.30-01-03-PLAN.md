Extend `bin/budget-tracker.cjs` with cooldown logic to prevent downgrade oscillation.

**Changes to triggerProfileDowngrade(cwd, options):**

Add an optional `options` parameter: `{ cooldownRounds: 3, cooldownMs: 300000 }` (defaults from config).

1. Before downgrading, read `downgrade_history` from `.planning/config.json`. This is an array of `{ ts: ISO8601, from: string, to: string }` objects.

2. Check cooldown: if the most recent entry in `downgrade_history` has a `ts` within `cooldownMs` of now, return `{ downgraded: false, reason: 'cooldown_active', current }` without downgrading.

3. Detect oscillation: if the last 3 entries in `downgrade_history` show alternating directions (down-up-down or up-down-up) within 10 minutes, freeze at current level. Return `{ downgraded: false, reason: 'oscillation_detected', current }`.

4. On successful downgrade, append to `downgrade_history` array in config.json. Keep only last 10 entries (prune older ones).

**New exported function: `checkCooldown(cwd, cooldownMs)`**
Returns `{ active: boolean, lastDowngradeTs: string|null, remainingMs: number }`.

**Update computeBudgetStatus** to accept an additional `cooldownActive` boolean. When cooldownActive is true, set `shouldDowngrade: false` regardless of budget percentage.

**Add compaction threshold config key** to DEFAULT_CONFIG in `hooks/config-loader.js`:
```javascript
smart_compact_threshold_pct: 65,  // Proactive compaction at 65% context (midpoint of 60-70% range from research)
```
This is the default percentage at which boundary-aware compaction triggers. After editing `hooks/config-loader.js`, copy to `hooks/dist/config-loader.js`.

**Tests** to add in `bin/budget-tracker.test.cjs`:
- triggerProfileDowngrade with recent downgrade_history entry returns cooldown_active
- triggerProfileDowngrade with old downgrade_history entry proceeds with downgrade
- triggerProfileDowngrade with oscillation pattern (3+ alternating in 10min) returns oscillation_detected
- checkCooldown with no history returns { active: false }
- checkCooldown with recent history returns { active: true, remainingMs > 0 }
- downgrade_history is pruned to 10 entries max

**Critical:** Preserve backward compatibility. The existing `triggerProfileDowngrade(cwd)` call with no options must work identically to before (cooldown defaults apply, but with no history it proceeds normally).
