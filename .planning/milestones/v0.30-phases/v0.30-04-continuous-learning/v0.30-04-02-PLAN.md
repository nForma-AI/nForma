**Create hooks/nf-session-end.js** following the exact pattern from RESEARCH.md code example:

1. `'use strict'` at top, CommonJS requires for fs, path, os
2. Hard 5-second timeout: `const HARD_TIMEOUT = setTimeout(() => process.exit(0), 5000); HARD_TIMEOUT.unref();`
3. Stdin accumulation pattern (same as nf-precompact.js): `let raw = ''; process.stdin.on('data', ...); process.stdin.on('end', ...)`
4. In the end handler:
   - Parse input JSON, extract `cwd`, `transcript_path`
   - Profile guard: `loadConfig(cwd)`, `shouldRunHook('nf-session-end', profile)`, exit 0 if inactive
   - Feature flag guard: if `config.learning_enabled === false`, exit 0
   - Verify transcript_path exists, exit 0 if not
   - Locate learning-extractor.cjs (try installed path `~/.claude/nf-bin/learning-extractor.cjs` first, then local `../bin/learning-extractor.cjs`)
   - Locate memory-store.cjs (same pattern as nf-session-start.js findMemoryStore)
   - If either module not found, exit 0
   - Call `extractor.readLastLines(transcriptPath, 500)` to get transcript lines
   - Call `extractor.extractErrorPatterns(lines)` -> for each pattern, check `memStore.isDuplicate(cwd, 'errors', 'symptom', pattern.symptom)`, if not duplicate call `memStore.appendEntry(cwd, 'errors', pattern)`
   - Call `extractor.extractCorrections(lines.slice(-200))` -> for each correction, check `memStore.isDuplicate(cwd, 'corrections', 'wrong_approach', correction.wrong_approach)`, if not duplicate call `memStore.appendEntry(cwd, 'corrections', correction)`
   - Log extraction counts to stderr: `[nf-session-end] Extracted N error patterns, M corrections`
5. Wrap everything in try/catch, always exit(0) (fail-open)
6. NEVER write to stdout (SessionEnd has no decision control)
7. Export `findLearningExtractor` and `findMemoryStore` for testing

4. Formulates candidate skills: `{ skill: description, evidence: [entry summaries], tags: [shared tags] }`
5. When `--validate` flag is passed:
   - For each candidate skill (max 5), output the candidate to stdout
   - Print instructions: "To validate via quorum, run: /nf:quick validate skill: <skill description>"

6. When `--validate` is NOT passed (dry run):
   - List candidate skills without validation
   - Show evidence count and tag clusters
7. When `--persist` flag with `--skill`, `--validated-by`, `--confidence`:
   - Directly append a validated skill to memory-store (for use after quorum validation)
   - Calls memory-store appendEntry for 'skills' category
8. Export clustering and candidate generation functions for testing

**3. Update hooks/nf-precompact.js:**

Extend formatMemoryInjection or add a new section after the memory block. In the `readMemoryInjection` function call area (around line 236), add:

```javascript
// Learning snapshot injection (LRNG-01, LRNG-04)
try {
  if (memoryStore && memoryStore.readLastN) {
    const recentFailures = memoryStore.readLastN(cwd, 'failures', 2);
    const recentCorrections = memoryStore.readLastN(cwd, 'corrections', 2);
    if (recentFailures.length > 0 || recentCorrections.length > 0) {
      const learningLines = ['', '## Learning Snapshot (auto-injected at compaction)', ''];
      if (recentFailures.length > 0) {
        learningLines.push('Known failures:');
        for (const f of recentFailures) {
          const approach = (f.approach || '').slice(0, 60);
          const conf = memoryStore.computeCurrentConfidence ? memoryStore.computeCurrentConfidence(f) : (f.confidence || 0.7);
          learningLines.push('  - ' + approach + ' (conf: ' + conf.toFixed(2) + ')');
        }
      }
      if (recentCorrections.length > 0) {
        learningLines.push('Recent corrections:');
        for (const c of recentCorrections) {
          const wrong = (c.wrong_approach || '').slice(0, 50);
          const correct = (c.correct_approach || '').slice(0, 50);
          learningLines.push('  - Not: ' + wrong + ' -> Instead: ' + correct);
        }
      }
      lines.push(...learningLines);
    }
  }
} catch (_) {}
```
