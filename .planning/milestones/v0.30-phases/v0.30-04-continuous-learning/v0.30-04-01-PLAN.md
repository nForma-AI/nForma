2. Add a `computeCurrentConfidence(entry)` function:
   - Takes an entry with `confidence` (number 0.0-1.0) and `last_confirmed` (ISO timestamp)
   - Computes weekly decay: `Math.floor(daysSinceConfirmed / 7) * 0.05`
   - Returns `Math.max(0.1, entry.confidence - weekDecay)`
   - If entry has no `confidence` or `last_confirmed`, return `entry.confidence || 0.7`

3. Add a `boostConfidence(cwd, approach)` function:
   - Reads failures.jsonl, finds entry matching `approach` (case-insensitive substring)
   - If found: sets `confidence = Math.min(1.0, confidence + 0.2)`, updates `last_confirmed` to now, increments `confirmation_count`
   - Rewrites the JSONL file with updated entry
   - Returns the updated entry or null if not found

4. Add `queryWithConfidence(cwd, category, field, keyword, minConfidence, limit)`:
   - Like queryByField but filters by `computeCurrentConfidence(entry) >= minConfidence`
   - Sorts results by computed confidence descending
   - Used for failures category primarily

5. Add CLI commands (in the switch block):
   - `append-correction`: requires `--wrong` and `--correct`, optional `--context`, `--tags`. Entry schema: `{ type: 'correction', wrong_approach, correct_approach, context, tags }`

   - `decay-failures`: no args. Reads all failures, applies computeCurrentConfidence, rewrites only entries where stored confidence differs from computed by > 0.01. Reports count updated.

  <files>bin/learning-extractor.cjs, bin/learning-extractor.test.cjs</files>

Create bin/learning-extractor.cjs as a CommonJS module with `'use strict'` at top.

1. `extractErrorPatterns(lines, maxPatterns = 10)`:
   - `lines` is an array of raw JSONL strings from a transcript file
   - Parse each line as JSON, skip malformed
   - Scan for error indicators in user entries with tool_result blocks:
     - `block.is_error === true`
     - `block.type === 'tool_error'`
     - Content string containing 'Error:', 'ENOENT', 'EACCES', 'EPERM', 'TypeError', 'ReferenceError', 'SyntaxError', or stack trace pattern (`at .+\(`)
   - For each error found at index i, look forward (up to 20 lines) for resolution:
     - Next user entry with a tool_result for the same tool (not is_error) = fix found
     - Or next assistant text containing 'fixed', 'resolved', 'solution', 'the issue was'
   - Extract symptom from error content (first 200 chars of error text)
   - Extract fix from resolution content (first 200 chars)
   - If both symptom and fix found, push `{ type: 'error_resolution', symptom, root_cause: '', fix, tags: [], ts: new Date().toISOString() }`
   - Stop after maxPatterns extractions
   - Return array of extracted patterns

2. `extractCorrections(lines, maxCorrections = 5)`:
   - `lines` is an array of raw JSONL strings
   - Only scan last 200 lines (corrections are more recent-conversation focused)
   - Find user messages that follow an assistant message (reaction pattern)
   - User message must match at least 2 of these CORRECTION_INDICATORS:
     ```
     /\bdon'?t\b/i, /\binstead\b/i, /\bwrong\b/i, /\bshould\b/i,
     /\bactually\b/i, /\bnot\b/i, /\bprefer\b/i, /\bover\b/i,
     /\bnot like that\b/i, /\binstead of\b/i, /\bI said\b/i,
     /\bthat's not\b/i, /\buse\s+\w+\s+not\b/i
     ```
   - Requiring 2+ indicators reduces false positives (conservative approach per research)
   - User message must be > 20 chars (skip "ok", "yes")
   - Extract: `{ type: 'correction', wrong_approach: assistantSummary, correct_approach: userMessage, context: '', tags: [], ts: new Date().toISOString() }`
   - `assistantSummary`: first 150 chars of the preceding assistant text block
   - `userMessage`: first 200 chars of the user's correction text
   - Stop after maxCorrections
   - Return array of corrections

3. `readLastLines(filePath, n)`:
   - Read file with fs.readFileSync, split by newline, filter empty, slice(-n)
   - Return array of line strings
   - If file does not exist or read fails, return []

4. Helper `extractTextFromEntry(entry)`:
   - If entry.message.content is string, return it
   - If array, find first block with type 'text', return its text
   - Return '' otherwise

Export: `{ extractErrorPatterns, extractCorrections, readLastLines, extractTextFromEntry }`

No CLI interface needed -- this module is consumed by hooks/nf-session-end.js (Plan 02).
