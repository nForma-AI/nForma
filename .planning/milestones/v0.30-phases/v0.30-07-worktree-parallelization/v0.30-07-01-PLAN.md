  <action>
1. Create `agents/nf-worktree-executor.md` with frontmatter:
   - `name: nf-worktree-executor`
   - `description: Executes plan tasks in an isolated git worktree for parallel execution. Spawned by execute-plan Pattern D for independent tasks.`
   - `tools: Read, Write, Edit, Bash, Grep, Glob`
   - `isolation: worktree`
   - `color: green`
   - The markdown body should be nearly identical to `agents/nf-executor.md` with these additions:
     a. Note that this executor operates in an isolated git worktree (branched from origin/main)
     b. Executor must report its worktree branch name in a STRUCTURED JSON BLOCK in its completion output (Q-R3.6-4: reliable parsing format). The executor MUST output a fenced JSON block at the end of its completion message:
        ```json
        {"worktree_branch": "<branch-name>", "task_number": <N>, "status": "complete|failed", "files_modified": ["path1", "path2"]}
        ```
        This format is parsed by the orchestrator -- do NOT use free-form text for branch reporting.
     c. Note the constraint: subagents cannot spawn other subagents, so this executor must execute tasks directly
     d. State updates (STATE.md, ROADMAP.md) should NOT be performed by worktree executors -- the orchestrator handles those after merge (Q-R3.6-6: executors must skip state_updates and final_commit steps)
     e. The executor must NOT perform final git commits of planning state -- only commit code changes within its task scope

2. Create `bin/worktree-merge.cjs` following the code example from RESEARCH.md with these functions:
   - `ensureCleanState(cwd, targetBranch)` (Q-R3.6-2: explicit precondition check) -- Before any merge operations:
     a. Run `git status --porcelain` to check for uncommitted changes; if dirty, throw Error('Uncommitted changes detected -- commit or stash before merge')
     b. Run `git rev-parse --abbrev-ref HEAD` to confirm current branch matches `targetBranch`; if not, throw Error(`Expected branch ${targetBranch}, got ${actual}`)
     c. Note: worktrees branch from origin/main, but merges happen into the working branch (which may differ) -- targetBranch makes this explicit
   - `mergeBranches(cwd, branches, options)` -- options includes `{ targetBranch }` (defaults to current branch). Before merging:
     a. Call `ensureCleanState(cwd, options.targetBranch)` (Q-R3.6-2)
     b. Record pre-merge checkpoint: `const checkpoint = execFileSync('git', ['rev-parse', 'HEAD'], { cwd }).toString().trim()` (Q-R3.6-3: rollback point)
     c. For each branch, attempt `git merge --no-ff <branch>` into current branch. On conflict: `git merge --abort`, mark as conflict. Return `{ branches: [{ branch, status, error? }], checkpoint }`.
   - `verifyMergedState(cwd, checkpoint)` (Q-R3.6-3: rollback on failure) -- Run `npm test` with 120s timeout. If tests fail:
     a. Log rollback warning to stderr: `Verification failed, rolling back to ${checkpoint}`
     b. Run `git reset --hard ${checkpoint}` to restore pre-merge state
     c. Return `{ pass: false, error: ..., rolledBack: true, checkpoint }`
     d. If tests pass, return `{ pass: true }`
   - `cleanupWorktreeBranches(cwd, branches)` -- Best-effort `git branch -d <branch>` for each. Return array of results. Also run `git worktree prune` to clean stale worktree entries.
   - CLI interface: `node bin/worktree-merge.cjs <merge|verify|cleanup|ensure-clean> [branches...]`
   - Follow project conventions: `'use strict'`, CommonJS, fail-open (process.exit(0) on error), stderr for diagnostics
   - Set MERGE_TIMEOUT_MS = 30000 (30s per merge operation)

3. Add `.claude/worktrees/` to `.gitignore` if not already present (append at end, with comment `# Worktree isolation (parallel execution)`)
  </action>
