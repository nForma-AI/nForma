  <name>Task 1: Create bin/continuous-verify.cjs verification engine</name>
  <files>bin/continuous-verify.cjs, bin/continuous-verify.test.cjs</files>
  <action>
Create bin/continuous-verify.cjs as a CommonJS module with 'use strict'. Zero new npm dependencies.

**Verification state management:**
- `initVerifyState(phase)` -- returns default state object: `{ version: 1, phase, max_runs: 3, runs_used: 0, timeout_ms: 5000, accumulated_files: [], last_run: null, runs: [] }`
- `getVerifyState(cwd)` -- reads `.planning/continuous-verify.json`, returns parsed JSON or null. Fail-open: parse errors return null.
- `saveVerifyState(cwd, state)` -- writes state to `.planning/continuous-verify.json` with `mkdirSync({ recursive: true })`. Fail-open: wrap in try/catch.

**Boundary detection:**
- `shouldTriggerVerification(toolName, toolInput, verifyState)` -- returns boolean.
  - Returns false if `verifyState.runs_used >= verifyState.max_runs` (budget exhausted).
  - Returns false if `toolName` is not 'Write' or 'Edit' (use a Set for WRITE_TOOLS).
  - Extracts `file_path` from `toolInput` (handle both string and object forms).
  - Accumulates unique file paths into `verifyState.accumulated_files`.
  - Returns true if: accumulated_files.length >= 5, OR file matches TEST_FILE_RE (`/\.(test|spec)\.(js|ts|cjs|mjs|jsx|tsx)$/`), OR basename is in CONFIG_FILES Set (`package.json`, `tsconfig.json`, `.eslintrc.json`, `vitest.config.ts`).
  - Returns false otherwise (no boundary reached).

**Check runner:**
- `runChecks(cwd, files, timeoutMs)` -- runs lint/test checks on accumulated files.
  - For test files in the accumulated list (matching TEST_FILE_RE): run `spawnSync('node', ['--test', ...testFiles], { cwd, timeout: timeoutMs, encoding: 'utf8' })`. Use node:test runner (project convention), NOT vitest, for the inline check. The test:ci script already uses `node --test`.
  - For non-test JS/CJS files: check if eslint exists at `node_modules/.bin/eslint`. If so, run `spawnSync` with those files. If eslint not found, skip lint (pass=true, reason='no linter').
  - Return structured result: `{ timestamp, checks: [{ type, pass, output, duration_ms }], files_checked: [...] }`.
  - Hard timeout via spawnSync `timeout` option. On timeout (ETIMEDOUT): return `{ type, pass: true, reason: 'timeout' }` (fail-open).
  - Wrap everything in try/catch, fail-open on any error.

**done_conditions evaluator:**
- `evaluateCondition(cwd, condition, timeoutMs)` -- evaluates a single condition. Switch on `condition.type`:
  - `file_exists`: `fs.existsSync(path.join(cwd, condition.path))` -- return `{ pass: boolean, type }`.
  - `test_pass`: `spawnSync('node', ['--test', condition.pattern], { cwd, timeout: timeoutMs || 30000, encoding: 'utf8' })` -- return `{ pass: status===0, type, output: last 500 chars of stdout }`. Use `node --test` (project convention).
  - `lint_clean`: check eslint exists, run on `condition.files`. If no linter, return pass=true with reason.
  - `typecheck_pass`: check tsc exists at `node_modules/.bin/tsc`, run `tsc --noEmit`. If no tsc, return pass=true with reason.
  - `command_pass`: split `condition.command` on spaces, run via spawnSync. Return pass based on exit code.
  - Default/unknown type: return `{ pass: true, type, reason: 'unknown condition type' }`.
  - Wrap entire function in try/catch, fail-open: return `{ pass: true, type, reason: 'error: ' + e.message }`.

- `evaluateAllConditions(cwd, conditions, timeoutMs)` -- maps over conditions array, returns `{ all_pass: boolean, results: [...] }`. If conditions is empty/null/undefined, return `{ all_pass: true, results: [] }`.

**CLI interface (if run as main):**
- `run-checks --cwd <dir> --files <comma-separated>` -- runs checks and prints JSON result.
- `evaluate --cwd <dir> --conditions <json-string>` -- evaluates done_conditions and prints JSON result.
- Fail-open on CLI errors: stderr warning + exit 0.

**Module exports:** `{ runChecks, shouldTriggerVerification, evaluateCondition, evaluateAllConditions, getVerifyState, saveVerifyState, initVerifyState }`

  <files>bin/execution-progress.cjs, bin/execution-progress.test.cjs</files>
  <action>
Extend bin/execution-progress.cjs to support done_conditions in the task schema.

**Changes to initProgress:**
- Accept optional `doneConditions` parameter in the options object: `initProgress(cwd, { planFile, totalTasks, taskNames, doneConditions })`.
- `doneConditions` is an array of arrays: `doneConditions[i]` is the conditions array for task `i`. If not provided or if `doneConditions[i]` is undefined, default to `[]`.
- Add `done_conditions` field to each task in the tasks array: `task.done_conditions = doneConditions[i] || []`.

**Changes to completeTask:**
- Before marking task complete, if `task.done_conditions.length > 0`, require the `continuous-verify.cjs` module and call `evaluateAllConditions(cwd, task.done_conditions)`.
- If `all_pass` is false, do NOT mark the task as complete. Instead set `task.status = 'blocked'` and `task.block_reason = 'done_conditions_failed'` and `task.condition_results = results`. Return the progress object (caller can inspect).
- If `all_pass` is true (or no done_conditions), proceed as before: mark complete.
- Use try/catch around the evaluateAllConditions call. On error, fail-open: proceed with marking complete (do not block on evaluation failure).
- Use dual-path require pattern for continuous-verify.cjs (try relative path, then installed path): `try { return require(path.join(__dirname, 'continuous-verify.cjs')); } catch { return null; }`. If module not found, skip evaluation (fail-open).

**New export:** Add `BLOCKED_STATUS = 'blocked'` constant to exports for test assertions.

**Extend tests in bin/execution-progress.test.cjs:**
- Add new `describe('done_conditions')` block:
  - `initProgress with doneConditions stores conditions on tasks`: init with `doneConditions: [[{ type: 'file_exists', path: 'foo.txt' }], []]`, verify `tasks[0].done_conditions.length === 1` and `tasks[1].done_conditions.length === 0`.
  - `initProgress without doneConditions defaults to empty arrays`: init without doneConditions, verify each task has `done_conditions: []`.
  - `completeTask with passing done_conditions marks complete`: create a temp file, init with `[{ type: 'file_exists', path: '<temp-file-relative>' }]`, complete task, verify status === 'complete'.
  - `completeTask with failing done_conditions sets blocked status`: init with `[{ type: 'file_exists', path: 'nonexistent-file-xyz.txt' }]`, complete task, verify `task.status === 'blocked'` and `task.block_reason === 'done_conditions_failed'`.

Register bin/continuous-verify.test.cjs in package.json `test:ci` script. Add it after the existing `bin/learning-extractor.test.cjs` entry in the `node --test` command list.
