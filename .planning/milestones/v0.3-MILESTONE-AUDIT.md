## Integration Findings

### Critical: runner field not propagated (EXEC-01, EXEC-02)

`cmdMaintainTestsBatch` does not include a `runner` field in `batches[N]`. `cmdMaintainTestsRunBatch` falls back to `'jest'` when absent. On any Playwright or Pytest project, run-batch will invoke `npx jest` on every file — wrong framework, 100% failure rate, all 3 flakiness re-runs are wasted.

**Fix:** One-line addition in `cmdMaintainTestsBatch` at ~line 5508 — propagate `runners[0]` (or the `--runner` flag value) into each batch entry as `runner`.

### Medium: No --batch-index flag (future ITER-01 blocker)

`cmdMaintainTestsRunBatch` reads `batches[0]` from a full manifest. The Phase 20 orchestrator needs to iterate N batches. Either `run-batch` needs `--batch-index N`, or the orchestrator must extract batch objects manually before passing them.

### Medium: resume-work routing table missing maintain-tests rows (INTG-02)

`resume-project.md` has no routing rows for maintain-tests sub-activities. An interrupted `fix-tests` session will not route to recovery — it will appear orphaned in resume-work.

### Low: --output-file dual-mode contract undocumented

With `--output-file`, run-batch returns `{ written, path, executed_count }` to stdout (not the full results). Without it, full results go to stdout. Phase 20's orchestrator must know which mode to use.

### Ready: Circuit breaker disable/enable (INTG-01)

`npx qgsd --disable-breaker` and `--enable-breaker` fully implemented in `bin/install.js`. Phase 20 can invoke directly — no gap.

### Pre-compliant: INTG-03

`maintain-tests` is not in `DEFAULT_CONFIG.quorum_commands`. Phase 20 simply must not add it.
