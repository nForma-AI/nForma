# Phase v0.24-02: Dispatch Reliability - Verification

## Phase Goal

> The dispatch list is built from live health data rather than static configuration -- dead providers are excluded, cooling-down slots are skipped, and the most reliable slots are tried first

## Must-Haves Verification

### SC1: Health Probe (DISP-01) -- PASSED

**Criterion:** Before building the dispatch list, `qgsd-prompt.js` runs a fast health probe (<3s) per provider and excludes all slots on providers that fail the probe -- observable via dispatch log showing "excluded: provider X down"

**Evidence:**
- `triggerHealthProbe()` function defined at line 164 of `hooks/qgsd-prompt.js`
- Called at line 417 before `getDownProviderSlots()` in the dispatch flow
- Uses `spawnSync('node', [checkPath, '--json'], { timeout: 3000 })` -- 3s timeout
- `getDownProviderSlots()` reads `~/.claude/qgsd-provider-cache.json` and excludes all slots backed by DOWN providers
- Skip note emits: `SKIP (PROVIDER DOWN): [slots] -- entire provider unreachable, skip all backed slots.`

**Verdict:** PASSED

### SC2: Availability Window (DISP-02) -- PASSED

**Criterion:** Slots whose `available_at` timestamp in the scoreboard is in the future are excluded from the dispatch list -- a slot marked unavailable until 14:30 is not dispatched at 14:25

**Evidence:**
- `getAvailableSlots(slots, cwd)` function defined at line 179 of `hooks/qgsd-prompt.js`
- Reads `.planning/quorum-scoreboard.json` availability section (NOT slots section)
- Compares `available_at_iso` timestamp against `Date.now()` -- future timestamps excluded
- Handles NaN dates via explicit `isNaN(ts)` check (fail-open)
- Called at line 426: `cappedSlots = getAvailableSlots(cappedSlots, cwd)`
- Debug log: `[qgsd-dispatch] AVAILABILITY EXCLUDE: ${slot} -- available_at_iso=... is in the future`
- Skip note: `SKIP (COOLING DOWN): [slots] -- available_at in future, skipping.`
- Unit tests verify filtering with future/past timestamps, malformed dates, missing scoreboard

**Verdict:** PASSED

### SC3: Success Rate Ordering (DISP-03) -- PASSED

**Criterion:** The dispatch list is ordered by descending recent success rate from scoreboard slot stats -- the slot with the highest success rate appears first in DISPATCH_LIST

**Evidence:**
- `sortBySuccessRate(slots, cwd)` function defined at line 215 of `hooks/qgsd-prompt.js`
- Reads `.planning/quorum-scoreboard.json` slots section with composite key aggregation
- Extracts slot name via `key.split(':')[0]` from composite keys (e.g., `claude-1:deepseek-ai/DeepSeek-V3.2`)
- Aggregates tp/fn across all model entries per slot name
- Computes `rate = totalTp / (totalTp + totalFn)` with 0.5 default for unknown slots
- Sorts by descending rate (highest success rate first)
- Called at line 432: `cappedSlots = sortBySuccessRate(cappedSlots, cwd)`
- Debug log: `[qgsd-dispatch] SUCCESS RATE: ${slot} = ${rate} (tp=N, fn=N, models=N)`
- Debug log: `[qgsd-dispatch] DISPATCH ORDER: [ordered slots]`
- Unit tests verify sorting, aggregation, default rates, fail-open

**Verdict:** PASSED

### SC4: Graceful Fallback -- PASSED

**Criterion:** When all preferred slots are excluded (all providers down or cooling), dispatch falls back gracefully to any remaining slot rather than failing with zero candidates

**Evidence:**
- Graceful fallback guard at line 435: `if (cappedSlots.length === 0 && orderedSlots.length > 0)`
- First tries non-DOWN-provider slots: `orderedSlots.filter(s => !skipSet.has(s.slot))`
- Last resort: `cappedSlots = [orderedSlots[0]]` -- any slot at all
- Debug log: `[qgsd-dispatch] FALLBACK: all slots filtered, restored ${slot}`
- Unit test confirms `gracefulFallback()` never returns empty array

**Verdict:** PASSED
