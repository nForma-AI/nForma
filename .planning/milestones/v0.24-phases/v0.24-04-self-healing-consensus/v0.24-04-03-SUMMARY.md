### Task 1: Add suggestMaxDeliberation and applyMaxDeliberationUpdate to verify-quorum-health.cjs

**File:** `/Users/jonathanborduas/code/QGSD/bin/verify-quorum-health.cjs` (268 lines)

**Result:** ✓ PASS

**Key Changes:**

1. **Extracted pure functions:**
   - `suggestMaxDeliberation(pPerRound, targetConfidence)` — Computes recommended maxDeliberation using geometric distribution: `k = ceil(log(1 - targetConfidence) / log(1 - pPerRound))`
   - `computeRates(slot, rounds)` — Extracted from main flow; calculates per-agent tp_rate and unavail_rate
   - `pMajorityExternal(agents)` — Extracted; computes P(≥2 of 4 external agents approve)
   - `applyMaxDeliberationUpdate(newValue, options)` — Atomically updates machine.ts and config.json with rollback safety

2. **Refactored CLI structure:**
   - Wrapped top-level code in `main()` function
   - Added `require.main === module` guard to prevent CLI execution on module import
   - Added `--auto-apply` flag support to bypass interactive approval

3. **Atomic update pattern:**
   - Backs up original files before any modifications
   - Updates machine.ts maxDeliberation field (regex: `/maxDeliberation:\s*\d+/`)
   - Updates config.json workflow.maxDeliberation if file exists
   - Regenerates formal specs via `node bin/generate-formal-specs.cjs`
   - Verifies spec sync via `node bin/check-spec-sync.cjs`
   - Rolls back all files on any failure

4. **Module exports:**
   ```javascript
   module.exports = { suggestMaxDeliberation, applyMaxDeliberationUpdate, computeRates, pMajorityExternal };
   ```
