**File:** `/Users/jonathanborduas/code/QGSD/bin/quorum-consensus-gate.cjs` (150 lines, +91)

**Result:** ✓ PASS

**Implementation:**

1. **readEarlyEscalationThreshold(configPaths)** — reads config with fail-open pattern
   - Searches configPaths in order (defaults to .planning/config.json, .planning/qgsd.json)
   - Returns value from workflow.early_escalation_threshold if valid (0 < val <= 1.0)
   - Falls back to 0.10 on missing/malformed/invalid files
   - Fail-open on JSON parse errors — no crashes

2. **computeEarlyEscalation(slotRates, minQuorum, remainingRounds, threshold)** — computes early escalation decision
   - Uses Poisson binomial CDF to compute P(at least minQuorum succeed in 1 round)
   - Extends to remaining rounds: P(consensus within k) = 1 - (1 - pPerRound)^k
   - Returns { shouldEscalate, probability, threshold, remainingRounds, pPerRound }
   - Handles edge case: remainingRounds <= 0 → escalate immediately

3. **CLI routing with --remaining-rounds** — backward compatible
   - With --remaining-rounds=N: calls computeEarlyEscalation, exits 1 for escalate / 0 for continue
   - Without --remaining-rounds: calls checkConsensusGate (original behavior), exits 0/1 for proceed/defer
   - Reads scoreboard for slotRates; falls back to priors if missing

4. **Config field** — early_escalation_threshold in workflow section (default 0.10)

```

  Synthesize and deliberate round...

  const result = node quorum-consensus-gate.cjs --remaining-rounds=$remainingRounds
```
