<task type="auto">
  <name>Task 1: Implement bin/quorum-slot-dispatch.cjs — prompt construction, output parsing, and CLI wrapper</name>
  <files>bin/quorum-slot-dispatch.cjs</files>
  <action>
Create bin/quorum-slot-dispatch.cjs with the following structure:

**Argument parsing (use getArg pattern from call-quorum-slot.cjs):**
- `--slot` (required) — slot name
- `--round` (required) — round number (integer)
- `--timeout` (optional) — timeout in ms for CQS subprocess
- `--cwd` (optional) — working directory for CQS subprocess
- `--mode` (required) — "A" or "B"
- `--question` (required) — question text
- `--artifact-path` (optional) — path to artifact file
- `--review-context` (optional) — one-sentence framing for artifact evaluation
- `--prior-positions-file` (optional) — path to tempfile containing prior_positions YAML
- `--traces-file` (optional) — path to tempfile containing execution traces (Mode B)
- `--request-improvements` (optional, boolean flag) — when present, include improvements instructions

**Exported pure functions (for testability — these are also used internally by main()):**

1. `buildModeAPrompt({ round, repoDir, question, artifactPath, reviewContext, priorPositions, requestImprovements })`
   - Returns string matching the EXACT Mode A template from agents/qgsd-quorum-slot-worker.md Step 2
   - Round 1 path: includes "IMPORTANT: Before answering..." instructions
   - Round 2+ path: includes prior_positions, "do you maintain your answer or revise it?"
   - Conditional sections: artifact (=== Artifact ===), review_context, review_context reminder (R2+), request_improvements
   - Citations instruction always appended

2. `buildModeBPrompt({ round, repoDir, question, artifactPath, reviewContext, traces, priorPositions })`
   - Returns string matching the EXACT Mode B template from agents/qgsd-quorum-slot-worker.md Step 2
   - Always includes "=== EXECUTION TRACES ===" section with traces content
   - Round 2+ includes prior_positions
   - Always ends with "verdict: APPROVE | REJECT | FLAG" format instructions

3. `parseVerdict(rawOutput, mode)`
   - Mode A: return first 500 chars of rawOutput as free-form position (simple truncation)
   - Mode B: extract "verdict: APPROVE|REJECT|FLAG" via regex; default to "FLAG" if not found

4. `parseReasoning(rawOutput)`
   - Extract "reasoning: ..." line via regex
   - Fallback: first 400 chars of rawOutput

5. `parseCitations(rawOutput)`
   - Extract "citations: |" block (indented content)
   - Return null if absent

6. `parseImprovements(rawOutput)` — MOVE from bin/gsd-quorum-slot-worker-improvements.test.cjs
   - Copy the exact implementation (lines 21-82 of the test file) including stripQuotes helper
   - This becomes the canonical location

7. `emitResultBlock({ slot, round, verdict, reasoning, citations, improvements, rawOutput, isUnavail, unavailMessage })`
   - Produces YAML-formatted result block matching agents/qgsd-quorum-slot-worker.md Step 4
   - Returns string (does NOT write to stdout directly — main() handles that)

**main() function (CLI entry point):**
1. Parse CLI args
2. Read tempfiles if --prior-positions-file or --traces-file provided (fs.readFileSync)
3. Build prompt: mode === 'A' ? buildModeAPrompt(...) : buildModeBPrompt(...)
4. Spawn call-quorum-slot.cjs as child process: `spawn(process.execPath, [cqsPath, '--slot', slot, '--timeout', timeout, '--cwd', cwd], { stdio: ['pipe', 'pipe', 'pipe'] })`
5. Write prompt to child.stdin, end stdin
6. Capture stdout and stderr, wait for close
7. If exit code !== 0 or stdout contains 'TIMEOUT': emit UNAVAIL result block
8. Otherwise: parse output (parseVerdict, parseReasoning, parseCitations, parseImprovements) and emit result block
9. Write result to process.stdout

**Module exports:** Export all pure functions so tests can import them:
```javascript
if (typeof module !== 'undefined') {
  module.exports = { buildModeAPrompt, buildModeBPrompt, parseVerdict, parseReasoning, parseCitations, parseImprovements, emitResultBlock, stripQuotes };
}
```

Only call main() when run directly (not when required as module):
```javascript
if (require.main === module) {
  main().catch(err => { process.stderr.write(`[quorum-slot-dispatch] Fatal: ${err.message}\n`); process.exit(1); });
}
```

**Critical details from research:**
- Use spawn (NOT exec) — handles large trace outputs safely
- CQS stdin: `child.stdin.write(prompt, 'utf8'); child.stdin.end()` — CQS already trims with .trim()
- Note: traces up to 35KB are safe within Node.js stdin pipe — CQS .trim() is idempotent. Mode B dispatch with large execution output will not break shell limits because we use spawn (not exec) and write to child.stdin.
- TIMEOUT detection: check both exit code !== 0 AND stdout.includes('TIMEOUT')
- Raw output truncation: first 5000 chars in result block's raw: field
- UNAVAIL block: first 500 chars in unavail_message
- IMPORTANT: buildModeAPrompt and buildModeBPrompt are intentionally separate functions with different conditional branches. Do NOT merge them — Mode A handles Round 1 vs Round 2+ branching with question-answer format; Mode B always includes EXECUTION TRACES section with verdict format.
