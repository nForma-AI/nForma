## Tasks Executed

### Task 1: Extend scoreboard with delivery stats and flakiness scoring

**Status:** COMPLETE (verified from git commit 2d6fb920)

Implementation includes:

1. **emptyData() extended** with delivery_stats object:
   - `total_rounds`: 0
   - `target_vote_count`: 3
   - `achieved_by_outcome`: {} (populated at runtime)

2. **computeDeliveryStats(data) function** (lines 278-326):
   - Iterates over data.rounds, counts valid votes per round
   - countValidVotes helper excludes UNAVAIL, TIMEOUT, empty string, null/undefined
   - Computes percentages: `pct = parseFloat(((count / total) * 100).toFixed(1))`
   - Ensures pct is JSON number (not string from .toFixed())
   - Handles empty rounds gracefully (total_rounds: 0, empty achieved_by_outcome)
   - Returns data for chaining

3. **computeFlakiness(data, windowSize=10) function** (lines 337-425):
   - Builds set of all unique slot names from all rounds
   - Handles both model-keyed (e.g., 'claude') and slot-keyed (e.g., 'gemini-1:model-id') votes
   - Collects verdictWindow: last 10 rounds where slot has a vote
   - Counts failures: UNAVAIL, TIMEOUT, empty string, falsy values
   - Computes flakiness_score = failures / window.length (0.0 = reliable, 1.0 = always fails)
   - Stores as number via parseFloat(...toFixed(2))
   - Stores in data.slots[key].flakiness_score and recent_verdicts
   - Returns data for chaining

4. **Integration into merge-wave** (lines 917-918):
   - After recomputeStats and recomputeSlots
   - Calls: computeDeliveryStats(data); computeFlakiness(data);
   - Automatic recomputation every merge-wave — no separate CLI command needed

5. **Export block** (lines 1085-1087):
   - Guard pattern: `if (typeof module !== 'undefined')`
   - Exports: computeDeliveryStats, computeFlakiness, emptyData
   - Allows testing without interfering with CLI script execution

### Task 2: Add flakiness-aware dispatch ordering to qgsd-prompt.js

**Status:** COMPLETE (verified from source code lines 215-255)

Implementation includes:

1. **sortBySuccessRate function** extended with flakiness scoring (lines 215-255):
   - getFlakiness helper reads flakiness_score from scoreboard.slots
   - Handles both model-keyed and slot-keyed entries
   - Filters by slotName (before ':' in composite keys)
   - Returns max flakiness across any model for the slot
   - Fails open: missing flakiness → defaults to 0 (treat as reliable)

2. **Updated sort logic** (lines 245-249):
   - Primary: flakiness ascending (lower = more reliable = dispatched first)
   - Secondary: success rate descending (higher = better = dispatched first)
   - Tiebreaker prevents ambiguity when multiple slots have same flakiness

3. **Enhanced stderr logging** (lines 251-252):
   - Includes flakiness values with 2-decimal precision for debugging
   - Format: `slot(f=0.30,r=0.850)` showing both metrics

4. **Function name unchanged**:
   - Backward compatible: sortBySuccessRate name retained for existing code references
   - Actually sorts by flakiness first, success rate second
