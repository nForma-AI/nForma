Implement retry-with-exponential-backoff in call-quorum-slot.cjs so that transient provider
failures (timeouts, network errors) are retried before recording UNAVAIL. This directly
addresses FAIL-01: "call-quorum-slot.cjs retries up to 2 times with exponential backoff
(1s, 3s) before recording UNAVAIL in quorum-failures.json."

The retry logic must NOT interfere with the existing OAuth rotation path. For non-OAuth slots,
the retry wraps the single runSubprocess/runHttp call. For OAuth slots, the retry is applied
per-attempt INSIDE the OAuth rotation loop (each oauth attempt gets retry protection).

<feature>
  <name>Retry with Exponential Backoff for Slot Calls</name>
  <files>bin/call-quorum-slot.cjs</files>
  <behavior>
    Add a retryWithBackoff() function to call-quorum-slot.cjs that wraps slot dispatch calls
    with transient-failure retry logic.

    RETRY BEHAVIOR:
    - MAX_RETRIES = 2 (total attempts = 3: initial + 2 retries)
    - BACKOFF_DELAYS = [1000, 3000] (1s for first retry, 3s for second retry)
    - Retryable errors: TIMEOUT, network/connection errors, non-zero exit codes with transient patterns
    - Non-retryable errors (fail immediately): CLI_SYNTAX (usage/unknown flag), spawn errors
    - Each retry attempt logs to stderr: '[call-quorum-slot] retry N/2 for slot <name> after <delay>ms'

    FUNCTION SIGNATURE:
    async function retryWithBackoff(fn, slotName, maxRetries = 2, delays = [1000, 3000])

    Where fn is an async function that either resolves with output string or rejects with Error.

    CLASSIFICATION LOGIC (isRetryable):
    - Error message contains 'TIMEOUT' → retryable
    - Error message contains 'ECONNREFUSED', 'ENOTFOUND', 'ECONNRESET', 'ETIMEDOUT' → retryable
    - Error message contains 'spawn error' → NOT retryable
    - Error message matches CLI_SYNTAX pattern (usage|unknown flag|unknown option|invalid flag|unrecognized) → NOT retryable
    - Default (unknown error pattern) → retryable (fail-open: retry when uncertain)

    INTEGRATION INTO main():
    In the main() function, wrap the dispatch calls:

    For non-OAuth subprocess slots:
      result = await retryWithBackoff(() => runSubprocess(provider, prompt, effectiveTimeout), slot);

    For OAuth rotation subprocess slots:
      The oauth rotation loop already exists. Inside runSubprocessWithRotation, wrap the inner
      runSubprocess call:
        const out = await retryWithBackoff(() => runSubprocess(provider, prompt, timeoutMs), provider.name);
      This means each OAuth rotation attempt gets retry protection.

    For HTTP slots:
      result = await retryWithBackoff(() => runHttp(provider, prompt, effectiveTimeout), slot);

    FAILURE RECORDING:
    writeFailureLog() and appendTokenSentinel() are called ONLY after all retries are exhausted
    (in the catch block of main, which is unchanged). The retry wrapper throws after exhausting
    retries, so the existing error handling in main() catches the final failure.

    SLEEP IMPLEMENTATION:
    function sleep(ms) { return new Promise(resolve => setTimeout(resolve, ms)); }
    Place this near the top of the file with other utility functions.
  </behavior>
  <implementation>
    1. Add sleep() utility function after the existing utility functions (findProjectRoot, writeFailureLog).

    2. Add isRetryable(error) function that classifies errors:
       - Returns false for CLI_SYNTAX patterns and spawn errors
       - Returns true for TIMEOUT, network errors, and unknown errors (fail-open)

    3. Add retryWithBackoff(fn, slotName, maxRetries, delays) function:
       - Loop from attempt 0 to maxRetries
       - On attempt 0, just call fn()
       - On subsequent attempts, log retry to stderr, sleep(delays[attempt-1]), then call fn()
       - If fn() succeeds, return result
       - If fn() fails and isRetryable is false, throw immediately
       - If fn() fails and isRetryable is true and more retries remain, continue loop
       - If fn() fails and no more retries, throw the last error

    4. In main(), wrap the dispatch calls:
       - Line ~383 (non-OAuth subprocess): wrap with retryWithBackoff
       - Line ~385 (OAuth subprocess): modify runSubprocessWithRotation to use retryWithBackoff internally
       - Line ~387 (HTTP): wrap with retryWithBackoff

    5. For the OAuth path, modify runSubprocessWithRotation to wrap the inner call:
       Change: const out = await runSubprocess(provider, prompt, timeoutMs);
       To: const out = await retryWithBackoff(() => runSubprocess(provider, prompt, timeoutMs), provider.name);

    IMPORTANT: Do NOT change the exit code behavior, writeFailureLog, or appendTokenSentinel calls.
    The retry is transparent to the caller — it either succeeds or throws the final error.

    Commit message: feat(v0.24-01-02): add retry with exponential backoff to call-quorum-slot.cjs (FAIL-01)
  </implementation>
</feature>
