- `fireworks`: claude-4, claude-6 (Kimi-K2, GLM-5 via Fireworks)

**Dispatch Skip Logic:**
1. Reads provider health cache at ~/.claude/qgsd-provider-cache.json
2. For each DOWN provider (healthy=false and within TTL):
   - Extracts hostname from baseUrl (e.g., api.akashml.com → akashml)
   - Finds all slots with matching provider field
   - Adds those slots to skip list
3. Skipped slots are removed from cappedSlots before building dynamic steps
4. Generates "SKIP (PROVIDER DOWN): [slot-list]" note in quorum instructions
5. Fail-open: any error returns empty skip list (dispatch proceeds normally)

**TTL Handling:**
- Healthy cache entries: 180s TTL
- Unhealthy cache entries: 300s TTL
- Expired entries are ignored (no cascading skips from stale data)
