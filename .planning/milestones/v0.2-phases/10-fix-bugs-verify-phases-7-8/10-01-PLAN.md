<task type="auto">
  <name>Task 2: Fix --reset-breaker path resolution to use git root (RECV-01)</name>
  <files>bin/install.js</files>
  <action>
In bin/install.js, locate the --reset-breaker handler at lines 2002-2013:

```javascript
if (hasResetBreaker) {
  const stateFile = path.join(process.cwd(), '.claude', 'circuit-breaker-state.json');
  ...
}
```

The bug: process.cwd() resolves relative to wherever the user invoked the command. If invoked from a subdirectory (e.g., src/), the state file path is wrong and the command reports "no active state" without actually clearing the breaker. The hook writes the state file relative to gitRoot (via getGitRoot(cwd) in qgsd-circuit-breaker.js line 149).

Fix: Replace the hard-coded process.cwd() path with a git-root-aware resolution:

```javascript
if (hasResetBreaker) {
  const { spawnSync } = require('child_process');
  const gitResult = spawnSync('git', ['rev-parse', '--show-toplevel'], {
    cwd: process.cwd(),
    encoding: 'utf8',
    timeout: 5000,
  });
  const projectRoot = (gitResult.status === 0 && !gitResult.error)
    ? gitResult.stdout.trim()
    : process.cwd();
  const stateFile = path.join(projectRoot, '.claude', 'circuit-breaker-state.json');
  if (fs.existsSync(stateFile)) {
    fs.rmSync(stateFile);
    console.log(`  ${green}✓${reset} Circuit breaker state cleared. Claude can resume Bash execution.`);
    console.log(`    Removed: ${stateFile.replace(os.homedir(), '~')}`);
  } else {
    console.log(`  ${dim}No active circuit breaker state found at ${stateFile.replace(os.homedir(), '~')}${reset}`);
  }
  process.exit(0);
}
```

Note: `spawnSync` is already available in the Node.js stdlib and is used in the hook itself. Check whether `require('child_process')` is already at the top of install.js — if so, only add the variable, not the require. The `const { spawnSync } = require('child_process');` line inside the if block is self-contained and safe even if spawnSync is not used elsewhere; alternatively, hoist it to a const outside the block if cleaner.

Fallback behavior: if git rev-parse fails (not a git repo or git unavailable), falls back to process.cwd() — identical to current behavior, no regression.

Replace with a two-step backfill that handles both missing top-level key AND missing individual sub-keys:

```javascript
// INST-10: Add missing circuit_breaker block or missing sub-keys without touching existing user values
if (!existingConfig.circuit_breaker) {
  existingConfig.circuit_breaker = { oscillation_depth: 3, commit_window: 6 };
  fs.writeFileSync(qgsdConfigPath, JSON.stringify(existingConfig, null, 2) + '\n', 'utf8');
  console.log(`  ${green}✓${reset} Added circuit_breaker config block to qgsd.json`);
} else {
  // Backfill individual missing sub-keys without touching values the user has set
  let subKeyAdded = false;
  if (existingConfig.circuit_breaker.oscillation_depth === undefined) {
    existingConfig.circuit_breaker.oscillation_depth = 3;
    subKeyAdded = true;
  }
  if (existingConfig.circuit_breaker.commit_window === undefined) {
    existingConfig.circuit_breaker.commit_window = 6;
    subKeyAdded = true;
  }
  if (subKeyAdded) {
    fs.writeFileSync(qgsdConfigPath, JSON.stringify(existingConfig, null, 2) + '\n', 'utf8');
    console.log(`  ${green}✓${reset} Added missing circuit_breaker sub-keys to qgsd.json`);
  }
}
```

This preserves user-modified values (the user's oscillation_depth: 5 stays untouched) while backfilling only the absent sub-keys. The check is === undefined, not falsy, so a user-set value of 0 would still be preserved (validateConfig() will warn at runtime — correct behavior).
