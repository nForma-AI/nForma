  <feature>
    <name>Two-layer config load with validation and stderr warnings</name>
    <files>hooks/config-loader.js, hooks/config-loader.test.js</files>
    <behavior>
      loadConfig(projectDir) behavior:
      - No files exist → returns DEFAULT_CONFIG, warns "No qgsd.json found ... using hardcoded defaults"
      - Only global exists and is valid → returns { ...DEFAULT_CONFIG, ...global }
      - Only project exists and is valid → returns { ...DEFAULT_CONFIG, ...project }
      - Both exist and are valid → returns { ...DEFAULT_CONFIG, ...global, ...project } (project wins)
      - Global exists but malformed → warns about global layer, falls back to DEFAULT_CONFIG for that layer, continues with project layer if present
      - Project exists but malformed → warns about project layer, uses global (or default) config without project overlay
      - Both malformed → warns for both, returns DEFAULT_CONFIG
      - Global has invalid quorum_commands type (not array) → validateConfig() corrects to DEFAULT_CONFIG.quorum_commands, warns
      - Global has invalid required_models type (not object) → validateConfig() corrects to DEFAULT_CONFIG.required_models, warns
      - Global has invalid fail_mode value → validateConfig() corrects to "open", warns
      - All warnings write to process.stderr — no stdout output ever

      Cases (input → expected):
      - loadConfig('/no/project') with no files → returns DEFAULT_CONFIG object
      - loadConfig() with global only: {quorum_commands: ['custom-cmd'], fail_mode: 'open', required_models: DEFAULT_CONFIG.required_models} → merged config with custom-cmd in quorum_commands
      - loadConfig('/project') with global + project: project has {fail_mode: 'closed'} → merged config has fail_mode: 'closed'
      - Merge strategy is SHALLOW REPLACE (standard JS spread): if project config has required_models, it entirely replaces global required_models. There is no deep merge or union of entries. This is intentional — project config is meant to fully override global for any key present.
      - loadConfig() with malformed global → stderr contains "[qgsd] WARNING:", returns DEFAULT_CONFIG
      - validateConfig() with quorum_commands: "string" (not array) → corrects to DEFAULT_CONFIG.quorum_commands
    </behavior>
    <implementation>
      Create hooks/config-loader.js with:

      1. DEFAULT_CONFIG constant (copy from qgsd-stop.js — the canonical default):
         {
           quorum_commands: ['plan-phase', 'new-project', 'new-milestone', 'discuss-phase', 'verify-work', 'research-phase'],
           fail_mode: 'open',
           required_models: {
             codex:    { tool_prefix: 'mcp__codex-cli__',  required: true },
             gemini:   { tool_prefix: 'mcp__gemini-cli__', required: true },
             opencode: { tool_prefix: 'mcp__opencode__',   required: true },
           },
         }

      2. readConfigFile(filePath) → returns parsed object or null. On missing file: returns null silently. On malformed JSON: calls process.stderr.write("[qgsd] WARNING: Malformed config at " + filePath + ": " + e.message + "\n"), returns null.

      3. validateConfig(config) → validates in-place, emits warnings for invalid fields, returns config:
         - if !Array.isArray(config.quorum_commands): warn, set to DEFAULT_CONFIG.quorum_commands
         - if typeof config.required_models !== 'object' || config.required_models === null: warn, set to DEFAULT_CONFIG.required_models
         - if !['open', 'closed'].includes(config.fail_mode): warn, set to 'open'
         All warnings: process.stderr.write("[qgsd] WARNING: " + message + "\n")

      4. loadConfig(projectDir) → main export:
         - globalPath = path.join(os.homedir(), '.claude', 'qgsd.json')
         - projectPath = path.join(projectDir || process.cwd(), '.claude', 'qgsd.json')
         - globalObj = readConfigFile(globalPath)
         - projectObj = readConfigFile(projectPath)
         - if (!globalObj && !projectObj): process.stderr.write warning about no config found, return { ...DEFAULT_CONFIG }
         - else: config = { ...DEFAULT_CONFIG, ...(globalObj || {}), ...(projectObj || {}) }
         - validateConfig(config)
         - return config

      5. module.exports = { loadConfig, DEFAULT_CONFIG }

      CRITICAL: Never write to stdout. Stdout is the hook decision channel in qgsd-stop.js.
      Use process.stderr.write() exclusively. Never use console.log().
      All paths use os.homedir() and path.join() — no ~ expansion.

      Test file hooks/config-loader.test.js:
      - Use node:test + node:assert/strict
      - Tests write temp config files to os.tmpdir() and pass projectDir to loadConfig()
      - Test cases cover all behavior cases listed above
      - Clean up temp files in finally blocks
      - Run: node --test hooks/config-loader.test.js
    </implementation>
