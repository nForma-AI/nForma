# Summary: v0.19-02-02 — Cold-Start Governance in run-prism.cjs

## What Was Built

Extended bin/run-prism.cjs with policy-driven cold-start governance and observation_window metadata. PRISM calibration checks will no longer fire false alarms on fresh installs.

## Key Files

### Modified
- `bin/run-prism.cjs` — Added:
  - `require('./read-policy.cjs')` at top
  - Policy read via `readPolicy(policyPath)` after scoreboardPath; fail-fast with clear error message if policy.yaml missing
  - `computeColdStartState(pol, sbPath, crPath)` function: counts CI runs (ndjson lines) + quorum rounds (scoreboard rounds[]) + days since first round timestamp; `inColdStart = !allThresholdsMet`
  - Cold-start detection block before Build argument list: logs which thresholds are unmet
  - `finalResult` override: result=fail → result=warn when `!passed && coldStartState.inColdStart` (CALIB-02)
  - `observation_window` metadata on ALL writeCheckResult calls: window_start/end (ISO 8601), n_rounds, n_events (CALIB-03)
  - `process.exit(passed ? 0 : (finalResult === 'warn' ? 0 : 1))` — warn exits 0
  - Early-exit calls (binary/model not found, policy load fail, launch fail) include observation_window with zero values
- `bin/run-prism.test.cjs` — Appended 7 new tests (CALIB-02, CALIB-03):
  1. all thresholds unmet → cold-start active in stderr
  2. min_ci_runs boundary: 4 lines < 5 stays in cold-start
  3. min_quorum_rounds boundary: 9 rounds < 10 stays in cold-start
  4. all thresholds met (10 rounds + 5 CI runs + 2 days) → no cold-start message
  5. warn suppression context: cold-start detected when PRISM would fail
  6. observation_window metadata present in NDJSON entry
  7. observation_window timestamps are ISO 8601 format

## Test Results

## Decisions

- Policy path read from `__dirname + '/../formal/policy.yaml'` (absolute, not cwd-relative) — ensures consistent behavior regardless of working directory
- computeColdStartState reads scoreboard only once (for both tp_rate injection and cold-start detection); avoids double-parse
- `inColdStart = !allThresholdsMet` explicit negation pattern (per CALIB pitfall 2 in plan)
- `daysSinceFirst` uses floating-point: `>= min_days` (boundary: exactly min_days = OUT of cold-start)
- `observation_window` is a fresh object literal per call (not mutated across calls — per CALIB pitfall 4)
