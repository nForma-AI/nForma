### Task 2 — GREEN implementation

**formal/prism/mcp-availability.pm** (new, 75 lines):
DTMC model with 4 independent two-state Markov chain modules (codex, gemini, opencode, copilot).
- Per-slot availability constants (codex_1_avail, gemini_1_avail, opencode_1_avail, copilot_1_avail) default to 0.85 prior; overridden at runtime by run-prism.cjs
- Labels: `min_quorum_available` (≥1 slot available), `total_outage` (all 0), `majority_available` (≥2)
- Each module has two guard transitions to model stationary availability

**formal/prism/mcp-availability.props** (new):
3 steady-state property checks (S=? queries): min_quorum_available, total_outage, majority_available

**bin/run-prism.cjs**:
- Added `readMCPAvailabilityRates(sbPath?)` function:
  - Reads quorum-scoreboard.json from sbPath or default cwd path
  - Computes per-slot availability = 1.0 - (unavail_count / total_count) excluding 'claude'
  - Returns rate map or null on empty/missing/malformed scoreboard
- Added `--model mcp-availability` flag handling:
  - Selects mcp-availability.pm and mcp-availability.props instead of quorum.pm/quorum.props
  - Strips --model <value> from extraArgs before forwarding to PRISM
  - Injects per-slot rates as -const <slot>_avail=<rate> flags (e.g. -const codex_1_avail=0.8)
  - Logs scoreboard rates or warns about missing rates (priors used)
  - Emits model='mcp-availability' and per_slot_rates in writeCheckResult metadata
  - Existing quorum.pm path completely unchanged (CALIB regression prevention)
- module.exports: adds readMCPAvailabilityRates export

**bin/run-prism.test.cjs**:
