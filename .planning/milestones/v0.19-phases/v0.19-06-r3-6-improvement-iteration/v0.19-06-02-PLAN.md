**Pattern to detect (file-read instructions, not mere mentions):**

The detection regex distinguishes between lines that _instruct_ an agent to read CLAUDE.md as a file versus lines that merely _mention_ CLAUDE.md in prose or comments. Use this exact pattern:

```javascript
// FILE_READ_PATTERN — matches lines that instruct reading CLAUDE.md as a file:
//   "Read `./CLAUDE.md`"  →  matches
//   "read CLAUDE.md"      →  matches
//   "load `CLAUDE.md`"    →  matches
//   "check CLAUDE.md"     →  matches
//   "open CLAUDE.md"      →  matches
// Does NOT match:
//   "CLAUDE.md references" (no verb prefix)
//   "# CLAUDE.md audit"   (comment/heading, no verb)
//   "see CLAUDE.md for"   ("see" not in verb list)
// MAINTAINABILITY NOTE: If new reading verbs appear in CLAUDE.md references
// across the codebase (e.g., `@`, `import`, `source`, `include`), extend the
// verb list above and update this pattern accordingly. Re-run the failing test
// to confirm the new variant is caught before committing.
const FILE_READ_PATTERN = /(?:Read|read|load|check|open)\s+[`'"]?(?:\.\/)?CLAUDE\.md[`'"]?/;
```

This pattern is self-contained — a future maintainer can replicate the detection logic from this definition alone, without consulting RESEARCH.md.

**Test logic:**
- For each file in FILES_TO_AUDIT: read file contents, split by line, check each line against FILE_READ_PATTERN
- Collect all matches as `{ file, lineNumber, line }`
- Assert violations.length === 0; if not, print each violation and fail

**Test name:** 'CLAUDE.md file-read references audit — all workflow/agent files'
