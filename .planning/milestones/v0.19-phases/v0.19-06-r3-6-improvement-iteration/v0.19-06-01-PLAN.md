    - parseImprovements(outputWithSuggestionHavingQuotedString) -> [{suggestion:"quoted value", rationale:"..."}]
    - Result block omits `improvements:` field entirely when array is empty (field not present = correct behavior)
  </behavior>

    Since parseImprovements is defined as a spec in a markdown agent file (not as an importable JS module), implement tests that:
    1. Define parseImprovements inline as a pure function implementing the spec from qgsd-quorum-slot-worker.md lines 251-260. Use the pseudocode from RESEARCH.md as reference: scan lines for 'Improvements:', then match `- suggestion:` and `rationale:` pairs.

    3. Export nothing; file is standalone test suite.

    parseImprovements implementation approach (from RESEARCH.md):
    - Split rawOutput on '\n'
    - Scan for line startsWith 'Improvements:'
    - In improvements section: match `^- suggestion:` → start new entry
    - Match `^\s+rationale:` → complete entry
    - Blank line or non-indented non-empty line → section ends
    - Push entry to array only when both suggestion AND rationale present
    - Quote stripping: trim leading/trailing whitespace and quotes from suggestion/rationale values
    - Return improvements array (empty array if nothing found)

<feature>
  <name>Signal Emission — Quorum Improvements Unit Tests (IMPR-02)</name>
  <files>bin/quorum-improvements-signal.test.cjs</files>
  <behavior>
    The quorum signal emission logic (specified in commands/qgsd/quorum.md lines 449-471) must:
    - Collect improvements: fields from final-round worker result blocks where verdict != UNAVAIL and improvements field is present and non-empty
    - De-duplicate by suggestion text (keep first occurrence when duplicate suggestion strings found)
    - Emit `<!-- QUORUM_IMPROVEMENTS_START\n[...JSON array...]\nQUORUM_IMPROVEMENTS_END -->` signal
    - Emit empty array `[]` when no improvements collected
    - Do NOT emit signal at all when request_improvements was false/absent

    Test cases:
    - collectImprovements([blockA_withImpr, blockB_withImpr]) -> deduplicated array (e.g. [{model, suggestion, rationale}, ...])
    - collectImprovements([blockA_withImpr, blockB_samesuggestion]) -> only first occurrence kept
    - collectImprovements([blockA_UNAVAIL_withImpr, blockB_withImpr]) -> UNAVAIL block skipped
    - collectImprovements([]) -> []
    - formatSignal([{model, suggestion, rationale}]) -> "<!-- QUORUM_IMPROVEMENTS_START\n[...]\nQUORUM_IMPROVEMENTS_END -->"
    - formatSignal([]) -> "<!-- QUORUM_IMPROVEMENTS_START [] QUORUM_IMPROVEMENTS_END -->"
    - Parsing the emitted signal: extract JSON between delimiters, JSON.parse -> same array as input
  </behavior>
  <implementation>
    Define collectImprovements(workerBlocks) and formatSignal(improvements) inline in the test file.

    collectImprovements:
    - Filter blocks: verdict !== 'UNAVAIL', improvements field is array with length > 0
    - Flatten all improvements from passing blocks
    - De-duplicate: iterate, track seen suggestion strings in a Set, skip duplicates
    - Add model field from the worker block's slot name to each improvement entry

    formatSignal:
    - If improvements.length === 0: return "<!-- QUORUM_IMPROVEMENTS_START [] QUORUM_IMPROVEMENTS_END -->"
    - Otherwise: return `<!-- QUORUM_IMPROVEMENTS_START\n${JSON.stringify(improvements)}\nQUORUM_IMPROVEMENTS_END -->`

    Signal parsing roundtrip test: given a signal string, extract substring between START and END delimiters, trim, JSON.parse → must equal original array.

    Use same Node.js assert pattern as other test files (bin/check-trace-redaction.test.cjs pattern).
  </implementation>
</feature>
