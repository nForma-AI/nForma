Create formal/check-result.schema.json as a JSON Schema draft-07 document with:
- "$schema": "http://json-schema.org/draft-07/schema#"
- "type": "object"
- "required": ["tool", "formalism", "result", "timestamp"]
- Properties: tool (string), formalism (string, enum: ["tla","alloy","prism","trace","redaction"]), result (string, enum: ["pass","fail","warn","inconclusive"]), timestamp (string, format: date-time), metadata (object)

Create bin/write-check-result.cjs as a CJS module:
- 'use strict'
- VALID_RESULTS = ['pass', 'fail', 'warn', 'inconclusive']
- VALID_FORMALISMS = ['tla', 'alloy', 'prism', 'trace', 'redaction']
- NDJSON_PATH = process.env.CHECK_RESULTS_PATH || path.join(__dirname, '..', 'formal', 'check-results.ndjson')
  (Use CHECK_RESULTS_PATH env var so tests can redirect to tmpDir without polluting real output)
- writeCheckResult(entry) validates: entry.tool must be string, entry.formalism must be in VALID_FORMALISMS, entry.result must be in VALID_RESULTS. Throws descriptive error on validation failure.
- Builds record = { tool, formalism, result, timestamp: new Date().toISOString(), metadata: entry.metadata || {} }
- Calls fs.appendFileSync(NDJSON_PATH, JSON.stringify(record) + '\n', 'utf8')
- module.exports = { writeCheckResult, NDJSON_PATH, VALID_RESULTS, VALID_FORMALISMS }

Create bin/write-check-result.test.cjs with node:test + assert. Use spawnSync pattern from existing tests for tests that need filesystem isolation (set CHECK_RESULTS_PATH env var to a temp file). Tests must cover:
1. 'module loads without error' — require() does not throw
2. 'writeCheckResult throws on missing tool field' — pass {} expect throw
3. 'writeCheckResult throws on invalid formalism' — pass { tool: 'x', formalism: 'bad', result: 'pass' } expect throw containing 'formalism'
4. 'writeCheckResult throws on invalid result' — pass { tool: 'x', formalism: 'tla', result: 'bad' } expect throw containing 'result'
5. 'writeCheckResult appends one line to NDJSON file' — use tmpDir + CHECK_RESULTS_PATH env, call writeCheckResult, read file, split by newline, assert exactly 1 non-empty line
6. 'writeCheckResult record has all required fields' — parse the appended JSON, assert tool/formalism/result/timestamp/metadata all present
7. 'NDJSON_PATH exported correctly' — assert typeof NDJSON_PATH === 'string' && NDJSON_PATH.endsWith('check-results.ndjson')

Use os.tmpdir() + fs.mkdtempSync for isolation. Clean up in try/finally.

Create bin/check-results-exit.cjs as a standalone CI exit script:
- 'use strict'
- const { NDJSON_PATH } = require('./write-check-result.cjs')
- If NDJSON_PATH file does not exist: write to stderr "[check-results-exit] No check-results.ndjson found — nothing to check\n", process.exit(0)
- Read file, split by '\n', filter empty lines, JSON.parse each line
- Filter for entries where r.result === 'fail'
- If any fails: write to stderr "[check-results-exit] N fail(s) found:\n" then for each fail write "  tool=X formalism=Y ts=Z\n", then process.exit(1)
- If no fails: write to stdout "[check-results-exit] All N check(s) pass/warn/inconclusive.\n", process.exit(0)
- Support CHECK_RESULTS_PATH env var override (it flows through write-check-result.cjs's NDJSON_PATH automatically)

Create bin/check-results-exit.test.cjs using node:test + assert + spawnSync pattern. Each test writes a temp NDJSON file, then spawns check-results-exit.cjs with CHECK_RESULTS_PATH env var pointing to that file, checks exit code. Tests:
1. 'module loads without error' — require() does not throw (note: script calls process.exit — load via isolated child, or guard the require with a sentinel)
   NOTE: Since the script calls process.exit at top level, use spawnSync approach for all tests rather than require(). Use 'exits 0 when no ndjson file exists' by setting CHECK_RESULTS_PATH to a path that doesn't exist.
2. 'exits 0 when no ndjson file exists' — spawn with nonexistent path, assert status=0
3. 'exits 0 when all results are pass' — write {"tool":"t","formalism":"tla","result":"pass","timestamp":"x","metadata":{}}, assert status=0
4. 'exits 1 when any result is fail' — write {"tool":"t","formalism":"tla","result":"fail","timestamp":"x","metadata":{}}, assert status=1
5. 'exits 0 when all results are warn' — write result=warn entry, assert status=0
6. 'exits 0 when all results are inconclusive' — write result=inconclusive entry, assert status=0
7. 'prints fail details to stderr on failure' — write fail entry, assert stderr contains 'tool=' and 'formalism='

Clean up tmpDir in try/finally.
  </action>

  <done>All 7 tests pass. node bin/check-results-exit.cjs exits 0 when no file exists. With a file containing result=fail it exits 1. With only pass/warn/inconclusive entries it exits 0.</done>
