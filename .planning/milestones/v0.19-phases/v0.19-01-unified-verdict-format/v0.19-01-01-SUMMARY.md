# Plan v0.19-01-01 Summary: NDJSON Infrastructure

## What Was Built

Created the shared NDJSON infrastructure that all runner scripts and the CI exit gate depend on.

### Files Created

| File | Purpose |
|------|---------|
| `formal/check-result.schema.json` | JSON Schema draft-07 contract — required fields: tool, formalism, result, timestamp; formalism enum: tla/alloy/prism/trace/redaction; result enum: pass/fail/warn/inconclusive |
| `bin/write-check-result.cjs` | Shared CJS helper — validates entry fields (throws descriptive errors on bad input), appends one JSON line to check-results.ndjson; supports CHECK_RESULTS_PATH env var for test isolation |
| `bin/write-check-result.test.cjs` | 7 unit tests: load, missing-tool throw, bad-formalism throw, bad-result throw, single-line append, required-fields, NDJSON_PATH export |
| `bin/check-results-exit.cjs` | CI exit reader — reads NDJSON_PATH, exits 0 on no file or all pass/warn/inconclusive, exits 1 on any result=fail; prints fail details to stderr |
| `bin/check-results-exit.test.cjs` | 7 unit tests using spawnSync: no-file, all-pass, any-fail, warn-only, inconclusive-only, stderr-details, empty-file |

## Decisions Made

- CHECK_RESULTS_PATH env var override used for test isolation — avoids polluting formal/check-results.ndjson during test runs
- check-results-exit.test.cjs uses spawnSync for all tests since the script calls process.exit() at top level (cannot be require()'d safely)
- Empty NDJSON file → exit 0 (no fails present, nothing to fail on)
