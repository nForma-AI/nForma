# Phase v0.19-07 Research: Wire Liveness Check to All TLC Runners

## Summary

Add `detectLivenessProperties()` calls to 4 TLC runners (`run-oscillation-tlc.cjs`, `run-breaker-tlc.cjs`, `run-protocol-tlc.cjs`, `run-account-manager-tlc.cjs`) so that all 6 unchecked liveness properties emit `result=inconclusive` when their companion `invariants.md` has no fairness declaration, closing the LIVE-02 requirement gap.

---

## Problem Statement

The `detectLivenessProperties()` function exists and is exported from `bin/run-tlc.cjs`. It is already called in `run-tlc.cjs` (the quorum runner) after a successful TLC run, producing `result=inconclusive` when fairness declarations are missing from `formal/spec/<surface>/invariants.md`. However, 4 other TLC runners execute liveness `.cfg` files but never call `detectLivenessProperties()` after a successful run — they emit `result=pass` unconditionally when TLC exits 0, even if no fairness declaration exists.

### Affected Runners and Their Liveness Configs

| Runner | Configs | Liveness Properties | Surface Key |
|--------|---------|--------------------|----|
| `run-oscillation-tlc.cjs` | MCoscillation.cfg, MCconvergence.cfg | AlgorithmTerminates, ConvergenceEventuallyResolves | oscillation, convergence |
| `run-breaker-tlc.cjs` | MCbreaker.cfg | MonitoringReachable | breaker |
| `run-protocol-tlc.cjs` | MCdeliberation.cfg, MCprefilter.cfg | ProtocolTerminates, PreFilterTerminates | deliberation, prefilter |
| `run-account-manager-tlc.cjs` | MCaccount-manager.cfg | IdleReachable | account-manager |

All 6 liveness properties have companion `invariants.md` files with valid `## PropertyName` headers under `formal/spec/<surface>/invariants.md`. The fairness declarations exist — the runners just never check them.

## Reference Implementation: `run-tlc.cjs`

`bin/run-tlc.cjs` already implements the correct pattern:

```js
const { detectLivenessProperties } = require('./run-tlc.cjs');
// ... after TLC exits 0 ...
const missingDeclarations = detectLivenessProperties(configName, cfgPath);
if (missingDeclarations.length > 0) {
  writeCheckResult({
    tool: 'run-tlc',
    formalism: 'tla',
    result: 'inconclusive',
    metadata: {
      config: configName,
      reason: 'Fairness declaration missing for: ' + missingDeclarations.join(', '),
    }
  });
  process.stdout.write('[run-tlc] Result: inconclusive — fairness declaration missing for: ' + missingDeclarations.join(', ') + '\n');
  process.exit(0); // inconclusive is not a failure
}
// else: write pass result
```

The function signature:
```js
detectLivenessProperties(configName, cfgPath, specDir?)
// configName: e.g., 'MCoscillation'
// cfgPath: absolute path to the .cfg file
// specDir: override for formal/spec directory (optional, for test injection)
// Returns: string[] — property names missing ## header in invariants.md
// Returns [] if no PROPERTY/PROPERTIES lines in cfg (not a liveness config)
```

`detectLivenessProperties` is already exported: `module.exports = { detectLivenessProperties };`

The `SURFACE_MAP` inside `run-tlc.cjs` already covers all 6 surfaces:
```js
const SURFACE_MAP = {
  'MCliveness':            'quorum',
  'MCdeliberation':        'deliberation',
  'MCprefilter':           'prefilter',
  'MCrecruiting-liveness': 'recruiting',
  'MCbreaker':             'breaker',
  'MCconvergence':         'convergence',
  'MCoscillation':         'oscillation',
  'MCaccount-manager':     'account-manager',
};
```

## Implementation Pattern Per Runner

Each runner needs these changes after TLC exits 0 (where currently only `writeCheckResult pass` is called):

1. **Import** `detectLivenessProperties` at the top:
   ```js
   const { detectLivenessProperties } = require('./run-tlc.cjs');
   ```

2. **After TLC exits 0**, before writing `pass`, call:
   ```js
   const missingDeclarations = detectLivenessProperties(configName, cfgPath);
   if (missingDeclarations.length > 0) {
     try {
       writeCheckResult({
         tool: 'run-XXX-tlc',
         formalism: 'tla',
         result: 'inconclusive',
         metadata: {
           config: configName,
           reason: 'Fairness declaration missing for: ' + missingDeclarations.join(', '),
         }
       });
     } catch (e) { ... }
     process.stdout.write('[run-XXX-tlc] Result: inconclusive — ...\n');
     process.exit(0);
   }
   // existing pass path
   ```

3. **`cfgPath`** is already computed in each runner before TLC invocation (step 4 in each file). No new variable needed.

## Test Strategy

Each runner's existing test file needs new tests for the `detectLivenessProperties` integration. The pattern from `run-tlc.test.cjs` applies:

- Test that the runner imports and calls `detectLivenessProperties` (source code assertion)
- Test that it passes `configName` and `cfgPath` with correct surface mapping
- Test that when `invariants.md` is missing, the runner would emit `result=inconclusive` (inject via `specDir` parameter — but since runners call the function directly with default `specDir`, test via source inspection OR via spawning with a crafted environment)

Simplest testable pattern: source-inspection tests (read the `.cjs` file and assert it contains the `detectLivenessProperties` call). This follows the same pattern used in `run-oscillation-tlc.test.cjs` for `workers = '1'` assertions.

More comprehensive: verify the inconclusive path via integration test — but that requires Java + TLC. Recommend source-inspection tests for Wave 0, matching the existing test style.

## Risks and Pitfalls

1. **`workers = 'auto'` in `run-breaker-tlc.cjs`**: This is pre-existing behavior and is not the subject of LIVE-02. Do not change it — leave `workers = 'auto'` as-is.

2. **Circular require warning**: `run-oscillation-tlc.cjs` requiring `run-tlc.cjs` creates a dependency. Node.js handles circular requires but it is cleaner to verify `run-tlc.cjs` does not require any of the 4 runners. From inspection: `run-tlc.cjs` only requires `child_process`, `fs`, `path`, and `write-check-result.cjs` — no circular dependency.

3. **`require.main === module` guard**: `run-tlc.cjs` wraps main execution in `if (require.main === module)` — so `require('./run-tlc.cjs')` from other runners ONLY loads the module (exports `detectLivenessProperties`), does NOT execute TLC. This is safe.

4. **Tool name in `writeCheckResult`**: Each runner must use its own tool name (e.g., `'run-oscillation-tlc'`, not `'run-tlc'`) in the inconclusive result.

5. **`metadata` field for runners without existing metadata object**: `run-breaker-tlc.cjs` and `run-account-manager-tlc.cjs` currently pass `metadata: {}`. The inconclusive path should add `config` and `reason` to `metadata`. The pass path can keep `metadata: {}` or add `config: configName` for consistency with other runners — follow existing pattern per file.
