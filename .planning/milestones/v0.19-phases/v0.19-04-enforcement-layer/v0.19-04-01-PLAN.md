**formal/trace/redaction.yaml** — Write the redaction policy file:

```yaml
# formal/trace/redaction.yaml
# Redaction policy for QGSD trace event payloads.
# Forbidden keys: any trace event containing these field names is a violation.
# Forbidden patterns: any trace event field VALUE matching these regex is a violation.

forbidden_keys:
  - "api_key"
  - "secret"
  - "password"
  - "token"
  - "authorization"
  - "bearer"
  - "private_key"
  - "access_key"

forbidden_patterns:
  - name: "aws_access_key_id"
    regex: "^AKIA[0-9A-Z]{16}$"
  - name: "github_pat"
    regex: "^ghp_[A-Za-z0-9_]{36}$"
  - name: "email"
    regex: "^[\\w\\.+-]+@[\\w\\.-]+\\.\\w+$"
  - name: "bearer_token"
    regex: "^Bearer\\s+[A-Za-z0-9._~+/-]+=*$"
```

**bin/check-trace-redaction.cjs** — Implement the validator:

Structure:
1. `require('fs')`, `require('path')`, `require('./write-check-result.cjs')` — built-ins only, no new deps.
2. `parseRedactionPolicy(policyPath)` — parse redaction.yaml using line-by-line regex/split (follow formal/policy.yaml parsing pattern from v0.19-02; no external YAML lib). Returns `{ forbidden_keys: string[], forbidden_patterns: Array<{name, regex, compiled: RegExp}> }`. Pre-compile all regex objects at parse time (not per-event) to avoid regex performance issues per RESEARCH pitfall 1.
3. `validateTraceEvent(event, policy)` — iterate keys in the event object; check each key against `forbidden_keys`; check each string value against each `forbidden_patterns[i].compiled` regex. Return array of violation objects: `{ key, value?, pattern_name?, violation_type: 'forbidden_key'|'forbidden_pattern' }`.
4. Main execution block (wrapped in `if (require.main === module)` guard for testability):
   - Accept optional `--trace-dir <path>` CLI arg; default to `formal/trace/` relative to `process.cwd()`.
   - If trace directory does not exist: call `writeCheckResult({ tool: 'check-trace-redaction', formalism: 'redaction', result: 'pass', metadata: { reason: 'no-trace-directory', directory: traceDir } })`; `process.exit(0)`.
   - Find all `.json` and `.jsonl` files in trace dir (non-recursive, `fs.readdirSync`).
   - If no files found: `writeCheckResult({ ..., result: 'pass', metadata: { reason: 'no-trace-events' } })`; `process.exit(0)`.
   - For each file: read and parse as NDJSON (split on `\n`, `JSON.parse` each non-empty line). Accumulate violations. Do NOT fail fast — collect all violations.
   - If violations found: `writeCheckResult({ ..., result: 'fail', metadata: { violations: violations.slice(0, 10), total_violations: violations.length } })`; print violations to stdout; `process.exit(1)`.
   - If no violations: `writeCheckResult({ ..., result: 'pass', metadata: { files_checked: fileCount, events_checked: eventCount } })`; `process.exit(0)`.
5. `module.exports = { parseRedactionPolicy, validateTraceEvent }` at end of file for test access.

Key pitfall avoidance:
- Pre-compile regex at startup (not per-event check).
- Handle missing `formal/trace/` gracefully (pass, not crash).
- Handle YAML parse via manual line-by-line regex, not an external YAML parser.
- Use `require.main === module` guard so tests can `require('./check-trace-redaction.cjs')` without triggering CLI execution.
