### bin/validate-traces.cjs (extended)

Added near the top (after requires):

**CONFIDENCE_THRESHOLDS constant** (exported):
```js
{ low: { min_rounds: 0, min_days: 0 }, medium: { min_rounds: 500, min_days: 14 }, high: { min_rounds: 10000, min_days: 90 } }
```

**computeConfidenceTier(n_rounds, window_days)** (exported):
- Returns 'high' when n_rounds >= 10000 AND window_days >= 90
- Returns 'medium' when n_rounds >= 500 AND window_days >= 14
- Returns 'low' otherwise

**readScoreboardMeta()** (internal):
- Reads `.planning/quorum-scoreboard.json` once before divergence loop
- Returns `{ n_rounds, window_days }` — defaults to 0/0 on error or absent file

**Divergence objects** now include `n_rounds`, `window_days`, `confidence` fields for all three divergence types: `json_parse_error`, `unmappable_action`, `state_mismatch`.

**require.main guard** added so tests can `require('./validate-traces.cjs')` without triggering CLI execution.

**module.exports** = `{ computeConfidenceTier, CONFIDENCE_THRESHOLDS }` at end of file.
