## What Was Built

### bin/check-trace-schema-drift.cjs

Schema drift guard using `git diff --name-only HEAD~1`:

**KNOWN_EMITTERS constant** (exported):
- `bin/validate-traces.cjs`
- `hooks/qgsd-stop.js`
- `hooks/qgsd-prompt.js`
- `hooks/dist/qgsd-stop.js`
- `hooks/dist/qgsd-prompt.js`

**checkSchemaDrift(changedFiles)** (exported pure function):
- `pass / no-schema-change`: schema.json NOT in changedFiles
- `pass / schema-change-atomic`: schema.json changed AND validate-traces.cjs changed AND at least one non-validator KNOWN_EMITTER changed
- `fail / schema-drift-detected`: schema.json changed but validator OR non-validator emitters NOT updated; includes `validator_updated` and `emitter_updated` boolean flags

Key design note: `bin/validate-traces.cjs` satisfies the validator condition only. The emitter check requires a separate hook file (`hooks/qgsd-stop.js` etc.). This reflects that updating only the validator without also updating emitters is still a drift.

**CLI**: calls `execFileSync('git', ['diff', '--name-only', 'HEAD~1'])`, splits on newlines, runs `checkSchemaDrift`, writes NDJSON via `writeCheckResult`, exits 0/1. Git errors caught and written as `result=fail` with `reason=git-error`.
