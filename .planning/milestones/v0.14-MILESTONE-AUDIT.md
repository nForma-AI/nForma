## Integration Findings

### BROKEN-01 — Critical: xstate-to-tla.cjs output structurally incompatible with existing TLA+ model

**Severity:** Critical (blocking full pipeline execution)
**Affects:** INTG-04

When `run-formal-verify.cjs` executes STEPS[0] without `--dry`, it calls `xstate-to-tla.cjs` which **writes to `.formal/tla/QGSDQuorum.tla`**, overwriting the hand-authored spec.

| Attribute | Hand-authored `QGSDQuorum.tla` | `xstate-to-tla.cjs`-generated |
|-----------|-------------------------------|-------------------------------|
| State variable | `phase` | `state` |
| Constants | `Agents`, `MaxDeliberation` | none |
| Symmetry | `AgentSymmetry` | none |
| Semantic invariants | `MinQuorumMet`, `NoInvalidTransition`, `LivenessEventually` | `TypeOK` only |
| Config file generated | — (uses MCsafety.cfg, MCliveness.cfg) | `MCQGSDQuorum.cfg` (not used by run-tlc.cjs) |

After STEPS[0] executes:
1. `MCsafety.cfg` references `SYMMETRY AgentSymmetry` → TLC fails (symbol not found)
2. `MCsafety.cfg` references `INVARIANT MinQuorumMet` → TLC fails (predicate not found)
3. `check-spec-sync.cjs` parses `phase \in {…}` from the overwritten file → fails to find it
4. All failures masked by `continue-on-error: true` in CI

**Current protection:** The repo has never run `run-formal-verify.cjs` non-dryly in a context that triggers STEPS[0]. Tests use `--only=bogus` (exits 1), `--only=generate` (accepted without TLA+ write in test context), and `node --check` syntax smoke only.

**Resolution needed in v0.14-02 planning:** Choose one canonical TLA+ model for QGSDQuorum — either:
- (A) Keep hand-authored model: rename xstate-to-tla.cjs output to `QGSDQuorum_xstate.tla` and add a merge/sync step
- (B) Replace with generated model: update MCsafety.cfg, MCliveness.cfg, check-spec-sync.cjs, and run-tlc.cjs for the new variable/constant schema

### MISSING-01 — Warning: check-spec-sync.cjs not in npm test (planned gap for v0.14-02)

`check-spec-sync.cjs` is wired in CI (formal-verify.yml line 86) but not in `npm test`. A developer can add an XState state, run `npm test`, get 326 passing tests, and ship without detecting spec drift. This is the planned work for DRFT-01 in v0.14-02 — flagged here as baseline context for gap-closure planning.

### MISSING-02 — Warning: new v0.14-01 scripts absent from CI path triggers

`bin/xstate-to-tla.cjs`, `bin/run-formal-verify.cjs`, and 3 other new scripts are not in the `formal-verify.yml` `paths:` block. Edits to these files do not auto-trigger CI. `workflow_dispatch` provides manual triggering as fallback. Recommend adding these paths when BROKEN-01 is resolved in v0.14-02.
