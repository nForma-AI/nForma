# Requirements: nForma v0.30 — Advanced Agent Patterns

**Defined:** 2026-03-07
**Core Value:** Planning decisions are multi-model verified by structural enforcement, not instruction-following

## v0.30 Requirements

### Token Optimization & Model Routing

- [x] **TOKN-01**: Auto-compaction at clean boundaries — PreCompact hook preserves execution state and memory across compaction events
- [x] **TOKN-02**: Thinking budget scaling per task type — task complexity classifier adjusts thinking budget directive based on plan vs research vs quick task
- [x] **TOKN-03**: Token usage dashboard — `/nf:tokens` displays per-slot and per-session token consumption from telemetry JSONL
- [x] **TOKN-04**: Task-complexity model routing — TIER_SLOT_MAP filters quorum slots by task complexity tier (T1/T2/T3)

### Verification & Execution State

- [x] **VERF-01**: File-based execution state — execution-progress.cjs tracks plan/task completion state to disk, survives compaction
- [x] **VERF-02**: Continuous test verification — PostToolUse hook triggers boundary-detection checks during execution with budget cap
- [x] **VERF-03**: Machine-verifiable completion — done_conditions in task schema enable automated evaluateAllConditions check

### Memory Persistence

- [x] **MEMP-01**: Decisions persist across compaction — memory-store.cjs JSONL append for decisions category injected by PreCompact hook
- [x] **MEMP-02**: Proactive session reminders — SessionStart hook generates memory reminder from recent entries across all categories
- [x] **MEMP-03**: Error resolution memory — error patterns persist to JSONL with queryable interface
- [x] **MEMP-04**: Quorum decision memory — quorum verdicts persist to JSONL for cross-session learning

### Continuous Learning

- [x] **LRNG-01**: Auto error extraction at session end — SessionEnd hook extracts error patterns from transcript via learning-extractor.cjs
- [x] **LRNG-02**: User correction capture — SessionEnd hook detects and persists user corrections to memory store
- [x] **LRNG-03**: Quorum-validated skill extraction — skill-extractor.cjs generates skill candidates persisted to `skills` JSONL category
- [x] **LRNG-04**: Failure catalog with confidence — failure entries persist with confidence scoring (weekly decay, reconfirmation boost)

### Subagent Orchestration

- [x] **ORCH-01**: Domain-specific context retrieval — context-retriever.cjs analyzes task domain and fetches relevant context artifacts
- [x] **ORCH-02**: Phase-to-phase context accumulation — context-stack.cjs maintains JSONL phase index for cross-phase context injection
- [x] **ORCH-03**: Specialized retrieval agents — quorum dispatch enriches prompts with domain-relevant context via enrichPromptWithRetrieval

### Parallel Execution

- [x] **PARA-01**: Worktree-isolated executor subagents — nf-worktree-executor agent runs plans in isolated git worktrees
- [x] **PARA-02**: Parallel plan execution with merge orchestration — Pattern D in execute-plan.md dispatches independent plans to worktrees with merge queue coordination
