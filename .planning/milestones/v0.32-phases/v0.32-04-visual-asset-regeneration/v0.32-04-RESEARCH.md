### VHS Tape Structure (existing)

The tape at `scripts/tui-screenshot.tape` follows this pattern:
1. Set shell, font, dimensions, theme, padding
2. Export PATH with Claude Code binary, unset CLAUDECODE
3. Launch `node bin/nForma.cjs`
4. Navigate modules via Tab (F-keys unsupported in VHS)
5. Navigate items via Up/Down arrows
6. Capture screenshots at each view
7. End with disconnect and quit

### Pattern: Version-Agnostic PATH

**Current (hardcoded, brittle):**
```
Type@500ms "export PATH=$PATH:/Users/jonathanborduas/Library/Application\ Support/Claude/claude-code/2.1.51/ && unset CLAUDECODE"
```

**Fixed (dynamic):**
```
Type@500ms "export PATH=$PATH:$(ls -d ~/Library/Application\\ Support/Claude/claude-code/*/  2>/dev/null | tail -1) && unset CLAUDECODE"
```

### Anti-Patterns to Avoid
- **Hardcoding version numbers in paths:** The current tape pins `2.1.51`; this breaks on every Claude Code update
- **Forgetting to unset CLAUDECODE:** Without this, TUI detects it is running inside Claude Code and alters behavior
- **Insufficient sleep times:** VHS needs adequate sleep between commands for the TUI to render; the existing times (8s startup, 2-5s for views) are tuned and should not be reduced

## Common Pitfalls

### Pitfall 1: Claude Code Version Path Drift
**What goes wrong:** VHS tape fails because hardcoded path `/claude-code/2.1.51/` no longer exists after update
**Why it happens:** Claude Code auto-updates, old version directory is removed
**How to avoid:** Use glob/ls to dynamically resolve the latest version directory
**Warning signs:** `command not found: claude` errors in VHS output

### Pitfall 2: CLAUDECODE Environment Variable Leak
**What goes wrong:** TUI renders differently when CLAUDECODE is set (detects embedded mode)
**Why it happens:** VHS inherits the parent shell's environment
**How to avoid:** Explicitly `unset CLAUDECODE` before launching TUI (already in tape)
**Warning signs:** TUI shows different layout or missing features

### Pitfall 3: Non-Deterministic Health Data
**What goes wrong:** `tui-agents-health.png` shows different latency values on each run
**Why it happens:** Live Health view makes real API calls with varying response times
**How to avoid:** Accept that health view data varies; determinism check should exclude this screenshot OR mock the health endpoint. The existing tape captures health after a 5s sleep, which should stabilize rendering but not values.
**Warning signs:** `cmp` fails on health screenshot between runs

### Pitfall 4: Session Screenshot Requires Real Claude
**What goes wrong:** `tui-session-claude.png` fails because Claude Code is not available or takes too long to start
**Why it happens:** The tape creates a new session named "demo" which launches real Claude Code
**How to avoid:** Increase sleep time or accept this screenshot may need special handling
**Warning signs:** Blank or partial terminal in session screenshot

### Pitfall 5: Orphaned SVG Files in Repository Root
**What goes wrong:** Untracked `tui-*.svg` files in repo root confuse contributors
**Why it happens:** Quick-213 generated SVGs to `docs/assets/`, but VHS tape also outputs SVGs to root during recording
**How to avoid:** Delete root-level `tui-*.svg` files and add to `.gitignore`
