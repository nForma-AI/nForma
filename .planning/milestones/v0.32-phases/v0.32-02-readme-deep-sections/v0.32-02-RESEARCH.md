### Pattern 1: Mermaid Flow Diagram for "How It Works"
**What:** A horizontally-oriented flow diagram showing the nForma quorum pipeline
**When to use:** When explaining a multi-step process with clear directional flow
**Example:**
```markdown
```mermaid
flowchart LR
    A["User Prompt"] --> B["Orchestrator"]
    B --> C["Dispatch to\n5 Models"]
    C --> D["Debate &\nConsensus"]
    D --> E["Execute\nwith Fresh Context"]
    E --> F["Atomic\nCommit"]

    style A fill:#f9f,stroke:#333
    style D fill:#bbf,stroke:#333
    style F fill:#bfb,stroke:#333
```​
```

**Key considerations:**
- Use `flowchart LR` (left-to-right) for horizontal flow
- Keep node labels short (2-3 words max per line)
- Use `\n` for line breaks within node labels
- Avoid complex styling -- GitHub's Mermaid renderer may not support all CSS
- Test rendering on GitHub after commit (local preview may differ)

### Anti-Patterns to Avoid
- **Images between table rows:** Any content between `|...|` rows breaks the GFM table parser; images must be before or after the complete table
- **Over-complex Mermaid diagrams:** GitHub's Mermaid renderer has size limits and may not support all Mermaid features (subgraphs work, but complex styling may not)
- **Burying the critical path in collapsibles:** Install + quorum setup + verify should never be hidden behind a `<details>` click
- **Anchor link to `<details>` section:** Clicking a nav link to a `<details>` section does NOT auto-expand it on GitHub; the Community section must be a real `##` heading, not inside `<details>`

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Architecture diagram | ASCII art or external SVG | GitHub-native Mermaid fenced code block | Natively rendered, text-editable, no asset pipeline needed |
| Discord badge | Custom HTML badge | shields.io Discord badge (already used in header) | Consistent styling, auto-updates member count |
| Collapsible sections | Custom JavaScript | `<details><summary>` HTML | Native browser support, works everywhere including GitHub |

**Key insight:** GitHub's native Mermaid support eliminates the need for any external diagram tooling. The diagram lives as text in the README and renders as a visual flowchart.

## Common Pitfalls

### Pitfall 1: Mermaid Diagram Not Rendering
**What goes wrong:** Mermaid code block shows as raw text instead of a diagram
**Why it happens:** Syntax errors in Mermaid code; GitHub only renders valid Mermaid
**How to avoid:** Test with the [Mermaid Live Editor](https://mermaid.live) before committing; use simple node shapes (rectangles, rounded) and avoid advanced features
**Warning signs:** Raw text visible instead of diagram on GitHub

### Pitfall 2: Mermaid Version Mismatch
**What goes wrong:** Diagram features work in Mermaid Live Editor but not on GitHub
**Why it happens:** GitHub pins a specific Mermaid version that may lag behind the latest
**How to avoid:** Stick to basic flowchart syntax (`flowchart LR/TD`, simple nodes, arrows, styles). Avoid newer features like `:::className` notation.
**Warning signs:** Diagram renders locally but shows error box on GitHub

### Pitfall 3: Breaking the Observability Table Worse
**What goes wrong:** Moving the image fixes one break but introduces another (e.g., missing closing `|` row)
**Why it happens:** GFM table parsing is strict -- any non-table line between `|...|` rows terminates the table
**How to avoid:** After fixing, verify the complete table renders by counting rows. The table should have 7 command rows (health, observe, triage, solve, session-insights, settings, set-profile).
**Warning signs:** Commands rendered as plain text instead of table rows

### Pitfall 4: Getting Started Section Becoming Too Long
**What goes wrong:** Making the critical path visible makes Getting Started visually overwhelming
**Why it happens:** Quorum setup has nested details (CLI agents, API agents, registration) that expand the visible section
**How to avoid:** Show only the wizard command (`/nf:mcp-setup`) and its one-line description in the visible path. Keep manual setup instructions inside a nested `<details>` within the collapsed "Agent Manager TUI" or a new "Manual Setup" collapsed section.
**Warning signs:** Getting Started is longer than How It Works when fully visible

### Pitfall 5: Community Section Discord Link Duplication
**What goes wrong:** Discord badge appears both in the header badge row and in the Community section
**Why it happens:** The header already has a Discord badge (line 9)
**How to avoid:** This is intentional -- the header badge is for immediate visibility, the Community section badge is for the call-to-action context. Both serve different purposes. But use the same badge style for consistency.
**Warning signs:** N/A -- this is acceptable duplication

### Pitfall 6: Nav Bar Not Updated for New Community Section
**What goes wrong:** New Community section is added but not linked from the nav bar
**Why it happens:** Nav bar on line 28 was expanded in v0.32-01 but does not include Community
**How to avoid:** Add `Community` anchor to the nav bar: `[Community](#community)`
**Warning signs:** No way to jump to Community from the nav bar

const { describe, it } = require('node:test');
const { strict: assert } = require('node:assert');
const fs = require('node:fs');

const README = fs.readFileSync(path.join(__dirname, '..', 'README.md'), 'utf8');

describe('RDME-05: Architecture diagram in How It Works', () => {

  it('diagram appears in or near How It Works section', () => {
    const howItWorksIdx = README.indexOf('## How It Works');
    const mermaidIdx = README.indexOf('```mermaid');
    assert.ok(howItWorksIdx > -1, 'How It Works section must exist');
    assert.ok(mermaidIdx > -1, 'Mermaid block must exist');
    assert.ok(mermaidIdx > howItWorksIdx, 'Mermaid block should be after How It Works heading');
    // Should be within ~50 lines of heading
    const linesBetween = README.substring(howItWorksIdx, mermaidIdx).split('\n').length;
    assert.ok(linesBetween < 50, `Mermaid block should be near How It Works heading (found ${linesBetween} lines away)`);
  });
});

describe('RDME-08: Community/Contributing section', () => {
  it('has a Community heading', () => {
    assert.ok(/^## Community/m.test(README), 'README should have ## Community section');
  });
  it('contains Discord link', () => {
    assert.ok(README.includes('discord.gg/M8SevJEuZG'), 'Community section should have Discord invite link');
  });
  it('appears before Star History', () => {
    const communityIdx = README.indexOf('## Community');
    const starHistoryIdx = README.indexOf('## Star History');
    assert.ok(communityIdx > -1 && starHistoryIdx > -1, 'Both sections must exist');
    assert.ok(communityIdx < starHistoryIdx, 'Community must appear before Star History');
  });
});

describe('RDME-09: Getting Started rebalanced', () => {
  it('install command is visible (not inside details)', () => {
    // npx command should appear before any <details> in Getting Started
    const gsIdx = README.indexOf('## Getting Started');
    const nextDetails = README.indexOf('<details>', gsIdx);
    const installCmd = README.indexOf('npx @nforma.ai/nforma', gsIdx);
    assert.ok(installCmd > gsIdx, 'Install command must be in Getting Started');
    assert.ok(installCmd < nextDetails, 'Install command must appear before first <details> block');
  });
  it('quorum setup mention is visible by default', () => {

describe('RDME-10: Observability table not broken', () => {
  it('no image between table rows in Observability section', () => {
    const obsIdx = README.indexOf('Observability');
    const obsSection = README.substring(obsIdx, obsIdx + 1000);
    // Find all table rows and images
    const lines = obsSection.split('\n');
    let inTable = false;
    for (let i = 0; i < lines.length; i++) {
      const line = lines[i].trim();
      if (line.startsWith('|') && line.endsWith('|')) {
        inTable = true;
      } else if (inTable && line.startsWith('![')) {
        assert.fail(`Image found between table rows at line ${i}: "${line}"`);
      } else if (inTable && !line.startsWith('|')) {

      }
