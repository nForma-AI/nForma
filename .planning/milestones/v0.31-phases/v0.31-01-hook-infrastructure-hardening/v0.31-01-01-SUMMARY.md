## Decisions Made
- Statusline hook uses 'Notification' as default event type since it receives a non-standard payload format (model, workspace) rather than standard PostToolUse events
- nf-check-update.js received a full stdin reader + validation block even though it previously didn't read stdin at all, to maintain uniform validation coverage across all 14 hooks
- Validation errors emit structured JSON on stderr then exit 0 (fail-open) -- consistent with existing hook error handling patterns
