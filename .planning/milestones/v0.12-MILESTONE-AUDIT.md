## Integration Findings

### New Issue: GAP-5 Workers Mismatch (v0.12-04)

**File:** `bin/run-oscillation-tlc.cjs` line 101

```javascript
// MCconvergence is safety-only (INVARIANT only) — 'auto' workers is safe.
const workers = configName === 'MCoscillation' ? '1' : 'auto';
```

`.formal/tla/MCconvergence.cfg` line 11 declares:
```
PROPERTY ConvergenceEventuallyResolves
```

This is a **liveness PROPERTY**, not an INVARIANT. TLC's multi-worker liveness checking bug applies equally to MCconvergence. Running with `-workers auto` on a multi-core machine risks a false-pass or silent skip of liveness verification.

**Fix:** Change `'auto'` to `'1'` for MCconvergence — or add a PROPERTY-presence check.

### Coherence Gap: MaxDeliberation Constant (GAP-2 / TLA-01)

QGSDQuorum.tla: `MaxDeliberation = 7` (workflow-level)
QGSDDeliberation.tla: `MaxDeliberationRounds = 10` (CLAUDE.md R3 protocol-level)

Different constants at different abstraction levels — neither is wrong, but no documentation bridges the two. A developer reading both specs would not know which governs runtime behavior.

### Inherited Issues (Phases 01-03, Unchanged)

**1. quorum_block/quorum_complete single-step replay divergences (VAL-01, VAL-02)**

`validate-traces.cjs` creates a fresh IDLE actor per event. The XState machine only accepts DECIDE from DELIBERATING. 286 quorum_block/quorum_complete events (41.5%) produce structural divergences — not protocol violations. Combined with 252 circuit_break unmappable events (36.6%), only 21.9% of events validate cleanly. The conformance score is not a reliable signal.

**2. rates.const → quorum.pm variable name mismatch (PRM-02)**

`export-prism-constants.cjs` generates per-slot variables (`tp_gemini`, `tp_opencode`, etc.). `quorum.pm` uses `tp_rate` and `unavail` (aggregate). The E2E PRISM flow requires manual aggregation with no documentation.

## Audit Conclusion

All 30 requirements have working implementations. No requirement is completely absent. The 8 partial requirements fall into three categories:

**Category A — Design-level gaps (VAL-01, VAL-02, LOG-03):** The conformance event infrastructure has a fundamental single-step replay design that makes the conformance score unreliable as a metric. "DECIDING" is emitted but not in VALID_PHASES. These require architectural rework of validate-traces.cjs.

**Category B — Test environment contamination (TLA-04, ALY-02):** Two tests fail because the JARs they test for absence are physically present. The error paths work; the tests need conditional-skip guards (the pattern used correctly in phases 06-08).

**Category C — Semantic mismatches (PRM-02, PET-03, GAP-5):** rates.const variable names don't match quorum.pm; deadlock condition is permanently inert; MCconvergence uses wrong workers setting. Each is a single-line fix.

**Most impactful to fix:**
1. **GAP-5** — `run-oscillation-tlc.cjs` line 101: change `'auto'` to `'1'` for MCconvergence (1 line)
2. **TLA-04 / ALY-02** — add conditional-skip guards for JAR-not-found tests when JAR present (2 tests, ~5 lines each)
3. **PRM-02** — document aggregation step in VERIFICATION_TOOLS.md OR align rates.const variable names with quorum.pm constants
4. **VAL-01/VAL-02** — multi-step trace replay architecture (larger rework, own phase)
