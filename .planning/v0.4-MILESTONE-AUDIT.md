## Broken E2E Flows

### Flow 1: Context Reset During Oscillation Resolution

```
Circuit breaker fires
→ oscillation-resolution-mode.md Step 4 writes:
   {"activity":"circuit_breaker","sub_activity":"oscillation_diagnosis"}   ← missing phase!
→ Context reset
→ resume-project reads activity-get
→ Finds activity=circuit_breaker, sub_activity=oscillation_diagnosis
→ Routing table row 175: /qgsd:execute-phase {phase}
→ {phase} = undefined (not in JSON payload)
→ User sees: /qgsd:execute-phase undefined  ← unusable
```

**Root cause:** `oscillation-resolution-mode.md` lines 64–65 and 92–93 do not include `phase` field
**Fix:** Add `"phase":"${PHASE_NUMBER}"` to both activity-set calls in oscillation-resolution-mode.md
**Requirements affected:** ACT-02 (schema incomplete), ACT-04 (routing broken)
