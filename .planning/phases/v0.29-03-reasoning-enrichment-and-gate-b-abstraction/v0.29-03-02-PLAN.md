Gate B algorithm (from research Pattern 4):
1. Load all L3 reasoning artifacts from `.planning/formal/reasoning/`: hazard-model.json, failure-mode-catalog.json, risk-heatmap.json
2. Load all L2 semantics artifacts from `.planning/formal/semantics/`: observed-fsm.json, invariant-catalog.json, mismatch-register.jsonl, assumption-register.json
3. For each entry in each L3 artifact (hazards array, failure_modes array, transitions array):
   a. Check derived_from array exists and is non-empty
   b. For each derived_from link, RESOLVE it by loading the referenced L2 artifact and navigating to the ref path (e.g., `observed_transitions.IDLE.QUORUM_START` must exist in observed-fsm.json)
   c. Mark entry as grounded (all links valid) or orphaned (any link broken or missing)
4. Compute gate_b_score = grounded_entries / total_entries
5. Target: 100% (research says this is achievable since we control all L3 generation)

- `semantics/observed-fsm.json#observed_transitions.STATE.EVENT` -- navigate JSON path

- `semantics/mismatch-register.jsonl` -- verify file exists (line-level refs optional)
- `evidence/*` links (L1 references in derived_from are also valid)

Output gate-b-abstraction.json to `.planning/formal/gates/` with: schema_version, generated, gate_b_score, total_entries, grounded_entries, orphaned_entries count, orphans array (with artifact, entry_id, reason), target (1.0), target_met boolean.

Create `bin/promote-gate-maturity.cjs` following the bin/ script pattern.

CLI interface (from research Pattern 5):
- `node bin/promote-gate-maturity.cjs --model "alloy/quorum-votes.als" --level SOFT_GATE` -- promote specific model
- `node bin/promote-gate-maturity.cjs --check` -- validate all models meet their current level criteria
- `node bin/promote-gate-maturity.cjs --check --fix` -- demote models that no longer meet criteria
- `node bin/promote-gate-maturity.cjs --json` -- JSON output mode

Promotion criteria:
- **ADVISORY** (default): model exists in registry. No enforcement.
- **SOFT_GATE**: model has source_layer assigned AND (if L3: all its entries have derived_from links per Gate B). Violations logged as warnings.
- **HARD_GATE**: meets SOFT_GATE criteria AND model has been TLC/Alloy-checked successfully (check-result pass in check-results.ndjson). Violations are errors.

The validateCriteria function checks:
1. For SOFT_GATE: model.source_layer is not null/undefined
2. For HARD_GATE: SOFT_GATE criteria met AND at least one passing check-result exists for the model in `.planning/formal/check-results.ndjson`

The promoteModel function:
1. Validates target level criteria
2. Rejects if target <= current level (not a promotion)
3. Updates model.gate_maturity and model.last_updated in registry
4. Writes updated registry to disk

The checkAllModels function:
1. For each model in registry, validates it meets its current gate_maturity criteria
2. Reports violations (models that don't meet their level)
3. With --fix: demotes violating models to the highest level they DO meet

Export promoteModel, validateCriteria, checkAllModels for testing.

Create `bin/promote-gate-maturity.test.cjs`:
- Unit: ADVISORY always passes validation
- Unit: SOFT_GATE requires source_layer
- Unit: HARD_GATE requires source_layer + passing check-result
- Unit: promotion rejected if target <= current
- Unit: demotion works in --check --fix mode
- Integration: reads real model-registry.json, --check reports current state
- Integration: promotes a model to SOFT_GATE and verifies registry updated
