Create `bin/test-recipe-gen.cjs` that transforms L3 failure modes into executable test recipe JSON.

**Input sources:**
- `.planning/formal/reasoning/failure-mode-catalog.json` (32 failure modes: 16 omission, 11 commission, 5 corruption)
- `.planning/formal/reasoning/hazard-model.json` (16 hazards with RPN scores)
- `.planning/formal/reasoning/risk-heatmap.json` (16 transitions with risk_tier)

**Core logic — per failure mode type:**
- OMISSION: setup={initial_state: fm.state}, input_sequence=[{event: fm.event}], expected_outcome={final_state: fm.to_state}, oracle={type: "state_assertion", check: `machine.state === '${fm.to_state}'`, failure_indicates: "Omission: transition did not fire"}
- COMMISSION: setup={initial_state: fm.state}, input_sequence=[{event: fm.event}], expected_outcome={rejection: true}, oracle={type: "guard_rejection", check: `machine.state === '${fm.state}'`, failure_indicates: "Commission: unmodeled event was accepted"}
- CORRUPTION: setup={initial_state: fm.state}, input_sequence=[{event: fm.event}], expected_outcome={final_state: fm.to_state, not_state: "any_other"}, oracle={type: "state_equality", check: `machine.state === '${fm.to_state}'`, failure_indicates: "Corruption: wrong target state reached"}

**Each recipe includes:**
- `id`: "TR-" + failure_mode.id
- `failure_mode_id`: the source failure mode ID
- `title`: human-readable description
- `setup`: initial_state + preconditions array
- `input_sequence`: array of {event, description}
- `expected_outcome`: final_state or rejection + assertions array
- `oracle`: type, check expression, failure_indicates message
- `risk_context`: severity_class, rpn (from hazard-model), risk_tier (from risk-heatmap)
- `derived_from`: array of L3 source references

**Output schema:** JSON with schema_version "1", generated timestamp, source attribution, recipes array, and summary (total_recipes, by_failure_mode counts, by_risk_tier counts).

**CLI interface:** Follow existing bin/ pattern (shebang, PROJECT_ROOT env var override, --json flag for machine-readable output, --project-root= arg). Write output to `.planning/formal/test-recipes/test-recipes.json`. Create the `test-recipes/` directory if it does not exist.

Create `bin/gate-c-validation.cjs` that verifies every L3 finding maps to at least one test recipe.

**Algorithm:**
1. Load L3 artifacts: failure-mode-catalog.json (failure_modes array)
2. Load test-recipes.json (recipes array)
3. Build lookup: recipe.failure_mode_id -> recipe[] (Map)
4. For each failure mode: check if at least one recipe references it. Mark as "validated" or "unvalidated".
5. Compute gate_c_score = validated_entries / total_entries
6. Report unvalidated entries as gaps with failure_mode_id, state, event, failure_mode type, severity_class
7. Emit check-result to NDJSON via write-check-result.cjs (formalism: "gate-c", result: pass/fail based on target >= 0.8)

**Output schema:** JSON with schema_version "1", generated, gate_c_score (0-1 fraction), total_entries, validated_entries, unvalidated_entries count, gaps array, target (0.8), target_met boolean.

**Write output to:** `.planning/formal/gates/gate-c-validation.json`

**CLI interface:** Follow existing bin/ pattern. Support --json flag (print JSON to stdout instead of human-readable). Support --project-root= arg. Exit 0 if target met, exit 1 if not.

**Follow gate-b-abstraction.cjs pattern** for output format consistency, check-result emission, and CLI flags.

**Tests (bin/gate-c-validation.test.cjs):**
