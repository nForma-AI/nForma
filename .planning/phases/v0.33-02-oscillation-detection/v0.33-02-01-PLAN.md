  <name>Task 1: Implement oscillation-detector.cjs with Mann-Kendall, credits, and cascade grace</name>
  <files>bin/oscillation-detector.cjs</files>
  <action>
Create `bin/oscillation-detector.cjs` with the following exports:

1. **Constants:**
   - `WINDOW_SIZE = 20` — number of recent entries to analyze
   - `MIN_POINTS = 5` — minimum entries before trend reporting
   - `Z_THRESHOLD = 1.96` — 95% confidence threshold for Mann-Kendall
   - `LAYER_KEYS` — import from `./solve-trend-helpers.cjs`
   - `LAYER_DEPS` — upstream dependency DAG as an object:
     ```
     {
       f_to_t: ['r_to_f', 'p_to_f'],
       t_to_c: ['f_to_t'],
       f_to_c: ['r_to_f', 'c_to_f'],
       d_to_c: ['r_to_d'],
     }
     ```
     Layers not listed have no upstream dependencies.

2. **`mannKendall(values)`** — Pure function implementing the Mann-Kendall test:
   - Input: array of numbers (residual values over time)
   - Compute S = sum of sgn(x_j - x_i) for all i < j
   - Compute variance: Var(S) = n*(n-1)*(2*n+5)/18
   - Compute Z score: Z = (S-1)/sqrt(Var(S)) if S>0, Z=0 if S===0, Z=(S+1)/sqrt(Var(S)) if S<0
   - Return `{ S, Z, trend }` where trend is:
     - `'DECREASING'` if Z <= -Z_THRESHOLD
     - `'INCREASING'` if Z >= Z_THRESHOLD
     - `'STABLE'` otherwise
   - Handle edge case: if values.length < 2, return `{ S: 0, Z: 0, trend: 'INSUFFICIENT_DATA' }`

3. **`countOscillations(values)`** — Count sign changes in consecutive deltas:
   - Compute deltas: diff[i] = values[i+1] - values[i]
   - Count sign changes: when sgn(diff[i]) !== sgn(diff[i-1]) and neither is 0
   - Return the count of sign changes

4. **`readTrendWindow(trendPath, windowSize)`** — Read last N entries from solve-trend.jsonl:
   - Read file, split by newline, parse each line as JSON (skip malformed)
   - Return last `windowSize` valid entries
   - Fail-open: return [] if file missing or empty

5. **`readLastSessionActions(sessionsDir)`** — Read actions from most recent solve-session file:
   - List files in sessionsDir matching `solve-session-*.md`
   - Read the latest one, extract layer names that had remediation actions
   - Return Set of remediated layer names
   - Fail-open: return empty Set on any error

6. **`updateVerdicts(options)`** — Main orchestration function:
   - `options.root` — project root
   - `options.trendPath` — override path (for testing)
   - `options.verdictsPath` — override path (for testing)
   - `options.sessionsDir` — override path (for testing)
   - Read trend window using `readTrendWindow()`
   - If entries < MIN_POINTS: write all layers as `INSUFFICIENT_DATA` with credits=1, return
   - Read previous verdicts from `oscillation-verdicts.json` (fail-open to empty)
   - Read last session actions for cascade grace period
   - For each of the 19 LAYER_KEYS:
     a. Extract that layer's residual series (filter out -1 fast-mode entries)
     b. If fewer than MIN_POINTS valid values: mark as `INSUFFICIENT_DATA`
     c. Run `mannKendall()` on the series
     d. Run `countOscillations()` on the series
     e. Determine `last_direction` from most recent delta (up/down/flat)
     f. Check cascade grace: if any upstream layer (from LAYER_DEPS) was remediated in last session, set `grace_period: true`
     g. Compute oscillation credits:
        - Load previous credits from existing verdict (default: 1)
        - If trend direction changed from previous verdict's `last_direction` AND NOT grace_period:
          - Decrement credit by 1
        - If credits <= 0: set `blocked: true`
     h. Override trend to `'OSCILLATING'` if oscillation_count >= 2 AND Mann-Kendall trend is `'STABLE'`
   - Write verdicts to `oscillation-verdicts.json` atomically:
     ```json
     {
       "generated": "ISO timestamp",
       "window_size": 20,
       "min_points": 5,
       "entry_count": N,
       "layers": { ... per-layer verdicts ... }
     }
     ```
   - Return the verdicts object

7. **Module exports:** `{ mannKendall, countOscillations, readTrendWindow, readLastSessionActions, updateVerdicts, WINDOW_SIZE, MIN_POINTS, Z_THRESHOLD, LAYER_DEPS, LAYER_KEYS }`

Important: Do NOT modify nf-solve.cjs in this task. Integration is in Plan 02.
