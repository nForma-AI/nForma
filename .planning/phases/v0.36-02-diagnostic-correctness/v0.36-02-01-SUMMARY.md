# Phase v0.36-02 Plan 01: Focus Filter Propagation & Missing-File Residuals Summary

**All 19 sweep layers now propagate focusSet via scoped:false flag, and 12 missing-file return paths report residual:-1 instead of false-zero**

## Performance

- **Duration:** 11 min
- **Started:** 2026-03-14T22:00:00Z
- **Completed:** 2026-03-14T22:11:00Z
- **Tasks:** 4
- **Files modified:** 3

## Accomplishments
- 17 sweep functions in nf-solve.cjs and 1 in sweepPtoF.cjs now include `scoped: false` in their detail object when focusSet is active, preventing downstream consumers from treating unscoped results as focus-filtered
- sweepFtoT() now filters its gap list by focusSet, reducing noise when running focused diagnostics
- 12 missing-file return paths changed from residual:0 to residual:-1 with `missing:` prefix reasons, preventing false convergence signals when formal artifacts are absent
- 8 new tests (4 focus, 4 missing-file) covering both source-level verification and integration behavior

3. **Task 3: Change missing-file returns from residual 0 to -1** - `6ca00f1e` (fix)

## Files Created/Modified
- `bin/nf-solve.cjs` - Added scoped flag to 17 sweep functions, focus filter to sweepFtoT, changed 12 missing-file returns to residual:-1
- `bin/sweepPtoF.cjs` - Accepts focusSet option, includes scoped flag in detail
- `bin/nf-solve.test.cjs` - 8 new tests covering TC-FOCUS and TC-MISSING categories

## Decisions Made
- Used `scoped: focusSet ? false : undefined` pattern so the key is absent from JSON when no focus is active (keeps output clean)
- sweepTtoR empty-test-files case changed to residual:-1 with skipped:true - consistent with the "missing data = unknown" principle
- Integration tests use --fast --report-only flags to avoid spawning expensive child processes during testing
