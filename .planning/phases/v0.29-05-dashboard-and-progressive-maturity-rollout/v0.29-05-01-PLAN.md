<task type="auto">
  <name>Task 1: Create cross-layer dashboard script with gate aggregation and terminal rendering</name>
  <files>bin/cross-layer-dashboard.cjs</files>
  <action>
Create `bin/cross-layer-dashboard.cjs` that aggregates four data sources and renders a terminal dashboard.

**Data collection (default: re-run gates for freshness; `--cached` to read existing JSON):**
1. Use the `spawnTool` pattern from nf-solve.cjs (child_process.spawnSync with PROJECT_ROOT env, 60s timeout). Accept exit code 0 OR 1 as valid (gate scripts exit 1 when target not met but still produce valid JSON stdout).
2. Spawn `gate-a-grounding.cjs --json` -> extract `grounding_score` (float 0-1), `target`, `target_met`, `explained`, `total`
3. Spawn `gate-b-abstraction.cjs --json` -> extract `gate_b_score` (float 0-1), `target`, `target_met`, `grounded_entries`, `total_entries`
4. Spawn `gate-c-validation.cjs --json` -> extract `gate_c_score` (float 0-1), `target`, `target_met`, `validated_entries`, `total_entries`
5. Read `instrumentation-map.json` from `.planning/formal/evidence/` -> extract `coverage.coverage_pct` (float 0-100)
6. Optionally spawn `promote-gate-maturity.cjs --check --json` -> extract `total`, `by_level` distribution

**Graceful degradation:** If any gate script fails or artifact is missing, show "N/A" for that metric instead of crashing. Log the error to stderr.

**Terminal rendering:** Follow nf-solve.cjs display conventions:
- Plain text with box-drawing characters (no ANSI colors, no chalk)
- Title: "Cross-Layer Alignment Dashboard"
- Sections: Layer Health (L1 coverage %, Gate A/B/C scores as percentages), health indicators ([PASS]/[WARN]/[FAIL]), detail counts (e.g., "29678/36047 traces explained")
- Gate maturity distribution section: ADVISORY/SOFT_GATE/HARD_GATE counts
- Composite summary line at bottom

**CLI flags:**
- `--json` — Output aggregated data as JSON (for machine consumption and testing)
- `--cached` — Read existing gate JSON files from `.planning/formal/gates/` instead of re-running gate scripts (fast path)
- No flag (default) — Re-run all gate scripts for fresh data, then display

**Health indicator thresholds:**
- L1 Coverage: PASS >= 50%, WARN >= 40%, FAIL < 40%
- Gate A: PASS >= 80% (its target), WARN >= 64%, FAIL < 64%
- Gate B: PASS >= 100% (its target), WARN >= 80%, FAIL < 80%
- Gate C: PASS >= 80% (its target), WARN >= 64%, FAIL < 64%

**Exit code:** 0 if all gates meet targets, 1 if any gate is below target or unavailable.

Do NOT import gate modules directly -- use child_process spawn for isolation (gate scripts have heavy initialization). Do NOT add any npm dependencies -- use only Node.js built-ins (fs, path, child_process).
