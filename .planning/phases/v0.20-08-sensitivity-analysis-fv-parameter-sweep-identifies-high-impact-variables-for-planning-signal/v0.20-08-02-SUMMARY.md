# Plan 02 Summary — run-sensitivity-sweep.cjs (SENS-01)

## What Was Built

`bin/run-sensitivity-sweep.cjs` — sensitivity sweep runner that varies key model parameters across ≥3 values each, records outcome deltas in `formal/sensitivity-report.ndjson`.

## Sweep Parameters

| Parameter | Model | Values | Baseline |
|-----------|-------|--------|----------|
| MaxSize   | TLA+  | 1, 2, 3 | 3 |
| tp_rate   | PRISM | 0.5, 0.75, 0.95 | 0.85 |

## Key Design Decisions

- **Direct `fs.appendFileSync` writes** (not `writeCheckResult()`) — avoids coupling sensitivity surface to VALID_FORMALISMS and check-results.ndjson pipeline
- **`surface: 'sensitivity'`** — distinct from `'safety'`/`'liveness'`/`'ci'` surfaces; reports appear in `sensitivity-report.ndjson` only
- **`check_id: 'sens:tla-maxsize'` / `'sens:prism-tp-rate'`** — namespaced for quick filter
- **Graceful degradation**: TLC jar absent → `result=inconclusive`, `triage_tags=['no-tlc']`; PRISM binary absent → `result=inconclusive`, `triage_tags=['no-prism']`; always exits 0
- **Delta tracking**: records include `metadata.delta` (`stable`, `flip-to-fail`, `unknown`) vs baseline value — enables outcome flip counting in the report generator
- **TLC temp cfg override**: reads MCsafety.cfg, regex-replaces `MaxSize = N`, writes to `os.tmpdir()`, cleans up after each run
