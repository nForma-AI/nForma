# Plan 03 Summary — sensitivity-report.cjs (SENS-03)

## What Was Built

`bin/sensitivity-report.cjs` — report generator that reads `formal/sensitivity-report.ndjson`, groups records by parameter, ranks by outcome-flip count, and writes `formal/sensitivity-report.md`.

## Key Design Decisions

- **Fail-open**: if NDJSON is empty or missing, writes "No Data Available" section to MD and exits 0
- **No-flips branch**: if all records are `inconclusive` or `stable`, writes "No Outcome Flips Detected" with a per-parameter summary (count + reason)
- **Ranked list**: parameters sorted descending by flip count; each entry shows description, flip values, code path, recommended test cases, monitoring metrics
- **`PARAM_ANNOTATIONS` map**: hardcoded code-path annotations for MaxSize, tp_rate, MaxDeliberation, unavail — provides actionable context even before tools are installed
- **`SENSITIVITY_REPORT_PATH` / `SENSITIVITY_MD_PATH` env vars**: both paths overridable — enables test isolation without touching repo `formal/` directory
- **`process.stderr` for progress**: stdout clean; all operational messages go to stderr
