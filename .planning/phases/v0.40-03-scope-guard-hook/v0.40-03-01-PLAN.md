Create `hooks/nf-scope-guard.js` following the nf-destructive-git-guard.js pattern exactly. The hook is a PreToolUse warn-only advisory.

**Hook structure** (use nf-destructive-git-guard.js as the template):

```
'use strict';
const fs = require('fs');
const path = require('path');
const { spawnSync } = require('child_process');
const { loadConfig, shouldRunHook, validateHookInput } = require('./config-loader');
```

**Five helper functions to export:**
1. `getGitRoot(cwd)` — spawnSync git rev-parse --show-toplevel, return null if not git repo. Copy verbatim from nf-destructive-git-guard.js.
2. `getCurrentBranch(cwd)` — spawnSync git rev-parse --abbrev-ref HEAD; return null if status !== 0, if error, or if result is empty string or "HEAD" (detached state).
3. `readScopeContract(gitRoot)` — reads `path.join(gitRoot, '.claude', 'scope-contract.json')`. Return null if file missing. Try/catch JSON.parse — return null on any error (fail-open). Return parsed object.
4. `extractTargetPath(toolInput)` — handles all three tools:
   - Edit: return `toolInput.file_path`
   - Write: return `toolInput.file_path`
   - MultiEdit: return `toolInput.files[0].file_path` if `toolInput.files` is a non-empty array and first entry has `file_path`, else null
   - Return null if toolInput is missing/not object
5. `isFileInScope(targetPath, branchEntry)` — receives the branch entry object from the contract (not raw patterns array). Logic:
   - If branchEntry is null/undefined, return true (in-scope, no contract = no restriction)
   - Extract scope patterns: the v0.40-02 quick.md schema uses `approach` and `out_of_scope` fields (NOT `scope_patterns`). Derive scope from `out_of_scope`: if `out_of_scope` is a non-empty array of strings, check if targetPath matches any `out_of_scope` entry as a literal prefix. If it does → out of scope (return false). If `out_of_scope` is empty or not present → all paths are in scope (return true).
   - Normalize paths: strip trailing slashes from both targetPath and each pattern before comparison.
   - Return true (in scope) or false (out of scope).

**main() function flow:**
1. Read stdin (same pattern as git guard: `let raw = ''; process.stdin.on('data'...) process.stdin.on('end'...)`)
2. Wrap in try/catch; exit(0) on any error (fail-open)
3. If empty stdin: exit(0)
4. JSON.parse(raw) → input
5. validateHookInput(eventType, input): if !valid → stderr warning + exit(0)
6. cwd = input.cwd || process.cwd()
7. gitRoot = getGitRoot(cwd); if null → exit(0)
8. loadConfig(gitRoot); check shouldRunHook('nf-scope-guard', profile); if not → exit(0)
9. toolName = input.tool_name || input.toolName || ''; if not in ['Edit', 'Write', 'MultiEdit'] → exit(0)
10. targetPath = extractTargetPath(input.tool_input); if null → exit(0)
11. branch = getCurrentBranch(cwd); if null → exit(0) (detached HEAD = no-op)
12. scopeContract = readScopeContract(gitRoot); if null → exit(0) (no contract = no-op, SCOPE-03)
13. branchEntry = scopeContract[branch]; if !branchEntry → exit(0) (no entry for this branch = no-op)
14. inScope = isFileInScope(targetPath, branchEntry)
15. If inScope → exit(0)
16. If !inScope:
    - Build warning text: `[nf-scope-guard] WARNING: Editing '${targetPath}' is outside declared scope. Out-of-scope items: ${JSON.stringify(branchEntry.out_of_scope || [])}. Task approach: "${branchEntry.approach || 'unknown'}". Consider: is this edit within the task's intended scope?`
    - Check if `.planning/current-activity.json` exists in gitRoot
    - If yes (nForma active): write JSON to stdout: `{ hookSpecificOutput: { hookEventName: 'PreToolUse', additionalContext: warningText } }`
    - If no: write to stderr
17. exit(0) — always (warn-only, SCOPE-02)

End with:
```javascript
if (require.main === module) main();
module.exports = { getGitRoot, getCurrentBranch, readScopeContract, isFileInScope, extractTargetPath };
```
