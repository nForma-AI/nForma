# Phase v0.40-03 Plan 01: Scope Guard Hook Summary

**One-liner:** PreToolUse hook that warns Claude when Edit/Write/MultiEdit calls target files outside the scope declared in `.claude/scope-contract.json` (branch-keyed, warn-only advisory, fail-open).

## Overview

Plan v0.40-03-01 implemented the nf-scope-guard PreToolUse hook, completing the intent declaration loop that began in Phase v0.40-02:

1. **Phase v0.40-02** (prior): `quick.md` automatically writes a scope contract to `.claude/scope-contract.json` for each quick task (Step 2.7), capturing the task's declared approach and out-of-scope items.

2. **Phase v0.40-03-01** (this plan): The scope guard hook reads that contract and emits a non-blocking warning via `additionalContext` whenever Claude edits a file that matches an out-of-scope pattern.

**Result:** Claude now has real-time scope guidance at edit time — helping prevent accidental scope creep while maintaining flexibility (warns, never blocks).

## Technical Notes

### Scope Contract Schema (from v0.40-02)

The hook expects a scope contract at `.claude/scope-contract.json` with structure:

```json
{
  "quick-task-branch-name": {
    "task_id": 333,
    "task_description": "...",
    "approach": "...",
    "out_of_scope": ["bin/", "docs/", ".github/"],
    "branches_affected": ["quick-task-branch-name"],
    "created_at": "2026-03-19T...",
    "planner_model": "opus",
    "created_by": "quick-orchestrator"
  }
}
```

The hook reads `out_of_scope` as the source of truth (not `scope_patterns` from ROADMAP — v0.40-02 quick.md Step 2.7 uses `out_of_scope`).

### Hook Priority

Set to 50 (Normal) — same as destructive-git-guard and mcp-dispatch-guard. Hooks with priority 50 run after critical safety hooks (1000) but before low-priority hooks (10).

### Fail-Open Semantics

The hook never blocks tool calls. Error paths:
- Empty stdin → exit 0
- Invalid JSON → exit 0
- Missing contract → exit 0
- Detached HEAD (no branch) → exit 0
- Not in git repo → exit 0
- Hook profile disabled → exit 0
- Any catch block → exit 0

This design ensures the hook is purely advisory with zero risk of breaking workflows.

### Hook Profile Placement

- **Minimal profile:** NOT included (scope guard requires reading contract files + branch detection; not suitable for constrained environments)
- **Standard profile:** Included (default for all users)
- **Strict profile:** Included (superset of standard)
