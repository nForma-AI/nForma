## Summary

Phase v0.15-03 is a pure filesystem housekeeping task. Twenty-two legacy numeric phase directories (`18-cli-foundation` through `39-rename-and-migration`) currently live under `.planning/phases/`. They were created before QGSD adopted the `v0.X-YY-name` versioned naming convention. The goal is to move them to `.planning/archive/legacy/` so `.planning/phases/` contains only versioned directories.

**Important context about W007:** The SAFE-02 requirement was authored before Phase v0.15-01 was complete. At that time, the health checker's ROADMAP extractor regex (`/#{2,4}\s*Phase\s+(\d+(?:\.\d+)?)\s*:/gi`) did NOT match the checkbox-format ROADMAP entries for phases 18–39, causing W007 false positives. After v0.15-01 fixed the phasePattern to also match checkbox-format entries (`[-*]\s+\[[^\]]\]\s+\*{1,2}`), W007 is now already 0 for all 22 legacy dirs. They match because ROADMAP.md contains their `[x] **Phase 18: ...` entries.

After the move, since the dirs are removed from `.planning/phases/`, W007 cannot fire for them (W007 requires a dir to be on disk AND not in ROADMAP). Success criteria 2 ("zero W007 for archived dirs") is satisfied both before and after the move. The W006 code ("in ROADMAP, not on disk") WILL fire for phases 18–39 post-move, which is the same accepted behavior as phases 1–17 (already archived in `.planning/milestones/v0.2-phases/`).

**No code changes** to `gsd-tools.cjs` are needed. No install sync required. This is a `mkdir` + `mv` operation plus test scaffolding.

**Primary recommendation:** Create `.planning/archive/legacy/`, move all 22 legacy dirs in one shell command, write SAFE-02 test cases to `gsd-tools.test.cjs` (Wave 0 scaffolding), then verify with `validate health`.

### Pattern: Existing Milestone Archive Precedent

The project already archives phases via `milestone complete --archive-phases` which creates `.planning/milestones/v0.X-phases/` subdirectories. Phases 1–17 were archived to `.planning/milestones/v0.2-phases/` and phases v0.7-01..04 to `.planning/milestones/v0.7-phases/`. The `.planning/archive/legacy/` path is a new location specified by the requirement — not routed through the existing milestone archive tooling.

The `getArchivedPhaseDirs()` function in `gsd-tools.cjs` (line 4177) only looks for `v*-phases` directories in `.planning/milestones/`. It does NOT scan `.planning/archive/legacy/`. This means the archived dirs will NOT appear in `phases list --include-archived` or `history-digest` commands. This is acceptable — they are pre-versioning era artifacts with no active role in QGSD operations.

### Anti-Patterns to Avoid

- **Do not use `milestone complete --archive-phases`**: That command moves to `.planning/milestones/vX.Y-phases/`. SAFE-02 requires `.planning/archive/legacy/` specifically.
- **Do not remove phases 18–39 from ROADMAP.md**: This would create a different mess (completed history removed). The W006 warnings that appear after archiving are the accepted behavior, matching phases 1–17.
- **Do not update `getArchivedPhaseDirs()` to scan `.planning/archive/legacy/`**: Out of scope for this phase. The requirement is about archiving, not extending the scan.

## Common Pitfalls

### Pitfall 1: Partial Move on Shell Error
**What goes wrong:** If a glob pattern matches no files, `mv` on some shells exits non-zero and stops. E.g., if any dir was already moved.
**Why it happens:** Shell glob expansion fails silently or noisily depending on shell options.
**How to avoid:** Run `mkdir -p .planning/archive/legacy` first, then move dirs one at a time or verify all 22 exist before running `mv`.
**Warning signs:** `mv: rename .planning/phases/18-*: No such file or directory`

### Pitfall 2: W006 Confusion After Move
**What goes wrong:** After moving dirs out of `.planning/phases/`, 22 new W006 warnings appear ("Phase 18..39 in ROADMAP.md but no directory on disk"). Team might think the phase broke something.
**Why it happens:** W006 fires for any phase in ROADMAP not on disk. Phases 18–39 remain in ROADMAP.
**How to avoid:** Document this as expected behavior. Phases 1–17 already have the same W006 state (archived in milestones). The requirement only specifies zero W007, not zero W006.
**Warning signs:** Health output shows 22 new W006 entries for phases 18..39 — this is correct.

### Pitfall 3: Assuming getArchivedPhaseDirs() Will Find the Moved Dirs
**What goes wrong:** `phases list --include-archived` and `history-digest` won't find the moved dirs after archiving.
**Why it happens:** `getArchivedPhaseDirs()` only scans `.planning/milestones/v*-phases/` — not `.planning/archive/legacy/`.
**How to avoid:** Accept this limitation. The 22 dirs are pre-versioning era artifacts not used in current QGSD operations. Extending `getArchivedPhaseDirs()` is out of scope.
**Warning signs:** `phases list --include-archived` returns 0 results for phases 18–39 after move — this is expected.

### Pitfall 4: Sequential Numbering Check
**What goes wrong:** After moving phases 18–39 off disk, the sequential integer phase check in `validate consistency` will see an empty `integerPhases` set (no integers on disk at all), so no gap warnings fire. This is correct.
**Why it happens:** The sequence check only runs on integer-named dirs in `.planning/phases/`. With 18–39 removed, no integers remain.
**How to avoid:** No action needed — this is the desired outcome.

// Source: qgsd-core/bin/gsd-tools.test.cjs (following SAFE-01 pattern)
describe('SAFE-02: legacy numeric phase dirs archived', () => {

    );
    fs.writeFileSync(
      path.join(tmpDir, '.planning', 'STATE.md'),
      '# Project State\n\n## Current Position\n\nPhase: v0.15-03\nPlan: 1\nStatus: in-progress\n'
    );
    // Create a versioned phase dir so STATE.md ref doesn't trigger W002
    fs.mkdirSync(path.join(tmpDir, '.planning', 'phases', 'v0.15-03-test'), { recursive: true });
  });

  afterEach(() => {
    cleanup(tmpDir);
  });

  test('SAFE-02-TC-01: W007 not emitted for numeric phases present in both ROADMAP and disk', () => {
    // Simulate pre-archive state: dir on disk + in ROADMAP
    fs.mkdirSync(path.join(tmpDir, '.planning', 'phases', '18-cli-foundation'), { recursive: true });
    const result = runGsdTools('validate health', tmpDir);
    const data = JSON.parse(result.output);
    const w007 = (data.warnings || []).some(
      i => i.code === 'W007' && i.message && i.message.includes('18')
    );
    assert.ok(!w007, `SAFE-02 pre-condition: W007 false positive for phase 18: ${JSON.stringify(data.warnings)}`);
  });

  test('SAFE-02-TC-02: W007 not emitted for numeric phases after move to archive/legacy/', () => {
    // Simulate post-archive state: dir NOT in phases/, in archive/legacy/
    // Numeric phase 18 is in ROADMAP but NOT on disk under phases/
    // → W006 fires (in ROADMAP, not on disk) but W007 must NOT fire
    const result = runGsdTools('validate health', tmpDir);
    const data = JSON.parse(result.output);
    const w007 = (data.warnings || []).some(
      i => i.code === 'W007' && i.message && i.message.includes('18')
    );
    assert.ok(!w007, `SAFE-02-TC-02: W007 must not fire for archived phase 18: ${JSON.stringify(data.warnings)}`);
  });

  test('SAFE-02-TC-03: archive/legacy/ contains numeric phase dirs after archive', () => {
    // Simulate the archive dir creation and move

    // Verify archive dir exists and contains the moved dir
    const archiveExists = fs.existsSync(path.join(tmpDir, '.planning', 'archive', 'legacy'));
    const dirMoved = fs.existsSync(path.join(tmpDir, '.planning', 'archive', 'legacy', '18-cli-foundation'));
    const notInPhases = !fs.existsSync(path.join(tmpDir, '.planning', 'phases', '18-cli-foundation'));
    assert.ok(archiveExists, 'SAFE-02-TC-03: .planning/archive/legacy/ must exist');
    assert.ok(dirMoved, 'SAFE-02-TC-03: 18-cli-foundation must exist in archive/legacy/');
    assert.ok(notInPhases, 'SAFE-02-TC-03: 18-cli-foundation must NOT be in phases/ after archive');
  });
});
