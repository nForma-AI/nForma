Create bin/invariant-catalog.cjs following the existing bin/ CJS script pattern (#!/usr/bin/env node, 'use strict', PROJECT_ROOT env var, --json flag support).

The script must extract invariants from THREE sources:

**Source A -- TLC .cfg files (80+ INVARIANT declarations):**
- Read all MC*.cfg files from .planning/formal/tla/
- Parse INVARIANT/INVARIANTS lines and block format (INVARIANTS keyword followed by indented names until next keyword or blank line)
- For each, record: name, source "tla_cfg", source_file, type "declared", formalism "tla", config (cfg filename without .cfg)
- Handle both single-line "INVARIANT Name" and block "INVARIANTS" format

**Source B -- spec/*/invariants.md files (15 files):**
- Read all spec/*/invariants.md files from .planning/formal/
- Parse ## section headers to get invariant names, extract **Property:** backtick content for property_expression
- For each, record: name, source "spec_invariants_md", source_file, type "declared", formalism (check for PROPERTY/liveness keywords -> "liveness", else "safety"), property_expression

**Source C -- Observed invariants mined from traces:**
- Read trace-corpus-stats.json for session data
- Implement 3-5 hardcoded pattern checks (per research recommendation, NOT a general mining framework):
  1. "quorum_start always precedes quorum_complete within a session"
  2. "circuit_break events only occur during active quorum sessions"
  3. "no session has more quorum_complete events than quorum_start events"
- For each observed invariant that holds, record: name, source "trace_mining", type "observed", property_expression, confidence "high", evidence_sessions count

**Deduplication:**
- Deduplicate by (invariant_name, source_model_name) pair. Multiple .cfg references to the same invariant in the same model collapse to one catalog entry with a check_references array.
- Report total_raw vs total_deduplicated in the output.

**Output format (invariant-catalog.json):**
```json
{
  "schema_version": "1",
  "generated": "ISO timestamp",
  "invariants": [ { name, source, source_file, type, formalism, ... } ],
  "summary": { "total_raw": N, "total_deduplicated": N, "by_type": { "declared": N, "observed": N }, "by_source": { "tla_cfg": N, "spec_invariants_md": N, "trace_mining": N } }
}
```

- Integration test: reads real .cfg files and invariants.md, verifies total_deduplicated is between 50-120 (reasonable range, not hardcoded exact count)
- Test that no invariant has source "unclassified"
- Test observed invariant mining produces at least 1 invariant

Run the script to generate initial invariant-catalog.json.

4. For each entry, create a structured record:
   - id: integer from # column
   - source: source column value
   - name: name column value
   - type: type column value (invariant, assume, constant, etc.)
   - coverage: coverage column value (uncovered, covered, partial)
   - proposed_metric: proposed metric column with backtick stripping
   - metric_type: metric type column value
   - validation_status: "untested" (default for all -- will be updated by future phases)
   - linked_l2_states: [] (empty array -- will be populated when cross-referencing with invariant catalog in Gate A or later)
5. Report parsing statistics: total_parsed, by_type breakdown, by_coverage breakdown

**Output format (assumption-register.json):**
```json
{
  "schema_version": "1",
  "generated": "ISO timestamp",
  "assumptions": [ { id, source, name, type, coverage, proposed_metric, metric_type, validation_status, linked_l2_states } ],
  "summary": { "total_parsed": N, "by_type": {...}, "by_coverage": {...}, "by_validation_status": { "untested": N, "validated": 0, "invalidated": 0 } }
}
```
