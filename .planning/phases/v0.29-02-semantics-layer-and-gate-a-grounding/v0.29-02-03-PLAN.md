Create bin/gate-a-grounding.cjs following the existing bin/ CJS script pattern.

**CRITICAL: Lock the "explains" definition FIRST before any scoring logic.**

A trace event is "explained" if and only if BOTH conditions hold:
1. Its action maps to a known entry in event-vocabulary.json (vocabulary_mapped = true)
2. Its XState transition is EITHER:
   a. Validated by fresh actor replay (xstate_valid = true), OR
   b. Correctly skipped as a mid-session event under H1 methodology (methodology_skip = true)

The H1 methodology skip applies when: event.phase !== 'IDLE' AND event.action !== 'quorum_start'. Mid-session events cannot be validated with a fresh actor (which starts in IDLE), so they are explained by methodology, not by replay.

**Unexplained traces fall into exactly one of three buckets:**
- instrumentation_bug: action NOT in vocabulary (e.g., quorum_fallback_t1_required = ~5,649 events)
- model_gap: action IS in vocabulary AND mapToXStateEvent returns a valid event BUT fresh actor replay produces wrong state
- genuine_violation: event violates a declared invariant from invariant-catalog.json (cross-reference with invariant catalog)

**Implementation steps:**
1. Read conformance-events.jsonl via planning-paths.cjs
2. Read event-vocabulary.json for vocabulary check
3. Read invariant-catalog.json (from Plan 01) for violation classification
4. Read mismatch-register.jsonl (from Plan 02) for known disagreements
5. Import mapToXStateEvent from validate-traces.cjs
6. Import createActor and nfWorkflowMachine from dist/machines/nf-workflow.machine.js
7. For each event, apply the explains definition:
   - Not in vocabulary -> instrumentation_bug (unexplained)
   - In vocabulary but mapToXStateEvent returns null -> instrumentation_bug (in vocab but no XState mapping)
   - Mid-session event (H1 skip) -> explained (methodology_skip)
   - Fresh actor replay matches -> explained (xstate_valid)
   - Fresh actor replay does not match -> model_gap (unexplained)
   - Check model_gap events against invariant catalog -> reclassify as genuine_violation if an invariant is violated
8. Compute grounding_score = explained_count / total_traces
9. Emit a check result via write-check-result.cjs with source_layer "L2" and gate "A"

**EARLY WARNING:** If grounding_score < 0.50 during development, STOP and re-examine the explains definition. The prior regression (69% -> 0% -> 17.9%) was caused by a wrong definition. Expected score: ~84.2%.
