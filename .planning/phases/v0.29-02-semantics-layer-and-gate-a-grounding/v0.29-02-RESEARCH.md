# Phase v0.29-02: Semantics Layer and Gate A Grounding - Research

**Researched:** 2026-03-06
**Domain:** Formal verification semantic layer (L2) -- invariant cataloging, mismatch tracking, assumption registration, observed FSM derivation, and quantitative grounding gate
**Confidence:** HIGH

## Summary

Phase v0.29-02 builds on the Layer 1 evidence artifacts produced by v0.29-01 (35,825 events across 349 sessions, 567 assumption gaps, 92 classified models, event vocabulary with 7 action types) to create Layer 2 semantics infrastructure. The phase has four L2 scripts and one gate script, producing five new JSON/JSONL artifacts in `.planning/formal/semantics/` and `.planning/formal/gates/`.

The hardest design problem is Gate A's "explains" definition. STATE.md explicitly warns: "Gate A metric definition is hardest design decision -- 'explains' must be formally specified; prior attempt regressed (69% -> 0% -> 17.9%)." The root cause of prior regression was that `validate-traces.cjs` uses a fresh XState actor per event, which means mid-session events (phase !== 'IDLE') cannot be validated standalone. The H1 methodology fix already skips these events (returning null = valid). Gate A must build on this corrected methodology: a trace is "explained" if (a) it maps to a known vocabulary action, AND (b) its XState transition is either validated by the actor or correctly skipped as a mid-session event. Unexplained traces are those with unmappable actions (`quorum_fallback_t1_required` = 5,649 events) or genuine state mismatches after replay.

The invariant catalog (SEM-01) aggregates from two sources: 15 `spec/*/invariants.md` files (declared invariants) containing liveness/fairness properties, and 80+ TLC `.cfg` INVARIANT declarations (TypeOK, semantic invariants). Observed invariants are mined from trace patterns (e.g., "quorum_start always precedes quorum_complete within a session"). The mismatch register (SEM-02) extends the existing `.divergences.json` pattern into a persistent JSONL ledger. The assumption register (SEM-03) structurally catalogs the 567 entries in `assumption-gaps.md`. The operational FSM derivation (SEM-04) uses the XState trace walker to build an observed state machine from traces, comparable to the hand-written `nf-workflow.machine.ts`.

**Primary recommendation:** Define "explains" as `vocabulary_mapped AND (xstate_valid OR methodology_skip)` BEFORE implementing any Gate A logic. Build the invariant catalog first (it has no dependencies on other SEM scripts), then the mismatch register (depends on understanding divergences), then the assumption register (depends on understanding the invariant catalog structure), and finally Gate A (consumes all three).

// Mine trace patterns that hold across all sessions
// Example observed invariant: "quorum_start always precedes quorum_complete in a session"
function mineObservedInvariants(sessions) {
  const observed = [];
  // Check: quorum_start always before quorum_complete
  let startBeforeComplete = true;
  for (const session of sessions) {
    const actions = session.events.map(e => e.action);
    const firstComplete = actions.indexOf('quorum_complete');
    const firstStart = actions.indexOf('quorum_start');
    if (firstComplete >= 0 && (firstStart < 0 || firstStart > firstComplete)) {
      startBeforeComplete = false;
      break;
    }
  }
  if (startBeforeComplete) {
    observed.push({
      name: 'StartPrecedesComplete',
      source: 'trace_mining',
      type: 'observed',
      property_expression: 'In every session, quorum_start occurs before quorum_complete',
      confidence: 'high',
      evidence_sessions: sessions.length,
    });
  }
  return observed;
}

### Pattern 4: Gate A "Explains" Definition
**What:** A trace is "explained" if it meets BOTH conditions:
1. Its action maps to a known vocabulary entry (event-vocabulary.json)
2. Its XState transition is either (a) validated by fresh actor replay, OR (b) correctly skipped as a mid-session event (H1 methodology)

**Critical:** Unexplained traces fall into exactly one of three buckets:
- `instrumentation_bug`: Action not in vocabulary (e.g., `quorum_fallback_t1_required` -- 5,649 events)
- `model_gap`: Action is in vocabulary but XState transition produces wrong state
- `genuine_violation`: Event violates a declared invariant

**Grounding score formula:**
```
explained = vocabulary_mapped AND (xstate_valid OR methodology_skip)
grounding_score = explained_count / total_traces
target >= 0.80
```

**Current baseline estimate:**
- Total traces: 35,825
- Unmappable (instrumentation_bug): ~5,649 (quorum_fallback_t1_required)
- State mismatches: ~10 (from .divergences.json)
- Methodology skips (valid): majority of mid-session events
- Estimated score: ~(35,825 - 5,649 - 10) / 35,825 = ~84.2% (above 80% target)

### Pattern 5: Observed FSM Derivation
**What:** Build a state machine from observed trace transitions, then diff against the hand-written XState machine.
**When to use:** SEM-04 implementation.
**Algorithm:**
```javascript
// 1. Replay all traces through xstate-trace-walker to extract observed transitions
// 2. Build adjacency map: { fromState -> { event -> toState } }
// 3. Compare against nf-workflow.machine.js states/transitions
// 4. Report: missing_in_observed (model has but traces don't cover),
//            missing_in_model (traces show but model doesn't have),
//            matching (both agree)

function deriveObservedFSM(events) {
  const transitions = new Map(); // "fromState" -> Map("eventType" -> { toState, count })
  const { mapToXStateEvent } = require('./validate-traces.cjs');
  const { createActor, nfWorkflowMachine } = require('../dist/machines/nf-workflow.machine.js');

  for (const event of events) {
    const xEvt = mapToXStateEvent(event);
    if (!xEvt) continue;
    const actor = createActor(nfWorkflowMachine);
    actor.start();
    const beforeState = actor.getSnapshot().value;
    actor.send(xEvt);
    const afterState = actor.getSnapshot().value;
    actor.stop();

    const key = String(beforeState);
    if (!transitions.has(key)) transitions.set(key, new Map());
    const tMap = transitions.get(key);
    const tKey = xEvt.type;
    if (!tMap.has(tKey)) tMap.set(tKey, { toState: String(afterState), count: 0 });
    tMap.get(tKey).count++;
  }
  return transitions;
}
```

### Anti-Patterns to Avoid
- **Redefining "explains" without considering H1 methodology:** The prior regression (69% -> 0% -> 17.9%) happened because mid-session events were counted as divergences. The H1 fix in validate-traces.cjs already handles this. Gate A MUST use the same methodology or delegate to validate-traces.
- **Building Gate A before defining the metric:** The metric definition must be locked in task 1 before any implementation. The 80% target is achievable with current data but only if `instrumentation_bug` traces are counted as "unexplained" (not explained).
- **Parsing TLA+ AST for invariants:** The TLA+ specifications are complex, but invariant extraction only needs `.cfg` file parsing (simple line-by-line). Do NOT attempt to parse `*.tla` files for invariant definitions -- the `.cfg` files are the authoritative source of which invariants are checked.
- **Treating assumption-gaps.md as structured data:** The file is markdown with a pipe-delimited table. Parse it as markdown, not as JSON. The 567 entries have columns: #, Source, Name, Type, Coverage, Proposed Metric, Metric Type.
- **Creating a new XState machine for observed FSM:** The observed FSM should be a JSON data structure (states + transitions + counts), NOT an XState machine definition. It is meant for comparison, not execution.

## Common Pitfalls

### Pitfall 1: Gate A Score Regression (THE Critical Pitfall)
**What goes wrong:** The grounding score drops to near-zero because the "explains" definition is too strict, counting methodology-skipped events as unexplained.
**Why it happens:** STATE.md documents a prior regression from 69% -> 0% -> 17.9%. The root cause was counting mid-session events as divergences when they cannot be validated with a fresh actor.
**How to avoid:** Define "explains" as `vocabulary_mapped AND (xstate_valid OR methodology_skip)`. The H1 methodology fix in validate-traces.cjs already classifies mid-session events as valid (null expectedState = skip). Gate A must use the SAME classification, not reimplement it.
**Warning signs:** grounding_score < 50% in early testing -- this indicates the "explains" definition is wrong.

### Pitfall 2: Double-Counting Invariants
**What goes wrong:** The same invariant appears multiple times in the catalog because it is declared in both a .cfg file AND a spec/*/invariants.md file.
**Why it happens:** TypeOK appears in ~25 .cfg files and may appear in invariants.md. Many invariants are declared once but checked in multiple configs.
**How to avoid:** Deduplicate by (invariant_name, source_model_name) pair. A unique invariant is defined by its name + the model it belongs to. Multiple .cfg references to the same invariant in the same model collapse to one catalog entry with multiple check references.
**Warning signs:** Catalog shows 200+ invariants when only ~80 unique ones exist.

### Pitfall 3: Assumption Register Schema Mismatch
**What goes wrong:** Parsing assumption-gaps.md fails because the markdown table has inconsistent whitespace or the column count varies.
**Why it happens:** The file is generated by a script but may have been manually edited. The pipe-delimited format is fragile.
**How to avoid:** Use a robust line parser: split on `|`, trim each cell, skip lines that don't produce the expected column count. The 567 entries should parse down to exactly 567 assumption records (minus header/separator rows).
**Warning signs:** Parser produces fewer than 500 or more than 600 entries.

### Pitfall 4: Observed FSM Completeness Illusion
**What goes wrong:** The observed FSM shows high coverage because it only considers events that mapToXStateEvent handles. The 5,649 unmapped events represent transitions the FSM does NOT cover.
**Why it happens:** mapToXStateEvent returns null for unknown actions, and these events are silently excluded from the FSM.
**How to avoid:** Track two metrics: (a) observed FSM coverage of the hand-written model (which transitions are exercised), and (b) trace coverage of the vocabulary (what % of events map to known actions). Report both.
**Warning signs:** Observed FSM shows 100% model coverage but vocabulary coverage is only 84%.

### Pitfall 5: JSONL vs JSON Confusion
**What goes wrong:** The mismatch register is specified as JSONL but gets written as a JSON array, or vice versa.
**Why it happens:** JSONL (one JSON object per line) and JSON array (`[{...}, {...}]`) look similar but have different append semantics.
**How to avoid:** The mismatch register MUST be JSONL (one entry per line, no wrapping array). This enables append-only updates in future phases without rewriting the whole file. Use `fs.appendFileSync` for new entries.
**Warning signs:** File starts with `[` instead of `{`.

### Extracting TLC INVARIANT declarations from .cfg files
```javascript
// Source: derived from direct inspection of 25+ MC*.cfg files
const fs = require('fs');
const path = require('path');

const TLA_DIR = path.join(ROOT, '.planning', 'formal', 'tla');

function extractCfgInvariants() {
  const invariants = [];
  const cfgFiles = fs.readdirSync(TLA_DIR).filter(f => f.startsWith('MC') && f.endsWith('.cfg'));

  for (const cfgFile of cfgFiles) {
    const content = fs.readFileSync(path.join(TLA_DIR, cfgFile), 'utf8');
    const lines = content.split('\n');
    let inInvariantBlock = false;

    for (const line of lines) {
      const trimmed = line.trim();
      // Handle both "INVARIANT Name" and "INVARIANTS" block format
      const singleMatch = trimmed.match(/^INVARIANT\s+(\S+)/);
      if (singleMatch) {
        invariants.push({
          name: singleMatch[1],
          source_file: path.join(TLA_DIR, cfgFile),
          config: cfgFile.replace(/\.cfg$/, ''),
        });
        continue;
      }
      if (trimmed === 'INVARIANTS') {
        inInvariantBlock = true;
        continue;
      }
      if (inInvariantBlock) {
        if (trimmed === '' || trimmed.startsWith('\\*') || /^[A-Z_]+\s/.test(trimmed)) {
          inInvariantBlock = false;
          continue;
        }
        invariants.push({
          name: trimmed,
          source_file: path.join(TLA_DIR, cfgFile),
          config: cfgFile.replace(/\.cfg$/, ''),
        });
      }
    }
  }
  return invariants;
}
```

### Parsing assumption-gaps.md table
```javascript
// Source: derived from direct inspection of assumption-gaps.md format
function parseAssumptionGaps(filePath) {
  const content = fs.readFileSync(filePath, 'utf8');
  const lines = content.split('\n');
  const assumptions = [];

  for (const line of lines) {
    // Skip non-table lines (headers, separators, empty)
    if (!line.startsWith('|') || line.includes('---') || line.includes('Source')) continue;
    const cells = line.split('|').map(c => c.trim()).filter(c => c.length > 0);
    if (cells.length < 7) continue;

    const [num, source, name, type, coverage, proposedMetric, metricType] = cells;
    assumptions.push({
      id: parseInt(num, 10),
      source,
      name,
      type,         // "invariant", "assume", "constant"
      coverage,     // "uncovered", "covered", "partial"
      proposed_metric: proposedMetric.replace(/`/g, ''),
      metric_type: metricType,
      validation_status: 'untested',  // default for SEM-03
      linked_l2_states: [],           // populated by cross-reference with invariant catalog
    });
  }
  return assumptions;
}
```

### Gate A grounding score computation
```javascript
// Source: derived from validate-traces.cjs + Gate A spec
function computeGroundingScore(events, vocabulary) {
  const vocabActions = new Set(Object.keys(vocabulary.vocabulary));
  const { mapToXStateEvent } = require('./validate-traces.cjs');
  const { createActor, nfWorkflowMachine } = require('../dist/machines/nf-workflow.machine.js');

  let explained = 0;
  let total = events.length;
  const unexplained = { instrumentation_bug: [], model_gap: [], genuine_violation: [] };

  for (const event of events) {
    // Check 1: vocabulary mapping
    if (!vocabActions.has(event.action)) {
      unexplained.instrumentation_bug.push(event);
      continue;
    }

    // Check 2: XState transition
    const xEvt = mapToXStateEvent(event);
    if (!xEvt) {
      // In vocabulary but not in XState mapper -- instrumentation gap
      unexplained.instrumentation_bug.push(event);
      continue;
    }

    // Check 3: H1 methodology -- mid-session events are explained by default
    if (event.phase && event.phase !== 'IDLE' && event.action !== 'quorum_start') {
      explained++; // methodology skip = explained
      continue;
    }

    // Check 4: fresh actor replay
    const actor = createActor(nfWorkflowMachine);
    actor.start();
    actor.send(xEvt);
    const snapshot = actor.getSnapshot();
    actor.stop();

    const expected = expectedStateForGateA(event);
    if (expected === null || snapshot.matches(expected)) {
      explained++;
    } else {
      unexplained.model_gap.push(event);
    }
  }

  return {
    grounding_score: total > 0 ? explained / total : 1.0,
    explained,
    total,
    unexplained_counts: {
      instrumentation_bug: unexplained.instrumentation_bug.length,
      model_gap: unexplained.model_gap.length,
      genuine_violation: unexplained.genuine_violation.length,
    },
    target_met: total > 0 ? (explained / total) >= 0.80 : true,
  };
}
```

## Open Questions

1. **How many unique invariants exist after deduplication?**
   - What we know: 80+ INVARIANT lines across 25+ .cfg files, but many are TypeOK repeated per model. 15 invariants.md files exist with liveness properties.
   - What's unclear: The exact unique count after dedup. Estimate: ~60-80 unique invariants.
   - Recommendation: Build the parser, run it, let the data answer. The catalog script should report total_raw vs total_deduplicated.

2. **Should observed invariants be auto-generated or manually curated?**
   - What we know: Trace mining can discover temporal ordering patterns (e.g., start before complete). But "mining" at 35K events with 7 action types will produce very few interesting patterns.
   - What's unclear: Whether the effort of implementing sophisticated pattern mining is worth it at this scale.
   - Recommendation: Implement 3-5 hardcoded pattern checks (ordering invariants, action co-occurrence) rather than a general mining framework. Mark as `type: "observed"` with specific `evidence_sessions` count.

3. **Should Gate A reuse validate-traces.cjs directly or reimplement?**
   - What we know: validate-traces.cjs already computes a deviation score. Gate A needs a "grounding score" which is conceptually similar but includes vocabulary coverage.
   - What's unclear: Whether to import and extend, or to build a separate script that calls validate-traces as a module.
   - Recommendation: Import `mapToXStateEvent` from validate-traces.cjs (it is already exported). Reimplement the scoring loop with the broader "explains" definition. Do NOT shell out to validate-traces.cjs.

4. **What is the expected mismatch count?**
   - What we know: .divergences.json currently has 10 entries (all circuit_break events with expectedState DECIDED but actualState IDLE). These are likely H1 methodology artifacts, not genuine bugs.
   - What's unclear: Whether a full mismatch scan will produce more entries from the 35K+ events.
   - Recommendation: Run Gate A first to get the full classification. The mismatch register should capture ALL divergences from validate-traces output (including the methodology-skip ones as "explained").
