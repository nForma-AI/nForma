**Step A: Update write-check-result.cjs**

Read bin/write-check-result.cjs and make these changes:

1. In the `writeCheckResult(entry)` function, after the existing optional field handling (triage_tags, observation_window), add handling for `requirement_ids`:

```javascript
// requirement_ids: optional array of requirement IDs this check covers
// Defaults to [] when not provided (backward compatible)
record.requirement_ids = Array.isArray(entry.requirement_ids) ? entry.requirement_ids : [];
```

2. Add validation: if `requirement_ids` is provided and is an array, verify every element is a string. If any element is not a string, throw:
```javascript
if (entry.requirement_ids !== undefined) {
  if (!Array.isArray(entry.requirement_ids)) {
    throw new Error('[write-check-result] requirement_ids must be an array');
  }
  for (const id of entry.requirement_ids) {
    if (typeof id !== 'string') {
      throw new Error('[write-check-result] requirement_ids must contain only strings (got: ' + typeof id + ')');
    }
  }
}
```

3. Place `requirement_ids` in the record object AFTER `triage_tags` and BEFORE `metadata` to keep consistent ordering.

4. Update the JSDoc comment for `writeCheckResult` to document the new field:
```
 * @param {string[]} [entry.requirement_ids] - Optional array of requirement IDs this check covers. Defaults to [].
```
