  <action>
Create `bin/analyze-state-space.cjs` — a Node.js script that statically analyzes TLA+ models to estimate state-space size and classify risk.

**Discovery and Model Enumeration:**

1. Read `formal/model-registry.json` to find all TLA+ model files (entries whose keys start with `formal/tla/` and end with `.tla`).
2. Filter out TTrace files (filenames containing `_TTrace_`) — these are counter-example traces, not models.
3. For each model .tla file, find its corresponding MC*.cfg file. The mapping convention is:
   - `QGSDQuorum_xstate.tla` → `MCQGSDQuorum.cfg` (the module name from the .tla file's `---- MODULE X ----` header, prefixed with `MC`)
   - `QGSDOscillation.tla` → `MCoscillation.cfg` (some use lowercase after MC)
   - `QGSDCircuitBreaker.tla` → `MCbreaker.cfg` (some use abbreviated names)
   - Strategy: For each .tla file, scan all .cfg files in `formal/tla/` and find the one whose `SPECIFICATION` directive references a Spec that matches (or whose filename is a plausible MC-prefixed variant). Pragmatic approach: build a lookup map of all .cfg files, and for each .tla file try: `MC` + moduleName + `.cfg`, `MC` + moduleName.toLowerCase() + `.cfg`, then fall back to scanning .cfg content for `SPECIFICATION Spec` references that import this module.
   - If no .cfg found, the model still gets analyzed from .tla source alone (no constant bounds available).

**Parsing .cfg files:**

Extract from each .cfg:
- `CONSTANTS` block: Parse `Name = Value` pairs. Values can be:
  - Integer: `MaxDeliberation = 10` → `{ name: "MaxDeliberation", value: 10, type: "integer" }`
  - Set literal: `Agents = {a1, a2, a3, a4, a5}` → `{ name: "Agents", value: ["a1","a2","a3","a4","a5"], type: "set", cardinality: 5 }`
  - Model value: `a1 = a1` → `{ name: "a1", value: "a1", type: "model_value" }` (skip these, they define symmetry set members)
- `INVARIANT` lines: count them
- `PROPERTY` lines: count them

**Parsing .tla files:**

Extract from each .tla:
- `VARIABLES` block: List all variable names (may span multiple lines, comma-separated after `VARIABLES` keyword).
- `TypeOK` invariant: For each variable in TypeOK, extract domain:
  - `variable \in {"A", "B", "C"}` → finite set, cardinality 3
  - `variable \in {TRUE, FALSE}` or `variable \in BOOLEAN` → boolean, cardinality 2
  - `variable \in 0..N` where N is a constant name → bounded integer range, cardinality = N+1 (resolve from .cfg constants)
  - `variable \in Nat` → **UNBOUNDED** — flag as unbounded domain
  - `variable \in Int` → **UNBOUNDED** — flag as unbounded domain
  - `variable \in SUBSET S` → powerset, cardinality 2^|S|
  - `variable \in [S -> T]` → function space, cardinality |T|^|S|
  - `variable \in Seq(S)` → **UNBOUNDED** — sequences are unbounded
- `EXTENDS` line: Note if it extends `Naturals`, `Integers`, `FiniteSets`, `Sequences`, `TLC`.

**State-Space Estimation:**

For each model, compute an estimated state count:
- If ALL variables have finite, resolvable domains: `estimated_states = product of all variable cardinalities`
- If any variable has an unresolvable or unbounded domain: `estimated_states = null` and set `has_unbounded = true`
- If .cfg has CONSTANTS that bound otherwise-unbounded domains: use the bounded value

**Risk Classification:**

Apply configurable thresholds (defaults):
```javascript
const DEFAULT_THRESHOLDS = {
  MINIMAL: 1000,        // <= 1,000 states
  LOW: 100000,          // <= 100,000 states
  MODERATE: 10000000,   // <= 10,000,000 states
  // > 10,000,000 or unbounded = HIGH
};
```

Classification logic:
- If `has_unbounded === true` → **HIGH** (DECOMP-02: automatic, no manual intervention)
- Else if `estimated_states <= MINIMAL` → **MINIMAL**
- Else if `estimated_states <= LOW` → **LOW**
- Else if `estimated_states <= MODERATE` → **MODERATE**
- Else → **HIGH**

      ],

        // from .cfg, or empty array
      ],
