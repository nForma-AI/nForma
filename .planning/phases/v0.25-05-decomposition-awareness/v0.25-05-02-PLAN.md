  <action>
Add a `validateCoveragePreservation(newMatrix)` function to `bin/generate-traceability-matrix.cjs` that compares the newly generated matrix against the previous `formal/traceability-matrix.json` to detect coverage regressions per requirement.

**Function: `validateCoveragePreservation(newMatrix)`**

Parameters:
- `newMatrix` — the newly built matrix object (before it's written to disk)

Returns:
```javascript
{
  baseline_found: true,       // false if no previous matrix exists
  baseline_date: "2026-03-03T...",  // metadata.generated_at from baseline
  requirements_checked: N,
  regressions: [
    {
      requirement_id: "REQ-01",
      baseline_property_count: 3,
      current_property_count: 1,
      lost_count: 2,
      baseline_models: ["formal/tla/OldModel.tla"],
      current_models: ["formal/tla/NewSplit1.tla"],
      detail: "REQ-01 coverage decreased from 3 properties to 1 (lost 2) — possible model split dropped coverage"
    }
  ],
  summary: {
    total_regressions: N,
    affected_requirements: N,
    clean: true/false
  }
}
```

**Logic:**

1. **Load baseline:** Read `formal/traceability-matrix.json` (the existing file on disk BEFORE this run overwrites it). If the file does not exist, return `{ baseline_found: false, regressions: [], summary: { total_regressions: 0, affected_requirements: 0, clean: true } }` with an informational stderr message: `[generate-traceability-matrix] info: no previous traceability-matrix.json found — coverage preservation check skipped`.

2. **Read the file before buildMatrix() writes.** Important: call `validateCoveragePreservation(newMatrix)` AFTER `buildMatrix()` produces the matrix object but BEFORE `fs.writeFileSync()` writes it to disk. This way the baseline is still the old file on disk.

3. **Per-requirement comparison:** For each requirement ID that exists in BOTH the baseline and the new matrix:
   - Count properties in baseline: `baseline.requirements[reqId].properties.length`
   - Count properties in new matrix: `newMatrix.requirements[reqId].properties.length`
   - If `new_count < baseline_count`: record a regression.
   - Extract the model files from baseline and current for this requirement (the distinct `model_file` values from properties).

4. **Ignore new requirements:** If a requirement exists in the new matrix but NOT in the baseline, skip it — it's a newly added requirement, not a regression.

5. **Ignore removed requirements:** If a requirement exists in the baseline but NOT in the new matrix, skip it for this check — removal is detected by the existing coverage_summary.

**Integration into main():**

In the `main()` function, after `buildMatrix()` but before writing to disk:

```javascript
const coveragePreservation = validateCoveragePreservation(matrix);
matrix.coverage_preservation = coveragePreservation;
```

**Warnings to stderr:**

For each regression:
```
[generate-traceability-matrix] warn: coverage regression — REQ-01 had 3 properties, now has 1 (lost 2) — check if model split dropped coverage
```

**Summary output (when not --quiet):**

Add to the existing summary:
```
[generate-traceability-matrix]   Coverage preservation: N regressions (N requirements checked vs baseline)
```
Or if no baseline:
```
[generate-traceability-matrix]   Coverage preservation: no baseline (first run)
```

**Do NOT change exit code** — warnings only, generator still exits 0.

**Implementation notes:**
- Read the baseline file using `fs.readFileSync` and `JSON.parse`, wrapped in try/catch. If the file is corrupt JSON, treat it as "no baseline" with a warning.
- The baseline comparison must happen BEFORE `fs.writeFileSync(OUTPUT_PATH, ...)` in the main flow.
- Use the same `TAG` constant for stderr messages.
- The `coverage_preservation` section is always present in output (even when no baseline — shows `baseline_found: false`).
