Create NFSolveConvergence.tla following the exact patterns from RESEARCH.md Code Examples 1-3. The spec must model:

1. **State variables** (Pattern 1): session, residual, prevResidual, oscillation_count, blocked, gate_maturity, converged, graceActive. Use CONSTANTS Layers, DownstreamOf, MaxSessions, MaxResidual.

2. **Init**: session=0, all residuals at MaxResidual, prevResidual=-1 (no history), oscillation_count=0, blocked=FALSE, gate_maturity="ADVISORY", converged=FALSE, graceActive=FALSE.

3. **Actions**:
   - RunSession: Non-deterministically picks a non-blocked layer and assigns a new residual in 0..MaxResidual. Detects oscillation (increase after decrease) when no grace period active. Increments oscillation_count and blocks on count reaching 1 (not 2 -- block BEFORE it can reach 2, per Pitfall 3). Updates prevResidual and increments session counter.
   - ActivateGrace: When a layer's residual decreases, activate grace period on all downstream layers (using DownstreamOf constant).
   - DeactivateGrace: Non-deterministically deactivate grace period on any layer (models grace expiry).
   - GateTransition: Advance gate_maturity forward only (ADVISORY->SOFT_GATE->HARD_GATE) for any layer. UNCHANGED all other vars.
   - CheckConvergence: Set converged=TRUE when all layers have residual=0 OR are blocked.

4. **Safety invariant** (Pattern 4): `OscillationBounded == \A l \in Layers : oscillation_count[l] <= 1` with @requirement FV-02 comment.

5. **Liveness property** (Pattern 5): `EventualConvergence == <>(converged = TRUE)` with @requirement FV-03 comment.

6. **TypeOK invariant**: All variables in expected domains.

7. **Spec definition**: `Spec == Init /\ [][Next]_vars /\ WF_vars(RunSession) /\ WF_vars(CheckConvergence)` -- weak fairness on RunSession and CheckConvergence per Pattern 5.

8. Include module header comment with `Handwritten -- not generated from XState.` and `Source: bin/oscillation-detector.cjs, bin/gate-stability.cjs, bin/nf-solve.cjs` following NFOscillation.tla pattern. Tag with `@requirement FV-01`, `@requirement FV-02`, `@requirement FV-03`.

CRITICAL modeling decisions:
- Use 3 abstract layers {L1, L2, L3} with linear DAG (L1->L2->L3) for tractable state space
- MaxSessions=5, MaxResidual=3 per Pitfall 1 guidance
- Block when oscillation_count reaches 1 (not after 2) per Pitfall 3 -- the safety invariant is `<= 1`
