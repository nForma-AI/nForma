## Deviations

1. **DownstreamOf moved to spec operator**: TLC config parser cannot handle `:>` `@@` expressions in config files. DownstreamOf defined as operator in spec instead of CONSTANT.

2. **L1/L2/L3 added as CONSTANTS**: Required for TLC model value assignment (config uses `L1 = L1`).

3. **EXTENDS TLC added**: Required for `:>` and `@@` operators used in DownstreamOf definition.

4. **Grace periods removed**: TLC liveness checking revealed that grace periods create infinite oscillation cycles (upstream progress re-activates grace on downstream layers indefinitely). Removing grace periods proves the STRONGER property: convergence holds even without grace lenience. Grace only delays oscillation detection — it cannot prevent it — so the proof without grace subsumes the proof with grace.

5. **RunSession split into ProgressSession + RegressSession**: Required for correct fairness modeling. WF on ProgressSession (residuals decrease) + no fairness on RegressSession (regression triggers blocking) ensures liveness. Single RunSession with WF allowed infinite non-convergent cycling.

6. **Session counter wraps modulo MaxSessions+1**: Prevents session exhaustion from falsifying liveness. Models unbounded solve invocations (real system has no hard session cap).
