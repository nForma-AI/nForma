  <action>
Rewrite `formal/check-result.schema.json` to add all 7 v2.1 fields as OPTIONAL properties (do NOT add any to the `required` array — backwards compatibility requires the `required` array stays as `["tool", "formalism", "result", "timestamp"]`).

Add the following to the `properties` object:
- `check_id`: `{ "type": "string" }` — unique step identifier (e.g. `"tla:quorum-safety"`)
- `surface`: `{ "type": "string" }` — STEPS tool group name (e.g. `"tla"`, `"alloy"`, `"prism"`, `"ci"`)
- `property`: `{ "type": "string" }` — human-readable check description
- `runtime_ms`: `{ "type": "integer", "minimum": 0 }` — CRITICAL measurement field — wall-clock elapsed time in milliseconds from tool start to writeCheckResult call; downstream triage bundles (v0.20-06) and timeout-adequacy decisions (planning gate v0.20-03) depend on this field being an integer >= 0, NOT a float; use `Math.round()` in the writer
- `summary`: `{ "type": "string" }` — one-line outcome (e.g. `"pass: MCsafety verified in 1823ms"`)
- `triage_tags`: `{ "type": "array", "items": { "type": "string" } }` — annotation convention for anomaly surfacing; callers should emit `["timeout-risk"]` when `runtime_ms` exceeds a reasonable threshold for that tool type (TLA+ > 120000ms, Alloy > 60000ms, PRISM > 300000ms — these thresholds indicate a check ran near system timeout boundaries); `["slow-verify"]` for checks taking > 50% of typical; leave empty in this phase but schema must support it
- `observation_window`: `{ "type": "object" }` — stochastic check metadata; for PRISM checks this is especially critical: the object MUST contain `window_start` (ISO timestamp), `window_end` (ISO timestamp), `n_traces` (integer count of observed quorum rounds), `n_events` (integer count of individual vote events), and `window_days` (float days in window); this field enables downstream timeout-adequacy analysis by correlating PRISM result reliability with the window size

Keep `additionalProperties: false` in place (required for strict validation).
Keep the existing `metadata` property unchanged.
