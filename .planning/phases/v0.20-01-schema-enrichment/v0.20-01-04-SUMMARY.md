## Decisions Made

1. **observation_window in run-prism.cjs:** Promoted from metadata to top-level argument to writeCheckResult. This satisfies v0.20-05 (EVID-02) requirements for evidence confidence tracking. The struct includes n_traces (quorum rounds observed) and window_days for downstream calculations.

2. **PRISM triage_tags logic:** Timeout thresholds at >300s (timeout-risk) and >120s (slow-verify). Low-confidence annotation automatically added when window_days < 7 OR n_traces < 30. This enables automatic self-annotation of insufficient-data results.

3. **Multi-config patterns:** CHECK_ID_MAP and PROPERTY_MAP in each Alloy and PRISM runner decouples check_id from hardcoded single values. Determined by CLI args (--spec for Alloy, --model for PRISM).

4. **Timer placement:** Placed immediately before spawnSync/exec call (not at module start) to measure only tool execution time, not policy/config loading.
