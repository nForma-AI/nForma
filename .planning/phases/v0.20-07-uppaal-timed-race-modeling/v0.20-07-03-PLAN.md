  <files>formal/uppaal/quorum-races.xml, formal/uppaal/quorum-races.q</files>
  <behavior>
    **UPPAAL XML model** (`formal/uppaal/quorum-races.xml`):

    The model uses 3 templates and parameterized constants. The constants MIN_SLOT_MS, MAX_SLOT_MS,
    TIMEOUT_MS, and MIN_GAP_MS are declared as UPPAAL global `const int` declarations — run-uppaal.cjs
    will use verifyta's `-C name=value` flags to override them with empirical values from
    check-results.ndjson at runtime.

    Default constant values (fallbacks when no empirical data):
    - MIN_SLOT_MS = 50     (minimum slot response time in ms)
    - MAX_SLOT_MS = 500    (maximum slot response time in ms)
    - TIMEOUT_MS = 1500    (planning gate deadline)
    - MIN_GAP_MS = 10      (minimum inter-slot response gap for race-free operation)
    - N_SLOTS = 4          (number of quorum slots, matching QGSD default)
    - MIN_QUORUM = 3       (minimum votes for consensus, matching QGSD threshold)

    **Template 1: SlotWorker**
    - Clock x (time since dispatch)
    - Location Waiting (initial): invariant x <= MAX_SLOT_MS
    - Location Responded (committed)
    - Transition Waiting→Responded: guard x >= MIN_SLOT_MS, sync response!, reset nothing

    **Template 2: QuorumOrchestrator**
    - Clock t (time since start), variable votes int
    - Location Collecting (initial): invariant t <= TIMEOUT_MS
    - Location Decided (goal state, no invariant)
    - Location TimedOut (absorbing, no invariant)
    - Transition Collecting→Collecting: sync response?, votes++
    - Transition Collecting→Decided: guard votes >= MIN_QUORUM (when votes threshold reached)
    - Transition Collecting→TimedOut: guard t >= TIMEOUT_MS (when deadline fires)

    **Template 3: RaceDetector**
    - Boolean flag `race_fired`
    - Location Idle (initial)
    - Location Race (urgent — marks simultaneous responses)
    - When two response! channels fire at same instant, Race becomes reachable

    **System declaration:**
    ```
    slot1 = SlotWorker(); slot2 = SlotWorker(); slot3 = SlotWorker(); slot4 = SlotWorker();
    orch = QuorumOrchestrator();
    race = RaceDetector();
    system slot1, slot2, slot3, slot4, orch, race;
    ```

    **UPPAAL XML structure** (minimal valid UPPAAL 4.x format):
    ```xml
    <?xml version="1.0" encoding="utf-8"?>
    <!DOCTYPE nta PUBLIC '-//Uppaal Team//DTD Flat System 1.6//EN'
                         'http://www.it.uu.se/research/group/darts/uppaal/flat-1_6.dtd'>
    <nta>
      <declaration>
        <!-- Global constants and channels -->
        const int MIN_SLOT_MS = 50;
        const int MAX_SLOT_MS = 500;
        const int TIMEOUT_MS = 1500;
        const int MIN_GAP_MS = 10;
        const int N_SLOTS = 4;
        const int MIN_QUORUM = 3;
        broadcast chan response;
        int votes = 0;
        bool race_fired = false;
      </declaration>
      <template>
        <name>SlotWorker</name>
        <declaration>clock x;</declaration>
        <!-- locations and transitions -->
      </template>
      <template>
        <name>QuorumOrchestrator</name>
        <declaration>clock t; int v = 0;</declaration>
        <!-- locations and transitions -->
      </template>
      <template>
        <name>RaceDetector</name>
        <declaration></declaration>
        <!-- locations and transitions -->
      </template>
      <system>
        slot1 = SlotWorker();
        slot2 = SlotWorker();
        slot3 = SlotWorker();
        slot4 = SlotWorker();
        orch = QuorumOrchestrator();
        race = RaceDetector();
        system slot1, slot2, slot3, slot4, orch, race;
      </system>
    </nta>
    ```

    **Property query file** (`formal/uppaal/quorum-races.q`):
    ```
    // UPPAAL-01: No deadlock in the model
    A[] not deadlock

    // UPPAAL-03a: Quorum can reach consensus (liveness under timing assumptions)
    E<> orch.Decided

    // UPPAAL-03b: Consensus before timeout is reachable when slots respond in time
    E<> orch.Decided and orch.t < TIMEOUT_MS

    // UPPAAL-03: No race fires when inter-slot gap >= MIN_GAP_MS (annotated safety property)
    // Note: Race detection is structural — if RaceDetector.Race is reachable, races can fire
    A[] not race.Race
    ```

    Comments in .q file must reference UPPAAL-01 and UPPAAL-03 requirements.
