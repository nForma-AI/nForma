  <action>
Create `bin/candidate-discovery.cjs` that discovers unlinked (model, requirement) pairs using the existing proximity graph.

**Inputs:**
- `.planning/formal/proximity-index.json` — the graph (loaded via JSON.parse)
- `.planning/formal/model-registry.json` — models with their linked requirements arrays
- `.planning/formal/requirements.json` — requirement IDs to iterate over

**Core logic:**

1. Load proximity-index.json, model-registry.json, requirements.json
2. Compute SHA256 hash of proximity-index.json (first 8 chars) for idempotency tracking
3. Import `proximity` function from `./formal-proximity.cjs` via require
4. Iterate all (model, requirement) pairs:
   - For each model in model-registry.json `.models` object
   - For each requirement ID in requirements.json `.requirements` object
   - Skip if requirement is already in model's `.requirements` array (already linked)
   - Construct keys: `formal_model::${modelPath}` and `requirement::${reqId}`
   - Call `proximity(index, modelKey, reqKey, maxHops)` with maxHops=3
   - If score > threshold (default 0.6), add to candidates array
5. Sort candidates by proximity_score descending (deterministic output for idempotency)
6. Write candidates.json to `.planning/formal/candidates.json`

**Output format for candidates.json:**
```json
{
  "metadata": {
    "generated": "ISO timestamp",
    "proximity_index_hash": "first 8 chars SHA256",
    "threshold": 0.6,
    "max_hops": 3,
    "total_pairs_checked": N,
    "candidates_found": N
  },
  "candidates": [
    {
      "model": "path/to/model.als",
      "requirement": "REQ-ID",
      "proximity_score": 0.85
    }
  ]
}
```

**CLI interface:**
- `node bin/candidate-discovery.cjs` — run with defaults (threshold=0.6, maxHops=3)
- `node bin/candidate-discovery.cjs --min-score 0.7` — override threshold
- `node bin/candidate-discovery.cjs --max-hops 2` — override hop limit
- `node bin/candidate-discovery.cjs --json` — print summary to stdout as JSON (for piping)
- Log progress to stderr: `[candidate-discovery] Checking N models x M requirements...`
- Log histogram of candidate scores to stderr for threshold tuning

**Error handling:**
- If proximity-index.json missing: exit 1 with message "proximity-index.json not found — run formal-verify first"
- If model-registry.json missing: exit 1 with message
- If proximity() returns NaN or undefined for a pair: skip with stderr warning, do not crash

**Important:** Use `crypto.createHash('sha256')` for the index hash. Use `JSON.stringify(candidates, null, 2)` for pretty output. Ensure candidates array sort is stable (secondary sort by model+requirement string for ties).
