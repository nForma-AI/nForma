**Step 2: Add buildTTrace helper function** (add near the other helper functions, before `if (require.main === module)`).

`buildTTrace` MUST implement session-based guard evaluation: group conformance events by `round_id` before evaluating guards. This preserves session replay context — a guard that depends on context set during QUORUM_START (e.g., `slotsAvailable`) will see that value when evaluating VOTES_COLLECTED guards within the same round, rather than seeing the uninitialized default from a fresh actor.

```javascript
// buildTTrace: construct a structured TTrace record for a divergent conformance event.
// Uses xstate-trace-walker to get guard evaluation details.
// IMPORTANT: roundEvents is the full ordered array of conformance events for this round_id group
// (or just [event] for standalone events with no round_id). Guards are evaluated with the actor
// replayed through all preceding events in the group, so guard context at event N reflects
// context accumulated from events 1..N-1 — not from a fresh IDLE snapshot.
//
// STANDALONE EVENT TRADE-OFF (intentional approximation):
// When an event has no round_id, roundEvents = [event] — a single-event group with no
// preceding context. The actor starts fresh (IDLE) before evaluating the guard.
// This is an acceptable methodological approximation because:
//   1. Standalone events represent ad-hoc or non-session actions where cross-event context
//      is not meaningful — evaluating them in isolation accurately reflects their runtime
//      execution environment.
//   2. The accuracy cost is bounded: only guards that depend on context set by earlier
//      events in the same round will be misclassified. Guards that depend only on the
//      event's own payload (the majority) are evaluated correctly.
//   3. The alternative (silently dropping standalone events) would introduce undetected
//      divergence blind spots — the approximation is always preferable to omission.
// Consequence: a standalone guard failure may be classified as impl-bug when it is a
// methodology artifact. attribute-trace-divergence.cjs should treat standalone TTrace
// records with lower specBugConfidence when no preceding context is present.
//
// scoreboardMeta and confidence are passed from the outer loop.
// walker is the result of require('./xstate-trace-walker.cjs') — passed in to keep helper pure.
function buildTTrace(event, actualState, expectedState, divergenceType, scoreboardMeta, confidence, walker, machine, roundEvents) {
  let guardEvaluations = [];
  if (walker && machine) {
    try {
      // Find the index of this event in its round group
      const eventIndex = (roundEvents || [event]).indexOf(event);
      const precedingEvents = (roundEvents || [event]).slice(0, eventIndex);

      // Build the session context by replaying all events preceding this one through a single actor.
      // This is the key difference from the fresh-actor approximation: guards evaluated here
      // will see slotsAvailable, polledCount, etc. as set by QUORUM_START and earlier events.
      const xstateEvt = mapToXStateEvent(event);
      if (xstateEvt) {
        const { createActor } = require((function() {
          const p = require('path');
          const fs = require('fs');
          const repoDist    = p.join(__dirname, '..', 'dist', 'machines', 'qgsd-workflow.machine.cjs');
          const installDist = p.join(__dirname, 'dist', 'machines', 'qgsd-workflow.machine.cjs');
          return fs.existsSync(repoDist) ? repoDist : installDist;
        })());
        const actor = createActor(machine);
        actor.start();
        // Replay all preceding events to accumulate session context
        for (const prev of precedingEvents) {
          const prevEvt = mapToXStateEvent(prev);
          if (prevEvt) actor.send(prevEvt);
        }
        const beforeSnap = actor.getSnapshot();
        actor.stop();
        const walkerResult = walker.evaluateTransitions(beforeSnap, xstateEvt, machine);
        guardEvaluations = walkerResult.possibleTransitions.map(t => ({
          guardName:    t.guardName || 'none',
          passed:       t.guardPassed,
          context:      t.guardContext,
        }));
      }
    } catch (_) { /* fail-open — guard evaluation failure must not break validation */ }
  }
  return {
    event,
    actualState,
    expectedState,
    guardEvaluations,
    divergenceType,
    confidence,
    observation_window: {
      n_rounds:    scoreboardMeta.n_rounds,
      window_days: scoreboardMeta.window_days,
    },
  };
}
```

**Standalone event handling (no round_id):** When an event has no `round_id` (or `round_id` is null/undefined), treat `roundEvents` as `[event]` — a single-event group with no preceding context. The actor starts fresh (IDLE), and guard evaluation uses only the event's own payload. This must NOT silently drop the event from TTrace output — it is included with an empty `precedingEvents` array, which is the correct approximation for standalone events.

**Reference: Expected TTrace object shape**

Each object written to `formal/.divergences.json` MUST match this structure. Use this as the concrete reference when implementing `buildTTrace` and when writing test fixtures.

```json
{
  "event": {
    "action": "quorum_complete",
    "outcome": "APPROVE",
    "vote_result": 3,
    "slots_available": 3,
    "round_id": "rnd-abc123",
    "timestamp": "2026-02-28T14:32:07.441Z"
  },
  "actualState": "DECIDED",
  "expectedState": "DELIBERATING",
  "divergenceType": "state_mismatch",
  "confidence": "medium",
  "guardEvaluations": [
    {
      "guardName": "hasQuorum",
      "passed": true,
      "context": { "slotsAvailable": 3, "polledCount": 3, "approveCount": 3 }
    },
    {
      "guardName": "isApproved",
      "passed": true,
      "context": { "approveCount": 3, "requiredMajority": 2 }
    }
  ],
  "observation_window": {
    "n_rounds": 847,
    "window_days": 30
  }
}
```

Notes on this shape:
- `event` is the raw conformance event object as parsed from conformance-events.jsonl — no transformation.
- `actualState` is the string state name from `actor.getSnapshot().value` after replaying the round through the XState machine.
- `expectedState` is the state the machine was expected to be in according to the conformance log.
- `divergenceType` is always `"state_mismatch"` for TTrace records written to `.divergences.json` (other types are excluded by the `.filter()` in Step 4).
- `confidence` is the tier string from `computeConfidenceTier()` — one of `"low"`, `"medium"`, `"high"`.
- `guardEvaluations` is an empty array `[]` when `walker=null` (fail-open) or when no XState guard exists for the transition — this is the correct value, not `null` or `undefined`.
- `observation_window.n_rounds` is the total round count from the scoreboard metadata — used by attribute-trace-divergence.cjs to weight specBugConfidence.

test('buildTTrace: returns required TTrace fields', () => {
  const event = { action: 'quorum_complete', outcome: 'APPROVE', vote_result: 3, slots_available: 3 };
  const scoreboardMeta = { n_rounds: 5, window_days: 1 };
  // Pass walker=null and machine=null to test the structure without XState dependency

  assert.equal(result.divergenceType, 'state_mismatch');

  assert.equal(result.observation_window.window_days, 7);

test('buildTTrace: fail-open when walker is null — guardEvaluations is empty array, event is NOT dropped', () => {

  const standaloneEvent = { action: 'quorum_complete' }; // no round_id
