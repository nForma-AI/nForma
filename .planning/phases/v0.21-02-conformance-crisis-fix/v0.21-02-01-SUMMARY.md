## Decisions Made
- XState v5's `machine.transition()` pure API is not usable standalone (requires `actorScope`). Used actor approach instead — send event through actor, capture before/after snapshots.
- Guard fn signature is `({ context, event }) => boolean` in XState v5. Walker calls guards this way.
- `machine.implementations.guards` is the correct location for guard functions in the built CJS bundle.

## Deviations from Plan
None - plan executed exactly as written. Machine was found at repo dist path as expected.

## Issues Encountered
- `machine.transition()` threw `TypeError: Cannot read properties of undefined (reading 'actionExecutor')` — XState v5 pure transition API requires actor scope context. Resolved by using the actor-based approach (start actor, advance via send, capture snapshots).
