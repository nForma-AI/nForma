## DISP-01: Quorum W/L Column

### Scoreboard file location

The quorum scoreboard lives at `.planning/quorum-scoreboard.json` (relative to the **project directory**, not `~/.claude/`). It is gitignored per-project.

### Scoreboard schema

```json
{
  "models": {
    "claude": { "score": 107, "tp": 114, "tn": 1, "fp": 4, "fn": 0, "impr": 0 },
    "gemini": { "score": 55,  "tp": 45,  "tn": 0, "fp": 0, "fn": 0, "impr": 5 },
    "opencode": { "score": 102, ... },
    "copilot": { "score": 84, ... },
    "codex": { "score": 0, ... }
  }
}
```

Keys under `models` are **family names** (e.g., `claude`, `gemini`), NOT slot names (e.g., `claude-1`, `claude-2`).

### Mapping slot names â†’ family names

Current mcpServers entries have slot names like `codex-1`, `gemini-1`, `opencode-1`, `copilot-1`, `claude-1` through `claude-6`. The family name is the slot name with the trailing `-N` suffix stripped.

```js
function slotToFamily(slotName) {
  return slotName.replace(/-\d+$/, '');
}
// 'claude-3' â†’ 'claude', 'gemini-1' â†’ 'gemini'
```

### W/L display

- `tp` = wins (quorum approved this model's votes)
- `fn` = losses (quorum blocked, model was wrong)
- Display format: `14W/2L` or `â€”` when absent

### File absent (ENOENT) guard

**This is the core constraint from STATE.md**: "Scoreboard reads need existsSync guard â€” ENOENT on fresh install must show `â€”` not crash (DISP-01)"


### Deriving CCR provider for a slot

Given a slot's model (from `CLAUDE_DEFAULT_MODEL` or provider map), match it against `ccrConfig.providers[i].models[]`:

```js
function getCcrProviderForSlot(model, ccrConfig) {
  if (!ccrConfig || !model) return null;
  for (const provider of (ccrConfig.providers || [])) {
    if ((provider.models || []).includes(model)) {
      return provider.name;  // e.g., 'akashml', 'together'
    }
  }
  return null;
}
```

### Badge display logic


"A key is configured" means: `secretsLib.hasKey(keytarAccount)` OR `env.ANTHROPIC_API_KEY` is set (legacy plaintext).

### Why key_status survives restart

The badge reads from `qgsd.json` on disk â€” the health probe (v0.10-03) will write `key_status` there. This phase only adds the read-path display logic.

## `readQgsdJson()` and `writeQgsdJson()` Helpers

### Design requirements

1. Located in `manage-agents.cjs` (additive, no new file)
2. Exported via `module.exports._pure` for unit testing
3. Use atomic tmp-rename for writes
4. `readQgsdJson()` returns `{}` when file absent (never throws)
5. `writeQgsdJson(data)` writes with the same `.tmp` + `renameSync` pattern

```js
const QGSD_JSON_PATH = path.join(os.homedir(), '.claude', 'qgsd.json');
const QGSD_JSON_TMP = QGSD_JSON_PATH + '.tmp';

function readQgsdJson() {
  if (!fs.existsSync(QGSD_JSON_PATH)) return {};
  try {
    return JSON.parse(fs.readFileSync(QGSD_JSON_PATH, 'utf8'));
  } catch (_) {
    return {};
  }
}

function writeQgsdJson(data) {
  fs.writeFileSync(QGSD_JSON_TMP, JSON.stringify(data, null, 2), 'utf8');
  fs.renameSync(QGSD_JSON_TMP, QGSD_JSON_PATH);
}
```

### Refactor opportunity

The current `listAgents()` reads `qgsd.json` **twice** (lines 198â€“203 and 211â€“218) with two separate try/catch blocks. These should be replaced with a single `readQgsdJson()` call. This reduces duplication and validates the helper works in context.

## Column Width Impact

The current header width object `W` has 8 fields. Adding 3 new columns (W/L, CCR, key-invalid badge) requires:

**Option A**: Add new padded columns to the table (wider table, more horizontal space)
**Option B**: Append badges inline to existing columns (no table width change)

Recommendation: **Option B for key-invalid badge** (appended inline after slot name), **Option A for W/L and CCR** as small new columns. This keeps the table readable on 120-char terminals.

Suggested column widths for new columns:
- `wl`: 8 chars (e.g., `14W/2L` = 6 chars, padded to 8)
- `ccr`: 12 chars (e.g., `akashml` = 7, `together` = 8)

The key-invalid badge (`[key invalid]`) renders inline in the Slot column or as a trailing annotation â€” it does NOT need a new column since it's a conditional badge.
