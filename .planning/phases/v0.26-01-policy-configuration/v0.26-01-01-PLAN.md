Add two new pure validation functions to manage-agents-core.cjs:

1. `validateTimeout(value)` — validates a timeout input string:
   - Returns `{ valid: true, ms: <parsed integer> }` if value is a positive integer > 0
   - Returns `{ valid: false, error: "Timeout must be a positive number (got: <value>)" }` if:
     - Value is NaN after parseInt
     - Value is <= 0
     - Value is empty/null/undefined (return `{ valid: true, ms: null }` — means "keep current", not an error)
   - Blank/empty string returns `{ valid: true, ms: null }` (keep current, no change)

2. `validateUpdatePolicy(policy)` — validates a policy string:
   - Returns `{ valid: true, policy: <policy> }` if policy is one of: "auto", "prompt", "skip"
   - Returns `{ valid: false, error: "Unknown update policy '<policy>'. Valid values: auto, prompt, skip" }` otherwise
   - The valid set is derived from the existing `POLICY_MENU_CHOICES` constant (lines 46-50)
