## Summary

Phase v0.10-02 adds two features to `bin/manage-agents.cjs`: (1) a provider preset library that replaces the free-text ANTHROPIC_BASE_URL prompt in `addAgent()` and `editAgent()` with a named `list` prompt, and (2) a "Clone slot" menu option that copies an existing slot's provider URL and model config into a new slot with a fresh keytar-isolated credential entry.

The inquirer version is pinned at **8.2.7 CJS** (`inquirer@^8.2.7` in package.json, confirmed `8.2.7` installed). The `list` prompt type already exists in the codebase (used extensively in `editAgent`, `addAgent`, etc.). No package changes are required.

The key behavioral contract for PRST-01 is: "if the probe fails, no partial slot is written." The current `addAgent()` already has a `probeProviderUrl()` pre-flight pattern that is re-used here with a stricter cancellation path — `saveAnyway` confirm becomes a `retry / cancel` loop.

The key behavioral contract for PRST-02 is: keytar isolation — the clone NEVER reads or copies the source slot's API key. The new slot gets its own keytarAccount and prompts fresh. This matches existing keytar design (`ANTHROPIC_API_KEY_<SLOT_UPPER>`).

**Primary recommendation:** Extend the existing `addAgent()` and `editAgent()` base URL steps with a preset selector pure function; add `cloneSlot()` as a new async function parallel to `addAgent()`; wire "Clone slot" into `mainMenu()`.

### Pattern 1: Provider Preset Table (Pure Data + Pure Function)

**What:** A hardcoded constant table of named providers (name, base URL, display label) exported via `_pure` for testability. A `buildPresetChoices()` function converts the table into inquirer `list` choices.

**When to use:** In `addAgent()` and `editAgent()` wherever `ANTHROPIC_BASE_URL` is currently prompted as `type: 'input'`.

**Example (based on project MEMORY.md provider map):**

```js
// Source: project MEMORY.md Provider Map section (2026-02-22)
const PROVIDER_PRESETS = [
  { name: 'AkashML',      value: 'https://api.akashml.com/v1',          label: 'AkashML       (api.akashml.com/v1)' },
  { name: 'Together.xyz', value: 'https://api.together.xyz/v1',         label: 'Together.xyz  (api.together.xyz/v1)' },
  { name: 'Fireworks.ai', value: 'https://api.fireworks.ai/inference/v1', label: 'Fireworks.ai  (api.fireworks.ai/inference/v1)' },
];

/**
 * Build inquirer list choices for provider preset selector.
 * Returns choice array with preset entries + separator + Custom option.
 * Pure function — no side effects.
 */
function buildPresetChoices() {
  return [
    ...PROVIDER_PRESETS.map((p) => ({
      name: p.label,
      value: p.value,
      short: p.name,
    })),
    new inquirer.Separator(),
    { name: 'Custom (enter URL manually)', value: '__custom__', short: 'Custom' },
  ];
}
```

**In `addAgent()` base URL step — replace:**
```js
// BEFORE (free-text input):
{
  type: 'input',
  name: 'baseUrl',
  message: 'ANTHROPIC_BASE_URL:',
  default: '',
},
```

**With a two-step approach:**
```js
// Step 1: list prompt with presets
const { presetChoice } = await inquirer.prompt([
  {
    type: 'list',
    name: 'presetChoice',
    message: 'Provider (select preset or Custom to type manually):',
    choices: buildPresetChoices(),
  },
]);

// Step 2: if custom, prompt for manual input
let baseUrl = presetChoice;
if (presetChoice === '__custom__') {
  const { customUrl } = await inquirer.prompt([
    {
      type: 'input',
      name: 'customUrl',
      message: 'ANTHROPIC_BASE_URL:',
      default: '',
    },

  baseUrl = customUrl.trim();
}
```

### Pattern 2: Probe-or-Cancel Loop for Failed Pre-Flight

**What:** When a provider preset is selected and the probe fails, the user gets `retry / cancel` — NOT `save anyway`. This is a behavioral change from the existing `saveAnyway` pattern.

**When to use:** After the preset selection step in `addAgent()` and after the base URL edit step in `editAgent()`, when a URL is set.

**Failure behavior:**
- Probe healthy → continue to next step
- Probe unhealthy → show error, offer `retry / cancel` loop:
  - `retry` → re-run `probeProviderUrl()` on same URL
  - `cancel` → `return` immediately (no slot written)
- Loop repeats until health or cancel — no `saveAnyway` escape hatch on the preset path

**Note:** The existing `saveAnyway` behavior may be preserved for the `__custom__` URL path (since custom URLs might be localhost/dev setups that are intentionally unreachable), but success criterion 2 requires "no partial slot written on probe failure" unconditionally. The planner should treat the entire base URL step (preset AND custom) uniformly — probe failure always offers `retry / cancel`, never `save anyway`.

```js
// Probe-or-cancel loop pattern
async function probeWithRetryOrCancel(baseUrl, apiKey) {
  while (true) {
    process.stdout.write(`\n  Probing ${baseUrl} ...`);
    const probe = await probeProviderUrl(baseUrl, apiKey);
    if (probe.healthy) {
      process.stdout.write(`\n  \x1b[32m✓ Provider UP (${probe.latencyMs}ms)\x1b[0m\n`);
      return true; // proceed
    }
    process.stdout.write(`\n  \x1b[31m✗ Provider DOWN: ${probe.error || probe.statusCode || 'timeout'}\x1b[0m\n`);
    const { retryAction } = await inquirer.prompt([
      {
        type: 'list',
        name: 'retryAction',
        message: 'Provider probe failed:',
        choices: [
          { name: 'Retry probe', value: 'retry' },
          { name: 'Cancel — do not write slot', value: 'cancel' },
        ],
      },
    ]);
    if (retryAction === 'cancel') {
      console.log('\n  Cancelled.\n');
      return false; // caller must return immediately
    }
    // retryAction === 'retry' → loop continues
  }
}
```

### Pattern 3: Clone Slot Flow

**What:** A new async `cloneSlot()` function that (1) presents the slot selector, (2) copies the source slot's env (base URL + model), (3) prompts for new slot name with uniqueness validation, (4) optionally prompts for API key for the new slot (skippable), (5) writes via `writeClaudeJson()`.

**When to use:** Wired to "Clone slot" menu option in `mainMenu()`.

**Key invariants:**
- Source slot's API key is NEVER read or transferred. The clone gets a fresh keytarAccount.
- API key prompt is optional (skip allowed) — slot written with `[no key]` state if skipped.
- New slot name uniqueness validated in `validate()` callback.
- New slot's `env` is a shallow copy of source's `env` with key fields preserved: `ANTHROPIC_BASE_URL`, `CLAUDE_DEFAULT_MODEL`, `CLAUDE_MCP_TIMEOUT_MS`, `PROVIDER_SLOT` — but `ANTHROPIC_API_KEY` is explicitly excluded.

```js
async function cloneSlot() {
  const data = readClaudeJson();
  const mcpServers = getGlobalMcpServers(data);
  const slots = Object.keys(mcpServers);
  if (slots.length === 0) {
    console.log('\n  No agents to clone.\n');
    return;
  }

  let secretsLib = null;
  try { secretsLib = require('./secrets.cjs'); } catch (_) {}

  // Step 1: Select source slot
  const { sourceSlot } = await inquirer.prompt([
    {
      type: 'list',
      name: 'sourceSlot',
      message: 'Select slot to clone:',
      choices: slots.map((name) => {
        const cfg = mcpServers[name];
        const model = (cfg.env && cfg.env.CLAUDE_DEFAULT_MODEL) || cfg.command || '?';
        const url = (cfg.env && cfg.env.ANTHROPIC_BASE_URL) || '—';
        return { name: `${name.padEnd(14)} ${model.slice(0, 36)} (${shortProvider(cfg)})`, value: name, short: name };
      }),
    },
  ]);

  // Step 2: New slot name (unique validation)
  const { newSlotName } = await inquirer.prompt([
    {
      type: 'input',
      name: 'newSlotName',
      message: 'New slot name:',
      validate(val) {
        if (!val || !val.trim()) return 'Required';
        if (/\s/.test(val)) return 'No spaces allowed';
        if (slots.includes(val.trim())) return `"${val.trim()}" already exists`;
        return true;
      },
    },
  ]);

  const newName = newSlotName.trim();
  const sourceCfg = mcpServers[sourceSlot];
  const sourceEnv = sourceCfg.env || {};

  // Build new env — copy provider/model fields; explicitly exclude API key
  const newEnv = {};
  if (sourceEnv.ANTHROPIC_BASE_URL) newEnv.ANTHROPIC_BASE_URL = sourceEnv.ANTHROPIC_BASE_URL;
  if (sourceEnv.CLAUDE_DEFAULT_MODEL) newEnv.CLAUDE_DEFAULT_MODEL = sourceEnv.CLAUDE_DEFAULT_MODEL;
  if (sourceEnv.CLAUDE_MCP_TIMEOUT_MS) newEnv.CLAUDE_MCP_TIMEOUT_MS = sourceEnv.CLAUDE_MCP_TIMEOUT_MS;
  newEnv.PROVIDER_SLOT = newName; // new slot gets its own PROVIDER_SLOT

  // Step 3: Optional API key for new slot (keytar-isolated)
  const { setKey } = await inquirer.prompt([
    {
      type: 'confirm',
      name: 'setKey',
      message: 'Set an API key for the new slot?',
      default: false,
    },
  ]);

  if (setKey) {
    const { apiKey } = await inquirer.prompt([
      {
        type: 'password',
        name: 'apiKey',
        message: 'API key for ' + newName + ':',
        mask: '*',
        validate: (v) => (v && v.trim() ? true : 'Cannot be empty'),
      },
    ]);
    const newAccount = deriveKeytarAccount(newName);
    if (secretsLib) {
      await secretsLib.set('qgsd', newAccount, apiKey.trim());
    } else {
      newEnv.ANTHROPIC_API_KEY = apiKey.trim(); // plaintext fallback
    }
  }
  // If setKey === false: no key set — slot shows [no key] in list view

  // Write new slot
  const newEntry = {
    type: sourceCfg.type || 'stdio',
    command: sourceCfg.command,
    args: sourceCfg.args || [],
    env: newEnv,
  };

  data.mcpServers = Object.assign({}, mcpServers, { [newName]: newEntry });
  writeClaudeJson(data);

  if (setKey) {
    console.log(`\n  \x1b[32m✓ Cloned "${sourceSlot}" → "${newName}" (key set)\x1b[0m\n`);
  } else {
    console.log(`\n  \x1b[32m✓ Cloned "${sourceSlot}" → "${newName}" \x1b[90m[no key]\x1b[0m\n`);
  }
}
```

### Anti-Patterns to Avoid

- **Copying keytarAccount string and calling `secretsLib.get()`** on the source slot: Never read the source key. The clone must not inherit credentials. The new slot has a fresh `deriveKeytarAccount(newName)` — completely isolated from the source slot's keychain entry.
- **Passing `saveAnyway: true` as default for preset-triggered probes**: Success criterion 2 requires abort on failure with retry/cancel. The `saveAnyway` confirm in the existing code must be replaced for the preset path.
- **Using `Promise.all` for sequential keytar writes**: The project invariant (see CRED-01 in requirements) requires sequential `for...of` for keytar writes to avoid concurrency errors. The clone flow touches only one key at a time, so this is not a risk here, but do not inadvertently introduce parallel keytar calls.
- **Forking `addAgent()` into a separate function for preset mode**: The success criterion says "when adding or editing an agent, the base URL step presents a named provider list." This means the preset selector must be integrated into the existing `addAgent()` and `editAgent()` functions, not a new parallel function.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Inquirer list prompt | Custom readline-based selector | `inquirer.prompt([{ type: 'list', ... }])` | Already installed at 8.2.7 CJS; custom selector would require raw-mode management and conflict with future dashboard work |
| URL validation | Custom regex | `new URL(baseUrl)` (already used in `probeProviderUrl`) | Node built-in, handles edge cases; the probe itself validates reachability |
| Provider reachability | Custom HTTP client | `probeProviderUrl()` (already in `manage-agents.cjs` lines 92–135) | Handles HTTPS vs HTTP, timeout, status codes 200/401/403/404/422 as healthy — no duplication needed |
| Key storage | Plain JSON in `~/.claude.json` | `secretsLib.set('qgsd', account, value)` | Security invariant: keys in keytar only, never in plaintext `.claude.json` (except fallback when keytar unavailable) |
| Unique name validation | Post-write check | Inline `validate()` in inquirer prompt | inquirer `validate()` is synchronous and prevents submission with a user-visible error message — cleaner UX than a try/catch after write |

**Key insight:** `probeProviderUrl()` is already the right tool. It uses the `/models` endpoint, not a full LLM call, completes in ~2–7 seconds, and returns `{ healthy, latencyMs, statusCode, error }`. Reuse it directly in the preset flow.

## Common Pitfalls

### Pitfall 1: inquirer v9 ESM incompatibility
**What goes wrong:** If someone upgrades `inquirer` to v9+, `require('inquirer')` throws because v9 is ESM-only.
**Why it happens:** The package.json `^8.2.7` range theoretically caps at v8, but the success criterion explicitly states "inquirer@8.2.7 CJS is used — the package is not upgraded." This is a hard invariant.
**How to avoid:** Success criterion 1 locks it. The PLAN.md must NOT include any `npm install inquirer@latest` or version bump task.
**Warning signs:** Any plan task that touches `package.json` dependencies for inquirer.

### Pitfall 2: Partial write on probe failure
**What goes wrong:** The current `addAgent()` calls `writeClaudeJson()` even when the user chooses "Save anyway" on a failed probe. For the preset path, the success criterion requires "no partial slot is written on probe failure."
**Why it happens:** The current flow sequences: collect all answers → probe → optionally save. If the probe fails and the function `return`s, no write happens. The risk is calling `writeClaudeJson()` before the probe returns, or missing a `return` after a failed probe cancel.
**How to avoid:** Structure the function so `writeClaudeJson()` is the last statement in the happy path, called only after a successful probe. Use `return` explicitly after `probeWithRetryOrCancel()` returns `false`.
**Warning signs:** Any inverted flow where `writeClaudeJson()` appears before the probe call.

### Pitfall 3: PROVIDER_SLOT for cloned slot
**What goes wrong:** The cloned slot inherits the source slot's `PROVIDER_SLOT` value (e.g., `claude-3`), which maps to `providers.json` entries by that name. The new slot `claude-7` would then display the wrong model/provider in `listAgents()`.
**Why it happens:** Shallow-copying the entire `env` object without overriding `PROVIDER_SLOT`.
**How to avoid:** Explicitly set `newEnv.PROVIDER_SLOT = newName` (the new slot name) — see Pattern 3 above. The clone inherits the base URL and model, but `PROVIDER_SLOT` must be set to the new name.
**Warning signs:** Test `listAgents()` after clone — if the new slot shows the source slot's row data, `PROVIDER_SLOT` was not overridden.

### Pitfall 4: inquirer `list` choices with `value: ''` (empty string)
**What goes wrong:** The `__custom__` choice has `value: '__custom__'` as a sentinel. If the custom input prompt returns an empty string (user presses Enter with blank), `baseUrl` ends up as `''` and the probe is skipped (current `addAgent()` skips probe when `!answers.baseUrl.trim()`).
**Why it happens:** The preset flow adds a new state: preset selected → probe → or custom → optional probe. The empty custom URL case needs explicit handling.
**How to avoid:** When `presetChoice === '__custom__'` and `customUrl` is blank, treat it as "no URL configured" — skip probe, write slot without `ANTHROPIC_BASE_URL` (same as current behavior when user leaves URL blank in `addAgent()`).

### Pitfall 5: `editAgent()` base URL field is conditional
**What goes wrong:** In `editAgent()`, the base URL step only runs if `fields.includes('baseUrl')` is selected in the checkbox. Replacing the `input` with a `list` inside this conditional block must preserve the checkbox gating — the preset selector must NOT run unconditionally.
**Why it happens:** The checkbox UX collects what to change first, then runs only selected fields. Replacing the base URL step must stay inside the `if (fields.includes('baseUrl'))` block.
**How to avoid:** The preset selector replaces the inner `inquirer.prompt([{ type: 'input', name: 'baseUrl', ... }])` call (lines 706–714), not the outer `if` condition. Pre-fill the `list` to match the current URL if a preset matches it.

## Open Questions

1. **Should `editAgent()` preset selector pre-select the current URL if it matches a preset?**
   - What we know: inquirer `list` supports a `default` field set to a choice `value`. The current URL in `env.ANTHROPIC_BASE_URL` may or may not match a preset value.
   - What's unclear: If the current URL is `https://api.akashml.com/v1`, should the list start with `AkashML` highlighted? This is good UX but requires a lookup.
   - Recommendation: Add a `findPresetForUrl(url)` pure function that returns the matching preset `value` or `'__custom__'`. Use the result as the `default` for the `list` prompt. This is a small pure function testable in isolation.

2. **Does cloneSlot() need a probe pre-flight?**
   - What we know: The source slot was already validated when added. The provider URL is copied unchanged.
   - What's unclear: The success criteria do not mention a probe during cloning. PRST-02 says "copies provider URL and model config" — no probe requirement.
   - Recommendation: Do NOT add a probe to `cloneSlot()`. The source slot's URL is assumed valid (it was probed at add time). Adding a probe would complicate the flow unnecessarily and is not in the success criteria.

3. **What PROVIDER_SLOT value should the clone use?**
   - What we know: `PROVIDER_SLOT` is the key into `providers.json` (the subprocess provider table). If the source slot is an HTTP provider (uses `ANTHROPIC_BASE_URL`), `PROVIDER_SLOT` may not exist in `providers.json` and is used as a display key only.
   - What's unclear: Should the clone use the new slot name as `PROVIDER_SLOT` (safe default) or inherit the source `PROVIDER_SLOT`?
   - Recommendation: Set `PROVIDER_SLOT = newName` (the new slot name). This follows the pattern in `addAgent()` where `PROVIDER_SLOT` defaults to `slotName`. A cloned slot is a new entity and should have its own identity in `providers.json` lookups.
