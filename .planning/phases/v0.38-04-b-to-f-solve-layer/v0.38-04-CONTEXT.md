# Phase v0.38-04: B-to-F Solve Layer - Context

**Gathered:** 2026-03-18
**Status:** Ready for planning

<domain>
## Phase Boundary

The solve pipeline autonomously identifies bugs that formal models should explain but don't, and routes them to model creation or refinement. Adds `b_to_f` as the 20th solve layer with residual scoring, wave DAG integration, and remediation dispatch.

</domain>

<decisions>
## Implementation Decisions

### Bug classification thresholds
- `covered` = failing test maps to ≥1 formal model through requirement/recipe/annotation lineage AND that model was checked
- `covered_reproduced` = 0 residual (model reproduces the bug — no action needed)
- `covered_not_reproduced` = +1 residual (models exist but all pass — route to model refinement)
- `not_covered` = +1 residual (no linked formal model exists — route to model creation via close-formal-gaps)
- Route `covered_not_reproduced` to refinement by default; only route to creation if linked model set is empty or stale

### Cycle dispatch limits
- Defaults: max 3/cycle for `not_covered` gaps, max 2/cycle for blind spots (per success criteria)
- Config-backed limits matching existing per-layer caps pattern
- Overflow items remain in residual — never dropped, surfaced in `capped_layers`/detail reporting per CONV-03
- Carried forward to next solve cycle unchanged

### Solve layer reporting
- Use established unified table format (not custom one-off)
- Add `B → F` row with aggregate residual score
- Include compact breakdown: `{covered_reproduced, covered_not_reproduced, not_covered}` plus top bug ID examples
- Both text and JSON detail output available

### Remediation priority ordering
- Deterministic ordering matching existing `computeWaves()` bias
- Dispatch priority: `not_covered` before `covered_not_reproduced`
- Within each bucket: sort by repeated failure count, then requirement/test breadth
- Stable bug ID tiebreaker for deterministic output

### Claude's Discretion
- Exact wave position calculation for b_to_f in solve-wave-dag.cjs
- Internal data structures for classification tracking
- JSON detail schema for b_to_f layer output
- Error handling when formal-scope-scan or model checkers fail during classification

</decisions>

<specifics>
## Specific Ideas

- b_to_f is the 20th layer key — must follow existing layer registration pattern in layer-constants.cjs
- Depends on `t_to_c` layer (test-to-code traceability provides the test→requirement lineage needed for classification)
- Remediation uses close-formal-gaps for not_covered (model creation) and model-driven-fix for blind spots (model refinement) — both delivered in prior phases
- Residual scoring matches existing pattern: 0 = resolved, +1 = needs work

</specifics>
