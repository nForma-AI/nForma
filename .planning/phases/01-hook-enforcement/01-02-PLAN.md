Build the UserPromptSubmit hook that detects GSD planning commands and injects quorum instructions into Claude's context window before Claude processes the prompt.

Purpose: Without this injection, Claude has no instruction to run quorum tools at the start of a planning command. The Stop hook is the hard gate, but the UserPromptSubmit hook is the proactive signal that makes Claude run quorum correctly the first time rather than waiting to be blocked.

Output: `hooks/qgsd-prompt.js` — a self-contained Node.js script, no dependencies beyond stdlib.
</objective>

  <action>
    Create `hooks/qgsd-prompt.js` as a self-contained Node.js script. Follow the stdin-buffering pattern from `hooks/gsd-statusline.js` exactly.

    The hook must:

    1. Read stdin as UTF-8 string, JSON.parse the input
    2. Extract `input.prompt` (the raw user text)
    3. Load config from `~/.claude/qgsd.json` via loadConfig() — fall back to DEFAULT_QUORUM_COMMANDS and DEFAULT_QUORUM_INSTRUCTIONS if file missing or malformed
    4. Build the allowlist pattern as an anchored regex that matches the BEGINNING of the trimmed prompt:
       ```
       const cmdPattern = new RegExp('^\\s*\\/gsd:(' + commands.join('|') + ')(\\s|$)');
       ```
       This requires the `gsd:` prefix (mandatory — matches stop hook regex exactly) and the command at the start of the prompt (with optional leading whitespace) and a word boundary after (space or end of string). This prevents `/gsd:execute-phase` from matching when the allowlist contains `execute` and also prevents substring matches within longer strings.
    5. If cmdPattern does NOT match prompt → process.exit(0) (silent pass — UPS-05)
    6. If cmdPattern matches → write to stdout:
       ```json
       {
         "hookSpecificOutput": {
           "hookEventName": "UserPromptSubmit",
           "additionalContext": "<quorum instructions text>"
         }
       }
       ```
       Then process.exit(0)
    7. Wrap everything in try/catch → process.exit(0) on any error (fail-open)

    DEFAULT_QUORUM_COMMANDS (hardcoded, exactly 6 — UPS-02):
    ['plan-phase', 'new-project', 'new-milestone', 'discuss-phase', 'verify-work', 'research-phase']

    DEFAULT_QUORUM_INSTRUCTIONS text (UPS-04 — must name exact MCP tools):
    ```
    QUORUM REQUIRED (structural enforcement — Stop hook will verify)

    Before presenting any planning output to the user, you MUST:
      1. Call mcp__codex-cli__review with the full plan content
      2. Call mcp__gemini-cli__gemini with the full plan content
      3. Call mcp__opencode__opencode with the full plan content
      4. Present all model responses, resolve any concerns, then deliver your final output

    Fail-open: if a model is UNAVAILABLE (quota/error), note it and proceed with available models.
    The Stop hook reads the transcript — skipping quorum will block your response.
    ```

    loadConfig() reads `path.join(os.homedir(), '.claude', 'qgsd.json')`. If the file exists and parses, return `fileConfig.quorum_commands` and `fileConfig.quorum_instructions` (whichever keys are present). If file missing or malformed, return null (caller falls back to defaults).

    Add shebang: `#!/usr/bin/env node`

    Require only: fs, path, os (Node stdlib only — no npm deps).

    Do NOT use `systemMessage` — that field only shows a UI warning to the user and does NOT inject into Claude's context. Only `hookSpecificOutput.additionalContext` achieves context injection (confirmed bug workaround per PITFALLS.md Pitfall 2 and confirmed API behavior from STACK.md).

    Do NOT inject on execute-phase, debug, quick, update, map-codebase, or any other non-planning command. The anchored regex + explicit allowlist is the only mechanism. There is no secondary check — if it's not in the 6-item list, it gets exit 0 silently.
