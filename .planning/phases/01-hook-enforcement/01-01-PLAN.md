<feature>
  <name>qgsd-stop.js — Stop hook with all six guards</name>
  <files>hooks/qgsd-stop.js, templates/qgsd.json</files>
  <behavior>
    The hook reads JSON from stdin (Claude Code Stop event payload), applies guards in strict order, then scans the current-turn transcript window for quorum evidence.

    Guard order (mandatory — must be first two lines after stdin parse):
    1. if stop_hook_active === true → process.exit(0) [STOP-02]
    2. if hook_event_name === 'SubagentStop' → process.exit(0) [STOP-03]
    3. if transcript_path missing or file not found → process.exit(0) [fail-open]
    4. getCurrentTurnLines(): scan JSONL backward from end to find last user message boundary; return only lines from that point forward [STOP-04]
    5. hasQuorumCommand(currentTurnLines, config.quorum_commands): scan current-turn user entries for /gsd:(plan-phase|new-project|new-milestone|discuss-phase|verify-work|research-phase) [STOP-06]
       - If false → process.exit(0) [STOP-09]
    6. last_assistant_message fast-path [STOP-05]: if last_assistant_message contains at least one quorum tool prefix (mcp__codex-cli__, mcp__gemini-cli__, mcp__opencode__) as a substring → skip JSONL parse and check for all three in JSONL
       - This is a negative fast-path only: used to quickly skip the parse when the message clearly shows quorum happened, not to skip verification
       - Implementation: if last_assistant_message mentions all three tool prefixes → proceed to step 7 with optimistic check; otherwise parse JSONL
    7. findQuorumEvidence(currentTurnLines, config.required_models): for each assistant entry in scope, scan message.content[] for type:tool_use blocks whose name.startsWith(model.tool_prefix) [STOP-01]
       - Match by prefix NOT exact name (catches mcp__codex-cli__review AND mcp__codex-cli__codex) [MCP-06 — noted for Phase 2, but prefix matching implemented here]
    8. missingModels = required models where required:true and not found in step 7
    9. If missingModels is empty → process.exit(0) [STOP-09]
    10. If missingModels non-empty → stdout JSON: { decision: 'block', reason: 'QUORUM REQUIRED: Before completing this /gsd:[command] response, call [tool1], [tool2], [tool3] with your current plan. Present their responses, then deliver your final output.' } then process.exit(0) [STOP-07, STOP-08]
       - tool names derived from config: prefix + canonical tool name (e.g. mcp__codex-cli__review, mcp__gemini-cli__gemini, mcp__opencode__opencode)
       - command name extracted from the user message detected in step 5

    Config loading (loadConfig()):
    - Path: path.join(os.homedir(), '.claude', 'qgsd.json')
    - If file missing → return DEFAULT_CONFIG
    - If JSON.parse fails → return DEFAULT_CONFIG
    - Merge: { ...DEFAULT_CONFIG, ...fileConfig } (shallow spread, file values override defaults)

    DEFAULT_CONFIG hardcoded values:
    - quorum_commands: ['plan-phase', 'new-project', 'new-milestone', 'discuss-phase', 'verify-work', 'research-phase']
    - fail_mode: 'open'
    - required_models: { codex: { tool_prefix: 'mcp__codex-cli__', required: true }, gemini: { tool_prefix: 'mcp__gemini-cli__', required: true }, opencode: { tool_prefix: 'mcp__opencode__', required: true } }

    Error handling: any uncaught error in main() → process.exit(0) (fail-open — never crash the user's session)

    Test cases (input → expected output):
    - { stop_hook_active: true, ... } → exit 0, no stdout
    - { hook_event_name: 'SubagentStop', stop_hook_active: false, ... } → exit 0, no stdout
    - { hook_event_name: 'Stop', stop_hook_active: false, transcript_path: '/nonexistent' } → exit 0, no stdout
    - { hook_event_name: 'Stop', stop_hook_active: false, transcript with no planning command in current turn } → exit 0, no stdout
    - { hook_event_name: 'Stop', stop_hook_active: false, transcript with /gsd:plan-phase in current turn + all 3 quorum tool_use blocks } → exit 0, no stdout
    - { hook_event_name: 'Stop', stop_hook_active: false, transcript with /gsd:plan-phase in current turn + only codex tool_use } → stdout: { decision:'block', reason: 'QUORUM REQUIRED: ...' } + exit 0
    - { hook_event_name: 'Stop', stop_hook_active: false, transcript with /gsd:plan-phase in OLD turn (before current turn boundary) + no quorum in current turn } → exit 0, no stdout (scope filter)
    - Malformed JSONL lines in transcript → skip gracefully, continue scanning
    - Config file missing → use DEFAULT_CONFIG, hook still works correctly
  </behavior>
  <implementation>
    1. Write templates/qgsd.json first (defines the schema concretely):

    ```json
    {
      "quorum_commands": [
        "plan-phase", "new-project", "new-milestone",
        "discuss-phase", "verify-work", "research-phase"
      ],
      "fail_mode": "open",
      "required_models": {
        "codex":    { "tool_prefix": "mcp__codex-cli__",  "required": true },
        "gemini":   { "tool_prefix": "mcp__gemini-cli__", "required": true },
        "opencode": { "tool_prefix": "mcp__opencode__",   "required": true }
      },
      "quorum_instructions": "QUORUM REQUIRED (structural enforcement — Stop hook will verify)\n\nBefore presenting any planning output to the user, you MUST:\n  1. Call mcp__codex-cli__review with the full plan content\n  2. Call mcp__gemini-cli__gemini with the full plan content\n  3. Call mcp__opencode__opencode with the full plan content\n  4. Present all model responses, resolve concerns, then deliver final output\n\nFail-open: if a model is UNAVAILABLE (quota/error), note it and proceed with available models.\nThe Stop hook reads the transcript — skipping quorum will block your response."
    }
    ```

    2. Write hooks/qgsd-stop.js following the exact pattern of hooks/gsd-statusline.js (stdin buffering, JSON parse, error handling, process.exit).

    Use the implementation skeleton from 1-RESEARCH.md Pattern 1 as the base. The hook must be a self-contained Node.js script with no require() calls beyond fs, path, os (all Node stdlib).

    Add a shebang line: `#!/usr/bin/env node`

    The getCurrentTurnLines() function scans BACKWARD from end of lines array looking for the last entry where entry.type === 'user'. Returns lines.slice(lastUserIdx) if found, or all lines if no user message found (conservative: treat whole transcript as current turn).

    The hasQuorumCommand() function builds a regex from config.quorum_commands array (escape hyphens): new RegExp('\\/gsd:(' + commands.map(c => c.replace(/-/g, '\\-')).join('|') + ')') and tests it against JSON.stringify(entry.message) for each user entry in currentTurnLines.

    The findQuorumEvidence() function iterates assistant entries in currentTurnLines, digs into message.content[], finds tool_use blocks, checks block.name.startsWith(modelDef.tool_prefix) for each required_models entry.

    The block reason message format (STOP-08):
    "QUORUM REQUIRED: Before completing this /gsd:[command] response, call [tool1], [tool2], [tool3] with your current plan. Present their responses, then deliver your final output."
    - Extract [command] from the user message that matched the quorum command pattern
    - Derive [tool1/2/3] from config: prefix + 'review' for codex, prefix + 'gemini' for gemini, prefix + 'opencode' for opencode
