## Files Created/Modified
- `agents/qgsd-quorum-worker.md` — Parallel quorum slot worker spawned per active slot. Reads repo context, calls call-quorum-slot.cjs, returns structured YAML-like output. Supports Mode A (free-form position) and Mode B (APPROVE/REJECT/FLAG) verdicts. Handles UNAVAIL on non-zero exit or TIMEOUT.
- `agents/qgsd-quorum-synthesizer.md` — Barrier synthesizer spawned once per round after all workers complete. Parses worker results, emits UNAVAIL_HINT lines, checks consensus per mode rules, outputs SYNTHESIS_RESULT: CONSENSUS REACHED or SYNTHESIS_RESULT: DELIBERATION NEEDED with verbatim cross-pollination bundle.

## Decisions Made
- Worker tools are `Read, Bash, Glob, Grep` — no Write. Workers never call update-scoreboard.cjs; that remains the orchestrator's responsibility at the barrier.
- Synthesizer tools are `Read` only — it receives all worker data via $ARGUMENTS.workers[], no Bash needed.
- Mode A verdicts from workers are free-form position summaries (not APPROVE/REJECT/FLAG) — the plan explicitly noted this distinction.
- UNAVAIL_HINT lines are emitted before SYNTHESIS_RESULT: so the orchestrator can process set-availability calls sequentially without parsing the full synthesis output first.
- CROSS_POLLINATION_BUNDLE content is the verbatim `Prior positions:` block so orchestrators can paste it directly into Round 2 worker $ARGUMENTS.prior_positions.
