<task type="auto">
  <name>Task 2: Create qgsd-quorum-synthesizer.md</name>
  <files>agents/qgsd-quorum-synthesizer.md</files>
  <action>
Create agents/qgsd-quorum-synthesizer.md as a new agent file. Use the same frontmatter structure as qgsd-quorum-test-worker.md.

Frontmatter:
- name: qgsd-quorum-synthesizer
- description: Quorum barrier synthesizer — spawned once per round after all worker Tasks complete. Reads all worker result blocks, checks consensus, and either emits a final verdict or builds the cross-pollination context bundle for the next round.
- tools: Read
- color: purple

$ARGUMENTS schema (document in a <arguments> block):
```
round: <integer>
mode: A | B
question: <question text>
workers:
  - slot: <slotName>
    verdict: <value>
    reasoning: <text>
    raw: |
      <first 2000 chars>
    [unavail_message: <text>]
  [... one entry per worker ...]
[artifact_path: <path>]
```

<role> section: Synthesizer steps:
1. Parse all worker result blocks from $ARGUMENTS.workers[]
2. For each entry with verdict=UNAVAIL and non-empty unavail_message: output a hint line:
   `UNAVAIL_HINT: <slot> | <first 500 chars of unavail_message>`
   (Orchestrator uses this to call set-availability sequentially at barrier)
3. Filter available slots: exclude any entry where verdict=UNAVAIL
4. Check consensus:
   - Mode A: consensus = all available slots gave equivalent positions (orchestrator judgment, synthesizer describes disagreements)
   - Mode B: consensus = all available slots have verdict=APPROVE, OR any has REJECT
5. If CONSENSUS: output `CONSENSUS REACHED` block with verdict + all positions
6. If DELIBERATION NEEDED: output `DELIBERATION NEEDED` block with the full cross-pollination Prior positions: bundle ready to paste into Round 2 worker prompts

<output_format> block: Two possible output forms.

CONSENSUS output:
```
SYNTHESIS_RESULT: CONSENSUS REACHED
round: <integer>
consensus_verdict: APPROVE | REJECT | FLAG | CONSENSUS
available_slots: <N>
unavail_slots: <M>

FINAL ANSWER:
[Full synthesis of available positions — detailed and actionable]

Prior positions:
• <slot1>: [summary]
• <slot2>: [summary]
[...]
[• <slot>: UNAVAIL]
```

DELIBERATION output:
```
SYNTHESIS_RESULT: DELIBERATION NEEDED
round: <integer>
available_slots: <N>
unavail_slots: <M>

CROSS_POLLINATION_BUNDLE:
Prior positions:
• <slot1>: [full reasoning from worker result]
• <slot2>: [full reasoning from worker result]
[...]
[• <slot>: UNAVAIL]
```

The orchestrator reads SYNTHESIS_RESULT: to decide next step. CROSS_POLLINATION_BUNDLE: content is verbatim-pasted into the prior_positions: field of Round 2 $ARGUMENTS.
  </action>
