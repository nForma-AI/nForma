## Decisions Made

- SEQUENTIAL CALLS ONLY exception is intentionally narrow: only worker Task spawns in one message turn are permitted as siblings. All Bash calls (set-availability, merge-wave, scoreboard) remain strictly sequential.
- Pre-resolved timeouts in Step 2 means workers are self-contained: they only need their `timeout_ms:` value from $ARGUMENTS and do not need to discover providers.json themselves.
- Mode B wave-barrier structure is symmetric with Mode A — same two-round, two-barrier pattern — making the orchestrator's control flow uniform across both modes.
- Mode B Consensus output section added explicitly (was previously inline with the verdicts display) to make the scoreboard merge-wave pattern clear for the Mode B path.
