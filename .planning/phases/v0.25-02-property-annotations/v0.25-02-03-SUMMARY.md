## Files Created/Modified
- `bin/extract-annotations.cjs` - Core extraction parser (~280 lines). Reads model-registry.json, parses @requirement annotations per formalism, outputs JSON. Supports --pretty, --summary, --validate flags.
- `test/extract-annotations.test.cjs` - 23 tests in 6 suites: TLA+ parsing (5), Alloy parsing (3), PRISM parsing (5), edge cases (3), integration (5), validation mode (2).

## Decisions Made
- **Section-based detection**: TLA+ files use section markers ("Safety invariants", "Liveness properties", "Actions", etc.) to distinguish properties from actions -- both use `Name ==` syntax but only properties should be tracked
- **Registry-driven file discovery**: Parser reads model-registry.json to discover files rather than globbing the filesystem, maintaining consistency with the formal verification pipeline
- **Path filtering**: Paths starting with `..` or `/` are excluded from scanning to skip test fixtures and external references in the registry
- **Props resolution**: PRISM .props files are discovered by finding .pm entries in the registry and resolving the corresponding .props sibling
