## What Was Built

### formal/trace/redaction.yaml
Redaction policy file with two sections:
- `forbidden_keys`: 8 entries (api_key, secret, password, token, authorization, bearer, private_key, access_key)
- `forbidden_patterns`: 4 entries with name+regex (aws_access_key_id, github_pat, email, bearer_token)

### bin/check-trace-redaction.cjs
Trace event redaction validator:
- `parseRedactionPolicy(policyPath)`: line-by-line YAML parser, pre-compiles all regex objects at startup
- `validateTraceEvent(event, policy)`: iterates event keys, checks forbidden_keys and forbidden_patterns, returns violations array
- CLI: accepts `--trace-dir <path>` (default: `formal/trace/`); exits 0 when trace dir absent or no events; exits 1 on violations
- `module.exports = { parseRedactionPolicy, validateTraceEvent }` for test access
- `writeCheckResult` integration with `formalism: 'redaction'`

### bin/check-trace-redaction.test.cjs
8 tests (5 unit + 3 integration):
- parseRedactionPolicy loads yaml with >= 4 forbidden_keys and >= 3 forbidden_patterns
- validateTraceEvent with forbidden key returns 1 violation
- validateTraceEvent with AWS access key pattern returns 1 violation (pattern match)
- validateTraceEvent with clean event returns 0 violations
- validateTraceEvent with both forbidden key and pattern returns >= 2 violations
- Integration: exits 0 with clean JSONL
- Integration: exits 1 with api_key event
- Integration: exits 0 with non-existent trace directory (graceful)
