## Decisions Made

- **Use `-A 4` not `-A 3`**: The `**Gap Closure:**` field is at line offset 4 from the `^### Phase X:` header in ROADMAP.md, confirmed across all 7 gap-closure phases (v0.10-07, v0.10-08, v0.12-09, v0.12-10, v0.13-03, v0.13-04, v0.13-05). Using `-A 3` would always miss it. Gemini's BLOCK in quorum Round 1 caught the original audit's `-A 3` recommendation as wrong.
- **Anchor to `^### Phase`**: Prevents cross-phase bleed where a downstream gap-closure phase's "Depends on: Phase X" line falls within 15 lines of a `**Gap Closure:**` entry, causing primary phase X to be incorrectly detected as a gap closure phase.

## Quorum Context

This fix was mandated by Gemini's BLOCK in the v0.13-05 quorum planning round (2026-02-25). The original audit recommendation specified `-A 3`, which Gemini correctly identified as insufficient — the Gap Closure field is at offset 4, not 3. The quorum deliberation resulted in the correct `-A 4` value being used in both the plan and the implementation.
