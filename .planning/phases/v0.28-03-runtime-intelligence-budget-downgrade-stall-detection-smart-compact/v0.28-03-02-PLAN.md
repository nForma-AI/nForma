<task type="auto">
  <name>Task 1: Create stall detector module</name>
  <files>
    bin/stall-detector.cjs
  </files>
  <action>
Create `bin/stall-detector.cjs` exporting:

1. `detectStalledSlots(cwd, config)`:
   - Reads `stall_detection` from config (using defaults if missing).
   - Reads quorum-failures.json via `planning-paths.cjs` resolveWithFallback(cwd, 'quorum-failures'). If file doesn't exist, return empty array.
   - Parses JSON. Groups records by slot name.
   - For each slot, counts consecutive TIMEOUT entries from the END of the array (most recent first). Stop counting when a non-TIMEOUT entry is found for that slot.
   - Returns array of `{ slot, consecutiveTimeouts, lastSeen }` for slots where consecutiveTimeouts >= 1.
   - Fail-open: try/catch around all I/O, returns [] on error.

2. `shouldEscalate(stalledSlots, config, cwd)`:
   - Takes output from detectStalledSlots, config, and cwd.
   - Gets threshold from `config.stall_detection.consecutive_threshold || 2`.
   - Filters stalledSlots to those with `consecutiveTimeouts >= threshold`.
   - If `config.stall_detection.check_commits !== false`, counts recent commits via `git rev-list --count HEAD --since="10 minutes ago"` (spawnSync, 3s timeout). If count > 0, return `{ escalate: false, reason: 'commits_active', stalledSlots: filtered }`.
   - If filtered is empty, return `{ escalate: false, reason: 'below_threshold', stalledSlots: [] }`.
   - Otherwise return `{ escalate: true, stalledSlots: filtered }`.

3. `formatStallReport(escalationResult)`:
   - Takes output from shouldEscalate.
   - If not escalate, returns null.
   - Returns a structured object:
   ```javascript
   {
     type: 'stall_report',
     ts: new Date().toISOString(),
     stalled_slots: escalationResult.stalledSlots.map(s => ({
       slot: s.slot,
       consecutive_timeouts: s.consecutiveTimeouts,
       last_seen: s.lastSeen,
       recommendation: 'Check provider health: node bin/check-mcp-health.cjs',
     })),
     recommendation: 'Stalled slots detected with no new commits. Consider checking provider status or adjusting quorum composition.',
   }
   ```
   - Also returns a string `message` property for additionalContext injection: "STALL DETECTED: {N} slot(s) stalled ({slotNames}). Run `node bin/check-mcp-health.cjs` to diagnose."

IMPORTANT: Stall detection is INFORMATIONAL ONLY. Never return a block decision. The formatted message is injected via additionalContext, not used to block.

IMPORTANT: Do NOT run stall detection when a cache hit occurred. The caller (Plan 03 will wire this in gsd-context-monitor.js) should skip stall checks on cache hits.

Use only node:fs, node:path, node:child_process. No external dependencies.
  </action>

  </verify>
