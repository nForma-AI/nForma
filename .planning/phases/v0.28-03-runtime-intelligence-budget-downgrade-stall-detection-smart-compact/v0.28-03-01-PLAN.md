**bin/budget-tracker.cjs:**

Create a new module exporting:

1. `DOWNGRADE_CHAIN = { quality: 'balanced', balanced: 'budget', budget: null }` -- the profile downgrade sequence.

2. `computeBudgetStatus(usedPct, budgetConfig, agentConfig)`:
   - Takes context window used percentage (0-100), budget config section, and agent_config section.
   - If `budgetConfig.session_limit_tokens` is null/undefined, returns `{ active: false }`.
   - Estimates tokens: `Math.round((usedPct / 100) * 200000)` (200K context window assumption).
   - Computes `budgetUsedPct = Math.round((estimatedTokens / budgetConfig.session_limit_tokens) * 100)`.
   - Returns `{ active: true, estimatedTokens, budgetUsedPct, shouldWarn: budgetUsedPct >= (budgetConfig.warn_pct || 60), shouldDowngrade: budgetUsedPct >= (budgetConfig.downgrade_pct || 85) }`.

3. `triggerProfileDowngrade(cwd)`:
   - Reads `.planning/config.json` from cwd.
   - Gets current `model_profile` (default 'balanced').
   - Looks up next in DOWNGRADE_CHAIN. If null (already at minimum), returns `{ downgraded: false, current: profile }`.
   - Writes new model_profile to config.json.
   - Returns `{ downgraded: true, from: current, to: next }`.
   - Wrapped in try/catch -- fail-open, returns `{ downgraded: false, error: message }`.

4. `formatBudgetWarning(status, downgradeResult)`:
   - Returns a string message suitable for additionalContext injection.
   - If shouldDowngrade and downgraded: "BUDGET ALERT: Context at {budgetUsedPct}% of token budget. Model profile downgraded from '{from}' to '{to}'. Consider running /compact."
   - If shouldWarn: "BUDGET WARNING: Context at {budgetUsedPct}% of token budget ({estimatedTokens} estimated tokens of {limit} limit). Monitor usage and consider /compact at next clean boundary."

Use only node:fs, node:path. No external dependencies.
