## Audit Findings

### 1. Run-collapse algorithm: CONFIRMED CORRECT
TLA+ `CollapseRuns` uses the same consecutive-duplicate-merging logic as JS `detectOscillation`. Abstract labels correctly model the `files.slice().sort().join('\0')` key behavior.

### 2. Depth threshold: CONFIRMED CORRECT
`Depth = 3` in MCoscillation.cfg matches JS default `oscillation_depth = 3`. The `>=` comparison is identical in both.

### 3. Net-diff second-pass (hasReversionInHashes): CONFIRMED CORRECT
TLA+ `SetNetChange(n)` nondeterministically sets netChange in `-(CommitWindow)..(CommitWindow)`, covering both positive (TDD progression) and non-positive (genuine reversion). This over-approximation correctly models `totalNetChange <= 0` in JS.

### 4. Flag condition: CONFIRMED CORRECT
`OscillationFlaggedCorrectly == flagged <=> (\E l \in Labels : flagCount[l] >= Depth /\ netChange <= 0)` precisely encodes the JS conjunction: `keyRunList.length >= depth && hasReversionInHashes() === true`.

### Overall: NO DIVERGENCES

## Known Gap (documented)

`hasReversionInHashes` and `detectOscillation` are not exported from `qgsd-circuit-breaker.js` — only `buildWarningNotice` and `buildBlockReason` are exported. Fixture tests for the oscillation algorithm itself are marked `test.todo()` with explanation. To enable direct fixture testing: export `hasReversionInHashes` from the hook.
