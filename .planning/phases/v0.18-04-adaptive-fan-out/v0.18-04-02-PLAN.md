**Implementation of mapRiskLevelToCount:**
```javascript
// Maps task envelope risk_level to a fan-out count (total participants including Claude).
// routine → 2 (Claude + 1 external: minimum R3.5 valid quorum)
// medium  → 3 (Claude + 2 external: balanced quorum)
// high    → maxSize (Claude + maxSize-1 external: full pool, unchanged behavior)
// absent/invalid → maxSize (fail-open: conservative)
//
// Note: 'low' is mapped like 'routine' (2 workers).
// maxSize from config.quorum.maxSize is the ceiling — never exceed it.
function mapRiskLevelToCount(riskLevel, maxSize) {
  if (riskLevel === 'routine' || riskLevel === 'low') return Math.min(2, maxSize);
  if (riskLevel === 'medium') return Math.min(3, maxSize);
  // 'high', undefined, null, invalid string → fail-open to maxSize
  return maxSize;
}
```

Add at the module level (not inside the stdin handler), after `parseQuorumSizeFlag`.

**Export for testability:** At the very end of qgsd-prompt.js, before the stdin handler or after all functions are defined, add:
```javascript
// Export helpers for unit testing (tree-shaken at runtime — no cost)
if (typeof module !== 'undefined') {
  module.exports = module.exports || {};
  module.exports.mapRiskLevelToCount = mapRiskLevelToCount;
  module.exports.parseQuorumSizeFlag = parseQuorumSizeFlag;
}
```

NOTE: The existing file does NOT use `module.exports` (it's a script, not a module). Add the export block AFTER the stdin handler closes, guarded by `typeof module !== 'undefined'` to be safe. The hook exits via `process.exit(0)` before the export assignment is reached in normal operation — the export is only accessible when the file is `require()`d in tests.

Actually, the correct pattern for a script-as-module: place `module.exports = { mapRiskLevelToCount, parseQuorumSizeFlag };` at the END of the file after the stdin `process.stdin.on('end', ...)` handler. Since the file is a script, it exits in the stdin handler path before reaching the export line when run directly. When `require()`d by tests, the stdin handler is registered but never fires (no stdin data), so module.exports is set.
