  <name>Task 1: Add --bug-mode flag to formal-scope-scan.cjs with formalism type and requirement coverage output</name>
  <files>bin/formal-scope-scan.cjs, test/bug-lookup.test.cjs</files>
  <action>
Extend the existing formal-scope-scan.cjs with a `--bug-mode` flag. When active, the tool changes behavior:

1. **Parse --bug-mode flag** in `parseArgs()`: add `bugMode: false` to args, set true when `--bug-mode` is present.

2. **Load model-registry.json** when in bug-mode (path: `.planning/formal/model-registry.json`). This provides formalism type (detect from file extension: `.tla` = tla, `.als` = alloy) and requirement coverage (from `requirements` array in each model entry).

3. **Add semantic/concept scoring** for bug-mode: In addition to the existing Layer 1 (scope.json matching) and Layer 2 (proximity enrichment), add a bug-mode-specific scoring function `scoreConceptMatch(bugDescription, modelMetadata)` that:
   - Tokenizes bug description (lowercase, split on whitespace, filter tokens > 2 chars)
   - Scores +0.1 for each matching token in model description
   - Scores +0.2 for matching requirement category prefix (e.g., "DETECT" in bug matches model with DETECT-01)
   - Caps score at 1.0
   - Models with score > 0 are added as matches with `matched_by: 'bug_pattern'`

4. **Enrich output schema** in bug-mode: Each match object gains:
   - `formalism`: "tla" or "alloy" (derived from model-registry key extension)
   - `requirement_coverage`: array of requirement IDs from model-registry entry
   - `bug_relevance_score`: combined score from concept matching + proximity

5. **Rank results** by `bug_relevance_score` descending, then by existing match-type priority.

6. **Update help text** to document `--bug-mode` flag usage.

7. **Fail-open**: If model-registry.json is missing, bug-mode falls back to standard mode with a stderr warning.

Reference the research pattern from v0.38-01-RESEARCH.md `scoreConceptMatch` example. Use the existing `enrichWithProximityIndex` function as the enrichment template.

Create `test/bug-lookup.test.cjs` using node:test framework (consistent with existing test files like `test/alloy-headless.test.cjs`):
- Test: --bug-mode flag parsed correctly
- Test: semantic scoring returns > 0 for matching description
- Test: formalism type derived correctly from .tla and .als extensions
- Test: requirement_coverage populated from model-registry
- Test: results ranked by bug_relevance_score descending
- Test: fail-open when model-registry.json is missing

<task type="auto">
  <name>Task 2: Create bug-model-gaps.json persistence layer</name>
  <files>.planning/formal/bug-model-gaps.json, bin/formal-scope-scan.cjs, test/bug-lookup.test.cjs</files>
  <action>
1. **Create initial `.planning/formal/bug-model-gaps.json`** with schema:
```json
{
  "version": "1.0",
  "entries": []
}
```

Each entry follows this schema (do NOT populate entries yet — this is the empty initial file):
```
{
  "bug_id": "string (nanoid or descriptive slug)",
  "description": "string (bug description from --description flag)",
  "timestamp": "ISO 8601 string",
  "status": "no_coverage | no_reproduction | reproduced",
  "matched_models": ["model-registry key paths"],
  "checked_models": [{ "model": "path", "result": "pass|fail|timeout", "trace": "string|null" }],
  "session_id": "string (optional, for cross-session tracking)"
}
```

2. **Add `--persist-gap` flag** to formal-scope-scan.cjs: When `--bug-mode --persist-gap` are both active, after the lookup completes:
   - Generate a bug_id from the first 8 chars of a hash of the description + timestamp
   - Determine status: if matches found, status = "no_reproduction" (checker hasn't run yet). If no matches, status = "no_coverage".
   - Append entry to bug-model-gaps.json `entries` array
   - Write file back with `JSON.stringify(data, null, 2)`
   - Deduplicate: if an entry with the same description already exists (exact match), update its timestamp and matched_models instead of creating a duplicate

3. **Add helper functions** in formal-scope-scan.cjs:
   - `loadBugModelGaps()`: read and parse bug-model-gaps.json, return `{ version, entries }`. If file missing, return `{ version: "1.0", entries: [] }`.
   - `saveBugModelGaps(data)`: write to disk with pretty JSON.
   - `hashBugId(description)`: use `require('crypto').createHash('sha256').update(description).digest('hex').slice(0, 8)`.

4. **Add tests** to test/bug-lookup.test.cjs:
   - Test: --persist-gap creates entry with correct schema
   - Test: no_coverage status when no models match
   - Test: no_reproduction status when models match but no checker ran
   - Test: deduplication updates existing entry instead of creating duplicate
   - Test: persistence survives simulated session boundary (write, re-read, verify)

Use a temporary directory for test isolation — write bug-model-gaps.json to a temp path, not the real `.planning/formal/` directory.
