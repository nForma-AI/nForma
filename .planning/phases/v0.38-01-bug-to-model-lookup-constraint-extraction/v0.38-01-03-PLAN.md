<task type="auto">
  <name>Task 1: Add --run-checkers flag with TLC/Alloy execution and timeout</name>
  <files>bin/formal-scope-scan.cjs</files>
  <action>
Extend formal-scope-scan.cjs (already modified by Plan 01) with a `--run-checkers` flag that executes model checkers on matched models.

1. **Parse --run-checkers flag** in `parseArgs()`: add `runCheckers: false` to args, set true when `--run-checkers` is present. Only valid when `--bug-mode` is also set.

2. **Add `runModelCheckers(matches, maxModels, timeoutMs)` function:**
   - Takes matched models from bug-mode lookup (max 3 per BML-02 requirement, sorted by bug_relevance_score)
   - For each matched model:
     - Determine formalism from `formalism` field (set by Plan 01)
     - For TLA+ models: Find the corresponding MC config file. Look in `.planning/formal/tla/` for `MC*.cfg` files. The model-registry key is the spec path (e.g., `.planning/formal/tla/NFCircuitBreaker.tla`). Find MC configs by scanning for cfg files that reference the spec's module name.
     - For Alloy models: The spec path from model-registry is the .als file directly.
     - Spawn checker with `child_process.spawnSync`:
       - TLA+: `node bin/run-tlc.cjs <config-name> --project-root=<ROOT>` with timeout
       - Alloy: `node bin/run-alloy.cjs --spec=<spec-name> --project-root=<ROOT>` with timeout
     - Set `timeout: timeoutMs` (default 60000ms = 60s per BML-02) on spawnSync options
     - Parse checker output to determine result:
       - **pass**: checker exits 0, no invariant violations
       - **fail**: checker exits non-zero OR stdout contains "Error:" / "Invariant .* is violated" / "Assertion .* may be violated" — this means the model reproduces a failure condition
       - **timeout**: spawnSync returns `status === null` and `signal === 'SIGTERM'`
     - Extract counterexample trace when available:
       - TLA+: Parse stderr/stdout for state trace between "Error:" and "Finished" markers
       - Alloy: Parse stdout for counterexample instance text
     - Record result: `{ model: path, formalism, result: 'pass'|'fail'|'timeout', trace: string|null, runtime_ms }`

3. **Integrate with --persist-gap**: When both `--run-checkers` and `--persist-gap` are active, update the bug-model-gaps.json entry:
   - Add `checked_models` array with checker results
   - Update status:
     - If any checker result is "fail" (model reproduces failure): status = "reproduced"
     - If all checkers pass or timeout: status = "no_reproduction"
     - If no models matched: status = "no_coverage" (unchanged from Plan 01)

4. **Enrich output**: When --run-checkers is active, each match in the output gains:
   - `checker_result`: "pass" | "fail" | "timeout" | "skipped" (skipped if beyond max 3)
   - `checker_trace`: counterexample trace string (null if none)
   - `checker_runtime_ms`: execution time in milliseconds

5. **Fail-open**: If Java is not installed or checker binary is missing, log warning to stderr and set result to "skipped" with reason. Do NOT crash.

6. **Update help text** to document `--run-checkers` flag.

**Important caveats:**
- Max 3 models per BML-02 requirement. If more than 3 match, run only top 3 by bug_relevance_score, mark rest as "skipped".
- 60s timeout per checker, not total. Three checkers could take up to 180s total.
- Do NOT modify run-tlc.cjs or run-alloy.cjs — use them as-is via subprocess spawning.
  </action>
