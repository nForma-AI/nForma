**Part A: Add auth_type to providers.json**

Add an `"auth_type"` field to every provider entry in `bin/providers.json`:

- Subscription CLI slots (auth_type: "sub"): codex-1, codex-2, gemini-1, gemini-2, opencode-1, copilot-1
- API-backed slots (auth_type: "api"): claude-1, claude-2, claude-3, claude-4, claude-5, claude-6

Place the field after `"type": "subprocess"` for readability. The classification rule is: if the slot uses a native CLI tool with its own subscription/login (codex, gemini, opencode, copilot), it is "sub". If the slot routes through `ccr` (claude-code-router) to a third-party API endpoint requiring an API key, it is "api".

**Part B: Wire auth_type into mcp-setup first-run flow**

In `commands/nf/mcp-setup.md`, find the first-run onboarding section (Step 2). After the existing step where each selected agent is written to `~/.claude.json` mcpServers, add a new sub-step that writes `auth_type` to `~/.claude/nf.json` agent_config.

Add a new step (Step 2e or equivalent, after keys are configured and before the "restart agents" step) with this Bash command pattern:

```bash
node -e "
const fs = require('fs');
const path = require('path');
const os = require('os');

// Read providers.json for auth_type lookup
const providersPath = path.join('INSTALL_DIR', 'bin', 'providers.json');
const providersData = JSON.parse(fs.readFileSync(providersPath, 'utf8'));
const providerMap = {};
for (const p of (providersData.providers || [])) {
  providerMap[p.name] = p;
}

// Read existing nf.json
const nfPath = path.join(os.homedir(), '.claude', 'nf.json');
let nfCfg = {};
try { nfCfg = JSON.parse(fs.readFileSync(nfPath, 'utf8')); } catch(e) {}

const agentConfig = nfCfg.agent_config || {};

// For each configured agent, set auth_type from providers.json
const selectedSlots = SELECTED_SLOTS; // array of slot names from earlier step
for (const slot of selectedSlots) {
  const provider = providerMap[slot];
  if (!agentConfig[slot]) agentConfig[slot] = {};
  agentConfig[slot].auth_type = provider ? provider.auth_type : 'api';
}

nfCfg.agent_config = agentConfig;
fs.writeFileSync(nfPath, JSON.stringify(nfCfg, null, 2));
console.log('auth_type written for: ' + selectedSlots.join(', '));
"
```

IMPORTANT: The `INSTALL_DIR` must resolve to the nForma install directory. Use the same path resolution pattern already used elsewhere in mcp-setup.md (look for how it resolves `bin/providers.json` or `bin/manage-agents-core.cjs` in existing steps). The `SELECTED_SLOTS` array comes from the user's selection in Step 2a.

Add clear prose instructions for the Claude executor to:
1. Collect the list of slot names the user selected during onboarding
2. Read auth_type from providers.json for each selected slot
3. Write auth_type to nf.json agent_config for each slot
4. Log which slots were configured
