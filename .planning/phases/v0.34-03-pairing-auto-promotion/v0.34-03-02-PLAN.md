**Auto-promotion eligibility (PROMO-03):**
A model is eligible for SOFT_GATE -> HARD_GATE when ALL of:
1. Current gate_maturity is "SOFT_GATE"
2. All 3 gates pass (gate_a.pass, gate_b.pass, gate_c.pass in per-model-gates.json)
3. `consecutive_clean_sessions >= 3` (read from solve-state.json)
4. Cooldown satisfied (use gate-stability.cjs `checkCooldown()`)

**Implementation:**

Add `--auto-promote` CLI flag parsing alongside existing `--check` and `--model` flags.

When `--auto-promote` is used:
1. Load solve-state.json to get consecutive_clean_sessions
2. If consecutive_clean_sessions < 3: print "Not enough clean sessions (N/3), no promotions" and exit 0
3. Load per-model-gates.json for gate pass info
4. Load model-registry.json
5. For each model with gate_maturity "SOFT_GATE" AND all 3 gates pass:
   - Check cooldown via gate-stability.cjs: `checkCooldown(modelPath, changelog)` — if returns false, skip with log
   - Check flip-flop via gate-stability.cjs: `countDirectionChanges(entries)` — if >= threshold, skip with log (PROMO-05)
   - Promote: set gate_maturity to "HARD_GATE" in model-registry.json
   - Append to promotion-changelog.json (PROMO-04):
     ```json
     {
       "model": "path/to/model.als",
       "from_level": "SOFT_GATE",
       "to_level": "HARD_GATE",
       "timestamp": "ISO",
       "session_id": null,
       "evidence_readiness": { "score": N, "total": 5 },
       "trigger": "auto_promotion",
       "consecutive_clean_sessions": 3
     }
     ```
6. Write updated model-registry.json
7. Print summary: "Promoted N models SOFT_GATE -> HARD_GATE"

**JSON output** (when combined with `--json`):

  "promoted": ["model1.als", "model2.als"],

**Step 8a code:**

```javascript
// Step 8a: Auto-promote eligible models (PROMO-01)
if (!reportOnly && solveState.consecutive_clean_sessions >= 3) {
  process.stderr.write(TAG + ' Step 8a: Checking auto-promotion eligibility (clean sessions: ' + solveState.consecutive_clean_sessions + ')\n');
  const promoResult = spawnTool('bin/promote-gate-maturity.cjs', ['--auto-promote', '--json']);
  if (promoResult.ok) {
    try {
      const promoData = JSON.parse(promoResult.stdout);
      if (promoData.promoted && promoData.promoted.length > 0) {
        process.stderr.write(TAG + ' Step 8a: Promoted ' + promoData.promoted.length + ' model(s) SOFT_GATE -> HARD_GATE\n');
        for (const m of promoData.promoted) {
          process.stderr.write(TAG + '   - ' + m + '\n');
        }
      } else {
        process.stderr.write(TAG + ' Step 8a: No models eligible for promotion\n');
      }
    } catch (e) {
      process.stderr.write(TAG + ' Step 8a: Could not parse promotion result: ' + e.message + '\n');
    }
  } else {
    process.stderr.write(TAG + ' Step 8a: promote-gate-maturity.cjs failed: ' + (promoResult.stderr || 'unknown') + '\n');
  }
} else if (!reportOnly) {
  process.stderr.write(TAG + ' Step 8a: Skipped — consecutive_clean_sessions=' + (solveState.consecutive_clean_sessions || 0) + ' (need 3)\n');
}
```

**Important:** This uses the existing `spawnTool()` helper already in nf-solve.cjs. The promotion runs as a subprocess, not inline, maintaining separation of concerns.
