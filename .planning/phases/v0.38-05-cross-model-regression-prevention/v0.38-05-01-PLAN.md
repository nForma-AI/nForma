1. **Create bin/resolve-proximity-neighbors.cjs** — a standalone CLI and importable module for resolving 2-hop formal_module neighbors from proximity-index.json.

**CLI interface:**
```
Usage:
  node bin/resolve-proximity-neighbors.cjs --model=account-manager [--max-hops=2] [--max-neighbors=10] [--format=json|csv|lines]
  node bin/resolve-proximity-neighbors.cjs --model=oscillation --project-root=/path/to/repo
```

**Core function: `resolveNeighbors(index, startModuleId, options)`**
- `index`: parsed proximity-index.json object
- `startModuleId`: the formal module name (e.g., "account-manager"), without the `formal_module::` prefix
- `options`: `{ maxHops: 2, maxNeighbors: 10 }`
- Returns: `{ neighbors: [{ id, hop_distance, shared_concepts }], warnings: [] }`

**Algorithm (bidirectional BFS):**

a. Build a reverse edge index at load time: for every node in `index.nodes`, for each edge `A -> B`, also record `B -> A` in a reverse map. This ensures traversal works regardless of edge direction.

b. Start BFS from `formal_module::${startModuleId}`. Initialize `visited = new Set([startKey])`, `frontier = [startKey]`.

c. For each hop (up to `maxHops`):
   - For each node in frontier, collect all outgoing edges (from `node.edges`) AND all reverse edges (from the reverse index)
   - Add unvisited targets to `nextFrontier`, mark as visited
   - Any target starting with `formal_module::` is a neighbor — add to results with `hop_distance = currentHop + 1`
   - Update frontier = nextFrontier

d. **Proximity scoring for cap:** When more than `maxNeighbors` found:
   - Primary sort: hop_distance ascending (1-hop neighbors before 2-hop)
   - Secondary sort: shared concept count descending (neighbors sharing more intermediate concept/code_file nodes rank higher)
   - Take first `maxNeighbors`

e. **Config-backed cap:** Read max_neighbors from config if available:
   ```javascript
   let maxNeighbors = options.maxNeighbors || 10;
   // Check project config for override
   try {
     const configPath = path.join(root, '.planning', 'config.json');
     if (fs.existsSync(configPath)) {
       const config = JSON.parse(fs.readFileSync(configPath, 'utf8'));
       if (config.cross_model_max_neighbors) maxNeighbors = config.cross_model_max_neighbors;
     }
   } catch (_) { /* fail-open */ }
   ```

f. **Error handling:**
   - proximity-index.json missing: return `{ neighbors: [], warnings: ['proximity-index.json not found. Run /nf:proximity to generate.'] }`
   - Start module not found in graph: return `{ neighbors: [], warnings: ['Model "X" not found in proximity-index.json'] }`
   - Malformed JSON: return with warning, empty neighbors

g. **CLI output formats:**
   - `--format=json` (default): full JSON object
   - `--format=csv`: comma-separated model IDs (e.g., "oscillation,breaker,convergence")
   - `--format=lines`: one model ID per line

h. **Module exports:**
   ```javascript
   module.exports = { resolveNeighbors, loadProximityIndex, buildReverseEdges };
   ```

2. **Apply scope filter after existing `--only` filter:**

After the `only` filter block (around line 443), add scope filtering:

```javascript
let steps = only
  ? STEPS.filter(s => s.tool === only || s.id === only)
  : STEPS;

// --scope: filter to specific model names (applies after --only)
if (scopeList) {
  steps = steps.filter(s => {
    // Match against step ID suffix (after the colon)
    const modelName = s.id.includes(':') ? s.id.split(':').slice(1).join(':') : s.id;
    return scopeList.some(m =>
      modelName === m ||
      s.id === m ||
      s.id.toLowerCase() === m
    );
  });
}
```

3. **Add scope validation error:**

After the existing `only` validation block, add:
```javascript
if (scopeList && steps.length === 0) {
  process.stderr.write(
    TAG + ' No models match --scope=' + scopeList.join(',') + '\n' +
    TAG + ' Available model IDs:\n'
  );
  for (const s of STEPS) {
    process.stderr.write(TAG + '   ' + s.id + '\n');
  }
  process.exit(1);
}
```

4. **Update summary output** to show scope when active:

In the `runOnce()` function, after the `only` display line, add:
```javascript
if (scopeList) {
  process.stdout.write(TAG + ' Scope: --scope=' + scopeList.join(',') + '\n');
}
```

5. **Precedence behavior:** `--scope` takes priority over `--only`. If both are given, apply `--only` first (filter by tool type), then `--scope` (filter by model name within that type). This is a natural consequence of the sequential filter order.

6. **Default behavior preserved:** When no `--scope` is given, `scopeList` is null, and the filter block is skipped entirely. The full suite runs as before.
