After the user confirms fix is applied (or auto mode proceeds):

```markdown
### Phase 5b — Cross-Model Regression Check

Display banner:
Phase 5b — Cross-Model Regression Check
  Resolving 2-hop proximity neighbors for: {$REPRODUCING_MODEL}

**Step 1: Resolve neighbors**

Extract model ID from $REPRODUCING_MODEL path. For TLA+ models, the ID is the config name (e.g., MCsafety from .planning/formal/tla/MCsafety.cfg). For Alloy, the spec name (e.g., quorum-votes from .planning/formal/alloy/quorum-votes.als).

NEIGHBORS=$(node bin/resolve-proximity-neighbors.cjs --model="$MODEL_ID" --format=json)

Parse JSON output. Extract neighbor IDs array.

If neighbors is empty (no proximity data or no neighbors found):
  Display: `No proximity neighbors found for ${MODEL_ID}. Skipping cross-model regression check.`
  Set NEIGHBOR_MODELS_PASS=null (inconclusive)
  Proceed to Phase 5c (result persistence).

If neighbors is non-empty:
  Display:
  Found {N} proximity neighbor(s) for {MODEL_ID}:
    {neighbor_id_1} (hop: {distance})
    {neighbor_id_2} (hop: {distance})
    ...

**Step 2: Build scope list and run scoped verification**

Build comma-separated scope from neighbor model IDs:
SCOPE=$(echo "$NEIGHBORS" | node -e "
  const input = require('fs').readFileSync('/dev/stdin','utf8');
  const data = JSON.parse(input);
  console.log(data.neighbors.map(n => n.id).join(','));
")

Run scoped formal verification:
node bin/run-formal-verify.cjs --scope="$SCOPE" --project-root="$(pwd)" 2>&1

Parse results. For each neighbor model:
- If checker passed: add to passed_neighbors array
- If checker failed: add to regressions array with violation details
- If checker timed out or errored: add to regressions with result="error"

**Step 3: Compute summary**

MODEL_PASS = true (the reproducing model was verified in Phase 6, renamed to be after 5b)
NEIGHBOR_MODELS_PASS = (regressions.length === 0)
REGRESSION_COUNT = regressions.length

**Step 4: Display summary table**

Display:
┌─────────────────────────────────────────────────────────┐
│ Cross-Model Regression Check                            │
├───────────────────────┬─────────────────────────────────┤
│ Primary model         │ {PASS/FAIL}                     │
│ Neighbor models       │ {PASS/FAIL/INCONCLUSIVE}        │
│ Regressions           │ {count}                         │
└───────────────────────┴─────────────────────────────────┘

If regressions found:
  Display each regression:
    WARNING: Regression in {model_id} ({formalism}): {violation summary}

  If $STRICT is true:
    Display:
    BLOCKED: --strict mode active. Fix cannot proceed until all neighbor model regressions are resolved.
    Regressions:
      - {model_id}: {violation}

    Fix is NOT declared done. User must resolve regressions and re-run.
    Exit workflow (do not proceed to Phase 6).

  If $STRICT is false (default):
    Display:
    Regressions detected but proceeding (fail-open mode).
    Re-run with --strict to gate fix completion on all neighbor models passing.
    Proceed to Phase 5c.

If no regressions:
  Display: All {N} neighbor model(s) pass. No cross-model regressions detected.
  Proceed to Phase 5c.
```

**Phase 5c — Persist Post-Fix Verification Results (NEW):**

```markdown
### Phase 5c — Persist Results

Write post_fix_verification results to bug-model-gaps.json entry.

Find the entry for the current bug (match by bug_id or model path):

ENTRY_UPDATE=$(node -e "
  const fs = require('fs');
  const gapsPath = '.planning/formal/bug-model-gaps.json';
  const gaps = JSON.parse(fs.readFileSync(gapsPath, 'utf8'));

  // Find or create entry for this bug
  let entry = gaps.entries.find(e => e.model_path === '$REPRODUCING_MODEL' || e.bug_id === '$BUG_ID');
  if (!entry) {
    entry = { bug_id: '$BUG_ID', model_path: '$REPRODUCING_MODEL' };
    gaps.entries.push(entry);
  }

  entry.post_fix_verification = {
    timestamp: new Date().toISOString(),
    model_pass: $MODEL_PASS,
    neighbor_models_pass: $NEIGHBOR_MODELS_PASS,
    neighbor_count: $NEIGHBOR_COUNT,
    regressions: $REGRESSIONS_JSON,
    passed_neighbors: $PASSED_JSON
  };

  fs.writeFileSync(gapsPath, JSON.stringify(gaps, null, 2));
  console.log('post_fix_verification persisted');
")
```
