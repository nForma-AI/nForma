Create `bin/observe-registry.cjs` exporting:

1. `registerHandler(sourceType, handlerFn)` — Adds a handler function to the registry map. Handler signature: `async handler(sourceConfig, options) -> { source_label, source_type, status, issues[], error? }`. Throws if sourceType already registered (prevents silent overwrite).

2. `getHandler(sourceType)` — Returns the registered handler function for a source type, or null if not registered.

3. `listHandlers()` — Returns array of registered source type strings.

4. `dispatchSource(sourceConfig, options, timeoutSeconds)` — Wraps handler call with Promise.race timeout:
   - Looks up handler via `getHandler(sourceConfig.type)`
   - If no handler registered, returns `{ source_label, source_type, status: "error", error: "No handler registered for type: X", issues: [] }`
   - Wraps handler invocation in Promise.race with timeout: `Promise.race([handlerFn(sourceConfig, options), timeoutPromise])`
   - On timeout, returns `{ source_label, source_type, status: "error", error: "Timeout after Ns", issues: [] }`
   - On handler error (thrown exception), catches and returns error status (fail-open)

5. `dispatchAll(sources, options)` — Takes array of source configs, calls dispatchSource for each in parallel via `Promise.allSettled`, returns array of results. This is the core parallel dispatch function per OBS-08.

The registry uses a simple `Map` internally. No external dependencies.

Create `bin/observe-registry.test.cjs` with node:test covering:
- registerHandler adds handler, getHandler retrieves it
- registerHandler throws on duplicate registration
- getHandler returns null for unregistered type
- listHandlers returns registered types
- dispatchSource calls handler and returns result
- dispatchSource returns error status when no handler registered
- dispatchSource returns timeout error when handler exceeds timeout (use a handler that returns a never-resolving promise, with 100ms timeout for test speed)
- dispatchSource catches thrown exceptions and returns error status
- dispatchAll runs multiple sources in parallel and returns all results
- dispatchAll includes both fulfilled and failed results (one handler succeeds, one times out)
