# v0.12-05-02 SUMMARY — TLA+ Formal Specs

## What Was Built

- Vars: `deliberationRound`, `improvementIteration`, `voteState`

### formal/tla/QGSDPreFilter.tla (GAP-6)
- Vars: `filterRound`, `modelAgreement`, `autoResolved`, `filterPhase`
- Atomic `EvaluateAndResolve(agree)` action prevents transient AutoResolutionSound violation
- Invariant `AutoResolutionSound`: autoResolved <=> modelAgreement
- Invariant `FilterRoundsBounded`: filterRound <= MaxFilterRounds
- Invariant `AutoResolvedPhaseConsistent`: AUTO_RESOLVED phase => autoResolved=TRUE /\ modelAgreement=TRUE
- Liveness `PreFilterTerminates`: <>(filterPhase = "AUTO_RESOLVED" \/ filterPhase = "USER_NEEDED")
- WF_vars for EvaluateAndResolve + EscalateToUser

### formal/tla/MCprefilter.cfg
- MaxFilterRounds=3
- INVARIANT TypeOK + AutoResolutionSound + FilterRoundsBounded + AutoResolvedPhaseConsistent
- PROPERTY PreFilterTerminates
- CHECK_DEADLOCK FALSE

## Decisions

- EvaluateAndResolve is atomic to avoid transient invariant violation (RESEARCH.md open question #2)
- No SYMMETRY in either config — FilterPhases and VoteStates are ordered enums, not symmetric sets
- CHECK_DEADLOCK FALSE in both — terminal states are by design, not deadlocks
- State space for QGSDDeliberation: 11 * 11 * 5 = 605 states max
- State space for QGSDPreFilter: 4 * 2 * 2 * 3 = 48 states max
