# Plan v0.25-04-02: CI Coverage Guard — Summary

## What was built

Created `bin/check-coverage-guard.cjs` — a CI-ready coverage regression guard that compares the current traceability matrix coverage percentage against a saved baseline and warns when the drop exceeds a configurable threshold (default 15 percentage points).

**Features:**
1. **Comparison mode (default):** Reads current matrix and baseline, computes the coverage percentage drop, and exits 1 if it exceeds the threshold.
2. **`--save-baseline` mode:** Saves the current traceability matrix as `formal/traceability-matrix.baseline.json`.
3. **`--threshold N` flag:** Overrides the default 15pp regression threshold.
4. **`--quiet` flag:** Suppresses stdout output (exit code only).
5. **Auto-create baseline:** When no baseline exists, creates one from the current matrix and reports no regression.
6. **Testability:** Env vars `COVERAGE_GUARD_MATRIX_PATH` and `COVERAGE_GUARD_BASELINE_PATH` override hardcoded paths for isolated testing.
