# Plan v0.14-02-01: TLA+ Canonicalization + CI Hardening

## Goal

Resolve the BROKEN-01 critical integration gap (xstate-to-tla.cjs overwrites the hand-authored QGSDQuorum.tla), and harden the CI pipeline by adding missing path triggers and removing continue-on-error masking.

## must_haves

- [ ] `node bin/xstate-to-tla.cjs src/machines/qgsd-workflow.machine.ts --module=QGSDQuorum --dry` does NOT produce output to `QGSDQuorum.tla` — it produces output to `QGSDQuorum_xstate.tla`
- [ ] `formal/tla/QGSDQuorum.tla` is never written by xstate-to-tla.cjs when `--module=QGSDQuorum` is passed
- [ ] `formal-verify.yml` `paths:` block includes all 5 new v0.14-01 scripts in both push and pull_request triggers
- [ ] `continue-on-error: true` is absent from the "Run full FV pipeline via master runner" step in formal-verify.yml
- [ ] `npm test` passes after these changes

<task id="TASK-00-test-output-path">
<title>Add output path assertion to xstate-to-tla.test.cjs</title>
<file>bin/xstate-to-tla.test.cjs</file>
<action>
Add a test that verifies `--dry` output references `QGSDQuorum_xstate.tla` and NOT `QGSDQuorum.tla`:

```js
test('--dry output references QGSDQuorum_xstate.tla, not QGSDQuorum.tla, for --module=QGSDQuorum', () => {
  const result = spawnSync(
    process.execPath,
    [XSTATE_TO_TLA, 'src/machines/qgsd-workflow.machine.ts', '--module=QGSDQuorum', '--dry'],
    { encoding: 'utf8', cwd: path.join(__dirname, '..') }
  );
  // --dry should exit 0 (or non-zero is OK if machine file not found in test context)
  // The important thing: stdout/stderr must NOT mention writing to QGSDQuorum.tla
  const combinedOutput = result.stdout + result.stderr;
  assert.ok(
    !combinedOutput.includes('QGSDQuorum.tla') || combinedOutput.includes('QGSDQuorum_xstate.tla'),
    'Output should reference QGSDQuorum_xstate.tla, not QGSDQuorum.tla: ' + combinedOutput
  );
});
```

Note: If the machine file is not available in the test context (CI without esbuild compilation), the test may need to use `--dry` with a fixture. Check existing test patterns in the file for how to handle this.
</action>
</task>
