# Phase v0.14-02 Plan 01: TLA+ Canonicalization + CI Hardening Summary

**One-liner:** Append `_xstate` suffix to xstate-to-tla.cjs output path and harden CI paths/error masking — resolves BROKEN-01 + MISSING-02.

## What Was Built

### TASK-00-fix-output-path (commit: 45e51a2)

Changed `bin/xstate-to-tla.cjs` to output `<ModuleName>_xstate.tla` instead of `<ModuleName>.tla`, preventing the generated spec from overwriting the hand-authored `formal/tla/QGSDQuorum.tla` canonical spec (BROKEN-01).

Three locations updated in xstate-to-tla.cjs:
- Header comment line 4: `.tla` → `_xstate.tla`
- Generated file header inside the TLA+ output block: `.tla` → `_xstate.tla`
- `tlaOutPath` assignment: `moduleName + '.tla'` → `moduleName + '_xstate.tla'`

`run-formal-verify.cjs` STEPS[0] label updated to mention `QGSDQuorum_xstate.tla`.

### TASK-00-test-output-path (commit: a423e06)

Added regression guard test to `bin/xstate-to-tla.test.cjs`:
- Test: `--dry` with `--module=QGSDQuorum` must reference `QGSDQuorum_xstate.tla`, not bare `QGSDQuorum.tla`
- Uses `spawnSync` with `cwd: path.join(__dirname, '..')` (matches existing test pattern)
- Passes: 3/3 tests after change

### TASK-04-ci-path-triggers + TASK-05-remove-continue-on-error (commit: cc71a2f)

Updated `.github/workflows/formal-verify.yml`:
- Added 5 v0.14-01 scripts to both `push: paths:` and `pull_request: paths:` blocks (MISSING-02):
  - `bin/xstate-to-tla.cjs`
  - `bin/run-formal-verify.cjs`
  - `bin/run-oauth-rotation-prism.cjs`
  - `bin/run-account-pool-alloy.cjs`
  - `bin/run-account-manager-tlc.cjs`
- Removed `continue-on-error: true` from "Run full FV pipeline via master runner" step
- Retained `continue-on-error: true` on quorum health check step (advisory `|| echo` guard — intentional)
