## Decisions Made

- Used `node --check` for syntax smoke test in `run-formal-verify.test.cjs`. The script has a top-level async IIFE that immediately starts executing child processes. Using `require()` in a child process would run the full pipeline. `node --check` validates syntax without execution.
- Integration smoke for `run-formal-verify.cjs` verifies `--only=generate` is accepted (does NOT trigger the "Unknown --only value" guard) without requiring Java. The step itself may fail but the pipeline wiring is verified.
- For `run-account-manager-tlc.cjs`, the `--config=invalid` test fires before the Java check because the VALID_CONFIGS guard is at the top of the script. This makes the test reliable even without Java installed.
