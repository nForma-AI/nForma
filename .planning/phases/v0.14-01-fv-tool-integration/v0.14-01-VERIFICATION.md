### Notes on CI Path Filter

The `formal-verify.yml` path filter does not include `bin/run-formal-verify.cjs` or `bin/xstate-to-tla.cjs`. Pushes touching only those files will not auto-trigger the workflow. The workflow does cover: `formal/tla/**`, `formal/alloy/**`, `formal/prism/**`, `src/machines/qgsd-workflow.machine.ts`, and the original runner scripts. `workflow_dispatch` provides manual triggering. This is an acceptable tradeoff for phase v0.14-01 (the goal is "CI runs the full pipeline on push and PR" for formal spec changes, which it does).
