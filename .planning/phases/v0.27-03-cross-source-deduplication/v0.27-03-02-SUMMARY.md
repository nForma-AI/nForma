# Plan v0.27-03-02 Summary — Dedup Engine & Formal Reference Linker

## What was built

### Task 1: Dedup engine (TDD)
- Created `bin/debt-dedup.cjs` with two-phase dedup strategy:
  - Phase 1: Fingerprint exact-match via hash map (O(n))
  - Phase 2: Levenshtein near-duplicate on remaining unmatched (O(n^2) on smaller set)
- `mergeDebtEntries(a, b)` — merges two entries preserving higher-occurrence as primary, concatenating source_entries, unioning environments, using min/max timestamps, advancing status
- `deduplicateEntries(entries, options)` — configurable threshold (default 0.85), returns `{ entries, mergeCount, mergeLog }`
- 28 unit tests covering merge logic, fingerprint dedup, Levenshtein dedup, mixed scenarios, edge cases

### Task 2: Formal reference auto-linker (TDD)
- Created `bin/formal-ref-linker.cjs` with layered resolution:
  - Layer 1: Manual — existing formal_ref preserved, source marked "manual"
  - Layer 2: Auto-detect — keyword matching against requirements.json
  - Layer 3: Spec-inferred — module name matching against .formal/spec/ directories
  - Layer 4: Null — no match found
- Tokenization strips short words (< 3 chars), scoring counts keyword overlap
- Fail-open on missing requirements or spec directory
- 13 unit tests covering all 4 layers, priority order, multiple matches, fail-open
