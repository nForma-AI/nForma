# Plan v0.27-03-01 Summary — Levenshtein Similarity & Schema Extension

## What was built

### Task 1: Levenshtein distance and similarity (TDD)
- Created `bin/levenshtein.cjs` — Wagner-Fischer algorithm with two-row space optimization
- `levenshteinDistance(a, b)` returns integer edit distance in O(m*n) time, O(min(m,n)) space
- `levenshteinSimilarity(a, b)` returns normalized score 0.0-1.0 via `1 - (distance / max(len_a, len_b))`
- Handles edge cases: empty strings, identical strings, unicode, long strings (256+ chars)
- 19 unit tests covering all edge cases

### Task 2: Debt schema extension with formal_ref_source (TDD)
- Extended `.formal/debt.schema.json` with `formal_ref_source` field: `["string", "null"]` with enum `["manual", "auto-detect", "spec-inferred", null]`
- Updated `bin/validate-debt-entry.cjs` — added formal_ref_source to allowedProps + validation logic
- Backward compatible: entries without formal_ref_source still pass validation
- 8 unit tests covering all valid values, invalid values, and backward compatibility
