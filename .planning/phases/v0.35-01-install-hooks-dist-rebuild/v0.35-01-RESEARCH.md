# Phase v0.35-01: Install hooks/dist rebuild - Research

**Researched:** 2026-03-12
**Domain:** Node.js installer build artifacts and dependency management
**Confidence:** HIGH

## Summary

nForma's installer (`bin/install.js`) expects `hooks/dist/` to contain pre-built hook files. Currently, this directory exists in the published npm package (via `prepublishOnly: npm run build:hooks`), but when developers clone from source, `hooks/dist/` is absent, forcing manual `npm run build:hooks` before `node bin/install.js --claude --global` works.

The phase must enable fresh source checkouts to work seamlessly: either the installer auto-rebuilds `hooks/dist/` on-demand, or it detects the condition and prints a clear, actionable error message with the exact command to run.

The formal installer model (`install-scope.als`) documents three key invariants: `NoConflict` (scopes never collide), `AllEquivalence` (--all flag = explicit flags), and `InstallIdempotent` (re-running install with same args yields identical result). This phase must preserve these invariants — auto-rebuild and warning messages must be idempotent operations.

**Primary recommendation:** Implement two-layer detection: (1) Before copying hooks, check if `hooks/dist/` exists; (2a) if missing, automatically run the same build logic as `npm run build:hooks` inline, (2b) if any errors occur during build, print clear instructions telling the user exactly which command to run. This preserves backward compatibility (published packages have dist/, already built) while fixing fresh clones.

### Recommended Build-on-Missing Pattern

**What:** If `hooks/dist/` does not exist during install, the installer either:
1. Silently runs the same hook-copy logic inline (idempotent, no network/subprocess)
2. Detects build errors and prints actionable guidance

**When to use:** On fresh clones where `npm install` was run but `npm run build:hooks` was skipped. Also handles cases where hooks are modified in hooks/ but dist/ not synced.

**Example pattern (from scripts/build-hooks.js):**
```javascript
// Source: /Users/jonathanborduas/code/QGSD/scripts/build-hooks.js
function buildHooksIfMissing(hooksDir, distDir) {
  if (fs.existsSync(distDir)) {
    // Already built, skip
    return { rebuilt: false, success: true };
  }

  const HOOKS_TO_COPY = [
    'nf-check-update.js',
    'nf-stop.js',
    'nf-prompt.js',
    'nf-circuit-breaker.js',
    'nf-session-start.js',
    'nf-precompact.js',
    'nf-spec-regen.js',
    'nf-token-collector.js',
    'nf-slot-correlator.js',
    'gsd-context-monitor.js',
    'nf-post-edit-format.js',
    'nf-console-guard.js',
    'nf-destructive-git-guard.js',
    'nf-mcp-dispatch-guard.js',
    'nf-node-eval-guard.js',
    'nf-session-end.js',
    'nf-resolve-bin.js',
    'config-loader.js',
    'conformance-schema.cjs',
  ];

  // Create dist directory
  try {
    fs.mkdirSync(distDir, { recursive: true });
  } catch (err) {
    return { rebuilt: false, success: false, error: err.message };
  }

  // Copy each hook
  for (const hook of HOOKS_TO_COPY) {
    const src = path.join(hooksDir, hook);
    const dest = path.join(distDir, hook);

    try {
      if (!fs.existsSync(src)) {
        return { rebuilt: false, success: false, error: `Missing source: ${src}` };
      }
      fs.copyFileSync(src, dest);
    } catch (err) {
      return { rebuilt: false, success: false, error: `Failed to copy ${hook}: ${err.message}` };
    }
  }

  return { rebuilt: true, success: true };
}
```

### Error Handling: Actionable Messages

**Scenario 1: Auto-rebuild succeeds** → Silent operation, no user message (fail-open pattern)

**Scenario 2: Auto-rebuild fails** → Print clear message:
```
Error: hooks/dist/ missing and rebuild failed:
  {{ERROR_DETAILS}}

To fix, run from the nForma source directory:
  npm run build:hooks

Then try install again:
  node bin/install.js --claude --global
```

This message provides:
- **What went wrong:** hooks/dist missing
- **Why:** Specific error (e.g., "file not found at hooks/nf-stop.js")
- **How to fix:** Exact command to run
- **Next step:** Exact re-run command

### Pattern: Idempotency via Existence Check

The formal spec `InstallIdempotent` assertion requires that `InstallOp[pre, mid, targets, sc] and InstallOp[mid, post, targets, sc] => SameState[mid, post]`.

For the rebuild layer:
- First run: `hooks/dist/` missing → rebuild, then copy
- Second run: `hooks/dist/` exists → skip rebuild, copy (same result)

Both paths produce identical final state in `~/.claude/hooks/`.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Detecting missing files in a complex directory | Custom stat loops | fs.existsSync() + path.join() | Standard library, tested across platforms |
| Running multiple file copies with error recovery | Spawn subprocess to bash/shell | Inline fs.copyFileSync() calls | Single process, no shell escaping issues, deterministic ordering |
| Documenting required hook list | Comments in multiple places | HOOKS_TO_COPY array in build-hooks.js + verify-hooks-sync.cjs | Single source of truth, CI guard ensures sync |
| Reproducing build-hooks.js logic | Custom code in installer | Factor the logic into shared function or inline copy | Avoid duplicate code paths that diverge over time |

**Key insight:** The hooks-sync CI guard (`scripts/verify-hooks-sync.cjs`) already detects drift between `HOOKS_TO_COPY` and what the installer tries to use. Leveraging the same manifest list for auto-rebuild prevents new divergence channels.

## Common Pitfalls

### Pitfall 1: Silent Failure on Missing hooks/dist

**What goes wrong:** Installer checks `fs.existsSync(hooksSrc)` (line 2007 in install.js) but if false, silently skips hooks. User gets incomplete installation with no error, hooks not functional, but no indication why.

**Why it happens:** Current code treats missing hooks as "optional" rather than "fatal but recoverable."

**How to avoid:** Make hooks mandatory: either auto-build or fail with actionable message. The test suite (`test/install-virgin.test.cjs` lines 119-127) expects `hooks/` directory to exist post-install.

**Warning signs:** `~/.claude/hooks/` empty after install; user reports `nf-stop.js: command not found` in hooks; hooks config in settings.json present but files absent.

### Pitfall 2: Divergence Between build-hooks.js and Installer Logic

**What goes wrong:** If the auto-rebuild logic in the installer is copy-pasted from `scripts/build-hooks.js`, future changes to the hook list break one path but not the other.

**Why it happens:** Two sources of truth for the same operation.

**How to avoid:** Extract the HOOKS_TO_COPY list to a shared location (or read it from build-hooks.js at runtime), or ensure the verify-hooks-sync.cjs guard runs on both paths.

**Warning signs:** CI passes but fresh clone fails; test suite tests published package (which has hooks/dist/) but not source clone.

### Pitfall 3: Recursive Sync on Subtree with node_modules

**What goes wrong:** If trying to copy all of `hooks/` instead of just listed files, might accidentally copy test files or node_modules symlinks, causing size bloat or path issues.

**Why it happens:** Overly broad directory copy instead of explicit file list.

**How to avoid:** Always iterate HOOKS_TO_COPY manifest, never glob or recursively copy `hooks/dist/`. Verify by checking file count: should be exactly N files (currently 19) per manifest.

**Warning signs:** hooks/dist/ size suddenly much larger; `.test.js` files in dist/; node_modules symlinks fail on different machines.

### Pitfall 4: Idempotency Broken by Conditional Messages

**What goes wrong:** First run prints "rebuilt hooks/dist", second run prints nothing. User script checks stdout to determine if rebuild happened — logic breaks on re-run.

**Why it happens:** Mixing silent rebuild with verbose logging without accounting for idempotency.

**How to avoid:** Rebuild silently (no stdout), or use stderr only for errors. If verbose logging needed, log to debug/trace level only (controlled by env var like `DEBUG=nf:*`).

**Warning signs:** Automation scripts check install output; CI logs show different message on retry; test assertions fail on re-run.

  'nf-session-end.js',
  'nf-resolve-bin.js',
  'config-loader.js',
  'conformance-schema.cjs',
];

function build() {
  const DIST_DIR = path.join(__dirname, '..', 'hooks', 'dist');
  if (!fs.existsSync(DIST_DIR)) {
    fs.mkdirSync(DIST_DIR, { recursive: true });
  }
  for (const hook of HOOKS_TO_COPY) {
    const src = path.join(__dirname, '..', 'hooks', hook);
    const dest = path.join(DIST_DIR, hook);
    if (!fs.existsSync(src)) {
      console.warn(`Warning: ${hook} not found, skipping`);
      continue;
    }
    fs.copyFileSync(src, dest);
  }
}

## Open Questions

1. **Should we rebuild silently or with debug output?**
   - Current thinking: Silent rebuild (fail-open pattern), errors print to stderr
   - Alternative: Add `--verbose` flag or `DEBUG=nf:install` env var support
   - Recommendation: Start silent, can add verbose flag in future phase if users request it

2. **Should stale dist/ be rebuilt or just used as-is?**
   - Current thinking: Only rebuild if dist/ completely missing
   - Alternative: Always rebuild if any hook in hooks/ is newer than dist/
   - Recommendation: Phase 1 (this phase) → missing only; Phase 2 (future) → staleness detection

3. **Should the error message suggest npm run build:hooks or node scripts/build-hooks.js?**
   - Current thinking: `npm run build:hooks` (user-friendly, hides script path)
   - Alternative: Direct path (more explicit)
   - Recommendation: Use npm script (less brittle, documented in package.json)
