# Plan v0.19-07-01: Wire detectLivenessProperties to All Four TLC Runners

## Objective

Add `detectLivenessProperties()` calls to 4 TLC runners so that all 6 liveness properties emit `result=inconclusive` when their companion `invariants.md` has no fairness declaration. Closes LIVE-02.

The function `detectLivenessProperties` is already implemented and exported from `bin/run-tlc.cjs`. The 4 runners need to import it and call it after a successful TLC run (exit status 0), before writing `result=pass`.

## Context

Reference implementation is in `bin/run-tlc.cjs` (after TLC exits 0, around line 170–196). The SURFACE_MAP in `run-tlc.cjs` already covers all 6 surfaces: oscillation, convergence, breaker, deliberation, prefilter, account-manager.

All companion `invariants.md` files already exist with the correct `## PropertyName` headers — so the current behavior is: TLC passes, runner writes `result=pass`, but it should write `result=inconclusive` only if the invariants.md header were absent. Since the headers ARE present, the call will return `[]` and the runners will continue to emit `result=pass` in normal operation. The wiring is what's required for LIVE-02, not a behavior change under normal conditions.
