# Phase v0.29-06 Plan 01: Fix nf-solve Data Contract and Exit-Code Bugs

## Objective
Fix two integration bugs in `bin/nf-solve.cjs` and verify bookkeeping sync is complete.

## Bug 1: Data Contract Mismatch in sweepL1toL2

**Root cause**: `sweepL1toL2()` (lines 1967-1970) reads `data.instrumentation_bug`, `data.model_gap`, `data.genuine_violation` — flat paths that do not exist in gate-a's JSON output. Gate-a nests these under `data.unexplained_counts.instrumentation_bug`, etc.

**Fix**: Change lines 1968-1970 to read from `data.unexplained_counts.*` instead of `data.*`.

**Before**:
```js
instrumentation_bug: data.instrumentation_bug || 0,
model_gap: data.model_gap || 0,
genuine_violation: data.genuine_violation || 0,
```

**After**:
```js
instrumentation_bug: (data.unexplained_counts && data.unexplained_counts.instrumentation_bug) || 0,
model_gap: (data.unexplained_counts && data.unexplained_counts.model_gap) || 0,
genuine_violation: (data.unexplained_counts && data.unexplained_counts.genuine_violation) || 0,
```

## Bug 2: Exit-Code Handling for Gate Scripts

**Root cause**: `spawnTool()` sets `ok: result.status === 0`. Gate-b and gate-c exit with code 1 when `target_met` is false, but still produce valid JSON on stdout. The sweep functions (`sweepL1toL2`, `sweepL2toL3`, `sweepL3toTC`) check `if (!result.ok)` and return an error response, discarding the valid JSON output.

**Fix**: Change the guard in all three gate sweep functions from `if (!result.ok)` to `if (!result.ok && !result.stdout)`. This mirrors the existing pattern used by `run-formal-verify` (line 974) which also handles non-zero exit codes gracefully.

**Affected functions**:
- `sweepL1toL2()` — gate-a (currently always exits 0, but fix is defensive)
- `sweepL2toL3()` — gate-b (exits 1 when target not met)
- `sweepL3toTC()` — gate-c (exits 1 when target not met)
