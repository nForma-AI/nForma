  <action>
Create `bin/validate-requirements-haiku.cjs` (CommonJS, 'use strict'). Follow `bin/update-scoreboard.cjs` Haiku API pattern.

1. **`buildValidationPrompt(requirements)`** -- Construct the Haiku prompt:
   - Accept array of requirement objects `[{ id, text, category, phase }]`
   - Build prompt with explicit rubrics:
     ```
     You are reviewing a requirements envelope for quality issues.

     Requirements:
     - ENV-01: <text>
     - ENV-02: <text>
     ...

     Analyze for exactly these three categories:

     1. DUPLICATES: Different IDs where the INTENT is the same (not just similar wording).
        Example: "Users can log in" and "Authentication allows user access" are duplicates.
        NOT duplicates: "Users can log in" and "Admins can log in" (different scope).

     2. CONTRADICTIONS: Two requirements that CANNOT BOTH be satisfied.
        Example: "Data is immutable" and "Data auto-updates daily" are contradictory.
        NOT contradictions: "Data is immutable" and "Amendments require approval" (compatible).

     3. AMBIGUITY: A single requirement that admits two or more INCOMPATIBLE interpretations.
        Example: "The system should be fast" (fast for whom? what metric?).
        NOT ambiguous: "Response time under 200ms for 95th percentile" (specific).

     Return ONLY valid JSON (no markdown, no explanation):
     {
       "findings": [
         {
           "type": "duplicate|contradiction|ambiguity",
           "requirement_ids": ["ENV-XX", "ENV-YY"],
           "description": "...",
           "severity": "high|medium|low",
           "suggested_resolution": "..."
         }
       ],
       "summary": "N duplicates, N contradictions, N ambiguities found"
     }

     If no issues found, return: {"findings": [], "summary": "No issues found"}
     ```
   - Return the prompt string

2. **`parseHaikuResponse(responseText)`** -- Parse Haiku JSON response:
   - Extract JSON from response text (handle possible markdown wrapping)
   - Validate structure: findings array with type/requirement_ids/description fields
   - Return parsed object or `{ findings: [], summary: "Parse error", error: true }` on failure

3. **`aggregateFindings(passResults)`** -- Aggregate multi-pass results:
   - Accept array of parsed responses from multiple Haiku passes
   - A finding is "confirmed" if it appears in 2+ of N passes
   - Match findings by: same `type` + overlapping `requirement_ids`
   - Return `{ confirmed: [...], total_passes, agreement_threshold }`

4. **`validateRequirements(options)`** -- Main pipeline:
   - Accept `{ envelopePath, passes, apiKey }` with defaults:
     - `envelopePath`: `formal/requirements.json`
     - `passes`: 3
     - `apiKey`: `process.env.ANTHROPIC_API_KEY`
   - Read envelope, extract requirements array
   - If envelope has `frozen_at` set: return `{ status: 'already-frozen', frozen_at }`
   - SDK availability guard (require.resolve pattern from update-scoreboard.cjs)
   - If SDK unavailable or no API key: return `{ status: 'skipped', reason: 'SDK or API key unavailable' }`
   - Run N Haiku passes: `client.messages.create({ model: 'claude-haiku-4-5-20251001', max_tokens: 4096, ... })`
   - Aggregate findings
   - Return `{ status: 'validated', confirmed, total_passes, raw_passes }`

5. **`freezeEnvelope(envelopePath)`** -- Freeze the envelope:
   - Read envelope JSON
   - Set `frozen_at` to current ISO timestamp
   - Write back atomically (temp + rename)
   - Return `{ frozen: true, frozen_at }`

6. **CLI entrypoint** (guarded by `require.main === module`):
   - Usage: `node bin/validate-requirements-haiku.cjs [--envelope=path] [--passes=3] [--freeze]`
   - Without `--freeze`: run validation only, print findings
   - With `--freeze`: run validation, then freeze if no high-severity findings
   - Print structured output:
     ```
     Validation Results (3 passes, 2+ agreement):
     
     CONFIRMED FINDINGS:
       [HIGH] CONTRADICTION: ENV-01 vs ENV-04 -- ...
       [LOW]  AMBIGUITY: ENV-03 -- ...
     
     Summary: 1 contradiction, 1 ambiguity (0 duplicates)
     
     To freeze: node bin/validate-requirements-haiku.cjs --freeze
     ```

7. **Exports:** `module.exports = { buildValidationPrompt, parseHaikuResponse, validateRequirements, aggregateFindings, freezeEnvelope }`
