1. **Create `formal/requirements.schema.json`** (JSON Schema draft-07):
   ```json
   {
     "$schema": "http://json-schema.org/draft-07/schema#",
     "type": "object",
     "required": ["schema_version", "source", "aggregated_at", "content_hash", "requirements"],
     "additionalProperties": false,
     "properties": {
       "schema_version": { "type": "string", "const": "1" },
       "source": { "type": "string" },
       "aggregated_at": { "type": "string", "format": "date-time" },
       "frozen_at": { "type": ["string", "null"], "format": "date-time" },
       "content_hash": { "type": "string", "pattern": "^sha256:[a-f0-9]{64}$" },
       "requirements": {
         "type": "array",
         "items": {
           "type": "object",
           "required": ["id", "text", "category", "phase", "status", "provenance"],
           "additionalProperties": false,
           "properties": {
             "id": { "type": "string", "pattern": "^[A-Z]+-\\d+$" },
             "text": { "type": "string", "minLength": 1 },
             "category": { "type": "string", "minLength": 1 },
             "phase": { "type": "string", "pattern": "^v[\\d.]+-\\d+$" },
             "status": { "type": "string", "enum": ["Pending", "Complete"] },
             "provenance": {
               "type": "object",
               "required": ["source_file", "milestone"],
               "properties": {
                 "source_file": { "type": "string" },
                 "milestone": { "type": "string" }
               }
             }
           }
         }
       }
     }
   }
   ```

2. **Create `bin/aggregate-requirements.cjs`** (CommonJS, 'use strict'):

   Follow `bin/task-envelope.cjs` patterns exactly. Structure:

   a. **`parseRequirements(content)`** -- Parse requirement bullets from REQUIREMENTS.md:
      - Track current category from `### Category Name` headers (the actual headers use em-dash, e.g., `### Requirements Envelope — ENV`; match both em-dash and double-dash variants)
      - Match bullets: `/^- \[[ x]\] \*\*([A-Z]+-\d+)\*\*:\s*(.+)$/`
      - Return array of `{ id, text, category, completed }` sorted by `id`

   b. **`parseTraceability(content)`** -- Parse the Traceability table:
      - Match table rows: `/^\|\s*([A-Z]+-\d+)\s*\|\s*(v[\d.]+-\d+)\s*\|\s*(\w+)\s*\|/gm`
      - Return map: `{ [reqId]: { phase, status } }`

   c. **`extractMilestone(content)`** -- Extract milestone from the title or first heading:
      - Match `# Requirements: QGSD v0.XX` or similar
      - Return milestone string (e.g., "v0.22")

   d. **`validateEnvelope(obj)`** -- Inline schema validation:
      - Check all required fields exist with correct types
      - Validate `schema_version` is `"1"`
      - Validate each requirement has required fields
      - Validate `id` format: `/^[A-Z]+-\d+$/`
      - Return `{ valid, errors }`

   e. **`aggregateRequirements(options)`** -- Main pipeline:
      - Accept `{ requirementsPath, outputPath }` with defaults:
        - `requirementsPath`: `.planning/REQUIREMENTS.md`
        - `outputPath`: `formal/requirements.json`
      - If `outputPath` exists and contains `frozen_at` (not null), exit with error: "Envelope is frozen -- use amendment workflow (ENV-04)"
      - Read REQUIREMENTS.md
      - Parse requirements and traceability
      - Merge: for each requirement, add `phase` and `status` from traceability table
      - Sort requirements by `id` (lexicographic) for determinism
      - Compute `content_hash`: SHA-256 of the sorted requirements JSON (before wrapping in envelope)
      - Build envelope: `{ schema_version: "1", source, aggregated_at, frozen_at: null, content_hash, requirements }`
      - Validate envelope
      - Write atomically (temp file + rename, following task-envelope.cjs pattern)
      - Return `{ valid, requirementCount, outputPath }`

   f. **CLI entrypoint** (guarded by `require.main === module`):
      - Usage: `node bin/aggregate-requirements.cjs [--requirements=path] [--output=path] [--dry-run]`
      - `--dry-run`: validate and print to stdout, don't write file
      - Print summary: "Aggregated N requirements to formal/requirements.json"

   g. **Exports:** `module.exports = { parseRequirements, parseTraceability, validateEnvelope, aggregateRequirements }`

   **CRITICAL for determinism (Success Criterion 4):**
   - Use a fixed `aggregated_at` timestamp when in `--deterministic` mode (for testing), or always regenerate in normal mode
   - Sort requirements by `id`
   - Use `JSON.stringify(envelope, null, 2)` with keys in insertion order (define object literal with keys in alphabetical order)
   - Content hash is computed from the sorted requirements array only (not the timestamp), so re-runs produce the same hash even with different timestamps
