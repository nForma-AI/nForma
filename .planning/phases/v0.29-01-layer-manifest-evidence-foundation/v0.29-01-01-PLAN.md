2. Auto-classify each model into L1/L2/L3 using deterministic rules from research:
   - L1 (Evidence): formalism "trace" or "redaction"; conformance-events.jsonl; debt.json; observe-handler outputs
   - L2 (Semantics): spec/*/invariants.md files; XState machine definition; assumption-gaps.md
   - L3 (Reasoning): formalism "tla", "alloy", "prism", "uppaal"; files in tla/, alloy/, prism/ directories
3. Assign gate_maturity "ADVISORY" and layer_maturity 0 to all models (Level 0: ungrounded)
4. For models with spec/*/invariants.md, assign layer_maturity 1 (has L2 semantic declarations)
5. Write updated model-registry.json with source_layer, gate_maturity, layer_maturity fields per model (INTG-05)
6. Generate layer-manifest.json containing:
   - schema_version "1"
   - generated timestamp
   - layers object with L1/L2/L3 arrays of artifact references (model path + description + grounding_status)
   - gate_relationships: which gates connect which layers (A: L1->L2, B: L2->L3, C: L3->TC)
   - summary with counts per layer and overall maturity distribution
7. Support --json for machine-readable output, default human-readable summary

The script must tolerate missing new fields gracefully (models without source_layer default to "unclassified"). Use path-based classification first, then check-result formalism as tiebreaker.
