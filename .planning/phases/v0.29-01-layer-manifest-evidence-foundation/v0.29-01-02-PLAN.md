Create bin/instrumentation-map.cjs following existing bin/ CJS pattern (#!/usr/bin/env node, 'use strict', PROJECT_ROOT, --json flag).

The script must:
1. Scan the three hooks that emit conformance events: hooks/nf-prompt.js, hooks/nf-stop.js, hooks/nf-circuit-breaker.js
2. For each hook file, use regex/AST-lite parsing to find:
   - All `appendFileSync` or `writeFileSync` calls that write to conformance-events (these are emission points)
   - The action type being emitted at each point (extract from the JSON object being written)
   - The XState state variable context (what state the hook is tracking)
3. Also scan the 6 observe handlers in bin/observe-handler-*.cjs for emission points
4. Load event-vocabulary.json and validate all discovered emission actions against it
5. When validating vocabulary entries, handle null xstate_event gracefully: entries like mcp_call have xstate_event: null intentionally. The validation logic must NOT treat null xstate_event as an error or missing mapping -- instead flag it as "no_xstate_mapping" in the emission point record. This prevents downstream NPE when consumers iterate emission points expecting xstate_event to be a string.
6. Calculate coverage: mapped_events / total_vocabulary_entries (excluding "undefined")
7. Produce instrumentation-map.json with:
   - schema_version "1", generated timestamp
   - emission_points: array of { file, line_number (approximate), action, state_variables, vocabulary_match, xstate_event (string|null) }
   - coverage: { total_vocabulary_actions, mapped_actions, coverage_pct }
   - unmapped_actions: actions in vocabulary not found in any hook
   - summary: human-readable coverage text

Create bin/instrumentation-map.test.cjs using node:test:
- Test hook file parsing with inline fixture strings (small mock hook code)
- Test vocabulary validation logic
- Test null xstate_event handling: verify that a vocabulary entry with xstate_event: null does not cause errors and is correctly represented in emission_points
- Test coverage calculation
- Integration test: run against real hooks directory, verify emission points found >= 3

Run the script to generate initial instrumentation-map.json.

Create bin/trace-corpus-stats.cjs following existing bin/ CJS pattern.

The script must:
1. Read conformance-events.jsonl using planning-paths.cjs resolve(cwd, 'conformance-events') -- NEVER hardcode the path (per research Pitfall 3)
2. Parse all JSONL lines (expect 35K+ events). Use the simple split/parse pattern from validate-traces.cjs
3. Infer sessions using timestamp gaps: a gap of >SESSION_GAP_MS (default 300000ms / 5 minutes) between consecutive events indicates a new session. Sort events by timestamp first. Accept a --session-gap-ms CLI flag to make this threshold configurable (e.g., `node bin/trace-corpus-stats.cjs --session-gap-ms 600000` for 10-minute gap). Parse from process.argv; if not provided, default to 300000.
4. For each session, compute: id, start_ts, end_ts, event_count, action_breakdown (count per action type)
5. Build action type index: for each action type, count total occurrences, list session IDs where it appears
6. Build state transition index: extract from/to state pairs where available in event data, count occurrences
7. Load event-vocabulary.json and validate all event actions against it. Count unknown actions
8. Produce trace-corpus-stats.json with:
   - schema_version "1", generated timestamp
   - session_gap_ms: the threshold used for session inference
   - total_events count
   - date_range: { earliest, latest }
   - sessions: array of { id, start, end, event_count, actions: { action_type: count } }
   - action_index: { action_type: { total, sessions: [ids] } }
   - transition_index: array of { from, to, count } (if transition data available)
   - vocabulary_validation: { known, unknown, unknown_actions: [] }
   - summary: human-readable text

Create bin/trace-corpus-stats.test.cjs using node:test:
- Test session inference algorithm with inline fixtures (events with varying timestamps)
- Test single-event session edge case: a session with exactly 1 event must still appear in output with start === end and event_count === 1
- Test identical timestamps edge case: multiple events with the exact same timestamp must be grouped into the same session (gap is 0, which is < threshold)
- Test --session-gap-ms flag parsing: verify that a custom threshold changes session boundaries
- Test action indexing with small event arrays
- Test vocabulary validation
- Integration test: run against real conformance-events.jsonl, verify total_events >= 35000

Run the script to generate initial trace-corpus-stats.json.
