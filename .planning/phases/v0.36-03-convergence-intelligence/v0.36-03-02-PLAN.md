<objective>
Add gate cap visibility to the solve remediation output (CONV-03) and implement baseline drift detection at report time (CONV-04). These are reporting-layer features that surface convergence pathologies to the user.
</objective>

<must_haves>
1. When any gate dispatch hits the max-3-per-cycle cap, the output JSON includes a `capped_layers` array — users can distinguish "residual remains because capped" from "residual remains because stuck"
2. At report time, the baseline is re-snapshotted and compared to session-start baseline; drift exceeding 10% is flagged to the user with an explicit warning — preventing misleading before/after deltas from mid-session external edits
</must_haves>

<task id="02-02" type="auto">
  <name>Create baseline-drift.cjs detection module</name>
  <files>bin/baseline-drift.cjs, test/baseline-drift.test.cjs</files>
  <action>
Create `bin/baseline-drift.cjs` with these exports:

1. `function detectBaselineDrift(sessionStartBaseline, currentSnapshot, options)`:
   - `sessionStartBaseline`: object with per-layer residual values from session start (e.g., `{ r_to_f: { residual: 5 }, f_to_t: { residual: 0 }, ... }`)
   - `currentSnapshot`: same structure, taken at report time
   - `options.threshold`: drift threshold as fraction (default 0.10 = 10%)
   - `options.requirementsPath`: path to requirements.json (optional, for req count comparison)

   Algorithm:
   - Import LAYER_KEYS from `./layer-constants.cjs`
   - For each layer in LAYER_KEYS:
     - Get baseline residual: `sessionStartBaseline[layer]?.residual ?? -1`
     - Get current residual: `currentSnapshot[layer]?.residual ?? -1`
     - Skip if either is -1 (missing/skipped)
     - Skip if baseline is 0 (can't compute % change from 0 — use absolute change > 2 instead)
     - Compute `pctChange = Math.abs(current - baseline) / baseline`
     - If baseline is 0 and `Math.abs(current - baseline) > 2`: flag as drift
     - If baseline > 0 and `pctChange > threshold`: flag as drift
   - If `options.requirementsPath` is provided, also compare requirement counts:
     - Read current requirements.json count
     - Compare with `sessionStartBaseline.requirement_count` if present
     - Flag if count changed (this indicates scope change during session)

   Returns:
   ```javascript
   {
     detected: boolean,
     layers: [
       { layer: string, baseline: number, current: number, pct_change: number }
     ],
     requirement_count_changed: boolean,
     warning: string | null  // human-readable warning if detected
   }
   ```

   Warning format: `"Baseline drift detected in {N} layer(s): {layer1} ({pct}%), {layer2} ({pct}%). Mid-session external edits may have affected the before/after comparison."`

2. Export: `{ detectBaselineDrift }`

Create `test/baseline-drift.test.cjs` using `node:test` and `node:assert/strict`:
- Test: no drift when baseline and current are identical
- Test: drift detected when a layer changes >10% (e.g., baseline=10, current=12)
- Test: no drift when change is exactly 10% (boundary)
- Test: drift detected when baseline is 0 and current > 2
- Test: missing layers (residual=-1) are skipped, not flagged
- Test: warning string includes layer names and percentages
- Test: multiple layers drifting produces correct count
- Test: requirement count change detection
  </action>
  <verify>node --test test/baseline-drift.test.cjs</verify>
  <done>bin/baseline-drift.cjs exports detectBaselineDrift. All tests pass covering threshold logic, edge cases, and warning formatting.</done>
</task>

2. Step 6.2 instructions:
   ```
   ## Step 6.2: Baseline Drift Check (CONV-04)

   Re-run a lightweight diagnostic to get a fresh residual snapshot:
   ```bash
   DRIFT_SNAPSHOT=$(node ~/.claude/nf-bin/nf-solve.cjs --json --report-only --fast --project-root=$(pwd) 2>/dev/null)
   ```
   If nf-bin path doesn't exist, fall back to bin/nf-solve.cjs.

   Then run baseline drift detection:
   ```bash
   node -e "
     const { detectBaselineDrift } = require('./bin/baseline-drift.cjs');
     const baseline = JSON.parse(process.env.BASELINE_JSON);
     const snapshot = JSON.parse(process.env.SNAPSHOT_JSON);
     const result = detectBaselineDrift(
       baseline.residual_vector || baseline,
       snapshot.residual_vector || snapshot,
       { threshold: 0.10, requirementsPath: '.planning/formal/requirements.json' }
     );
     console.log(JSON.stringify(result));
   "
   ```

   If `result.detected` is true:
   - Display warning banner:
     ```
     ─ Baseline Drift Warning ───────────────────────────
     {result.warning}

     Affected layers:
     {for each layer in result.layers:}
       - {layer.layer}: {layer.baseline} -> {layer.current} ({layer.pct_change}% change)
     ─────────────────────────────────────────────────────
     ```
   - Include `baseline_drift` in the report output

   If `result.detected` is false:
   - Log: `"Baseline drift check: clean — no external changes detected"`

   Fail-open: if the drift check script errors, log warning and continue.
   ```
