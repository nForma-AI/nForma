<objective>
Implement internal cycle detection for the solve convergence loop (CONV-01) and split the single residual total into three distinct buckets (CONV-02). These are foundational data model changes that downstream reporting depends on.
</objective>

<must_haves>
1. Per-layer residual history is tracked across iterations; when an A-B-A-B oscillation pattern spans 4+ iterations on any layer, that layer is reported as oscillating and excluded from further dispatch — consistent with the formal ConvergenceEventuallyResolves invariant that resolution must terminate rather than loop indefinitely
2. Solve output reports residual in three distinct buckets (`automatable`, `manual`, `informational`) — the user sees each bucket separately and never a misleading combined total
</must_haves>

<task id="01-01" type="auto">
  <name>Create solve-cycle-detector.cjs module</name>
  <files>bin/solve-cycle-detector.cjs, test/solve-cycle-detector.test.cjs</files>
  <action>
Create `bin/solve-cycle-detector.cjs` with these exports:

1. `class CycleDetector`:
   - Constructor: initializes `this.history = {}` (keyed by layer name, values are arrays of residual numbers per iteration)
   - `record(iteration, perLayerResiduals)`: for each layer key in the passed object, push the residual value to `this.history[layer]`. `perLayerResiduals` is an object like `{ r_to_f: 3, f_to_t: 0, ... }`
   - `detectOscillating()`: returns an array of layer keys that exhibit A-B-A-B patterns. Algorithm: for each layer with 4+ recorded values, check if `values[i] === values[i-2]` for at least 2 consecutive pairs starting from index 2 (meaning values[2]===values[0] AND values[3]===values[1], spanning 4 points). Return all such layer keys.
   - `getHistory()`: returns the full history object for debugging/reporting

2. `function detectCycles(history)`: standalone function taking `{ layer: number[] }` and returning `string[]` of oscillating layer keys (same algorithm as class method, for use without class)

Import LAYER_KEYS from `./layer-constants.cjs` for validation.

Create `test/solve-cycle-detector.test.cjs` using `node:test` and `node:assert/strict`:
- Test: empty history returns no oscillating layers
- Test: 3 iterations (below threshold) returns no oscillating layers
- Test: A-B-A-B pattern over 4 points detected (e.g., [5, 3, 5, 3])
- Test: A-B-A-B pattern over 6 points detected (e.g., [5, 3, 5, 3, 5, 3])
- Test: monotonically decreasing values NOT detected as oscillating
- Test: single layer oscillating among many stable layers
- Test: multiple layers oscillating simultaneously
- Test: values with A-B-A pattern (3 points) NOT detected (need 4+)
  </action>
