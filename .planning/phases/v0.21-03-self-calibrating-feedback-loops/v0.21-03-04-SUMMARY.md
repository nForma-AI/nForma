# Summary: v0.21-03-04 — LOOP-03: sensitivity-sweep-feedback.cjs

## What Was Built

Feedback wrapper that reads the sensitivity sweep report and detects new threshold violations based on the empirical TP rate from the scoreboard.

## Key Files

### Created
- `bin/sensitivity-sweep-feedback.cjs` (194 lines) — LOOP-03 feedback script with `DEVIATION_THRESHOLD = 0.05` constant. Algorithm:
  1. Read `formal/sensitivity-report.ndjson` (exit 0 if absent)
  2. Filter to PRISM tp_rate records; exit 0 if all inconclusive
  3. Read empirical rate from `quorum-scoreboard.md` (wins/losses table)
  4. Find nearest tested value from actionable records
  5. If deviation <= threshold: exit 0 (no action)
  6. If deviation > threshold AND flip-to-fail record within 2x threshold: exit 1 (new violation)
  7. Otherwise: re-run PRISM with updated rates and exit based on PRISM result

## Notable Deviations from Plan

The plan described checking PRISM exit code to determine violation. This was changed to first check for flip-to-fail records near the empirical rate (without requiring PRISM to fail), because PRISM is installed on this machine and exits 0 even in cold-start mode (failure suppressed). The conservative approach exits 1 when flip-to-fail boundary is near, regardless of PRISM result.
