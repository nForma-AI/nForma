Create `bin/propose-debug-invariants.cjs` with the following behavior:

**Debug artifact location:** `.planning/quick/quorum-debug-latest.md` relative to `process.cwd()`.

**--non-interactive flag:** When present, print all proposals to stdout and exit 0 without prompting. This is the CI mode.

**Mining heuristics** (extract candidates from the debug artifact text):

1. **State transition heuristic:** Find lines matching `<StateA> -> <StateB>` or `<StateA> → <StateB>`. For each unique `(A, B)` pair, generate:
   ```
   PROPERTY TransitionHint_<A>_to_<B> == [][phase = "<A>" => phase' \in {"<A>", "<B>"}]_vars
   ```

2. **Root cause heuristic:** Find lines containing `root_cause:` (from the `## worker responses` section). Extract the text after `root_cause:`. If it mentions a state name (IDLE, DECIDING, COMPLETE, BLOCKED), generate:
   ```
   INVARIANT RootCauseHint_<sanitized-text> == TRUE  \* Review: <original root_cause text>
   ```
   Sanitize for TLA+ name: replace non-alphanumeric with `_`, limit to 40 chars.

3. **Invariant candidate heuristic:** Find lines containing `Invariant candidate:` or `invariant:` (case-insensitive). Extract the text after the colon and present it as a comment with a TODO:
   ```
   PROPERTY InvariantCandidate_<N> == TRUE  \* TODO: Formalize: <extracted text>
   ```

**Non-interactive mode output format:**
```
=== propose-debug-invariants: N candidate(s) from quorum-debug-latest.md ===

[1] PROPERTY TransitionHint_IDLE_to_DECIDING == ...
    Source: line 12 "IDLE -> DECIDING"

[2] INVARIANT RootCauseHint_quorum_block_DECIDING == TRUE  \* Review: ...
    Source: line 18 "root_cause: quorum_block fired while phase=DECIDING"

Run without --non-interactive to accept/reject each individually.
```

**Interactive mode** (no --non-interactive flag): Present each candidate one at a time using `readline` from Node.js built-ins. For each:
- Print the candidate with its source
- Prompt: `Accept [a], Reject [r], or Skip [s]? `
- On 'a': generate a session ID using `` `debug-sess-${Math.floor(Date.now()/1000)}-${Math.random().toString(16).slice(2,10)}` `` and call `accept-debug-invariant.cjs` via spawnSync with args `[specPath, propertyName, propertyBody, sessionId]`.
  - `specPath` defaults to `formal/tla/QGSDQuorum.tla` (the canonical spec for quorum behavior).
  - Print result (accepted/rejected).
- On 'r' or 's': skip, print "Rejected/Skipped."
- After all candidates: print summary "X accepted, Y rejected, Z skipped."

**Fail-open cases:**
- Debug artifact absent: print "No debug artifact found at .planning/quick/quorum-debug-latest.md — run a debug session first." Exit 0.
- No candidates extracted: print "No invariant candidates detected in the debug artifact." Exit 0.
- accept-debug-invariant.cjs fails: print warning, continue to next candidate.

**File header:**
```javascript
#!/usr/bin/env node
'use strict';
// bin/propose-debug-invariants.cjs

//   node bin/propose-debug-invariants.cjs                  # Interactive: accept/reject each
//   node bin/propose-debug-invariants.cjs --non-interactive # CI: print all, exit 0
```

**Key constraints from research:**
- Session ID format MUST be: `` `debug-sess-${Math.floor(Date.now()/1000)}-${Math.random().toString(16).slice(2,10)}` `` — validated by accept-debug-invariant.cjs regex. Non-conforming IDs cause exit 1 from accept-debug-invariant.cjs.
- Call accept-debug-invariant.cjs as subprocess via `spawnSync(process.execPath, [scriptPath, specPath, name, body, sessionId])` — NOT via require() (the script uses `require.main === module` guard).
- TLA+ property body must use `==` not `=` for definition. The `PROPERTY` keyword (not `Property`) is correct TLA+ syntax.
- `--non-interactive` mode must NEVER call accept-debug-invariant.cjs — only print proposals.
