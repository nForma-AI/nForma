Read `bin/run-sensitivity-sweep.cjs` to understand the NDJSON record shape. Read `bin/export-prism-constants.cjs` to understand how it reads the scoreboard and computes TP rate.

Create `bin/sensitivity-sweep-feedback.cjs` using `spawnSync` from `child_process` (NOT exec — use the array-of-args form which is injection-safe). Use Node.js built-ins only: `fs`, `path`, `child_process.spawnSync`.

**Algorithm:**

1. Resolve `ROOT` as `path.join(__dirname, '..')` and use `process.cwd()` as the working directory for all file lookups (same pattern as other bin/ scripts).
2. Read `formal/sensitivity-report.ndjson` from cwd. If absent, print warning and exit 0.
3. Parse NDJSON: `raw.trim().split('\n').filter(l => l.length > 0).map(l => JSON.parse(l))`.
4. Filter to records where `formalism === 'prism'` and `metadata.parameter === 'tp_rate'`. Call these `prismTpRecords`.
5. If all `prismTpRecords` have `result === 'inconclusive'`, print warning and exit 0 (PRISM not available — no actionable data).
6. Compute empirical TP rate from `quorum-scoreboard.md` in cwd:
   - If file absent, exit 0 with warning.
   - Parse: use regex `/\|\s*\S+\s*\|\s*(\d+)\s*\|\s*(\d+)\s*\|/g` on each line. Sum group 1 as wins, group 2 as losses across all matching rows. `tp_rate = wins / (wins + losses)`. If total is 0, exit 0 with warning.
7. Find `nearestTestedValue`: from the `value` field of prismTpRecords (excluding inconclusive), pick the value minimizing `Math.abs(empiricalRate - value)`.
8. `DEVIATION_THRESHOLD = 0.05` (constant at top of file). If `Math.abs(empiricalRate - nearestTestedValue) <= DEVIATION_THRESHOLD`, print "No deviation detected" and exit 0.
9. If deviation detected:
   a. Print deviation summary to stdout.
   b. Run `spawnSync(process.execPath, [path.join(__dirname, 'export-prism-constants.cjs')], { encoding: 'utf8', cwd: process.cwd(), timeout: 10000 })` — fail-open: warn on non-zero, continue.
   c. Run `spawnSync(process.execPath, [path.join(__dirname, 'run-prism.cjs')], { encoding: 'utf8', cwd: process.cwd(), timeout: 60000 })`.
   d. If run-prism exits non-zero: check for any `flip-to-fail` delta in records near the empirical rate. If found: print "New threshold violation detected" and exit 1. Otherwise exit 0 (pre-existing failure, not new).
   e. If run-prism exits 0: print "PRISM re-run passed" and exit 0.

**Important constraints from research:**
- Only act on `result: 'pass'` or `result: 'fail'` records — skip all `result: 'inconclusive'` records.
- Do NOT hardcode `[0.5, 0.75, 0.95]` — derive the tested values from the report records (`record.metadata.value`).
- The `DEVIATION_THRESHOLD = 0.05` must be a named constant at the top of the file (not a magic number inline).
- Use `spawnSync` with the array-of-args form `[execPath, [scriptPath]]` — no shell interpolation.
- All subprocess calls are fail-open: warn on failure, do not throw.
