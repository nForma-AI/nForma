## E2E Flow 1: Token Collection → Health Display (OBSV Chain)

### Flow Structure

```
[Quorum Slot Execute]
    ↓
[SubagentStart Hook - qgsd-slot-correlator]
    → writes .planning/quorum-slot-corr-{agent_id}.json
    ↓
[Slot Work] (claude-1, codex-1, gemini-1, etc)
    → generates agent_transcript_path with token usage
    ↓
[SubagentStop Hook - qgsd-token-collector]
    → reads agent_transcript_path
    → sums input_tokens, output_tokens, cache tokens
    → resolves slot from correlation file or last_assistant_message
    → appends record to .planning/token-usage.jsonl
    ↓
[CLI Slot Dispatch - call-quorum-slot.cjs]
    → CLI invocation (codex-cli, gemini-cli, etc)
    → appendTokenSentinel(slotName) called 3 times (success/error/unknown-provider)
    → appends null-token record to token-usage.jsonl
    ↓
[User runs /qgsd:health]
    → health.md display_token_usage step executes
    → reads .planning/token-usage.jsonl (last 100 records)
    → aggregates by slot, sorts by total tokens
    → displays table with slot, input, output, rounds columns
    → shows "null (CLI)" for CLI slots with null tokens
    ↓
[User sees token consumption dashboard]
```

## E2E Flow 2: Tier Config → Model Resolution → Plan-Phase Dispatch (TIER Chain)

### Flow Structure

```
[Tier Config in ~/.qgsd.json (or defaults)]
    model_tier_planner: 'opus' (default)
    model_tier_worker: 'haiku' (default)
    model_overrides: { 'gsd-phase-researcher': 'sonnet' } (optional per-agent)
    ↓
[loadConfig(cwd) in hooks/config-loader.js]
    → reads ~/.qgsd.json (or uses DEFAULT_CONFIG)
    → validates model_tier_* values: type=string, value in ['haiku','sonnet','opus']
    → invalid values: stderr WARNING, delete from config
    → returns config with tier keys + model_overrides
    ↓
[resolveModelInternal(cwd, agentType) in gsd-tools.cjs]
    1. Check per-agent override: config.model_overrides[agentType]
       → if present, return it (highest priority)
    2. Map agentType to tier: agentToTier map
       → 'gsd-phase-researcher' → 'worker'
       → 'gsd-plan-checker' → 'worker'
       → 'gsd-planner' → 'planner'
    3. Build tierKey: 'model_tier_' + tier
    4. Check tierModel: config[tierKey]
       → if valid, return it
    5. Fall back to profile lookup (balanced/fast/lean)
    ↓
[init plan-phase in gsd-tools.cjs]
    → calls resolveModelInternal(cwd, 'gsd-phase-researcher') → 'haiku'
    → calls resolveModelInternal(cwd, 'gsd-plan-checker') → 'haiku'
    → calls resolveModelInternal(cwd, 'gsd-planner') → 'inherit'
    → returns JSON context:
       researcher_model: 'haiku'
       checker_model: 'haiku'
       planner_model: 'inherit'
    ↓
[plan-phase.md Task Dispatch]
    → Researcher Task: model="{researcher_model}" → model="haiku"
    → Plan-checker Task: model="{checker_model}" → model="haiku"
    → Planner Task: model="{planner_model}" → model="inherit"
    ↓
[Subagent Execution]
    → Researcher runs on haiku (15-20x cheaper than sonnet)
    → Checker runs on haiku (15-20x cheaper than sonnet)
    → Planner runs on opus (inherit = expensive, high-value decisions)
    ↓
[Cost Reduction Achieved]
    → Per-plan-phase cost drops 15-20x for two spawn sites
```
