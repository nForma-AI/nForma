## Summary

Extended `update-scoreboard.cjs` with a slot-keyed `slots{}` schema and `--slot`/`--model-id` CLI arguments, enabling per-slot-per-model tracking with composite keys.

## What Was Built

- `bin/update-scoreboard.cjs`:
  - USAGE string updated with `--slot` and `--model-id` documentation
  - `validate()` updated: `--slot` and `--model` are mutually exclusive; `--slot` requires `--model-id`; slot names are free-form strings
  - `emptyData()` updated: added `slots: {}` alongside existing `models{}`
  - `loadData()` updated: backward-compat ensures `slots: {}` initialized for old scoreboards
  - Added `emptySlotStats(slot, modelId)` helper
  - Added `recomputeSlots(data)` function that replays rounds looking for composite `slot:model-id` keyed votes
  - Slot-mode branch in `main()`: creates composite key, appends to rounds, calls `recomputeSlots()` (not `recomputeStats()`), prints confirmation
- `bin/update-scoreboard.test.cjs`: added SC-TC-SLOT-1..4 tests covering all three SCBD requirements
