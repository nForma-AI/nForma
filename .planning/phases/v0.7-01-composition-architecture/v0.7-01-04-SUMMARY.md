## Summary

Wired the installer, migration script, and health check binary to auto-populate and consume `quorum_active` from config, making the field self-maintaining.

## What Was Built

- `bin/install.js`:
  - Added `buildActiveSlots()` helper reading all mcpServer keys from `~/.claude.json`
  - Fresh install path: added `quorum_active: buildActiveSlots()` to `qgsdConfig` object
  - Reinstall path: added backfill block after circuit_breaker backfill — populates `quorum_active` if absent/empty; non-empty user value is never overwritten
- `bin/migrate-to-slots.cjs`:
  - Added `populateActiveSlots(qgsdJsonPath, claudeJsonPath, dryRun)` function (idempotent: skips if already set and non-empty)
  - CLI entrypoint calls `populateActiveSlots()` after migration and logs result
  - Exported `populateActiveSlots` in `module.exports`
- `bin/check-provider-health.cjs`:
  - Reads `quorum_active` from global + project `qgsd.json` (project takes precedence)
  - Creates `activeMcpServers` filtered by `quorum_active` (empty = all servers = fail-open)
  - Providers build loop now iterates `activeMcpServers` instead of `mcpServers`

## Key Files Created/Modified

- `bin/install.js` — buildActiveSlots() + fresh install + reinstall backfill
- `bin/migrate-to-slots.cjs` — populateActiveSlots() function + CLI call + export
- `bin/check-provider-health.cjs` — quorum_active filter + activeMcpServers
