## Problem Statement

Phase v0.13-05 fixed the IS_GAP_CLOSURE grep pattern in the SOURCE file (`qgsd-core/workflows/transition.md`, line 458) but did not include a sync/install step. The INSTALLED copy at `~/.claude/qgsd/workflows/transition.md` still uses the old unanchored pattern.

**Source (correct):** `grep -A 4 "^### Phase ${COMPLETED_PHASE}:" .planning/ROADMAP.md`
**Installed (broken):** `grep -A 15 "Phase ${COMPLETED_PHASE}" .planning/ROADMAP.md`

Claude Code reads the INSTALLED copy at runtime — not `qgsd-core/`. So LOOP-02 is broken at runtime.

## How the Installer Works

The installer (`bin/install.js`) copies `qgsd-core/` → `~/.claude/qgsd/` via `copyWithPathReplacement()`:

```javascript
const skillSrc = path.join(src, 'qgsd-core');
const skillDest = path.join(targetDir, 'qgsd');
copyWithPathReplacement(skillSrc, skillDest, pathPrefix, runtime);
```

Running `node bin/install.js --claude --global` will:
1. Copy all files from `qgsd-core/` to `~/.claude/qgsd/` (including `workflows/transition.md`)
2. Apply path prefix replacements (`~/.claude/` references)
3. Log `✓ Installed qgsd`

This is the idiomatic sync approach — it handles all path replacements correctly.

## Alternative: Direct Copy

A simpler alternative is:
```bash
cp /Users/jonathanborduas/code/QGSD/qgsd-core/workflows/transition.md ~/.claude/qgsd/workflows/transition.md
```

However, the installer is preferred because:
- It applies path prefix replacements (handles `~/.claude/` → correct config dir)
- It keeps all installed files consistent (not just transition.md)
- It is the canonical deployment mechanism

**Risk:** The installer also copies hooks from `hooks/dist/` and `bin/*.cjs` — running it may update other installed files. This is acceptable: the installed files should always match the source.
