<files>commands/qgsd/mcp-setup.md</files>
<action>
Insert a new `## Composition Screen` section into `commands/qgsd/mcp-setup.md` immediately before the `## Agent Sub-Menu` section (around line 776).

The section must implement the full WIZ-09 flow:

**Step CS-1: Read slots and quorum_active**
```bash
COMPOSITION_DATA=$(node -e "
const fs = require('fs'), os = require('os');
let slots = [];
let active = [];
try {
  const cj = JSON.parse(fs.readFileSync(os.homedir() + '/.claude.json', 'utf8'));
  slots = Object.keys(cj.mcpServers || {});
} catch(e) {}
try {
  const qgsd = JSON.parse(fs.readFileSync(os.homedir() + '/.claude/qgsd.json', 'utf8'));
  active = Array.isArray(qgsd.quorum_active) ? qgsd.quorum_active : [];
} catch(e) {}
process.stdout.write(JSON.stringify({ slots, active }) + '\n');
")
```
Parse `COMPOSITION_DATA` for `slots` (array of slot names) and `active` (current quorum_active array).

**Step CS-2: Display composition table**

Display banner:
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 QGSD ► QUORUM COMPOSITION
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

Render a numbered table. For each slot:
- If `active` array is non-empty AND slot IS in `active`: show `● ON`
- If `active` array is non-empty AND slot NOT in `active`: show `○ OFF`
- If `active` array is empty: show `● ON (all)` — all slots participate when quorum_active is empty (fail-open)

```
#   Slot              Status
──  ────────────────  ──────
1   codex-cli-1       ● ON
2   gemini-cli-1      ● ON
3   opencode-1        ○ OFF
4   copilot-1         ● ON
```

If `active` is empty, display a note below the table:
```
ℹ  quorum_active is empty — all slots participate (fail-open mode)
```

**Step CS-3: AskUserQuestion for composition actions**

Track an in-memory `PENDING_ACTIVE` array initialized from `active` (copy of it).

Use AskUserQuestion:
- header: "Quorum Composition"
- question: "Enter slot number to toggle ON/OFF, or choose an option:"
- options:
  - "1 — {slot-name} [{current status}]" (one per slot, showing current pending state)
  - "Apply — save changes to qgsd.json"
  - "Add new slot — add a slot to ~/.claude.json and quorum_active"
  - "Cancel — discard changes"

**Toggle handler:** When user selects a slot number N:
- If slot is currently in `PENDING_ACTIVE`: remove it → status becomes `○ OFF`
- If slot is not in `PENDING_ACTIVE` AND `PENDING_ACTIVE` is non-empty: add it → status becomes `● ON`
- If slot is not in `PENDING_ACTIVE` AND `PENDING_ACTIVE` is empty: `PENDING_ACTIVE = [slot]` (switching from fail-open to explicit list with this one slot ON)
- Special case: if removing a slot would make `PENDING_ACTIVE` empty, show warning: "Removing this slot will leave quorum_active empty — all slots will participate (fail-open). Continue?" — proceed only if confirmed.
- Re-display the AskUserQuestion with updated statuses after each toggle.

**Apply handler:** When "Apply" selected:
```bash
node -e "
const fs = require('fs'), os = require('os');
const qgsdPath = os.homedir() + '/.claude/qgsd.json';
let cfg = {};
try { cfg = JSON.parse(fs.readFileSync(qgsdPath, 'utf8')); } catch(e) {}
cfg.quorum_active = JSON.parse(process.env.PENDING_ACTIVE);
fs.writeFileSync(qgsdPath, JSON.stringify(cfg, null, 2) + '\n');
process.stdout.write(JSON.stringify({ written: true, count: cfg.quorum_active.length }) + '\n');
" PENDING_ACTIVE="{JSON.stringify(PENDING_ACTIVE)}"
```

If `written: true`: display:
```
✓ quorum_active updated — {N} slot(s) active.

  Changes take effect on next quorum call (no restart required).
```
Return to roster display.

If error: display error and return to roster.

**Cancel handler:** When "Cancel" selected:
Display "No changes made." Return to roster display.
