Purpose: SIG-04 requires that weak quorum rounds are deferred rather than proceeding silently. This prevents low-quality consensus decisions by gating on the mathematical probability of reaching quorum given current slot availability.
Output: `bin/quorum-consensus-gate.cjs` + tests

  <action>
Create `bin/quorum-consensus-gate.cjs` following existing QGSD patterns (CommonJS, 'use strict').

1. **`poissonBinomialCDF(probabilities, k)`** function:
   - Computes P(X >= k) where X is the number of successes with heterogeneous probabilities.
   - Uses the recursive DP approach for the Poisson binomial distribution:
     - `dp[0] = 1` (base case: 0 successes with 0 trials)
     - For each probability p_i: `dp[j] = dp[j] * (1-p_i) + dp[j-1] * p_i` for j from n down to 1
   - Returns P(X >= k) = sum of dp[k..n].
   - This is equivalent to the PRISM mcp-availability.pm model but computed in O(n^2) without needing Java.

2. **`computeConsensusProbability(slotRates, minQuorum)`** function:
   - Takes an object `{ slot_name: availability_rate }` and minimum quorum size.
   - Extracts probabilities array from slotRates values.
   - Calls `poissonBinomialCDF(probabilities, minQuorum)`.
   - Returns `{ probability, slotCount, minQuorum, rates: slotRates }`.

3. **`checkConsensusGate(options)`** function:
   - Accepts `{ scoreboardPath, configPath, minQuorum }` with defaults.
   - Reads scoreboard availability rates using `readMCPAvailabilityRates` from `run-prism.cjs`.
   - If no rates (empty/missing scoreboard): uses prior rates (0.85 per slot for 4 slots).
   - Reads threshold from config.json `workflow.consensus_probability_threshold` (default: 0.70).
   - Computes P(consensus_reached) via computeConsensusProbability.
   - **If P >= threshold:** returns `{ action: 'proceed', probability, threshold, message: 'Consensus probability {P} >= threshold {T} -- proceeding' }`.
   - **If P < threshold:** returns `{ action: 'defer', probability, threshold, message: 'WARNING: Consensus probability {P} < threshold {T} -- deferring quorum round. Slot availability too low for reliable consensus.' }`.

4. **CLI entrypoint** (guarded by `require.main === module`):
   - Usage: `node bin/quorum-consensus-gate.cjs [--min-quorum=2]`
   - Prints gate result as JSON to stdout.
   - Exit 0 for proceed, exit 1 for defer.

5. **Exports:** `module.exports = { checkConsensusGate, computeConsensusProbability, poissonBinomialCDF }`.

**Key design:** Uses closed-form math (Poisson binomial), NOT live PRISM invocation. This ensures the gate runs in <1ms, suitable for calling before every quorum round.
