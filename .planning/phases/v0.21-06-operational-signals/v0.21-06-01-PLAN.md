Output: `bin/detect-coverage-gaps.cjs` + tests

  <name>Task 1: Create detect-coverage-gaps.cjs -- TLC state coverage gap detector</name>
  <files>bin/detect-coverage-gaps.cjs</files>
  <action>
Create `bin/detect-coverage-gaps.cjs` following existing QGSD patterns (CommonJS, 'use strict', error handling via process.stderr.write).

1. **State mapping definitions:**
   Define a `STATE_MAPS` object mapping each TLA+ spec to its state variable name and value-to-name mapping. Use the state comments from the TLA+ specs as the source of truth:
   ```js
   const STATE_MAPS = {
     'QGSDQuorum': {
       variable: 's',
       values: { '0': 'COLLECTING_VOTES', '1': 'DECIDED', '2': 'DELIBERATING' }
     },
     'QGSDStopHook': {
       variable: 'phase',
       values: { '0': 'IDLE', '1': 'READING', '2': 'DECIDING', '3': 'BLOCKED' }
     },
     // ... other specs as needed
   };
   ```

2. **`parseTlcStates(specName)`** function:
   - Looks up the spec in STATE_MAPS.
   - Returns the full set of named states from the mapping (all values).
   - This represents the complete TLC-reachable state space for that spec.
   - Returns `{ specName, states: Set<string>, variable: string }` or null if spec unknown.

3. **`parseTraceStates(logPath)`** function:
   - Reads `.planning/conformance-events.jsonl` (or provided path).
   - Parses each JSON line, extracts the `state` or `action` field.
   - Maps action names to states using a reverse mapping (e.g., 'quorum_start' -> 'COLLECTING_VOTES').
   - Returns `Set<string>` of observed states.
   - If file doesn't exist, returns null.

4. **`detectCoverageGaps(options)`** function:
   - Accepts `{ specName, logPath, outputPath }` with defaults.
   - Calls parseTlcStates to get reachable states.
   - Calls parseTraceStates to get observed states.
   - If no trace log: returns `{ status: 'no-traces', reason: 'conformance log not found' }`.
   - Computes gap = reachable - observed.
   - If gap is empty: returns `{ status: 'full-coverage', gaps: [] }`.
   - If gaps exist: writes `formal/coverage-gaps.md` and returns `{ status: 'gaps-found', gaps: [...], outputPath }`.

5. **Coverage gaps markdown format:**
   ```markdown
   # TLC Coverage Gaps

   Generated: {timestamp}
   Spec: {specName}

   ## Unreached States

   | State | Variable Value | Description |
   |-------|---------------|-------------|
   | BLOCKED | phase=3 | State reachable by TLC but never observed in conformance traces |

   ## Coverage Summary

   - TLC reachable: {N} states
   - Trace observed: {M} states
   - Gaps: {N-M} states ({percentage}% coverage)

   ## Action Items

   Each gap represents a state that formal verification proves is reachable but production has never exercised.
   Add test cases that drive the system into these states.
   ```

6. **CLI entrypoint** (guarded by `require.main === module`):
   - Usage: `node bin/detect-coverage-gaps.cjs [--spec=QGSDQuorum] [--log=path]`
   - Default spec: QGSDQuorum (most commonly verified).
   - Prints summary to stdout.

7. **Exports:** `module.exports = { detectCoverageGaps, parseTlcStates, parseTraceStates }`.
