  <name>Task 1: Add buildRoadmapDot and computeCriticalPath to generate-petri-net.cjs</name>
  <files>bin/generate-petri-net.cjs</files>
  <action>
Extend `bin/generate-petri-net.cjs` with roadmap Petri net generation. Keep existing quorum Petri net functionality intact.

1. **`parseRoadmapPhases(roadmapContent)`** function:
   - Parses ROADMAP.md content to extract phases and dependencies.
   - Regex for phase headers: `/^### Phase (v[\d.]+-\d+):\s*(.+)/` (captures phase number and name).
   - Regex for depends_on: `/^\*\*Depends on\*\*:\s*(.+)/` (captures comma-separated phase references).
   - Regex for completion status: `/^\- \[(x| )\].*Phase (v[\d.]+-\d+)/` (captures checkbox state).
   - Returns array of `{ number, name, dependsOn: string[], completed: boolean }`.

2. **`buildRoadmapDot(phases)`** function:
   - Takes parsed phases array.
   - Generates a DOT digraph representing a Petri net:
     - Each phase is a transition (rectangle).
     - Between each dependency pair, a place (circle) represents the completion token.
     - A "start" place feeds phases with no dependencies.
     - A "done" place receives tokens from phases with no dependents.
   - Place naming: `p_{phase_from}_{phase_to}` for dependency tokens.
   - Completed phases are styled differently (green fill).
   - Returns DOT string.

3. **`computeCriticalPath(phases)`** function:
   - Builds a DAG from phase dependencies.
   - Performs topological sort.
   - Computes longest path from any source to any sink (each phase weight = 1).
   - Returns `{ path: string[], length: number }` where path is the sequence of phase numbers on the critical path.

4. **CLI changes:**
   - When `--roadmap` flag is present:
     - Read `$CWD/.planning/ROADMAP.md`.
     - Call `parseRoadmapPhases(content)`.
     - Call `buildRoadmapDot(phases)`.
     - Call `computeCriticalPath(phases)`.
     - Write `formal/petri/roadmap-petri-net.dot`.
     - Render SVG via @hpcc-js/wasm-graphviz (same pattern as existing quorum net).
     - Print critical path to stdout.
   - Without `--roadmap`: existing quorum Petri net behavior unchanged.

5. **Update exports:**
   ```js
   module.exports._pure = { buildDot, buildRoadmapDot, computeCriticalPath, parseRoadmapPhases };
   ```
