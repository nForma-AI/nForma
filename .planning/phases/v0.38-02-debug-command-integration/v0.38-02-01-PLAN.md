  <action>
Create `bin/debug-formal-context.cjs` — a testable helper that assembles formal context for the debug workflow. This script both exports functions for unit testing and runs as a CLI tool when invoked directly.

**CLI interface:**
```
node bin/debug-formal-context.cjs --description "failure description" --format json
```

**CLI JSON output schema** (this is the contract — Step A.5 in Task 2 parses this):
```json
{
  "verdict": "reproduced|not-reproduced|no-model",
  "constraints": [
    { "text": "...", "source_model": "...", "formalism": "tla|alloy" }
  ],
  "models": [
    { "name": "...", "reproduced": true|false }
  ]
}
```

The `verdict` field is one of exactly three strings: `"reproduced"`, `"not-reproduced"`, or `"no-model"`. The `constraints` array contains max 3 entries. The `models` array contains all matched models with their checker results.

**Exported functions:**

1. `assembleFormalContext(description, options)` — Main orchestrator:
   - Calls `runBugModeMatching(description)` from formal-scope-scan.cjs to find matching models
   - If matches found AND `options.runCheckers !== false`, calls `runModelCheckers(matches)` from formal-scope-scan.cjs with 30s timeout (NOT the default 60s — debug loop needs faster turnaround)
   - For reproducing models, calls `extractTlaConstraints` or `extractAlloyConstraints` from model-constrained-fix.cjs based on formalism type, then `renderConstraintSummary` to get English text
   - Returns `{ verdict, constraints, models }`:
     - `verdict`: "reproduced" if any model's checker reported failure, "not-reproduced" if models exist but none failed, "no-model" if no matches
     - `constraints`: array of `{ text, source_model, formalism }` — max 3, sorted by relevance score descending (per research pitfall 2 — avoid cognitive overload)
     - `models`: array of matched model objects with checker results
   - **Fail-open on ALL errors**: wrap entire function in try/catch, return `{ verdict: 'no-model', constraints: [], models: [] }` on any error. Log errors to stderr. NEVER throw — formal context is advisory, not required.
   - **Quorum invariant safety**: EventualConsensus (from QGSDQuorum.tla) requires the quorum to eventually reach DECIDED state. Fail-open semantics ensure formal consultation never blocks this.

2. `formatConstraintBlock(constraints)` — Formats constraints for worker prompt injection:
   - If constraints array is empty, return empty string (no block injected)
   - Otherwise return (using only the first 3 constraints, slice if more):
     ```
     [FORMAL CONSTRAINTS]
     The following constraints were extracted from formal models that cover this failure:
     - {constraint 1 text}
     - {constraint 2 text}
     - {constraint 3 text}

     These are verified properties of the system. Do NOT propose fixes that violate these constraints.
     [END FORMAL CONSTRAINTS]
     ```

3. `formatVerdictSummary(verdict, models)` — One-line summary for bundle:
   - "reproduced" -> "FORMAL: Bug reproduced by model {name} ({formalism})"
   - "not-reproduced" -> "FORMAL: {N} models exist but none reproduced this bug"
   - "no-model" -> "FORMAL: No formal model covers this failure"

**CLI mode (when require.main === module):**
- Parse --description and --format args
- Call assembleFormalContext(description, { runCheckers: true })
- Output JSON to stdout (matching the schema above exactly)
- Exit 0 on success, exit 1 on fatal error (but assembleFormalContext itself never throws)

**Module exports:**
```javascript
module.exports = { assembleFormalContext, formatConstraintBlock, formatVerdictSummary };
```

**Dependencies (all existing, no new installs):**
- `require('./formal-scope-scan.cjs')` for runBugModeMatching, runModelCheckers
- `require('./model-constrained-fix.cjs')` for extractTlaConstraints, extractAlloyConstraints, renderConstraintSummary

```
### Step A.5: Formal Model Consultation

Run formal model consultation to check if formal models cover this failure:

  node bin/debug-formal-context.cjs --description "$ARGUMENTS" --format json 2>/dev/null

Parse the JSON output. The output schema is:
  { "verdict": "reproduced|not-reproduced|no-model", "constraints": [...], "models": [...] }

Assign from parsed JSON:
  - $FORMAL_VERDICT = parsed.verdict (one of: "reproduced", "not-reproduced", "no-model")
  - $FORMAL_CONTEXT = the full parsed JSON object (used in Steps B, C, E, and G)

If the command fails or produces no output, set:
  - $FORMAL_VERDICT = "no-model"
  - $FORMAL_CONTEXT = { "verdict": "no-model", "constraints": [], "models": [] }
and continue (fail-open — formal context is advisory).
```

**Step B modification (assemble bundle):**

In the bundle assembly, add after the existing FAILURE CONTEXT / EXIT CODE / TEST OUTPUT lines:

```
  FORMAL VERDICT: $FORMAL_VERDICT (or "skipped" if Step A.5 failed)
```

**Step C modification (worker prompt template):**

Between the `</bundle>` closing tag and the "Given this failure, answer ONLY:" line, insert the constraint block conditionally:

```
  If $FORMAL_CONTEXT.constraints is non-empty, insert:

  - [constraint 3 text]

  These are verified properties of the system. Do NOT propose fixes that violate these constraints.
  [END FORMAL CONSTRAINTS]
```
