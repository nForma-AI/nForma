### SC3 — MULTI-03: Direct ~/.claude.json edit + `npx qgsd@latest` populates quorum_active for new slot

**Evidence in `bin/install.js` (lines 1915-1929):**

```javascript
// MULTI-03: Incremental quorum_active update — append new slots discovered since last install
// Only runs if quorum_active is already set (non-empty); new slots are appended, existing preserved
if (existingConfig.quorum_active && existingConfig.quorum_active.length > 0) {
  const { addSlotToQuorumActive } = require('./migrate-to-slots.cjs');
  const allCurrentSlots = buildActiveSlots();
  const newSlots = allCurrentSlots.filter(s => !existingConfig.quorum_active.includes(s));
  for (const newSlot of newSlots) {
    const result = addSlotToQuorumActive(newSlot, qgsdConfigPath);
    if (result.added) {
      console.log(`  ✓ Added new slot to quorum_active: ${newSlot}`);
      existingConfig.quorum_active.push(newSlot);
    }
  }
}
```

- `buildActiveSlots()` reads all mcpServers keys from `~/.claude.json`
- Difference against `existingConfig.quorum_active` identifies new slots
- `addSlotToQuorumActive()` is idempotent — safe to call on each reinstall

**Evidence in `bin/migrate-to-slots.cjs`:**
- `addSlotToQuorumActive(slotName, qgsdJsonPath, dryRun)` implemented
- Appends slot to `quorum_active` array; no-op if already present
- Returns `{ added, slot, skipped, reason? }`

**Test coverage (MS-TC-ADD-1..5):** 5/5 pass in `bin/migrate-to-slots.test.cjs`

**Status: PASS**
