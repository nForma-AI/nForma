# Phase v0.14-04 Plan 02 Summary: PRISM Scoreboard Injection Integration Tests

## One-liner

Integration tests for run-prism.cjs scoreboard-to-PRISM -const injection using fixture scoreboards and Args log capture.

## What Was Built

Created `bin/run-prism.test.cjs` with 4 integration tests verifying that `bin/run-prism.cjs` correctly reads `.planning/quorum-scoreboard.json` and injects empirical TP/unavailability rates as PRISM `-const` parameters — without requiring PRISM to be installed.

The tests work by setting `PRISM_BIN=prism` (the default sentinel value), which causes `run-prism.cjs` to skip the binary-existence check. The `[run-prism] Args: ...` log line is printed to stdout before `spawnSync` is called, so tests can assert the correct `-const` flags appear regardless of whether `prism` is on PATH.

Added `bin/run-prism.test.cjs` to the `npm test` script (after `run-oauth-rotation-prism.test.cjs`, before `check-spec-sync.test.cjs`).

## Test Coverage

| Test | Scenario | Key Assertions |
|------|----------|---------------|
| 1 | No scoreboard exists | Args has `-const tp_rate=0.85`, `-const unavail=0.15`; stderr says "No scoreboard"; no "Injected from scoreboard" |
| 2 | All-TP fixture (10 rounds) | Args has `-const tp_rate=1`, `-const unavail=0`; "Injected from scoreboard" appears |
| 3 | Mixed: 3x TP + 1x UNAVAIL per slot | Args has `-const tp_rate=0.75`, `-const unavail=0.25` |
| 4 | Caller override `-const tp_rate=0.42` | Override value appears; scoreboard value (1.0) absent; unavail still injected from scoreboard |
