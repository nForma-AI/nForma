The key testing challenge: run-prism.cjs exits non-zero when PRISM_BIN is not installed. However, the `Args:` log line is printed BEFORE PRISM is spawned. All tests must capture stdout/stderr and assert the `-const` flags appear in the Args log line (which prints before binary launch), regardless of whether PRISM actually runs.

Pattern for capturing the Args log: set `PRISM_BIN=/nonexistent/path` so the script exits with error 1, but the `[run-prism] Args: ...` line still appears in stdout because it is printed before the spawnSync call.

Wait â€” re-read run-prism.cjs carefully: it checks `if (prismBin !== 'prism' && !fs.existsSync(prismBin))` at the TOP and exits early before printing Args. So with a nonexistent PRISM_BIN, the Args line is never printed.

Correct approach: Set PRISM_BIN to a value that passes the existence check (i.e., point it at an existing but non-PRISM file like `/bin/false` or `/usr/bin/true` on macOS/Linux). The binary will be found (exists), spawnSync will run it, it will exit non-zero (not a PRISM invocation), but the `[run-prism] Args: ...` line will appear in stdout before the spawn.

Alternative (simpler): Use `PRISM_BIN=prism` (the default sentinel â€” skip the existence check). spawnSync will attempt to run `prism` from PATH, fail with ENOENT, and the Args line will still have been printed to stdout.

Use `PRISM_BIN=prism` (the literal string 'prism') â€” the existence check is skipped for this value. The Args line is printed before spawnSync. The binary invocation will fail (prism not on PATH in test env) but the test only checks stdout for the Args line.
