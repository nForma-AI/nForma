**Block 1: Session State Injection (SESSION-01, SESSION-02, SESSION-03)**

Add a helper function `extractStateContext(stateContent)`:
- Extract the "## Current Position" section from STATE.md content
- Also extract "### Decisions" section if present
- Cap extraction at 20 lines total
- Return the extracted text trimmed, or empty string on failure

In the main hook body, BEFORE the cmdPattern check, add the session state injection block:
```js
// ── SESSION STATE INJECTION (SESSION-01/02/03) ──
// Inject STATE.md summary on first message of each session only.
// Fail-open: missing STATE.md or read errors are silently skipped.
if (isFirstMessageOfSession(cwd, sessionId)) {
  try {
    const statePath = path.join(cwd, '.planning', 'STATE.md');
    if (fs.existsSync(statePath)) {
      const stateContent = fs.readFileSync(statePath, 'utf8');
      const stateContext = extractStateContext(stateContent);
      if (stateContext && stateContext.length > 0 && stateContext.length <= 500) {
        instructions += '\n\nSESSION CONTEXT (from STATE.md):\n' + stateContext;
      }
    }
  } catch (_) { /* fail-open: STATE.md read failure is non-fatal */ }
}
```

Note: `sessionId` is already available via `const sessionId = input.session_id || null;` earlier in the function. Use it directly.
