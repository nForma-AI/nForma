<task type="auto">
  <name>Task 1: Create predictive-power.cjs with bug-to-property linking and recall computation</name>
  <files>bin/predictive-power.cjs</files>
  <action>
Create bin/predictive-power.cjs following the established module pattern (see gate-stability.cjs, oscillation-detector.cjs).

Header: `'use strict';` with JSDoc comment referencing Requirements: PRED-01, PRED-02.

Implement these exported functions:

1. **linkBugsToProperties(debtPath, registryPath)** (PRED-01):
   - Read debt.json at debtPath, parse `debt_entries` array
   - Read model-registry.json at registryPath, parse `models` object
   - For each debt entry, extract `formal_refs` (array of requirement IDs -- already populated by formal-ref-linker.cjs from v0.27-03)
   - For each model in registry, check if model's `requirements` array overlaps with the debt entry's `formal_refs`
   - If overlap exists, the bug is "predictable" by that model
   - Build mapping with schema from RESEARCH.md Pattern 1 (schema_version, generated, total_bugs, total_linked, mappings array)
   - Each mapping entry: bug_id, bug_fingerprint, bug_title (truncated 120 chars), source_type, formal_refs, matching_models array, predicted boolean
   - Each matching_model: model_path, requirements_overlap, gate_maturity, layer_maturity
   - Entries WITHOUT formal_refs still appear in mappings (predicted: false, matching_models: []) -- these are counted as unlinked
   - Return the mapping object (caller decides whether to write it)

2. **writeBugToProperty(mappingObj, outputPath)** -- convenience writer:
   - Takes mapping object from linkBugsToProperties, adds `generated` timestamp
   - Writes atomically to outputPath as formatted JSON

3. **computePerModelRecall(mappings, perModelGatesPath)** (PRED-02):
   - Read per-model-gates.json at perModelGatesPath (written every solve run per INTG-01 from v0.33-03)
   - Group bug mappings by model_path
   - For each model: relevant = bugs whose formal_refs overlap with model requirements, predicted = subset where model's gate_a passes
   - Compute recall = predicted / relevant (0 if relevant is 0)
   - Return object keyed by model_path: { relevant, predicted, recall (4 decimal places) }
   - CRITICAL: Recall is INFORMATIONAL ONLY -- do NOT use as gate input (PRED-03 is deferred per REQUIREMENTS.md)

4. **formatRecallSummary(recallScores)** -- human-readable summary:
   - Report aggregate: total models scored, average recall, top-5 models by recall
   - Keep under 15 lines for solve report integration (Pitfall 5: avoid unbounded output)
   - Return string

Edge cases per RESEARCH.md pitfalls:
- debt_entries may be empty array (return total_bugs: 0, total_linked: 0, mappings: [])
- Entries without formal_refs: count separately, include in mappings as unlinked
- Missing files: fail-open with stderr warning, return empty results
- model-registry.json may have models without `.` prefix keys (filter to only model paths starting with `.`)

Module pattern: `module.exports = { linkBugsToProperties, writeBugToProperty, computePerModelRecall, formatRecallSummary };`
