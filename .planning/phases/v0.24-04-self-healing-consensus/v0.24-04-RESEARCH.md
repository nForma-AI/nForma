# Phase v0.24-04: Self-Healing Consensus - Research

**Researched:** 2026-03-03
**Domain:** Consensus probability prediction, early escalation trigger logic, configuration self-healing
**Confidence:** HIGH

## Summary

Phase v0.24-04 implements automatic detection and healing of degraded consensus conditions. The system uses the existing PRISM consensus probability model to predict P(consensus | remaining rounds) after each deliberation round, and fires escalation early when probability drops below a threshold, rather than exhausting all maxDeliberation rounds. Additionally, the verify-quorum-health tool gains the ability to recommend and auto-apply maxDeliberation adjustments to qgsd.json when empirical reliability drops below 95% target confidence.

The phase depends on v0.24-02 (dispatch reliability) and v0.24-03 (observability: telemetry, delivery stats, flakiness), which provide the data (per-round verdicts, per-slot reliability metrics) needed to compute accurate consensus probabilities.

**Primary recommendation:** Implement HEAL-01 (early escalation via consensus probability gate in deliberation loop) and HEAL-02 (auto-adjust maxDeliberation in XState machine and qgsd.json) as two coordinated tasks. HEAL-01 reuses quorum-consensus-gate.cjs logic already verified in v0.23; HEAL-02 extends verify-quorum-health.cjs to modify machine state and config files after user approval.

## Architecture Patterns

### Pattern 1: Conditional Consensus Probability (HEAL-01)

**What:** After each deliberation round votes are collected, compute P(consensus | remaining rounds) using the Poisson binomial distribution. If P falls below a threshold (default 10%), immediately trigger escalation instead of looping to the next deliberation round.

**When to use:** HEAL-01 requires a deliberation loop that can be interrupted mid-execution. The XState machine's DELIBERATING state has a guard `noInfiniteDeliberation` that checks `deliberationRounds < maxDeliberation`. HEAL-01 adds a second guard: `consensusProbable` that checks `P(consensus | remaining) >= threshold`.

**Data flow:**
1. Votes collected for round N
2. Update scoreboard with round verdicts
3. **NEW:** Read scoreboard rates for all slots
4. **NEW:** Compute P(consensus | maxDeliberation - N)
5. **NEW:** If P < threshold, fire DECIDE event (escalate immediately)
6. Otherwise, proceed to next deliberation round

**Current guard (XState):**
```javascript
// From src/machines/qgsd-workflow.machine.ts
guards: {
  noInfiniteDeliberation: ({ context }) =>
    context.deliberationRounds < context.maxDeliberation,
  // ...
}

// In DELIBERATING state:
{
  guard: 'noInfiniteDeliberation',
  target: 'DELIBERATING',
  // continue looping
}
```

**Extended guard (HEAL-01 addition):**
```javascript
guards: {
  noInfiniteDeliberation: ({ context }) =>
    context.deliberationRounds < context.maxDeliberation,

  // NEW for HEAL-01:
  consensusProbable: ({ context, event }) => {
    // Read from scoreboard rates or event payload
    const pConsensus = computeConsensusProbability(...);
    const threshold = context.consensusThreshold || 0.10;
    return pConsensus >= threshold;
  },
}
```

**Execution location:** The deliberation loop runs in qgsd-prompt.js (or new quorum orchestration module). After each round's vote merge:
1. Call `quorum-consensus-gate.cjs --min-quorum=3` to get P(consensus)
2. If result.action === 'defer' (i.e., P < threshold), break loop and trigger escalation
3. Otherwise, continue to next round

### Pattern 2: Interactive Config Auto-Adjuster (HEAL-02)

**What:** When verify-quorum-health.cjs detects that the empirical P(within maxDeliberation) is below the target confidence (95% default), instead of just reporting "FAIL", it recommends a new maxDeliberation value and asks for user approval to apply it.

**When to use:** HEAL-02 runs as a post-round health check or on explicit user command (`/qgsd:health-check-auto-adjust` or similar). It's not part of the deliberation loop; it's a batch operation that updates configuration after observing degradation.

**Execution flow (in verify-quorum-health.cjs):**
```
1. Read maxDeliberation from XState machine
2. Read empirical rates from scoreboard
3. Compute P(within maxDeliberation)
4. If P < TARGET_CONFIDENCE:
   a. Recommend new_maxDeliberation = kForTarget (calculated)
   b. Ask user: "Update maxDeliberation from 7 to 9? (yes/no/defer)"
   c. If yes:
      - Update src/machines/qgsd-workflow.machine.ts (maxDeliberation: 7 → 9)
      - Update context defaults in machine
      - Update qgsd.json workflow config if it exists
      - Re-run formal spec generation (node bin/generate-formal-specs.cjs)
      - Report success
   d. If no/defer: Report recommendation but do not update
```

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Consensus probability computation | Custom inclusion-exclusion logic | poissonBinomialCDF from quorum-consensus-gate.cjs | Already verified in v0.23; tested with heterogeneous slot rates; mathematically proven |
| Per-round verdict aggregation | Custom filtering/mapping | Existing scoreboard round structure (rounds[].votes) | Schema is established; already fed from v0.24-03 telemetry |
| Interactive user approval | Custom readline/prompt logic | Use existing CLI pattern (node bin/tool.cjs --confirm or similar) or integrate with qgsd-prompt.js workflow | QGSD has established patterns for structured user input; avoid reimplementing |
| XState machine modification | Custom regex replacement | Establish one authoritative update function in bin/update-machine.cjs | Prevents divergence between machine.ts, qgsd.json, and formal specs; single source of truth |
| Formal spec regeneration | Custom spec editing | node bin/generate-formal-specs.cjs after machine update | Specs are auto-generated; hand-editing creates drift and test failures |

**Key insight:** Consensus probability and reliability calibration are complex, formally-verified operations. Reusing the proven quorum-consensus-gate.cjs and verify-quorum-health.cjs modules (already tested) prevents bugs and ensures consistency with the formal model.

## Common Pitfalls

### Pitfall 1: Consensus Probability Computed from Wrong Data

**What goes wrong:** P(consensus | remaining_rounds) is computed from stale scoreboard rates, so the early escalation threshold is out of sync with actual reliability.

**Why it happens:** Scoreboard is only updated after merge-wave completes. If consensus gate runs before scoreboard is refreshed, it uses old rates.

**How to avoid:** Compute consensus probability AFTER update-scoreboard.cjs merge-wave completes, not before. Explicitly sequence: (1) collect votes, (2) merge-wave update, (3) compute P(consensus), (4) check threshold.

**Warning signs:** Escalation fires when consensus is actually likely; or escalation doesn't fire when it should. Verify via debug log: "P(consensus|3 remaining) = 0.92 >= 0.10 threshold → continue" vs actual outcome.

### Pitfall 2: maxDeliberation Updated Without Regenerating Specs

**What goes wrong:** XState machine context has maxDeliberation: 9, but the formal TLA+ spec still assumes MaxDeliberation=7, so verify-quorum-health.cjs reports wrong P(within 7 rounds) on the next run.

**Why it happens:** Updating machine.ts is a quick file edit, but forgetting to run node bin/generate-formal-specs.cjs afterward leaves specs out of sync.

**How to avoid:** HEAL-02 must run formal spec regeneration as part of the auto-apply sequence: (1) update machine.ts, (2) update qgsd.json, (3) run node bin/generate-formal-specs.cjs, (4) verify specs match via bin/check-spec-sync.cjs, (5) report success.

**Warning signs:** `check-spec-sync.cjs` reports "XState maxDeliberation=9 but TLA+ MaxDeliberation=7"; or next run of verify-quorum-health reports inconsistent P(within maxDeliberation).

### Pitfall 3: Escalation Threshold Hardcoded Instead of Configurable

**What goes wrong:** Early escalation threshold (default 10%) is hardcoded in quorum loop logic. User cannot override to be more or less aggressive without code changes.

**Why it happens:** Treating threshold as a constant instead of a config parameter.

**How to avoid:** Read early escalation threshold from config.json workflow.early_escalation_threshold (default 0.10). Pass as argument to consensus-probability gate. Store in XState context alongside maxDeliberation.

**Warning signs:** User asks "Can we be more aggressive (escalate at 15% instead of 10%)?" and the only answer is code changes.

### Pitfall 4: User Approval Flow Not Atomic

**What goes wrong:** User approves maxDeliberation update, HEAL-02 updates machine.ts successfully, but fails partway through formal spec generation. Machine and specs are now inconsistent.

**Why it happens:** No transactional wrapper around the multi-step update sequence.

**How to avoid:** Wrap the entire sequence (machine update, qgsd.json update, spec generation, verification) in a single operation with a rollback function. If any step fails, restore all files to pre-update state before returning to user.

**Warning signs:** `check-spec-sync.cjs` fails; verify-quorum-health.cjs and actual machine diverge; specs are unverifiable.

## Code Examples

### HEAL-01: Early Escalation in Deliberation Loop

**Location:** qgsd-prompt.js (quorum dispatch) or new quorum-orchestration.cjs module

```javascript
// Source: bin/quorum-consensus-gate.cjs (existing), extended for HEAL-01
const { checkConsensusGate } = require('./quorum-consensus-gate.cjs');

async function deliberationLoop(initialRound, maxDeliberation, earlyEscalationThreshold = 0.10) {
  let deliberationRounds = initialRound; // Start from round where we landed

  while (deliberationRounds < maxDeliberation) {
    // Collect votes for this round (existing quorum dispatch logic)
    const roundResult = await dispatchQuorumRound(deliberationRounds);
    const { votes, slotCount } = roundResult;

    // Update scoreboard with new round data
    await updateScorebord.cjs(...);

    // NEW (HEAL-01): Check consensus probability after scoreboard update
    const gateResult = checkConsensusGate({
      minQuorum: Math.ceil(slotCount / 2),
      // Uses updated scoreboard rates
    });

    if (gateResult.action === 'defer') {
      // P(consensus | remaining) < earlyEscalationThreshold
      process.stdout.write(
        `[HEAL-01] Early escalation: P(consensus | ${maxDeliberation - deliberationRounds} remaining) = ` +
        `${gateResult.probability.toFixed(4)} < ${earlyEscalationThreshold.toFixed(2)} threshold\n`
      );
      // Break loop and fire DECIDE event
      return { escalated: true, reason: 'early_escalation', rounds: deliberationRounds };
    }

    // Check unanimity (existing logic)
    const unanimous = votes.every(v => v === 'APPROVE');
    if (unanimous) {
      return { escalated: false, reason: 'unanimity', rounds: deliberationRounds };
    }

    // Continue to next round
    deliberationRounds++;
  }

  // Exhausted maxDeliberation without early escalation
  return { escalated: true, reason: 'max_rounds_exhausted', rounds: deliberationRounds };
}

// Usage in quorum dispatch:
const result = await deliberationLoop(1, 7, 0.10);
if (result.escalated) {
  xstateMachine.send({ type: 'DECIDE', outcome: result.reason });
}
```

### HEAL-02: Interactive maxDeliberation Adjustment

**Location:** bin/verify-quorum-health.cjs (extended)

```javascript
// Source: bin/verify-quorum-health.cjs extended for HEAL-02

// At the end of the health report, after detecting P(actual) < TARGET_CONFIDENCE:

async function recommendAndApplyAutoAdjustment(pActual, kForTarget, TARGET_CONFIDENCE, ROOT) {
  const machineFile = path.join(ROOT, 'src', 'machines', 'qgsd-workflow.machine.ts');
  const configFile = path.join(ROOT, '.planning', 'config.json');
  const qgsdJsonFile = path.join(ROOT, '.planning', 'qgsd.json');

  // Read current maxDeliberation
  const machineSrc = fs.readFileSync(machineFile, 'utf8');
  const currentMatch = machineSrc.match(/maxDeliberation:\s*(\d+)/);
  const currentMaxDelib = currentMatch ? parseInt(currentMatch[1], 10) : 7;

  process.stdout.write(
    '\n[HEAL-02] Recommended Action:\n' +
    '  Increase maxDeliberation from ' + currentMaxDelib + ' to ' + kForTarget +
    ' to achieve ' + (TARGET_CONFIDENCE * 100).toFixed(0) + '% confidence\n\n'
  );

  // Interactive approval (simplified; in reality use readline or Claude Code prompting)
  process.stdout.write('Apply this change? (yes/no) [no]: ');

  // For CLI, read from stdin; for Claude Code integration, accept --auto-apply flag
  const autoApply = process.argv.includes('--auto-apply');
  const userConfirms = autoApply || await getUserApprovalViaStdin();

  if (!userConfirms) {
    process.stdout.write('Deferred. Run again with --auto-apply to apply automatically.\n');
    return;
  }

  // Apply update atomically
  try {
    // 1. Update XState machine
    const updatedMachineSrc = machineSrc.replace(
      /maxDeliberation:\s*\d+/,
      'maxDeliberation: ' + kForTarget
    );
    fs.writeFileSync(machineFile, updatedMachineSrc, 'utf8');
    process.stdout.write('✓ Updated src/machines/qgsd-workflow.machine.ts (maxDeliberation = ' + kForTarget + ')\n');

    // 2. Update qgsd.json if it exists (project-level override)
    if (fs.existsSync(qgsdJsonFile)) {
      const qgsdConfig = JSON.parse(fs.readFileSync(qgsdJsonFile, 'utf8'));
      if (!qgsdConfig.workflow) qgsdConfig.workflow = {};
      qgsdConfig.workflow.maxDeliberation = kForTarget;
      fs.writeFileSync(qgsdJsonFile, JSON.stringify(qgsdConfig, null, 2), 'utf8');
      process.stdout.write('✓ Updated .planning/qgsd.json (workflow.maxDeliberation = ' + kForTarget + ')\n');
    }

    // 3. Update .planning/config.json (if that's the operational config)
    if (fs.existsSync(configFile)) {
      const config = JSON.parse(fs.readFileSync(configFile, 'utf8'));
      if (!config.workflow) config.workflow = {};
      config.workflow.maxDeliberation = kForTarget;
      fs.writeFileSync(configFile, JSON.stringify(config, null, 2), 'utf8');
      process.stdout.write('✓ Updated .planning/config.json\n');
    }

    // 4. Regenerate formal specs using safe spawning
    process.stdout.write('Regenerating formal specs...\n');
    const { spawnSync } = require('child_process');
    const genResult = spawnSync('node', ['bin/generate-formal-specs.cjs'], {
      cwd: ROOT,
      stdio: 'inherit',
      timeout: 30000
    });

    if (genResult.error || genResult.status !== 0) {
      process.stderr.write('✗ Formal spec generation failed (status=' + genResult.status + ')\n');
      throw new Error('Spec generation failed');
    }
    process.stdout.write('✓ Formal specs regenerated\n');

    // 5. Verify specs match machine
    const syncResult = spawnSync('node', ['bin/check-spec-sync.cjs'], {
      cwd: ROOT,
      timeout: 10000
    });

    if (syncResult.error || syncResult.status !== 0) {
      process.stderr.write('✗ Spec verification failed\n');
      throw new Error('Spec verification failed');
    }
    process.stdout.write('✓ Specs verified against XState machine\n');

    process.stdout.write(
      '\nSuccess! maxDeliberation is now ' + kForTarget + '.\n' +
      'Next quorum runs will aim for ' + (TARGET_CONFIDENCE * 100).toFixed(0) + '% confidence.\n'
    );

  } catch (err) {
    process.stderr.write(
      '\n✗ Auto-adjustment failed: ' + err.message + '\n' +
      'Files may be partially updated. Manual rollback may be needed.\n'
    );
    process.exit(1);
  }
}
```

### Reading Consensus Threshold from Config

**Location:** XState machine or quorum dispatch module

```javascript
// Read early escalation threshold from qgsd.json or .planning/config.json
function readEarlyEscalationThreshold() {
  const configPaths = [
    path.join(process.cwd(), '.planning', 'qgsd.json'),
    path.join(process.cwd(), '.planning', 'config.json'),
    path.join(process.env.HOME, '.claude', 'qgsd.json'),
  ];

  for (const cfgPath of configPaths) {
    try {
      if (fs.existsSync(cfgPath)) {
        const config = JSON.parse(fs.readFileSync(cfgPath, 'utf8'));
        if (config.workflow?.early_escalation_threshold !== undefined) {
          return config.workflow.early_escalation_threshold;
        }
      }
    } catch (_) {}
  }

  return 0.10; // Default threshold: 10%
}
```

## Open Questions

1. **Early escalation threshold placement**
   - What we know: Threshold (default 10%) is needed in deliberation loop to trigger early escalation
   - What's unclear: Should threshold be a constant in quorum-consensus-gate.cjs, or read from config every time?
   - Recommendation: Store in XState context alongside maxDeliberation. Read from qgsd.json on startup. Allow per-quorum override via environment variable for testing.

2. **Formal spec coverage for early escalation**
   - What we know: TLA+ spec currently models maxDeliberation as a hard upper bound on DELIBERATING rounds
   - What's unclear: How to formally specify the probabilistic early escalation trigger in TLA+? (TLA+ is non-probabilistic)
   - Recommendation: Document this as a "probabilistic refinement beyond TLA+" in formal/spec/mcp-calls/. Early escalation is an optimization that preserves EventualConsensus but is not provable in non-probabilistic logic. Verify via empirical smoke tests instead.

3. **Atomic update rollback for maxDeliberation**
   - What we know: HEAL-02 updates machine.ts, qgsd.json, and regenerates specs
   - What's unclear: If spec generation fails after machine update, should we rollback the machine file to pre-update state?
   - Recommendation: YES. Wrap entire sequence in try/catch. If any step fails (spec gen, verification), restore all files via git checkout or backup. Ensures no partial updates.

4. **maxDeliberation in XState context vs machine default**
   - What we know: Machine has context.maxDeliberation = 7 as default. Quorum dispatch sends QUORUM_START event to set slotsAvailable, polledCount, etc.
   - What's unclear: Can QUORUM_START also pass a custom maxDeliberation? Or is it always read from machine context defaults?
   - Recommendation: Allow both. If event.maxDeliberation is present, use it. Otherwise, fall back to context.maxDeliberation. This enables per-run override (useful for testing) while preserving the machine-level default.
