## What Was Built

### bin/task-envelope.cjs (361 lines)

CLI tool for managing task envelope JSON sidecars. Implements four commands:

- **init**: Creates new envelope with research section
  - Required: `--phase`
  - Optional: `--objective`, `--constraints`, `--target-files`, `--confidence`, `--risk-level` (default: medium)
  - Validates risk_level enum (low|medium|high); invalid → stderr warning, fallback to medium
  - Atomic write: tmpPath + fs.renameSync prevents mid-write corruption

- **update**: Appends/updates a section while preserving others
  - Command: `update --section plan` (other sections can be added in future)
  - Required: `--phase`, `--plan-path`, `--key-decisions` (comma-separated)
  - Optional: `--wave-count`
  - Preserves all existing sections (research, etc.) when adding plan section

- **read**: Outputs envelope as JSON to stdout
  - Required: `--phase`
  - Returns valid envelope or exits 1 if not found

- **validate**: Returns {valid: bool, errors: string[]} JSON structure
  - Required: `--phase`
  - Returns validation result without exiting (graceful fail-open)

### Exported Functions

- **validateEnvelope(obj)** → {valid, errors}: Core validation logic for external use
  - Validates schema_version = "1"
  - Validates phase matches `v\d+\.\d+-\d{2}` regex
  - Validates risk_level ∈ {low, medium, high}
  - Returns list of all validation failures

- **ENVELOPE_SCHEMA**: Constant object documenting required fields and types

## Key Design Decisions

1. **Atomic Write Pattern**: tmpPath + fs.renameSync (not writeFile + rename)
   - Reason: POSIX atomic rename prevents partial JSON if process crashes
   - Same pattern as update-scoreboard.cjs and existing tooling

2. **Invalid risk_level Handling**: Warn but continue with fallback
   - Reason: Graceful degradation; envelope still written with valid data
   - Stderr warning informs executor; ENV-03 can parse fallback and emit log

3. **Envelope Update Strategy**: Read → merge → write atomically
   - Reason: Preserves all sections when updating partial data
   - Prevents loss of research section when planner calls update --section plan

4. **Phase Directory Discovery**: Glob matching with fallback
   ```javascript
   const match = dirs.find(d => d === phaseId || d.startsWith(phaseId + '-'))
   ```
   - Reason: Handles both exact phase ID (v0.18-03) and phase names (v0.18-03-task-envelope)
   - Aligns with existing phase directory naming convention

5. **Validation on Write**: Validate envelope before write, emit stderr on error
   - Reason: Early feedback to user; ensures only valid envelopes persist
   - Different from silent fallback in read: write must be valid, read can be lenient
