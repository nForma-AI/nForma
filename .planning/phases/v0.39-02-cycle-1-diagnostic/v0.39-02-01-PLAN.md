Create bin/model-state-extractor.cjs that imports `parseITFTrace` from `./parse-tlc-counterexample.cjs`. Exports two functions:

1. `extractFinalStates(traceJsonPath)` — Reads an ITF JSON trace file, returns the final state(s).
   - If trace has a `loopPoint` (lasso trace), return `states.slice(loopPoint)` (the cycle states).
   - If no loop point, return `[states[states.length - 1]]` (just the last state).
   - If states array is empty or missing, return `[]`.
   - Fail-open: wrap in try/catch, return `[]` on any parse error with a stderr warning.

2. `analyzeStateSpace(traceJsonPath)` — Reads an ITF JSON trace file, returns a state space analysis object:
   - `{ initial_state, final_states, all_states, state_count }` where:
     - `initial_state` is `states[0]` (or null if empty)
     - `final_states` is the result of extractFinalStates logic
     - `all_states` is a deduplicated array of unique states (use `JSON.stringify` for equality, preserve first occurrence order)
     - `state_count` is the raw `states.length`
   - Fail-open: return `{ initial_state: null, final_states: [], all_states: [], state_count: 0 }` on any error.

Create bin/model-state-extractor.test.cjs with 6+ tests using `require('node:test')` and `assert`:
- extractFinalStates returns last state for linear trace (no loop)
- extractFinalStates returns cycle states for lasso trace (loopPoint set)
- extractFinalStates returns empty array for empty trace
- extractFinalStates returns empty array on file read error (fail-open)
- analyzeStateSpace returns correct structure for 3-state trace
- analyzeStateSpace deduplicates repeated states in all_states

Use temporary ITF JSON files written to /tmp for test fixtures. Each fixture should be a valid ITF JSON structure: `{ "states": [...], "loop": N }`.

  <action>
Create bin/diagnostic-proposal-generator.cjs that exports `generateCorrectionProposals(stateDiff, bugContext)`.

Input: `stateDiff` is the output of `generateStateDiff()` from diagnostic-diff-generator.cjs (Phase 1). It has shape: `{ summary: { changed_fields, first_divergence_index, total_changes }, per_state_diffs: [{index, changes: [{key, oldValue, newValue}]}] }`. `bugContext` is a string (the bug description text).

Output: Array of proposal objects sorted by priority (ascending), each with:
```
{
  id: string,           // e.g., "PROP-SV-0", "PROP-INV-1", "PROP-TEMP-1"
  type: string,         // "add_state_variable" | "add_invariant" | "add_temporal_property"
  target: string,       // The specific element to add (field name, invariant name, property name)
  confidence: number,   // 0.0–1.0
  reasoning: string,    // Human-readable explanation of why this correction is needed
  example: string,      // TLA+/Alloy skeleton example showing what to add
  priority: number,     // 1=highest (state variables), 2=medium (invariants), 3=lower (temporal)
  effort: string        // "low" | "medium" | "high"
}
```

Three proposal categories (generate all that apply):

1. **add_state_variable** (priority 1, confidence 0.8, effort "low") — One per diverged field in `summary.changed_fields`. ID: `PROP-SV-{i}`. Target: the field name. Reasoning: "Model does not track variable '{field}' which is critical to bug behavior". Example: `VARIABLE {field} \\ Track this field across states`.

2. **add_invariant** (priority 2, confidence 0.7, effort "medium") — One proposal if `per_state_diffs` is non-empty AND `first_divergence_index` is not null. ID: `PROP-INV-1`. Target: `Inv_StateConstraint_{index}`. Reasoning: "Divergence first occurs at state {index}; add invariant constraining valid state transitions". Example: `Inv_{index} == state = {index} => <valid condition>`.

3. **add_temporal_property** (priority 3, confidence 0.6, effort "high") — One proposal if `changed_fields` is non-empty. ID: `PROP-TEMP-1`. Target: `FinalStateCorrectness`. Reasoning: "Ensure model's final state satisfies correct termination condition". Example: `Prop_FinalCorrect == <>[](field = bugValue)` using first changed field.

Return empty array if stateDiff has no changes (`per_state_diffs` empty and `changed_fields` empty).

Sort output by `priority` ascending (1 before 2 before 3).

Create bin/diagnostic-proposal-generator.test.cjs with 8+ tests using `require('node:test')` and `assert`:
- Returns empty array for identical traces (no changes)
- Generates add_state_variable proposals for diverged fields (one per field)
- Generates add_invariant proposal when per_state_diffs is non-empty
- Generates add_temporal_property proposal when changed_fields is non-empty
- Proposals are sorted by priority ascending
- Each proposal has all required fields (id, type, target, confidence, reasoning, example, priority, effort)
- Multiple changed fields produce multiple PROP-SV proposals with sequential IDs
- Handles empty bugContext string gracefully (reasoning still makes sense)

Test fixtures: Construct stateDiff objects inline (no file I/O needed — these are pure function tests).

IMPORTANT: This module does NOT import diagnostic-diff-generator.cjs — it consumes its output format as a data contract. Verify no import of diagnostic-diff-generator with grep after creation.
