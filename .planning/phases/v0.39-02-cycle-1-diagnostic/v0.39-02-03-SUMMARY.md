## Architecture & Integration

### Diagnostic Flow

```
refinement-loop.cjs (INCOMPLETE outcome)
  ↓
generateDiagnosticFeedback()
  ↓ (Stage 1-5: extract, compare, propose)
  ↓
onDiagnosticGenerated callback fires
  ↓
caller (model-driven-fix workflow) serializes diagnostic as JSON
  ↓
passes as --review-context to quorum-slot-dispatch.cjs
  ↓
buildModeAPrompt / buildModeBPrompt detect JSON
  ↓
formatDiagnosticForPrompt() creates "## Model Diagnostic Feedback" section
  ↓
reviewContext nullified (prevent double-render)
  ↓
quorum worker receives structured diagnostic in prompt
```

### Key Design Decisions

1. **Fail-open diagnostic generation**: Failures in diagnostic pipeline (missing files, parse errors, etc.) log warnings but do NOT stop refinement loop. This ensures robustness.

2. **JSON detection in review-context**: Optional feature — if review-context is valid diagnostic JSON, format and inject. Otherwise, treat as plain text. This preserves backward compatibility.

3. **Double-render prevention**: After formatDiagnosticForPrompt formats diagnostic, the original reviewContext is set to null. This ensures the JSON string doesn't appear verbatim in the prompt alongside the formatted section.

4. **Callback-based injection**: refinement-loop.cjs fires onDiagnosticGenerated callback with the diagnostic object. Caller can decide what to do (serialize for quorum, log, etc.).

5. **Backward compatibility**: Existing callers without bugTraceJsonPath continue to work identically. No breaking changes.
