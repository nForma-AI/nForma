Update bin/refinement-loop.cjs to add diagnostic feedback generation when the model is INCOMPLETE. This extends the existing `verifyBugReproduction` function.

**New imports at top of file (add after existing imports):**
```javascript
const { extractFinalStates } = require('./model-state-extractor.cjs');
const { generateStateDiff, formatDiffAsMarkdown } = require('./diagnostic-diff-generator.cjs');
const { generateCorrectionProposals } = require('./diagnostic-proposal-generator.cjs');
```

**New internal function `generateDiagnosticFeedback(checkerOutput, bugTraceJsonPath, bugContext)`:**
- Takes checker stdout/stderr output, path to bug trace ITF JSON, and bug context string.
- Stage 1: Write checker output to a temp ITF file (in /tmp) if it looks like JSON, then call `extractFinalStates(tempPath)` to get model's final states. If checker output is not parseable JSON, return null (fail-open).
- Stage 2: Call `extractFinalStates(bugTraceJsonPath)` to get bug trace's final states. If bugTraceJsonPath is null/undefined or file doesn't exist, return null (fail-open — diagnostic requires bug trace).
- Stage 3: Call `generateStateDiff(modelFinalStates, bugFinalStates)` to compare.
- Stage 4: Call `formatDiffAsMarkdown(diffResult)` for human-readable output.
- Stage 5: Call `generateCorrectionProposals(diffResult, bugContext)` for structured proposals.
- Return `{ mismatch_diff: markdownString, correction_proposals: proposalArray, trace_alignment: { model_state_count, bug_state_count, first_divergence_index, diverged_fields } }`.
- Wrap entire function in try/catch; on error return null with stderr warning. This is fail-open — diagnostic failure must NOT break the refinement loop.

**Update `verifyBugReproduction` options:**
- Add new optional parameters: `options.bugTraceJsonPath` (string, path to bug trace ITF JSON) and `options.onDiagnosticGenerated` (callback function receiving `(diagnostic, attemptNumber)`).

**Update the INCOMPLETE branch inside the iteration loop (around line ~192):**
After recording `iteration = { attempt, passed: true, summary }`:
```javascript
// Generate diagnostic feedback if bug trace available
if (options.bugTraceJsonPath && attempt < maxAttempts) {
  const diagnostic = generateDiagnosticFeedback(output, options.bugTraceJsonPath, bugContext);
  if (diagnostic) {
    iteration.diagnostic = diagnostic;
    if (options.onDiagnosticGenerated) {
      options.onDiagnosticGenerated(diagnostic, attempt);
    }
  }
}
```

**Update the return value for 'not_reproduced' status:**
Add `final_diagnostic` field pulling from the last iteration with a diagnostic:
```javascript
const lastDiagnostic = iterations.filter(i => i.diagnostic).pop();
return {
  status: 'not_reproduced',
  attempts: maxAttempts,
  model_path: modelPath,
  counterexample: null,
  final_diagnostic: lastDiagnostic ? lastDiagnostic.diagnostic : null,
  iterations
};
```

**Update `formatIterationFeedback` to include diagnostic summary:**
If `iteration.diagnostic` exists, append a line: `\n  Diagnostic: {N} correction proposals generated ({fields} diverged fields)`.

**Update module.exports:** Add `generateDiagnosticFeedback` to exports for direct testing.

**Update CLI arg parsing:** Add `--bug-trace-json` flag that maps to `options.bugTraceJsonPath`.

- Bug trace fixture: `{ "states": [{ "pc": "Init", "count": 0 }, { "pc": "Step", "count": 5 }, { "pc": "Done", "count": 10 }] }` — represents observed bug behavior.
- The mock checker output will be different ITF JSON: `{ "states": [{ "pc": "Init", "count": 0 }, { "pc": "Step", "count": 3 }, { "pc": "Done", "count": 6 }] }` — represents model's (wrong) state space.

**Tests:**

1. **Callback receives diagnostic with mismatch_diff as markdown string:** Call verifyBugReproduction with bugTraceJsonPath and an onDiagnosticGenerated callback. Assert callback is called. Assert `diagnostic.mismatch_diff` is a string containing "State Divergence Report".

2. **Callback receives diagnostic with correction_proposals array:** Assert `diagnostic.correction_proposals` is an array with at least one element. Assert first proposal has `type`, `target`, `confidence`, `reasoning`, `example` fields.

3. **Diagnostic proposals include add_state_variable for diverged fields:** Assert proposals array contains at least one proposal with `type: 'add_state_variable'`. Assert the target references a field name that actually differs between the two traces (e.g., "count").

4. **Diagnostic context includes trace_alignment metadata:** Assert `diagnostic.trace_alignment` has `model_state_count`, `bug_state_count`, `first_divergence_index`, and `diverged_fields`. Assert `diverged_fields` is a non-empty array.

5. **Iteration history preserves diagnostic across attempts:** Run with maxAttempts=2, mock checker always passes. Assert `result.iterations[0].diagnostic` exists. Assert `result.final_diagnostic` is not null. Assert `result.final_diagnostic.correction_proposals.length > 0`.

6. **Backward compat: no diagnostic when bugTraceJsonPath not provided:** Call verifyBugReproduction WITHOUT bugTraceJsonPath. Assert no iteration has a `diagnostic` property. Assert `result.final_diagnostic` is null or undefined.

Clean up temp files after each test.

IMPORTANT: These tests validate the contract that quorum workers would consume. The diagnostic object shape (mismatch_diff string, correction_proposals array, trace_alignment object) is the interface between refinement-loop and quorum injection. If this shape changes, quorum workers break.
