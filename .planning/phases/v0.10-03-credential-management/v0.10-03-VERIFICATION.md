The following human tests must be confirmed with a real TTY:

#### 1. Batch Rotate Keys — Sequential Confirmation (CRED-01)

**Test:** Run `node bin/manage-agents.cjs`; select "11. Batch rotate keys".
**Expected:** Checkbox picker shows all configured slots with current key status ([key set]/[no key]). After selecting 2+ slots and pressing Enter, each slot is prompted sequentially: first slot's password prompt appears, after entry the confirmation "slot-name: key updated" prints, THEN the next slot's prompt appears. After all slots: "✓ N slot(s) updated" confirmation.
**Why human:** TTY-interactive inquirer checkbox + password prompts cannot be driven programmatically without a TTY driver.

#### 2. Menu Item 11 Visual (CRED-01)

**Test:** Run `node bin/manage-agents.cjs` and observe the main menu.
**Expected:** After v0.10-07-02 menu fix, menu shows: "11. Batch rotate keys", separator, "12. Live health dashboard", separator, "13. Update coding agents", separator, "14. Tune timeouts", "15. Set update policy". No gap between 11 and 12.
**Why human:** Visual menu rendering requires TTY.

#### 3. Single syncToClaudeJson (CRED-01)

**Test:** With 3 slots configured, run batch rotate, update all 3. Observe timing of file write (single final sync vs. per-slot sync).
**Expected:** A single `~/.claude.json` file write occurs after the entire batch loop completes, not after each individual slot.
**Why human:** File I/O timing requires live execution; multi-slot batch required to distinguish per-slot vs. single-sync behavior.

#### 4. key_status Persistence Across Restart (CRED-02)

**Test:** (1) Run "7. Check agent health" on a slot with an intentionally invalid key (expect 401 probe result). (2) Exit CLI. (3) Restart: `node bin/manage-agents.cjs`. (4) Select "1. List agents".
**Expected:** The slot that had a 401 probe shows "[key invalid]" badge without requiring a new probe. Check `cat ~/.claude/qgsd.json | node -e "const d=JSON.parse(require('fs').readFileSync('/dev/stdin','utf8')); console.log(JSON.stringify(d.agent_config, null, 2));"` to confirm `key_status.status === 'invalid'` is persisted.
**Why human:** Requires live keytar + real ~/.claude/qgsd.json state across process restart; 401 probe requires an actual invalid API key.
