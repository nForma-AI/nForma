  <name>Task 1: Implement cmdMaintainTestsDiscover in gsd-tools.cjs</name>
  <files>get-shit-done/bin/gsd-tools.cjs</files>
  <action>
Add a `cmdMaintainTestsDiscover(cwd, options, raw)` function and wire it into the command dispatch switch under `maintain-tests` → `discover`.

**Framework detection logic (config-file-first, never glob):**

Detection order per framework — check these paths relative to `cwd`:
- Jest: `jest.config.js`, `jest.config.cjs`, `jest.config.mjs`, `jest.config.ts`, `jest.config.json`. Also check `package.json` for a `"jest"` key or `devDependencies`/`dependencies` containing `"jest"`.
- Playwright: `playwright.config.js`, `playwright.config.cjs`, `playwright.config.mjs`, `playwright.config.ts`.
- Pytest: `pytest.ini`, `setup.cfg` (with `[tool:pytest]` section), `pyproject.toml` (with `[tool.pytest.ini_options]` section).

If `--runner jest|playwright|pytest` flag is passed, skip detection and use only that runner.

**CLI invocation per framework (use spawnSync, inherit cwd, capture stdout):**

- Jest: `npx jest --listTests` — output is newline-delimited absolute file paths. Split on `\n`, filter empty lines. If `npx` is unavailable, try `./node_modules/.bin/jest --listTests`.
- Playwright: `npx playwright test --list` — output is human-readable. Parse lines matching pattern `/\s+\S+\.spec\.(ts|js|tsx|jsx)/` to extract file paths. Resolve relative paths against cwd.
- Pytest: `python -m pytest --collect-only -q` — output has format `path/to/test.py::test_name`. Extract unique file paths by splitting on `::` and taking index 0. Filter lines starting with `ERRORS` or `=`.

**Deduplication:** After collecting paths from all detected frameworks, deduplicate by normalized absolute path. A file appearing in both jest and playwright output (e.g., a `.spec.ts` file in a project with both configs) is included once, attributed to the framework that listed it first.

**Output format (JSON to stdout or --output-file):**
```json
{
  "runners": ["jest", "playwright"],
  "test_files": ["/abs/path/to/test1.test.js", "/abs/path/to/test2.spec.ts"],
  "total_count": 2,
  "by_runner": {
    "jest": ["/abs/path/to/test1.test.js"],
    "playwright": ["/abs/path/to/test2.spec.ts"]
  }
}
```

**Flags:**
- `--runner auto|jest|playwright|pytest` (default: auto)
- `--dir <path>` — search for config files starting from this dir instead of cwd
- `--output-file <path>` — write JSON to file instead of stdout

**Error handling:** If a runner is detected but its CLI invocation fails (non-zero exit or empty output), include a `warnings` array in output with the runner name and error. Do not fail the whole command — return results from other runners. If NO runners are detected, return `{ "runners": [], "test_files": [], "total_count": 0 }` with a warning.

**Implementation placement:** Add `cmdMaintainTestsDiscover` just before the main() call, after the activity commands section. Add the switch case in the `maintain-tests` top-level case. The `maintain-tests` top-level case should be a new case that parses the sub-command from `args[1]` and dispatches to sub-command handlers.

**CRITICAL: Use spawnSync for CLI invocations, NOT execSync.** The existing pattern in qgsd-circuit-breaker.js uses spawnSync — follow that pattern. Do NOT use execSync with shell: true (shell injection risk). Do NOT use the Jest programmatic API (runCLI) — it breaks across major Jest versions.
