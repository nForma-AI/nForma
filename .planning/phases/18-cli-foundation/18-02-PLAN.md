**Seeded shuffle — implement Mulberry32 inline (no external dependency):**

```javascript
function mulberry32(seed) {
  return function() {
    seed |= 0; seed = seed + 0x6D2B79F5 | 0;
    let t = Math.imul(seed ^ seed >>> 15, 1 | seed);
    t = t + Math.imul(t ^ t >>> 7, 61 | t) ^ t;
    return ((t ^ t >>> 14) >>> 0) / 4294967296;
  };
}

function seededShuffle(arr, seed) {
  const rng = mulberry32(seed);
  const result = arr.slice();
  for (let i = result.length - 1; i > 0; i--) {
    const j = Math.floor(rng() * (i + 1));
    [result[i], result[j]] = [result[j], result[i]];
  }
  return result;
}
```

This is a standard Fisher-Yates shuffle with a deterministic PRNG. The same seed + same input array always produces the same output array.

**Batch generation:**

1. Read test_files from input
2. If `--exclude-file <path>` is provided, read a JSON array of already-processed file paths and filter them from the input before shuffling (enables resume: only unprocessed files are batched)
3. Apply seededShuffle(testFiles, seed)
4. Split shuffled array into chunks of `--size N` (default 100)
5. Assign batch_id = 1-based index to each chunk

**Output format (JSON to stdout or --output-file / --manifest-file):**
```json
{
  "seed": 42,
  "batch_size": 100,
  "total_files": 500,
  "total_batches": 5,
  "batches": [
    {
      "batch_id": 1,
      "files": ["/abs/path/test1.js", "/abs/path/test2.js"],
      "file_count": 100
    },
    {
      "batch_id": 2,
      "files": ["..."],
      "file_count": 100
    }
  ]
}
```

**Flags:**
- `--input-file <path>` — JSON file from discover output
- `--input-json <json>` — inline JSON string (alternative to --input-file)
- `--size N` — batch size (default 100, configurable)
- `--seed N` — PRNG seed (default: Date.now() % 2147483647, printed in output for reproducibility)
- `--exclude-file <path>` — JSON array of already-processed file paths to skip
- `--output-file <path>` — write manifest to file instead of stdout
- `--manifest-file <path>` — alias for --output-file (more semantic name for this use case)

**CRITICAL: Write manifest to disk before returning.** If both --manifest-file and --output-file are absent, write to stdout only. If either is present, write to disk AND to stdout (so the caller can read it either way). The manifest must be written BEFORE any caller can proceed to run-batch.

**Config integration:** Read batch_size from `config.maintain_tests.batch_size` if present in `.claude/qgsd.json`, with `--size` flag overriding config.

**Implementation placement:** Add `cmdMaintainTestsBatch` after `cmdMaintainTestsDiscover`. Add `seededShuffle` and `mulberry32` as module-level functions above `cmdMaintainTestsBatch`. Add `batch` case to the maintain-tests switch.
