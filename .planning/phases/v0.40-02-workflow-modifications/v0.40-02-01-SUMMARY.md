## Decisions Made

- **Haiku model choice:** Haiku is fast and cheap for intent extraction; parsing intent is non-trivial for hand-rolled patterns
- **Scope contract format:** JSON with branch name as key prevents concurrent task collision (one global file, per-branch entries)
- **Step positioning:** Step 2.7 must execute synchronously between branching and planner spawn so branch name is known and contract exists before execution
- **Fail-open on all errors:** Haiku unavailability, JSON parse failure, file write failure all produce warnings but allow quick task to proceed (scope guard is v0.40-03, not yet blocking)
- **Non-modal approach:** INTENT-03 requires automatic derivation, no user confirmation needed
- **APPROACH_BLOCK injection:** Planner context includes approach so planner plans within declared scope
