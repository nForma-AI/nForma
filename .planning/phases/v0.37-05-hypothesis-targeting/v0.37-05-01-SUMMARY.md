## Results

- hypothesis-layer-map.cjs: keyword-based mapping of TLA+ model filenames to DAG layer keys
- Transition detection: only UNMEASURABLE -> CONFIRMED/VIOLATED transitions count
- Weight computation: each transition adds +1 to mapped layers (additive for multiple transitions)
- solve-wave-dag.cjs: intra-wave sort by priority weight descending, alphabetical tiebreaker
- Fully backward compatible (no weights = identical behavior to before)
- All 38 tests pass (22 + 16)

## Decisions

- Used keyword-based mapping (not TLA+ property analysis) to keep the module simple and avoid parsing formal spec files
- Priority weights only affect intra-wave ordering, never cross-wave topology (dependency ordering is absolute)
- Sequential chain compaction preserves dependency order within compacted waves regardless of priority weights
