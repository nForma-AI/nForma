  <name>Task 1: Add review-only tool restriction to quorum-slot-dispatch.cjs</name>
  <files>bin/quorum-slot-dispatch.cjs, bin/quorum-slot-dispatch.test.cjs</files>
  <action>
Modify bin/quorum-slot-dispatch.cjs to restrict tool access for review-only dispatches:

1. **Add --review-only arg parsing** at the top with existing arg parsing (around line 57):
   ```javascript
   const reviewOnly = hasFlag('--review-only');
   ```

2. **Determine review mode** in main() before prompt construction:
   ```javascript
   const isReviewMode = mode === 'B' || reviewOnly;
   ```

3. **Modify the spawn call to call-quorum-slot.cjs** (around line 955) to pass --allowedTools when review mode is active and the slot is a ccr-based slot. To detect ccr slots, read providers.json to check if the slot's `display_type` is `claude-code-router` or if its `cli` contains `ccr`. Add the logic:
   ```javascript
   // Load provider info to check if ccr-based
   const providers = (() => {
     try {
       const pPath = path.join(__dirname, 'providers.json');
       return JSON.parse(fs.readFileSync(pPath, 'utf8')).providers || [];
     } catch { return []; }
   })();
   const provider = providers.find(p => p.name === slot);
   const isCcrSlot = provider && (provider.display_type === 'claude-code-router' || (provider.cli && provider.cli.includes('ccr')));
   ```

   If isReviewMode and isCcrSlot, add `'--allowedTools', 'Read,Grep,Glob'` to the spawn args array before the slot arg. Specifically, insert them into the args passed to call-quorum-slot.cjs.

   HOWEVER: note that call-quorum-slot.cjs does not forward --allowedTools to the subprocess. The tool restriction needs to be applied at the ccr subprocess level. Since quorum-slot-dispatch.cjs builds the prompt and passes it to call-quorum-slot.cjs which then spawns the provider CLI, the --allowedTools flag must be injected into the provider's args_template dynamically.

   Better approach: Instead of modifying call-quorum-slot.cjs, modify the prompt sent to call-quorum-slot.cjs. For ccr slots, prepend `--allowedTools Read,Grep,Glob\n` as a special directive line that call-quorum-slot.cjs can extract and add to the spawn args. OR simpler: modify quorum-slot-dispatch.cjs to spawn call-quorum-slot.cjs with an additional `--allowed-tools` flag that call-quorum-slot.cjs forwards to the provider subprocess.

   SIMPLEST approach matching the research recommendation: Modify the prompt for ALL review-mode dispatches:
   - For ccr slots: prepend a machine-readable directive `[REVIEW-ONLY: --allowedTools Read,Grep,Glob]` that the slot worker agent can parse (ccr slots run through the agent which respects tool restrictions)
   - For non-ccr slots: prepend `REVIEW-ONLY MODE: Do NOT modify any files. Use only read operations (read files, search code, list files). Do NOT use Write, Edit, or destructive Bash commands.\n\n`

   Actually the simplest correct approach: modify the prompt construction functions (buildModeAPrompt/buildModeBPrompt are already deterministic). Add a `reviewOnly` option to buildModeBPrompt that prepends the restriction text. For ccr slots, the restriction is structural (the ccr CLI's --allowedTools is set at dispatch level); for non-ccr, it is prompt-level.

   Since call-quorum-slot.cjs is the one that spawns the provider CLI with args_template, the cleanest integration is:
   - Add `--allowed-tools` arg to call-quorum-slot.cjs that injects into subprocess args for ccr slots
   - Pass this from quorum-slot-dispatch.cjs when isReviewMode

   But this creates a dependency on Plan 02 files. Since Plan 02 also modifies call-quorum-slot.cjs, and the wave is 1 (parallel), we should NOT modify call-quorum-slot.cjs here.

   FINAL APPROACH (no call-quorum-slot.cjs changes):
   - For ALL review-mode dispatches, prepend the review-only instruction to the prompt text.
   - For ccr slots specifically, ALSO add `--allowedTools Read,Grep,Glob` to the spawn args when spawning call-quorum-slot.cjs. call-quorum-slot.cjs ignores unknown args (getArg returns null for unknown flags), so this is safe. The ccr subprocess args_template includes `{prompt}` which is substituted. The `--allowedTools` needs to go into the provider's args, not call-quorum-slot's args.

   Since we cannot easily inject --allowedTools into the provider subprocess args from quorum-slot-dispatch.cjs without modifying call-quorum-slot.cjs, use prompt-level restriction for ALL slots as the primary mechanism, with a comment documenting that ccr structural enforcement requires a future call-quorum-slot.cjs change (or accept prompt-level as sufficient for v0.31-02 given the research recommendation that prompt-level is best-effort for non-ccr and "sufficient" given quorum vote catches unauthorized modifications).

   Implementation:
   - In buildModeBPrompt, when reviewOnly option is true, prepend after the header: `IMPORTANT: This is a READ-ONLY review task. You MUST NOT modify any files. Only use read operations (Read, Grep, Glob tools). Do NOT use Write, Edit, Bash(write), or any destructive commands.`
   - In main(), pass `reviewOnly: isReviewMode` to buildModeBPrompt.

4. **Extend tests in bin/quorum-slot-dispatch.test.cjs**:
   - Test that buildModeBPrompt with reviewOnly=true includes the REVIEW-ONLY restriction text
   - Test that buildModeBPrompt with reviewOnly=false (or undefined) does NOT include the restriction text
   - Test that Mode A prompts do NOT include the restriction text regardless of reviewOnly flag
