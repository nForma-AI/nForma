<objective>
Fix the TUI "Add Agent -> CLI Agent" flow to resolve CLI binary paths before writing MCP entries, matching the format mcp-setup produces.

Purpose: GitHub issue #7 — TUI writes bare CLI name (e.g. "codex") to mcpServers[].command instead of the resolved absolute path (e.g. "/opt/homebrew/bin/codex"). This breaks on Linux/WSL and diverges from what mcp-setup generates. The fix uses resolveCli() from resolve-cli.cjs (already production-tested in v0.35-03) to resolve the path at form submission time and validates the binary is executable before writing.

Output: Modified nForma.cjs with path resolution and validation in the CLI Agent form handler, plus unit tests proving resolution, validation, and format parity.
</objective>

  <action>
Wire `resolveCli` from `bin/resolve-cli.cjs` into the TUI's "Add Agent -> CLI Agent" form handler so the resolved absolute path is written to `mcpServers[slotName].command` instead of the bare CLI name.

1. **Import resolveCli** — After the existing requires at the top of nForma.cjs (line ~11, after `const NF_VERSION = ...`), add:
   ```javascript
   const { resolveCli } = require('./resolve-cli.cjs');
   ```

2. **Resolve CLI path after user input** — In the CLI Agent handler (line ~1691, after `if (!command) { toast(...); return; }`), add path resolution and validation:
   ```javascript
   // Resolve CLI path (TUI-01: cross-platform path resolution)
   const resolvedCommand = resolveCli(command);

   // Validate the resolved path is executable (TUI-01: prevent broken entries)
   try {
     fs.accessSync(resolvedCommand, fs.constants.X_OK);
   } catch (err) {
     toast(`CLI not found or not executable: ${resolvedCommand}`, true);
     return;
   }
   ```

3. **Write resolved path to MCP entry** — Change line ~1696 from:
   ```javascript
   data.mcpServers = { ...servers, [slotName]: { type: 'stdio', command, args: [] } };
   ```
   to:
   ```javascript
   data.mcpServers = { ...servers, [slotName]: { type: 'stdio', command: resolvedCommand, args: [] } };
   ```
   This ensures the MCP entry format matches what mcp-setup produces: `type='stdio'`, resolved absolute command path, `args=[]`.

4. **Update toast message** — Change line ~1709 from:
   ```javascript
   toast(`✓ Added CLI agent "${slotName}"`);
   ```
   to:
   ```javascript
   toast(`✓ Added CLI agent "${slotName}" at ${resolvedCommand}`);
   ```
   This gives the user immediate feedback showing the resolved path.

5. **Do NOT modify the providers.json metadata write** (lines 1700-1707) — that section is correct as-is.

6. **Do NOT change the API agent handler** (lines 1714+) — it correctly uses `command: 'node'` which needs no resolution.
