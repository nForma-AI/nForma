# Phase v0.35-04 Plan 01: Fix TUI CLI Agent Path Resolution Summary

**One-liner:** TUI "Add Agent -> CLI Agent" now resolves bare CLI names to absolute paths via resolveCli() and validates executability before writing MCP entries, ensuring cross-platform compatibility and format parity with mcp-setup.

## Objective Met

Fixed GitHub issue #7 — TUI was writing bare CLI names (e.g. "codex") to mcpServers[].command instead of resolved absolute paths (e.g. "/opt/homebrew/bin/codex"). This broke on Linux/WSL and diverged from mcp-setup's format. The fix integrates resolveCli() from resolve-cli.cjs (v0.35-03 production component) into the CLI Agent form handler with executable validation.

## Technical Details

### Path Resolution Strategy (from resolve-cli.cjs)
1. `which <name>` — system PATH lookup
2. Homebrew prefixes — /opt/homebrew/bin, /usr/local/bin
3. npm global bin — derived from `npm root -g`
4. Common system paths — /usr/bin
5. Fallback — return bare name (let OS resolve at spawn time, Rule 5 fail-open)

### Validation Gate
Before writing MCP entry, the resolved path is tested with `fs.accessSync(resolvedCommand, fs.constants.X_OK)`. Non-executable paths are rejected with user-visible error toast, preventing broken configurations.

### MCP Entry Format (TUI output)
```javascript
{
  type: 'stdio',
  command: '/opt/homebrew/bin/codex',  // resolved absolute path
  args: []
}
```
This now matches exactly what mcp-setup generates.
