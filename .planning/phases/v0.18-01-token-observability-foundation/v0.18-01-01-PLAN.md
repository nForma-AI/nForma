**hooks/qgsd-token-collector.js** (SubagentStop hook):

'use strict'; require fs, path. Read stdin, parse JSON. Guard: if input.agent_type !== 'qgsd-quorum-slot-worker', process.exit(0).

resolveSlot(input): Try correlation file at path.join(process.cwd(), '.planning', 'quorum-slot-corr-' + input.agent_id + '.json'). If exists: parse, delete with fs.unlinkSync (ignore errors), return data.slot if set. Fallback: parse "slot: <name>" from the first line matching /^slot:\s*(\S+)/m in input.last_assistant_message. Final fallback: return input.agent_id || 'unknown'.

appendRecord(input, inputTokens, outputTokens, cacheCreate, cacheRead): build record object {ts, session_id, agent_id, slot, input_tokens, output_tokens, cache_creation_input_tokens, cache_read_input_tokens}. Write to path.join(process.cwd(), '.planning', 'token-usage.jsonl') via fs.mkdirSync (recursive) then fs.appendFileSync. Wrap in try/catch that does nothing — observational, never fails.

Main flow:
1. If agent_transcript_path absent or file does not exist: call appendRecord(input, null, null, null, null) and exit 0.
2. Read transcript file. Split by '\n'. Filter empty lines. For each line: JSON.parse, skip if type !== 'assistant', skip if isSidechain === true, skip if isApiErrorMessage === true, skip if no message.usage. Accumulate input_tokens, output_tokens, cache_creation_input_tokens, cache_read_input_tokens (use || 0 for absent fields).
3. Call appendRecord with accumulated totals.
4. process.exit(0).
5. Outer try/catch: process.exit(0) on any error (fail-open).

IMPORTANT: Never write to stdout. Only stderr for debugging (optional). The hook must not block the subagent stop event — async:true is set in settings.json by install.js (Plan 02), not in this file.

'use strict'; require fs, path. Read stdin, parse JSON. Guard: if input.agent_type !== 'qgsd-quorum-slot-worker', exit 0. Guard: if !input.agent_id, exit 0.

Write correlation file: path.join(process.cwd(), '.planning', 'quorum-slot-corr-' + input.agent_id + '.json') with content { agent_id: input.agent_id, ts: new Date().toISOString(), slot: null }. Note: slot is null because SubagentStart does NOT include the prompt text (confirmed in research). The token collector resolves slot from last_assistant_message.

fs.mkdirSync for the parent dir (recursive), then fs.writeFileSync. Wrap in try/catch that exits 0 (fail-open). process.exit(0).

**Part A: Extend bin/call-quorum-slot.cjs with appendTokenSentinel()**

Add a function near the top of the file (after the failure log function, before the Args section):

```javascript
// ─── Token sentinel for CLI slots (OBSV-04) ───────────────────────────────────
function appendTokenSentinel(slotName) {
  try {
    const record = JSON.stringify({
      ts: new Date().toISOString(),
      session_id: null,
      agent_id: null,
      slot: slotName,
      input_tokens: null,
      output_tokens: null,
      cache_creation_input_tokens: null,
      cache_read_input_tokens: null,
    });
    const logPath = path.join(findProjectRoot(), '.planning', 'token-usage.jsonl');
    fs.mkdirSync(path.dirname(logPath), { recursive: true });
    fs.appendFileSync(logPath, record + '\n', 'utf8');
  } catch (_) {} // observational — never fails
}
```

Call appendTokenSentinel() in the main() function after the result is obtained (successful dispatch) AND after the catch block (failed dispatch). Both cases must log a sentinel for CLI slots so OBSV-04 is satisfied regardless of success/failure.

Specifically, modify main() to call appendTokenSentinel(slot) in three places:
1. After `process.stdout.write(result)` (success path) — before `process.exit(0)`
2. In the catch block after `writeFailureLog(slot, ...)` — before `process.exit(1)`
3. If provider type is unknown — before `process.exit(1)`

This ensures every CLI slot call (subprocess or http type) produces a sentinel record.

Note: appendTokenSentinel only runs for providers of type 'subprocess' or 'http' — all current providers are one of these types. The sentinel is safe to write for HTTP providers too (they also lack SubagentStop events since they are called from within call-quorum-slot.cjs, not as Claude Code Tasks).
