# v0.18-01 Plan 02 Summary — Hook Registration + Health Display

## What Was Built

**Task 1: Register SubagentStop and SubagentStart hooks in install.js**

Added two idempotent hook registrations to `bin/install.js` immediately after the PostToolUse context monitor hook registration (lines 1863–1888), following the exact pattern of existing registrations:

- **SubagentStop** (qgsd-token-collector): `matcher: 'qgsd-quorum-slot-worker'`, `async: true`
- **SubagentStart** (qgsd-slot-correlator): `matcher: 'qgsd-quorum-slot-worker'`, `async: true`

Both use idempotency checks via `.includes('qgsd-token-collector')` and `.includes('qgsd-slot-correlator')` to prevent duplicate registrations.

**Task 2: Token display in health.md + install sync**

Added `<step name="display_token_usage">` step to `qgsd-core/workflows/health.md` between `format_output` and `offer_repair`. The step runs an inline Node.js script that:
- Reads `.planning/token-usage.jsonl` (last 100 records file size guard)
- Aggregates token totals by slot
- Sorts by total tokens descending
- Displays a formatted table with slot, input, output, rounds columns
- Shows "null (CLI)" for CLI slots that have null token counts
- Handles absent file gracefully with "No token data yet."

Ran `node bin/install.js --claude --global` to register hooks in `~/.claude/settings.json` and install hook files to `~/.claude/hooks/`.
