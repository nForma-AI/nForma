## Summary

Phase v0.34-06 validates the complete semantic scoring → gate enrichment → auto-promotion pipeline end-to-end. This phase does NOT implement new functionality; instead, it creates integration tests that verify three prior phases work together correctly: v0.34-04 (semantic scoring pipeline wiring), v0.34-05 (auto-promotion state initialization), and their dependencies.

The phase validates:
1. **SEM-03, SEM-04**: compute-semantic-scores.cjs integrated into nf-solve.cjs pipeline and gate files enriched with semantic_score fields
2. **PROMO-02, PROMO-03, PROMO-04**: Auto-promotion eligibility correctly determined (3 consecutive clean sessions), promotions logged with metadata

**Primary recommendation:** Create E2E test file (`test/e2e-semantic-promotion-pipeline.test.cjs`) that:
- Runs compute-semantic-scores.cjs within nf-solve.cjs pipeline simulation
- Simulates 3 consecutive clean session cycles
- Verifies gate files contain semantic_score after each iteration
- Verifies auto-promotion fires at 3rd clean session
- Verifies promotion-changelog.json contains correct metadata

## Architecture Patterns

### Pattern 1: Multi-Cycle E2E Simulation

**What:** Create a temporary test fixture mimicking `.planning/formal/` directory structure, then simulate multiple nf-solve.cjs iterations. Each iteration should:
- Read current gate state (wiring scores)
- Simulate semantic scoring computation (update gate files with semantic_score)
- Check if session is "clean" (wiring >= 1.0 AND semantic >= 0.8 AND formal pass)
- Increment consecutive_clean_sessions if clean, reset if not
- Save state back to solve-state.json
- Attempt auto-promotion if consecutive_clean_sessions >= 3
- Verify expected state changes

**Example Test Structure:**
```javascript
// Source: Pattern from test/auto-promotion-integration.test.cjs (existing)
test('E2E: semantic-scoring -> gate enrichment -> auto-promotion', () => {
  const tmpDir = createFixtureDir({ /* initial state */ });

  // Iteration 1: Clean session, consecutive_clean_sessions -> 1
  simulateSolveIteration(tmpDir, { semantic: { A: 0.85, B: 0.9, C: 0.88 } });
  const state1 = loadSolveState(tmpDir);
  assert.equal(state1.consecutive_clean_sessions, 1);

  // Iteration 2: Clean session, consecutive_clean_sessions -> 2
  simulateSolveIteration(tmpDir, { semantic: { A: 0.85, B: 0.9, C: 0.88 } });
  const state2 = loadSolveState(tmpDir);
  assert.equal(state2.consecutive_clean_sessions, 2);

  // Iteration 3: Clean session, auto-promotion fires, consecutive_clean_sessions -> 3
  simulateSolveIteration(tmpDir, { semantic: { A: 0.85, B: 0.9, C: 0.88 } });
  const state3 = loadSolveState(tmpDir);
  assert.equal(state3.consecutive_clean_sessions, 3);

  // Verify promotion occurred
  const changelog = JSON.parse(fs.readFileSync(path.join(tmpDir, 'promotion-changelog.json'), 'utf8'));
  assert.ok(changelog.length > 0);
  assert.equal(changelog[0].trigger, 'auto_promotion');
  assert.equal(changelog[0].consecutive_clean_sessions, 3);
});
```

**When to use:** Validates full pipeline correctness across session boundaries.

### Pattern 2: State Persistence Across Iterations

**What:** Between iterations, verify state is correctly persisted and reloaded. This tests solve-state.json handling and checkCleanSession() behavior.

**Current Implementation** (nf-solve.cjs):
```javascript
// Read existing state (may not exist on first run)
let existingSolveState = {};
try {
  existingSolveState = JSON.parse(fs.readFileSync(solveStatePath, 'utf8'));
} catch (_) { /* first run */ }

// Increment or reset based on session cleanliness
const isCleanSession = checkCleanSession();
const prevClean = (existingSolveState && existingSolveState.consecutive_clean_sessions) || 0;
solveState.consecutive_clean_sessions = isCleanSession ? prevClean + 1 : 0;

// Persist
fs.writeFileSync(solveStatePath, JSON.stringify(solveState, null, 2) + '\n');
```

**Test Pattern:**
- Create initial state with `consecutive_clean_sessions: 0`
- Run iteration 1 (clean session) → state reloaded, counter incremented
- Verify reloaded state has counter = 1
- Run iteration 2 (clean session) → state reloaded with counter = 1, incremented to 2
- Verify persistence chain: 0 → 1 → 2 → 3

**When to use:** Ensures counter doesn't reset unexpectedly between iterations.

### Pattern 3: Semantic Score Enrichment Verification

**What:** Verify compute-semantic-scores.cjs is actually called and gate files are enriched with semantic_score fields. This is a critical integration point.

**Verification Steps:**
1. Create gate files with wiring_*_score fields (no semantic_score)
2. Call compute-semantic-scores.cjs (or simulate its effect)
3. Read gate files and verify semantic_score field exists and is numeric
4. Verify schema_version is '3' when semantic_score is present

**Test Pattern:**
```javascript
// Before compute-semantic-scores
const gateA = JSON.parse(fs.readFileSync(path.join(gatesDir, 'gate-a-grounding.json'), 'utf8'));
assert.equal(gateA.semantic_score, undefined, 'semantic_score should not exist yet');

// Simulate compute-semantic-scores enrichment
enrichGateWithSemanticScore(gatesDir, 'gate-a-grounding.json', 0.85);

// After enrichment
const gateAEnriched = JSON.parse(fs.readFileSync(path.join(gatesDir, 'gate-a-grounding.json'), 'utf8'));
assert.equal(gateAEnriched.semantic_score, 0.85, 'semantic_score should be enriched');
assert.equal(gateAEnriched.schema_version, '3', 'schema version should bump to v3');
```

**When to use:** Validates SEM-03 and SEM-04 requirements.

### Anti-Patterns to Avoid

- **Don't:** Test semantic scoring algorithm (that's v0.34-02's responsibility). **DO:** Test that gate files carry the fields.
- **Don't:** Create real candidates.json and run Haiku evaluation. **DO:** Mock gate enrichment or skip semantic scoring simulation.
- **Don't:** Test promote-gate-maturity.cjs logic (already tested in v0.34-05). **DO:** Test that auto-promotion fires when consecutive_clean_sessions >= 3.
- **Don't:** Fail test if semantic scoring step fails. **DO:** Test fail-open behavior (gates still valid without semantic_score).

## Common Pitfalls

### Pitfall 1: Semantic Score Fields Lost on Iteration 2

**What goes wrong:** Iteration 1 enriches gate files with semantic_score. Iteration 2 calls compute-per-model-gates.cjs which overwrites gates WITHOUT preserving semantic_score. On iteration 3, semantic_score is missing and checkCleanSession() fails.

**Why it happens:** v0.34-04 requires compute-per-model-gates.cjs to preserve semantic_score (read-preserve-write pattern). If that implementation is incomplete, test exposes the bug.

**How to avoid:**
- Verify compute-per-model-gates.cjs reads existing gate file before writing (line ~365)
- Test with multiple iterations and inspect gate files after iteration 2
- Assert that semantic_score persists across gate file rewrites

**Warning signs:**
- Iteration 1 gate files have semantic_score, iteration 2 have none
- Test passes with 1 iteration, fails with 2+
- `git diff .planning/formal/gates/` shows semantic_score disappearing

### Pitfall 2: Convergence Loop Never Calls compute-semantic-scores

**What goes wrong:** nf-solve.cjs integrates the compute-semantic-scores call (v0.34-04), but integration point is wrong (conditional that doesn't execute, wrong loop level). Gate files never enriched. Auto-promotion threshold fails. All sessions marked non-clean.

**Why it happens:** nf-solve.cjs is complex; easy to place call in wrong location or inside skipped conditional (e.g., `if (fastMode)` when test runs with `--fast`).

**How to avoid:**
- Verify compute-semantic-scores.cjs is called AFTER sweepPerModelGates() but BEFORE auto-promotion check
- Test logs should show "Step 7: Enriching gates with semantic scores" (or equivalent) before promotion attempt
- Verify gate files have semantic_score field BEFORE checkCleanSession() is called

**Warning signs:**
- Gate files have schema_version: '2' (should be '3' after v0.34-04)
- All sessions marked non-clean despite wiring >= 1.0 and formal pass
- checkCleanSession() output shows "semantic=[A:0, B:0, C:0]" (defaulted to 0)

### Pitfall 3: Auto-Promotion Eligibility Check Skipped

**What goes wrong:** nf-solve.cjs calls promote-gate-maturity.cjs but does NOT pass `consecutive_clean_sessions` or `--check --fix` flags. Promotion eligibility logic in promote-gate-maturity.cjs is never invoked. Test expects promotions but none occur.

**Why it happens:** v0.34-05 adds auto-promotion eligibility logic. If nf-solve.cjs doesn't wire the call correctly (missing parameters), the feature is silently disabled.

**How to avoid:**
- Verify nf-solve.cjs calls promote-gate-maturity.cjs with correct arguments after 3 clean sessions
- Check promote-gate-maturity.cjs receives consecutive_clean_sessions value from solve-state.json
- Test should simulate exactly this: run 3 clean iterations, then verify promote-gate-maturity.cjs eligibility check

**Warning signs:**
- consecutive_clean_sessions increments to 3 but no promotions occur
- promotion-changelog.json stays empty despite clean sessions
- promote-gate-maturity.cjs never receives auto-promotion eligibility check

### Pitfall 4: Regression Reset Not Working

**What goes wrong:** Iteration 1-3 pass (clean sessions), auto-promotion fires. Iteration 4 is NOT clean (semantic score drops below 0.8). consecutive_clean_sessions should reset to 0. If reset fails, counter stays at 3 forever.

**Why it happens:** checkCleanSession() may have logic error or state write fails silently. Reset is critical to prevent flip-flop models from being promoted.

**How to avoid:**
- Test explicitly: 3 clean → 1 non-clean → verify counter resets to 0
- Verify checkCleanSession() output shows "NOT_CLEAN" reason
- Verify solve-state.json actually contains 0 after non-clean iteration

**Warning signs:**
- consecutive_clean_sessions stays at 3 despite non-clean session
- Promotions occur even after regression (data integrity issue)
- Test passes with 3 clean → 1 clean, fails with 3 clean → 1 non-clean

### Pitfall 5: Changelog Metadata Incomplete

**What goes wrong:** Promotion-changelog.json entries exist but lack required fields (consecutive_clean_sessions, session_id, trigger). PROMO-04 not fully satisfied.

**Why it happens:** promote-gate-maturity.cjs generates changelog entries; if implementation omits a field, test exposes it.

**How to avoid:**
- Verify promote-gate-maturity.cjs includes ALL PROMO-04 fields: model, from_level, to_level, timestamp, session_id, evidence_readiness, trigger, consecutive_clean_sessions
- Test assertion must check every field, not just entry existence

**Warning signs:**
- Test passes but changelog entries are missing session metadata
- Audits of promotion-changelog.json show inconsistent field presence across entries

### Multi-Cycle E2E Test

Source: Adapt from test/auto-promotion-integration.test.cjs pattern

```javascript
// test/e2e-semantic-promotion-pipeline.test.cjs (new)
const { test } = require('node:test');
const assert = require('node:assert/strict');
const fs = require('fs');
const path = require('path');
const os = require('os');

// Helper from v0.34-05 test infrastructure
function createFixtureDir(opts = {}) {
  const tmpDir = fs.mkdtempSync(path.join(os.tmpdir(), 'nf-e2e-'));
  const gatesDir = path.join(tmpDir, 'gates');
  fs.mkdirSync(gatesDir, { recursive: true });

  // Create gate files with wiring scores (no semantic_score yet)
  const wiringA = opts.wiringA != null ? opts.wiringA : 1.0;
  const wiringB = opts.wiringB != null ? opts.wiringB : 1.0;
  const wiringC = opts.wiringC != null ? opts.wiringC : 1.0;

  fs.writeFileSync(path.join(gatesDir, 'gate-a-grounding.json'),
    JSON.stringify({ schema_version: '2', wiring_evidence_score: wiringA }));
  fs.writeFileSync(path.join(gatesDir, 'gate-b-abstraction.json'),
    JSON.stringify({ schema_version: '2', wiring_purpose_score: wiringB }));
  fs.writeFileSync(path.join(gatesDir, 'gate-c-validation.json'),
    JSON.stringify({ schema_version: '2', wiring_coverage_score: wiringC }));

  // Formal check results (no counterexamples by default)
  fs.writeFileSync(path.join(tmpDir, 'check-results.ndjson'),
    '{"result":"no_counterexample"}\n');

  // Initial model registry with SOFT_GATE models
  const registry = { '.alloy/quorum-votes.als': { gate_maturity: 'SOFT_GATE' } };
  fs.writeFileSync(path.join(tmpDir, 'model-registry.json'), JSON.stringify(registry));

  // Initial promotion changelog
  fs.writeFileSync(path.join(tmpDir, 'promotion-changelog.json'), '[]');

  // Initial solve state
  fs.writeFileSync(path.join(tmpDir, 'solve-state.json'),
    JSON.stringify({ consecutive_clean_sessions: 0 }));

  return tmpDir;
}

function enrichGateWithSemanticScore(gatesDir, filename, score) {
  const filePath = path.join(gatesDir, filename);
  const gate = JSON.parse(fs.readFileSync(filePath, 'utf8'));
  gate.semantic_score = score;
  gate.schema_version = '3';
  fs.writeFileSync(filePath, JSON.stringify(gate, null, 2));
}

function simulateSolveIteration(tmpDir, opts = {}) {
  // Read current state
  const stateFile = path.join(tmpDir, 'solve-state.json');
  const state = JSON.parse(fs.readFileSync(stateFile, 'utf8'));
  const prevClean = state.consecutive_clean_sessions || 0;

  // Optionally enrich gates with semantic scores
  if (opts.semanticA != null) {
    enrichGateWithSemanticScore(path.join(tmpDir, 'gates'), 'gate-a-grounding.json', opts.semanticA);
  }
  if (opts.semanticB != null) {
    enrichGateWithSemanticScore(path.join(tmpDir, 'gates'), 'gate-b-abstraction.json', opts.semanticB);
  }
  if (opts.semanticC != null) {
    enrichGateWithSemanticScore(path.join(tmpDir, 'gates'), 'gate-c-validation.json', opts.semanticC);
  }

  // Determine if session is clean (all wiring >= 1.0 AND all semantic >= 0.8)
  const gatesDir = path.join(tmpDir, 'gates');
  const gateA = JSON.parse(fs.readFileSync(path.join(gatesDir, 'gate-a-grounding.json'), 'utf8'));
  const gateB = JSON.parse(fs.readFileSync(path.join(gatesDir, 'gate-b-abstraction.json'), 'utf8'));
  const gateC = JSON.parse(fs.readFileSync(path.join(gatesDir, 'gate-c-validation.json'), 'utf8'));

  const wiringClean =
    (gateA.wiring_evidence_score || 0) >= 1.0 &&
    (gateB.wiring_purpose_score || 0) >= 1.0 &&
    (gateC.wiring_coverage_score || 0) >= 1.0;

  const semanticClean =
    (gateA.semantic_score || 0) >= 0.8 &&
    (gateB.semantic_score || 0) >= 0.8 &&
    (gateC.semantic_score || 0) >= 0.8;

  const isClean = wiringClean && semanticClean && (opts.formalPass !== false);

  // Update state
  state.consecutive_clean_sessions = isClean ? prevClean + 1 : 0;
  fs.writeFileSync(stateFile, JSON.stringify(state, null, 2));

  return { isClean, consecutive: state.consecutive_clean_sessions };
}

test('E2E-PROMO-1: 3 clean sessions trigger auto-promotion', () => {
  const tmpDir = createFixtureDir();

  // Iteration 1: Clean session → consecutive_clean_sessions = 1
  const iter1 = simulateSolveIteration(tmpDir, {
    semanticA: 0.85, semanticB: 0.9, semanticC: 0.88
  });
  assert.equal(iter1.isClean, true, 'iteration 1 should be clean');
  assert.equal(iter1.consecutive, 1, 'counter should be 1');

  // Iteration 2: Clean session → consecutive_clean_sessions = 2
  const iter2 = simulateSolveIteration(tmpDir, {
    semanticA: 0.85, semanticB: 0.9, semanticC: 0.88
  });
  assert.equal(iter2.isClean, true, 'iteration 2 should be clean');
  assert.equal(iter2.consecutive, 2, 'counter should be 2');

  // Iteration 3: Clean session → consecutive_clean_sessions = 3 → auto-promotion eligible
  const iter3 = simulateSolveIteration(tmpDir, {
    semanticA: 0.85, semanticB: 0.9, semanticC: 0.88
  });
  assert.equal(iter3.isClean, true, 'iteration 3 should be clean');
  assert.equal(iter3.consecutive, 3, 'counter should be 3 (auto-promotion eligible)');

  // Verify gate files have semantic_score
  const gateA = JSON.parse(fs.readFileSync(path.join(tmpDir, 'gates', 'gate-a-grounding.json'), 'utf8'));
  assert.equal(gateA.semantic_score, 0.85, 'gate-a should have semantic_score');
  assert.equal(gateA.schema_version, '3', 'gate schema should be v3');

  fs.rmSync(tmpDir, { recursive: true, force: true });
});

test('E2E-PROMO-2: regression resets consecutive_clean_sessions', () => {
  const tmpDir = createFixtureDir();

  // Build up to 3 clean sessions
  simulateSolveIteration(tmpDir, { semanticA: 0.85, semanticB: 0.9, semanticC: 0.88 });
  simulateSolveIteration(tmpDir, { semanticA: 0.85, semanticB: 0.9, semanticC: 0.88 });
  simulateSolveIteration(tmpDir, { semanticA: 0.85, semanticB: 0.9, semanticC: 0.88 });

  // Iteration 4: Regression (semantic drops below 0.8) → counter resets to 0
  const iter4 = simulateSolveIteration(tmpDir, {
    semanticA: 0.75, semanticB: 0.9, semanticC: 0.88  // A drops below threshold
  });
  assert.equal(iter4.isClean, false, 'iteration 4 should not be clean');
  assert.equal(iter4.consecutive, 0, 'counter should reset to 0');

  fs.rmSync(tmpDir, { recursive: true, force: true });
});
```
