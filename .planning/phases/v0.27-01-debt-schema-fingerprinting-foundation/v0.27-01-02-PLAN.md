Implement deterministic fingerprinting algorithms for both issues (hierarchical: exception type -> function name -> message hash) and drifts (formal parameter key hash).

Purpose: Stable fingerprints are the foundation for Phase 3 deduplication. Without deterministic hashing, the same error observed twice would create duplicate debt entries instead of incrementing occurrence counts.

Output: Issue fingerprinting module (bin/fingerprint-issue.cjs), drift fingerprinting module (bin/fingerprint-drift.cjs), and comprehensive test suites proving determinism and separation.
</objective>
