# Phase v0.27-01 Plan 02: Deterministic Fingerprinting Summary

**Hierarchical issue fingerprinting (exception type → function name → message hash) and formal parameter key fingerprinting for drift tracking**

## Overview

Implemented two deterministic fingerprinting algorithms that form the foundation for Phase 3 deduplication:

1. **Issue Fingerprinting (FP-01):** Hierarchical strategy combining exception type, function name, and normalized message hash
2. **Drift Fingerprinting (FP-02):** Single-key strategy hashing only the formal parameter identifier

Both algorithms guarantee:
- **Stability:** Same input always produces identical output (verified via 5x determinism tests)
- **Separation:** Different inputs produce different fingerprints
- **Immutability:** Normalization strips timestamps and line numbers, ensuring content-based stability

## Tasks Completed

### Task 1: Issue Fingerprinting with Hierarchical Strategy (TDD)

**Red Phase:** Created 20 comprehensive tests covering:
- Determinism (5x run verification)
- Separation by exception type, function name, message content
- Timestamp and line number normalization
- Case normalization

**Green Phase:** Implemented `bin/fingerprint-issue.cjs` with:

### Task 2: Drift Fingerprinting by Formal Parameter Key (TDD)

**Red Phase:** Created 18 comprehensive tests covering:

**Green Phase:** Implemented `bin/fingerprint-drift.cjs` with:

## Key Implementation Decisions

1. **SHA-256 with 16-char slice:** Follows project's existing crypto pattern (aggregate-requirements.cjs) and provides sufficient entropy for stable deduplication
2. **Fail-fast validation:** drift fingerprinting throws Error immediately if formal_parameter_key is missing or empty, preventing silent failures
3. **Message normalization regex:** Matches ISO8601 format (`\d{4}-\d{2}-\d{2}T[\d:.Z]+`) and line numbers (`:\d+`) precisely to handle real Sentry/GitHub error messages
4. **Hierarchical separator:** Uses colon (`:`) as separator in issue fingerprinting before final hash for readability and auditability
