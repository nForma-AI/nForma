  <name>Task 1: Create generate-proposed-changes.cjs — TLA+ delta synthesis from plan truths</name>
  <files>bin/generate-proposed-changes.cjs</files>
  <action>
Create `bin/generate-proposed-changes.cjs` that:

1. **Reuses** `parsePlanFrontmatter` and `classifyTruth` from `bin/generate-phase-spec.cjs` via `require()`. Do NOT duplicate these functions.

2. **CLI interface:**
   - Usage: `node bin/generate-proposed-changes.cjs <path-to-PLAN.md> [--dry-run]`
   - Accepts a single PLAN.md file path (NOT a directory — unlike generate-phase-spec.cjs which handles directories).
   - `--dry-run` prints output to stdout without writing files.

3. **`generateProposedChanges(planFilePath)`** function:
   - Reads the PLAN.md file content.
   - Calls `parsePlanFrontmatter(content)` to extract `phase` and `truths`.
   - If `truths` is empty, returns `{ generated: false, reason: 'no truths' }`.
   - Calls `classifyTruth(truth)` for each truth to determine INVARIANT vs PROPERTY.
   - **ReqNN naming scheme:** Property names MUST use zero-padded two-digit indices: `Req01`, `Req02`, ..., `Req99`. The index is 1-based and corresponds to the truth's position in the `must_haves: truths:` array (first truth = Req01, second = Req02, etc.). This zero-padded scheme is critical because PLAN-02's `formatTlcFeedback` uses a regex to parse the `NN` suffix back to a truth index. Mismatched naming breaks bidirectional traceability.
   - Generates a TLA+ MODULE named `ProposedChanges` (fixed name for TLC config stability).
   - The generated spec must include:
     - Module header with phase, timestamp, source file path as comments.
     - `EXTENDS Naturals, FiniteSets, TLC`
     - **ReqNN-to-Truth index mapping comment block** immediately after the EXTENDS line. This enables manual audit and lossless bidirectional traceability per RESEARCH.md:
       ```
       \* === ReqNN-to-Truth Mapping ===
       \* Req01 -> Truth[0]: "first truth text"
       \* Req02 -> Truth[1]: "second truth text"
       \* ================================
       ```
     - Minimal state machine: `VARIABLES state`, `Init == state = "INIT"`, `Next == state' = state` (trivial model so TLC can enumerate; the INVARIANT/PROPERTY stubs default to TRUE).
     - `Spec == Init /\ [][Next]_<<state>>`
     - For each truth: `ReqNN == TRUE \* "truth text here"` (stubbed to TRUE -- these are placeholder properties).
   - Determines output directory: derive `phaseDir` from planFilePath (its parent directory), then target is `{phaseDir}/formal/proposed-changes.tla`.
   - Creates `{phaseDir}/formal/` directory using `fs.mkdirSync(formalDir, { recursive: true })` (the `recursive: true` flag acts as `mkdir -p`, preventing ENOENT errors when intermediate directories are missing during parallel phase executions).
   - Writes the .tla file.
   - Returns `{ generated: true, specPath, truthCount, classifications: [{truth, kind, propName}] }`.

4. **`generateTlaCfg(specPath)`** function:
   - Writes `MCProposedChanges.cfg` in the same directory as specPath.
   - Content: `SPECIFICATION Spec` followed by `INVARIANT ReqNN` for each INVARIANT and `PROPERTY ReqNN` for each PROPERTY (parsed from the .tla file by reading back the generated content and extracting INVARIANT/PROPERTY lines).
   - Returns `{ cfgPath }`.

5. **CLI entrypoint** (guarded by `require.main === module`):
   - Parse args, validate path exists.
   - Call `generateProposedChanges(path)`.
   - If not generated (no truths), print message to stderr and exit 0.
   - Call `generateTlaCfg(specPath)`.
   - Print summary: spec path, cfg path, truth count, classifications.

6. **Exports:** `module.exports = { generateProposedChanges, generateTlaCfg }`.

**Follow existing patterns from `generate-phase-spec.cjs`**: CommonJS, `'use strict'`, `#!/usr/bin/env node`, error handling via `process.stderr.write()`, 2-space indentation.

**Key difference from generate-phase-spec.cjs:** This writes to the phase's `formal/` subdirectory (scoped to planning), NOT to `formal/tla/scratch/` (which is the canonical scratch area). This ensures proposed-changes.tla is ephemeral and phase-scoped per RESEARCH.md Pitfall 3.
