  <action>
Create `bin/quorum-formal-context.cjs` following existing QGSD patterns (CommonJS, 'use strict', error handling via process.stderr.write).

1. **`generateFormalSpecSummary(planFilePath)`** function:
   - Reads planFilePath, calls `parsePlanFrontmatter(content)` from generate-phase-spec.cjs.
   - For each truth, calls `classifyTruth(truth)` to get INVARIANT or PROPERTY.
   - **Special character handling for truths:** Truth strings may contain characters that could cause issues when injected into the quorum slot-worker prompt: double quotes (`"`), single quotes (`'`), newlines, angle brackets (`<`, `>`), and backslashes (`\`). The summary generator MUST sanitize truths before embedding:
     - Replace newlines with spaces (truths should be single-line in the summary).
     - Escape or preserve double quotes as-is within the summary (they appear inside already-quoted strings in the output format, so wrap each truth in backticks instead of double quotes to avoid nested quote confusion).
     - Strip or escape angle brackets (`<` -> `[`, `>` -> `]`) to prevent XML/tag injection in the slot-worker prompt (which uses XML-style `<task>` delimiters).
     - Backslashes: preserve as-is (they are valid in truth text referencing TLA+ operators).
   - Generates a plain-English summary (NOT raw TLA+ code -- per RESEARCH.md Pitfall 4):
     ```
     Proposed formal properties (from plan's must_haves: truths:):
     1. [INVARIANT] `truth text` -- This is a safety property (must always hold)
     2. [PROPERTY] `truth text` -- This is a liveness property (must eventually hold)
     ```
   - Returns `{ summary: string, truthCount: number }` or `null` if no truths found.
   - The summary uses the ORIGINAL truth strings from the PLAN.md frontmatter (lossless, per RESEARCH.md Open Question 3 recommendation), with the sanitization above applied only to the output format.

2. **`generateVerificationResult(tlcResult)`** function:
   - Takes a TLC result object (from run-phase-tlc.cjs's runPhaseTlc or iterativeVerify).
   - Maps to a string:
     - If tlcResult is null or undefined: `"INCONCLUSIVE: No verification was run"`
     - If tlcResult.status === 'skipped': `"INCONCLUSIVE: No truths in plan to verify"`
     - If tlcResult.status === 'passed': `"PASS: All {N} properties verified by TLC in {runtimeMs}ms"`
     - If tlcResult.status === 'failed': `"FAIL: {violations.length} properties violated -- {violations.join(', ')}"`
     - If tlcResult.violations includes 'Java not found': `"INCONCLUSIVE: Java/TLC not available for verification"`
   - Returns the string.

3. **`buildFormalEvidenceBlock(formalSpecSummary, verificationResult)`** function:
   - Takes the outputs of the two functions above.
   - If both are null/empty, returns null (no formal evidence to inject).
   - Produces a formatted text block matching the RESEARCH.md pattern:
     ```
     === Formal Evidence ===
     Proposed TLA+ properties (from plan's must_haves: truths:):
     {formalSpecSummary}

     Verification result: {verificationResult}
     ======================
     ```
   - Returns the block as a string.
   - This block is injected into the Mode A prompt in qgsd-quorum-slot-worker by the plan-phase.md orchestrator (injected AFTER the artifact section, BEFORE prior_positions).

4. **`getFormalContext(planFilePath, tlcResult)`** convenience function:
   - Combines all three steps: generate summary, generate verification result, build block.
   - Returns `{ formalSpecSummary, verificationResult, evidenceBlock }` or `{ evidenceBlock: null }` if no truths.
   - This is the primary API the orchestrator calls.

5. **CLI entrypoint** (guarded by `require.main === module`):
   - Usage: `node bin/quorum-formal-context.cjs <path-to-PLAN.md> [--tlc-result=<json>]`
   - If `--tlc-result` provided, parse as JSON and use as tlcResult.
   - Otherwise, pass null (INCONCLUSIVE).
   - Prints the evidence block to stdout.

6. **Exports:** `module.exports = { generateFormalSpecSummary, generateVerificationResult, buildFormalEvidenceBlock, getFormalContext }`.

**Key design decisions:**
- formal_spec_summary uses ORIGINAL truth strings (per RESEARCH.md recommendation -- lossless, bidirectional traceability).
- The module does NOT modify the quorum-slot-worker agent file. The orchestrator (plan-phase.md) is responsible for injecting the evidence block into the prompt. This module just generates the content.
- Fail-open: if anything fails, return null evidence block rather than crashing.
