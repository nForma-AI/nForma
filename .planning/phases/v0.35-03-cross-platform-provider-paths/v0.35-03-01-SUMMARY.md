# Phase v0.35-03 Plan 01: Cross-Platform Provider Path Resolution Summary

Integrated runtime CLI path resolution into nForma's provider dispatch pipelines to support Linux, WSL, and macOS without hardcoded Homebrew paths.

## Objective

Enable nForma to work on macOS, Linux (apt/snap), and WSL by resolving CLI paths at runtime rather than relying on hardcoded `/opt/homebrew/bin/` paths in providers.json. The existing `resolve-cli.cjs` utility already handles multi-strategy discovery; this plan wires it into the two code paths that spawn provider CLIs.

## Implementation Details

### Task 1: Wire resolveCli into unified-mcp-server.mjs

**Changes:**
- Imported `resolveCli` using `createRequire` from `resolve-cli.cjs`
- Added resolution loop after providers load from providers.json:
  - For each subprocess provider, resolve the CLI path using resolveCli(bareName)
  - Also resolve service commands (ccr start/stop/status)
  - Log resolved paths to stderr for observability
- Updated all spawn calls to use `provider.resolvedCli ?? provider.cli` (defensive fallback)
- Updated health check's `fs.accessSync` binary existence check to use resolved path
- Defensive pattern ensures API-type providers (no subprocess, no cli) still work

**File:** bin/unified-mcp-server.mjs
**Commits:** f919cb33

### Task 2: Wire resolveCli into call-quorum-slot.cjs and create integration tests

**Part A - call-quorum-slot.cjs Changes:**
- Imported `resolveCli` from `resolve-cli.cjs`
- Added resolution in `runSubprocess()` before spawn call
- Updated `buildSpawnArgs()` isCcr detection to use resolved path with fallback
- Updated spawn call to use `provider.resolvedCli ?? provider.cli`

**Part B - Integration Tests (test/resolve-cli-integration.test.cjs):**
Created comprehensive integration test suite with 18 tests validating:

3. resolvedCli fallback pattern works correctly (both set/unset cases)
4. Service command resolution works (ccr start/stop/status)
5. isCcr detection works with resolvedCli in both code patterns
6. Integration patterns from unified-mcp-server.mjs and call-quorum-slot.cjs

## Known Behavior Changes

1. **Startup logging:** When CLI paths differ from template, resolution is logged to stderr:
   ```
   [mcp] Resolved codex: /opt/homebrew/bin/codex -> /usr/bin/codex
   ```

2. **Path resolution strategy priority:**
   1. `which <name>` — system PATH lookup
   2. Homebrew prefixes — `/opt/homebrew/bin`, `/usr/local/bin`
   3. npm global bin — derived from `npm root -g`
   4. Common system paths — `/usr/bin`
   5. Bare fallback — return name unchanged (let OS resolve at spawn)

3. **API-type providers:** Unaffected; they have no `.cli` property so resolution loop skips them
