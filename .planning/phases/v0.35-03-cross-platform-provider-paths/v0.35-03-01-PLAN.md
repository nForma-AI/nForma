<objective>
Integrate runtime CLI path resolution into the two provider dispatch pipelines so nForma works on macOS, Linux, and WSL without hardcoded Homebrew paths.

Purpose: GitHub issue #6 -- providers.json hardcodes `/opt/homebrew/bin/` paths which break on Linux (apt -> /usr/bin, snap -> /snap/bin) and WSL. The existing `resolve-cli.cjs` utility already handles multi-strategy path discovery; this plan wires it into the two code paths that spawn provider CLIs.

Output: Modified unified-mcp-server.mjs and call-quorum-slot.cjs that resolve CLI paths at startup, plus integration tests proving the resolution is used.
</objective>

2. **Resolve paths at load time** — After `providers` is loaded from providers.json (line ~26), add a resolution step:
   ```javascript
   // Resolve CLI paths at startup (XPLAT-01: cross-platform path discovery)
   for (const provider of providers) {
     if (provider.type === 'subprocess' && provider.cli) {
       const bareName = provider.cli.split('/').pop();
       provider.resolvedCli = resolveCli(bareName);
       if (provider.cli !== provider.resolvedCli) {
         process.stderr.write(`[mcp] Resolved ${bareName}: ${provider.cli} -> ${provider.resolvedCli}\n`);
       }
       // Also resolve service commands if present (e.g., ccr start/stop/status)
       if (provider.service) {
         for (const key of ['start', 'stop', 'status']) {
           if (provider.service[key] && provider.service[key][0]) {
             const svcBareName = provider.service[key][0];
             const resolvedSvc = resolveCli(svcBareName);
             if (resolvedSvc !== svcBareName) {
               provider.service[key][0] = resolvedSvc;
             }
           }
         }
       }
     }
   }
   ```

3. **Replace all `provider.cli` spawn references with `provider.resolvedCli`** in:
   - `runProvider()` (line ~237): `spawn(provider.resolvedCli ?? provider.cli, args, ...)`
   - `runSubprocessWithArgs()` (line ~301): `spawn(provider.resolvedCli ?? provider.cli, args, ...)`
   - `runDeepHealthCheck()` (line ~573): `fs.accessSync(provider.resolvedCli ?? provider.cli, ...)` and the error message
   - The `model` fallback (line ~530): leave `provider.cli` as-is (display only)

   Use `provider.resolvedCli ?? provider.cli` pattern as defensive fallback so API-type providers (no resolvedCli) still work.

4. **Do NOT modify providers.json** — Keep hardcoded paths as documentation/template per research recommendation. Runtime code resolves them.

5. **Do NOT add platform-specific branching** — Let resolveCli() handle all platform differences internally.
