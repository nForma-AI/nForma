**Workflow (core/workflows/model-driven-fix.md):**

Create a workflow following the debug.md orchestrator pattern with 6 sequential phases. Use the `<purpose>`, `<process>`, `<constraints>`, `<success_criteria>` structure matching existing workflows.

**Purpose section:**
Orchestrate the full model-driven fix cycle: discover existing models, attempt reproduction, refine or create a model that captures the failure, extract constraints, apply constraints to guide fix, and verify the fix resolves the failure.

**Process section with 6 phases:**

### Phase 1 — Discovery

Find existing formal models that may cover the bug's affected code.

```
ARGS: Parse from $ARGUMENTS:
  - BUG_DESC: the main argument text (bug description)
  - --files: comma-separated affected file paths (optional)
  - --formalism: override formalism selection (optional)
  - --verbose: show full checker output (optional, default false)
  - --skip-fix: stop after constraint extraction, skip Phases 5-6 (optional)

Run model discovery:
  node bin/formal-scope-scan.cjs --bug-mode --description "$BUG_DESC" ${FILES:+--files "$FILES"} --format json

Parse JSON output:
  - If matches found: display matched models table, proceed to Phase 2
  - If no matches: display "No existing models cover this failure", skip to Phase 3
```

### Phase 2 — Reproduction

Attempt to reproduce the bug using existing models.

```
If existing models were found in Phase 1:

  node bin/formal-scope-scan.cjs --bug-mode --run-checkers --description "$BUG_DESC" ${FILES:+--files "$FILES"} --format json

  For each model result:
    - If checker found violation: model REPRODUCES the bug
      Display: "Model {name} reproduces the bug (invariant {X} violated)"
      Set $REPRODUCING_MODEL = model path
      Skip to Phase 4 (model already captures the failure)
    - If checker passed: model does NOT reproduce
      Display: "Model {name} passes -- does not capture failure"

  If no model reproduced the bug:
    Display: "Existing models do not reproduce this bug. Proceeding to refinement."
    Proceed to Phase 3
```

### Phase 3 — Refinement (uses Plan 01 deliverables)

Create or refine a formal model that captures the failure mode.

```
Display banner:
  Phase 3 -- Refinement
  Creating model to capture: {first 80 chars of BUG_DESC}...

Invoke close-formal-gaps workflow with --bug-context:

  /nf:close-formal-gaps --bug-context="$BUG_DESC" ${FORMALISM:+--formalism=$FORMALISM} --batch

  This triggers:
  - Step 5: Spec generation biased by bug context (MRF-01)
  - Step 6: Refinement loop with inverted verification (MRF-02)

Parse the result:
  - If a reproducing model was created:
    Set $REPRODUCING_MODEL = path to the new model
    Display: "Model created and bug reproduced after {N} attempt(s)"
    Proceed to Phase 4
  - If refinement exhausted (3 attempts, no reproduction):
    Display: "WARNING: Model created but does not reproduce the bug after 3 attempts"
    Set $REPRODUCING_MODEL = path to latest model (best effort)
    Proceed to Phase 4 with caveat

  If $VERBOSE, show full refinement iteration details.
  Otherwise show summary verdicts only (per user decision).
```

### Phase 4 — Constraint Extraction

Extract fix constraints from the reproducing model.

```
Display banner:
  Phase 4 -- Constraint Extraction
  Extracting constraints from: {$REPRODUCING_MODEL}

Run constraint extraction:
  node bin/model-constrained-fix.cjs --spec "$REPRODUCING_MODEL" --max-constraints 3 --format json

Parse JSON output to get constraints array.

Display extracted constraints:
  Constraints from {model name}:
  1. {constraint 1 english text}
  2. {constraint 2 english text}
  3. {constraint 3 english text}

Store $CONSTRAINTS for Phase 5.

If --skip-fix flag is active:
  Display: "Constraints extracted. Stopping here (--skip-fix)."
  Display the constraints as the final output.
  Exit workflow.
```

### Phase 5 — Constrained Fix

Apply constraints to guide the code fix.

```
Display banner:
  Phase 5 -- Constrained Fix
  Applying {N} constraint(s) to guide fix

Present constraints to the developer/quorum as fix guidance:

  [FIX CONSTRAINTS]
  The following constraints were extracted from formal model {model name}:
  - {constraint 1}
  - {constraint 2}
  - {constraint 3}

  Apply a fix that:
  1. Resolves the described bug: {BUG_DESC}
  2. Does NOT violate any of the above constraints
  3. Passes the formal model verification in Phase 6
  [END FIX CONSTRAINTS]

This phase presents the constraints and waits for the fix to be applied.
The fix can be applied manually by the developer or by another workflow.

After fix is applied (user confirms or automated fix completes):
  Proceed to Phase 6.
```

### Phase 6 — Pre-Verification

Verify the fix resolves the failure by re-running the model.

```
Display banner:
  Phase 6 -- Pre-Verification
  Running model against fixed code: {$REPRODUCING_MODEL}

Run ONLY the new/refined model (per user decision: pre-verification scope limited to new model):

  Determine formalism from model file extension (.tla or .als)
  If TLA+: node bin/run-tlc.cjs "$REPRODUCING_MODEL"
  If Alloy: node bin/run-alloy.cjs "$REPRODUCING_MODEL"

Parse result:
  - If checker PASSES (no violations):
    The fix resolves the failure -- model no longer finds the bug
    Display: "Pre-verification PASSED -- model passes after fix"
    Display: "The formal model confirms the bug is resolved."

  - If checker FAILS (violation found):
    The bug is still present or the fix introduced a new issue
    Display: "Pre-verification FAILED -- model still finds violation"
    Display the violation details
    Suggest: "Review the fix and re-run /nf:model-driven-fix or apply additional changes"

Do NOT run broader regression testing across other models (per user decision:
pre-verification scope limited to new/refined model only; cross-model regression
is deferred to Phase v0.38-05).
```

**Constraints section:**
- Run ONLY the new/refined model during pre-verification (Phase 6) -- broader regression deferred
- Auto-select formalism based on bug type unless --formalism override provided
- Show summary verdicts by default, full output via --verbose
- Each phase displays a banner with phase name before executing
- Fail-open: if any phase tool call fails, log error and continue to next phase where possible
- Phase 3 delegates to close-formal-gaps.md for spec generation (do not duplicate that logic)

**Success criteria section:**
- [ ] All 6 phases execute in sequence
- [ ] Phase 1 discovers existing models (or reports none)
- [ ] Phase 2 attempts reproduction with existing models
- [ ] Phase 3 creates/refines model with bug context bias via close-formal-gaps --bug-context
- [ ] Phase 4 extracts constraints from reproducing model
- [ ] Phase 5 presents constraints as fix guidance
- [ ] Phase 6 re-runs model to verify fix (ONLY the new/refined model)
- [ ] --skip-fix stops after Phase 4
- [ ] --verbose shows full checker output throughout
