  <action>
1. Create `agents/nf-worktree-executor.md` with frontmatter:
   - `name: nf-worktree-executor`
   - `description: Executes plan tasks in an isolated git worktree for parallel execution. Spawned by execute-plan Pattern D for independent tasks.`
   - `tools: Read, Write, Edit, Bash, Grep, Glob`
   - `isolation: worktree`
   - `color: green`
   - The markdown body should be nearly identical to `agents/nf-executor.md` with these additions:
     a. Note that this executor operates in an isolated git worktree (branched from origin/main)
     b. Executor must report its worktree branch name in its completion output: run `git rev-parse --abbrev-ref HEAD` and include as `worktree_branch: <branch>` in the completion format
     c. Note the constraint: subagents cannot spawn other subagents, so this executor must execute tasks directly
     d. State updates (STATE.md, ROADMAP.md) should NOT be performed by worktree executors -- the orchestrator handles those after merge

2. Create `bin/worktree-merge.cjs` following the code example from RESEARCH.md with these functions:
   - `mergeBranches(cwd, branches)` -- For each branch, attempt `git merge --no-ff <branch>` into current branch. On conflict: `git merge --abort`, mark as conflict. Return array of `{ branch, status, error? }`.
   - `verifyMergedState(cwd)` -- Run `npm test` with 120s timeout, return `{ pass, error? }`.
   - `cleanupWorktreeBranches(cwd, branches)` -- Best-effort `git branch -d <branch>` for each. Return array of results. Also run `git worktree prune` to clean stale worktree entries.
   - CLI interface: `node bin/worktree-merge.cjs <merge|verify|cleanup> [branches...]`
   - Follow project conventions: `'use strict'`, CommonJS, fail-open (process.exit(0) on error), stderr for diagnostics
   - Set MERGE_TIMEOUT_MS = 30000 (30s per merge operation)

3. Add `.claude/worktrees/` to `.gitignore` if not already present (append at end, with comment `# Worktree isolation (parallel execution)`)
  </action>
  <verify>
    - `grep 'isolation: worktree' agents/nf-worktree-executor.md` returns match
    - `grep 'worktree_branch' agents/nf-worktree-executor.md` returns match
    - `node bin/worktree-merge.cjs --help 2>&1` exits 0 (fail-open)
    - `node -e "const m = require('./bin/worktree-merge.cjs'); console.log(typeof m.mergeBranches, typeof m.verifyMergedState, typeof m.cleanupWorktreeBranches)"` prints "function function function"
    - `grep '.claude/worktrees/' .gitignore` returns match
  </verify>
  <done>Worktree executor agent definition exists with isolation: worktree in frontmatter. Merge orchestration module exports mergeBranches, verifyMergedState, cleanupWorktreeBranches. .gitignore updated.</done>
