Create bin/convergence-gate-runner.cjs implementing the three-gate convergence check.

**Main function: runConvergenceGates(models, config, checkerFns)**

Parameters:
- models: { consequenceModelPath, reproducingModelPath, neighborModelPaths (array of paths) }
- config: { bugTrace (path to ITF JSON), sessionId, formalism: 'tla'|'alloy' }
- checkerFns (optional): { runChecker(modelPath, options) } — dependency injection for testability. Defaults to real run-tlc/run-alloy calls via child_process.execFile (NOT exec, to avoid shell injection).

Returns: { gate1_invariants: {passed, details}, gate2_bug_resolved: {passed, details}, gate3_neighbors: {passed, regressions}, converged: boolean, writeOnceTimestamp: string (ISO8601), iteration: number }

Implementation steps:

1. **Load or initialize verdict log** at .planning/formal/cycle2-simulations/{sessionId}/gate-verdicts.json
   - loadOrInitializeVerdicts(verdictLogPath): read file, parse JSON array, return array.
   - **File missing:** return [] (fail-open — new session, no prior history).
   - **File exists but corrupt (JSON parse error):** THROW Error('Verdict log corrupt at {verdictLogPath}. Cannot guarantee write-once semantics. Manual inspection required.'). Do NOT return empty array — returning [] would erase any prior converged=true verdict, silently violating ResolvedAtWriteOnce. Corrupt log is a HARD error.
   - Compute iterationNumber = existingVerdicts.length + 1

2. **Check for prior convergence (ResolvedAtWriteOnce PRE-GATE enforcement)**
   - Scan existing verdicts for any with converged=true. Store priorConverged flag and the iteration number of the prior convergence.
   - This check happens BEFORE running gates, so we can block a write-once violation before any new verdict is persisted.

3. **Gate 1 — Invariants Hold** (runGate1InvariantsHold)
   - Run consequence model through checker with verification_mode='validation' (normal semantics — invariants SHOULD hold)
   - For TLA+: use child_process.execFile to run `node bin/run-tlc.cjs {consequenceModelPath}` with verification_mode=validation
   - For Alloy: use child_process.execFile to run `node bin/run-alloy.cjs {consequenceModelPath}` with verification_mode=validation
   - Or use injected checkerFns.runChecker if provided (for testing)
   - If checker passes (exit 0): gate1 = { passed: true, details: "All original invariants hold in consequence model" }
   - If checker fails (non-zero exit): gate1 = { passed: false, details: "Invariant violation: {parse violation from output}" }
   - **Dependency failure handling (HaikuUnavailableNoCorruption):** If checker throws (ENOENT, timeout, network), do NOT write verdict, do NOT delete state. Return { gate1_invariants: {passed: null, details: "UNAVAILABLE: {error}"}, ...all gates null, converged: null, unavailable: true, preservedState: true }

4. **Gate 2 — Bug Resolved** (runGate2BugResolved)
   - Run consequence model through checker with inverted verification: the bug trace should NOT appear in the consequence model
   - Use verification_mode='diagnostic' (inverted semantics — if checker finds NO violation, bug IS resolved)
   - Parse output: if checker passes (no violation found) -> bug is resolved (gate2.passed=true)
   - If checker finds violation -> bug still present (gate2.passed=false, details cite the trace)
   - Same dependency failure handling as gate 1.

5. **Gate 3 — No Neighbor Regressions** (runGate3NeighborRegressions)
   - For each neighbor model path: run checker against BOTH reproducing model and consequence model
   - Regression = violation in consequence run that was NOT in reproducing run (pre-existing failures are NOT regressions, per Pitfall 6 from research)
   - gate3 = { passed: boolean, regressions: [{neighborPath, violation, isNewRegression: true}] }
   - If no neighbor paths provided: gate3 = { passed: true, regressions: [], details: "No neighbors to check" }
   - Same dependency failure handling.

6. **Compute convergence and enforce write-once BEFORE persisting**
   - converged = gate1.passed && gate2.passed && gate3.passed (all three must pass)
   - **CRITICAL — Write-once check BEFORE disk write:** If priorConverged is true AND current converged is false, throw Error('ResolvedAtWriteOnce VIOLATED: cannot revert convergence verdict from iteration N') IMMEDIATELY. Do NOT append to the array. Do NOT write to disk. The non-converged verdict must never be persisted when a prior converged=true exists.
   - Only if the write-once check passes: build verdict object { iteration, timestamp: new Date().toISOString(), gate1, gate2, gate3, converged, mutable: false }, APPEND to verdict array, write full array to gate-verdicts.json with JSON.stringify(arr, null, 2).

7. **Return verdict**

Export: { runConvergenceGates, loadOrInitializeVerdicts }
Module pattern: CommonJS.

**Critical formal invariants to enforce:**
- ResolvedAtWriteOnce: once logWritten (converged=true), it cannot revert to false. Implementation: check BEFORE persisting — if prior converged and current not converged, throw without writing. Corrupt verdict log throws hard error (not fail-open) to prevent silent erasure of prior convergence.
- HaikuUnavailableNoCorruption: when dependency unavailable, logWritten and stateDeleted remain unchanged. Implementation: on dependency error, return early without writing verdict or deleting any state.
