# Phase v0.39-03 Plan 02: Three-Gate Convergence Verification Summary

Run three automated convergence gates with write-once verdict persistence and dependency failure safety.

## Objective Achieved

Implemented the verification engine of Cycle 2 solution simulation: a three-gate convergence checker that runs automated verification gates in sequence and persists verdicts using append-only write-once semantics. When external dependencies become unavailable, all accumulated state is preserved without corruption.

## Deliverables

### 1. bin/convergence-gate-runner.cjs

Main module implementing three-gate convergence verification with formal invariant enforcement.

**Exports:**
- `runConvergenceGates(models, config, checkerFns)` — Main entry point
- `loadOrInitializeVerdicts(verdictLogPath)` — Verdict log initialization

**Key Features:**

1. **Gate 1 — Invariants Hold**
   - Runs consequence model with verification_mode=validation (normal semantics)
   - Delegates to run-tlc.cjs or run-alloy.cjs based on formalism
   - Returns {passed, details} indicating if original invariants are preserved

2. **Gate 2 — Bug Resolved**
   - Runs consequence model with verification_mode=diagnostic (inverted semantics)
   - Inverted logic: checker passes (no violation) means bug is resolved
   - Returns {passed, details} indicating if bug no longer triggers

3. **Gate 3 — No Neighbor Regressions**
   - Runs each neighbor model against both reproducing and consequence models
   - Regression = violation in consequence that wasn't in reproducing (baseline comparison)
   - Distinguishes pre-existing failures from regressions (Pitfall 6 prevention)
   - Returns {passed, regressions: Array} with per-neighbor details

4. **Convergence Logic**
   - Converged = all three gates passed
   - Write-once check runs BEFORE persisting: if prior convergence exists and current would write non-converged, throws error and does NOT persist

5. **Verdict Persistence**
   - Path: .planning/formal/cycle2-simulations/{sessionId}/gate-verdicts.json
   - Format: JSON array of verdict objects with fields: iteration, timestamp, gate1_invariants, gate2_bug_resolved, gate3_neighbors, converged, mutable
   - Append-only: each iteration appended without overwriting previous entries
   - Each verdict marked mutable: false to enforce write-once contract

6. **Formal Invariant Enforcement**

   **ResolvedAtWriteOnce:**
   - Once converged verdict written (converged=true), no subsequent verdict with converged=false can be persisted
   - Check runs BEFORE writing to disk
   - If violation detected: throws error immediately without writing verdict
   - Test case: iteration 1 converges (written), iteration 2 fails gates → error + log unchanged

   **HaikuUnavailableNoCorruption:**
   - When dependency becomes unavailable (model checker timeout, ENOENT, network error)
   - Return early without writing verdict
   - Return early without deleting session state
   - Preserve all accumulated files and session directory
   - Return {unavailable: true, preservedState: true} to signal resumability

7. **Error Handling**
   - Missing verdict log: fail-open, return empty array (new session)
   - Corrupt verdict log: hard error, throw with message containing corrupt and write-once
   - Dependency failure: return unavailable signal, preserve state
   - Checker not found: throw ENOENT propagation

8. **Dependency Injection**
   - Optional checkerFns parameter for testing
   - Allows test mocks to replace actual run-tlc/run-alloy calls
   - Used to verify gate execution logic without requiring real model checkers

## Implementation Details

### Convergence Gate Sequence

runConvergenceGates accepts:
- models: {consequenceModelPath, reproducingModelPath, neighborModelPaths: [...]}
- config: {bugTrace, sessionId, formalism: 'tla'|'alloy', projectRoot}
- checkerFns: optional {runChecker(modelPath, options)} for testing

Returns:
- gate1_invariants: {passed, details}
- gate2_bug_resolved: {passed, details}
- gate3_neighbors: {passed, regressions: [...]}
- converged: boolean
- writeOnceTimestamp: ISO8601 string
- iteration: number
- unavailable: boolean (if dependency failed)
- preservedState: boolean (if dependency failed but state preserved)

### Verdict Log Structure

Each entry in gate-verdicts.json:
- iteration: number
- timestamp: ISO8601
- gate1_invariants: {passed, details}
- gate2_bug_resolved: {passed, details}
- gate3_neighbors: {passed, regressions, details}
- converged: boolean
- mutable: false (formal guarantee: once written, cannot change)

### Checker Interface

Each gate delegates to checker function (real or mock):
- checkerFns.runChecker(modelPath, options)
- Returns: {passed: boolean, details: string}
- Throws: error with .unavailable=true for dependency failures

Real implementation uses child_process.execFile to call:
- node bin/run-tlc.cjs {modelPath} --verification-mode={mode}
- node bin/run-alloy.cjs {modelPath} --verification-mode={mode}

## Formal Invariant Verification

### ResolvedAtWriteOnce

**Property:** Once converged verdict written (converged=true), it cannot revert to false in any subsequent iteration.

**Implementation:**
1. Load existing verdicts from disk
2. Scan for prior convergence before running gates
3. Run all three gates
4. BEFORE writing verdict: Check if prior converged AND current would write non-converged
5. If violation: throw error immediately without writing
6. Test case: First iteration converges → verdict written. Second iteration fails gates → error thrown, verdict log unchanged (still contains only iteration 1).

**Test Results:**
- Iteration 1 converges (written)
- Iteration 2 fails gates → throws ResolvedAtWriteOnce VIOLATED error
- Verdict log on disk: still contains only iteration 1 (iteration 2 never written)
- Formal invariant enforced

### HaikuUnavailableNoCorruption

**Property:** When external dependency (model checker, quorum slot) becomes unavailable, logWritten and stateDeleted remain unchanged (state is preserved without corruption).

**Implementation:**
1. Each gate has try-catch that recognizes unavailability signals (ENOENT, ETIMEDOUT, network errors)
2. On dependency error:
   - Return early without persisting verdict
   - Return early without deleting session state
   - Return {unavailable: true, preservedState: true} to indicate resumability
3. Test cases:
   - Checker throws ENOENT → unavailable=true returned, verdict not written
   - Session directory preserved, test files still exist after failure

**Test Results:**
- Dependency failure returns unavailable=true, preservedState=true
- Verdict log not created/written (or stays empty)
- Session directory and files preserved
- Ready for retry once dependency recovers
- Formal invariant enforced

## Key Design Decisions

1. **Hard Error on Corrupt Verdict Log**
   - Plan spec: Corrupt log throws hard error (not fail-open)
   - Reasoning: Returning empty array would silently erase write-once history
   - Implementation: Catch JSON parse error, throw with "corrupt" and "write-once" in message

2. **Write-Once Check BEFORE Persisting**
   - Plan spec: Check runs before any disk write
   - Reasoning: Ensures violation is never persisted; verdict log always satisfies invariant
   - Implementation: Check after gate execution, before verdict array push and file write
   - Test verifies: Error thrown AND verdict log unchanged

3. **Dependency Injection for Testing**
   - Plan spec: Supports optional checkerFns parameter
   - Reasoning: Allows test suite to validate gate logic without requiring model checkers
   - Implementation: Optional third parameter, defaults to real child_process.execFile calls
   - Test coverage: 9 tests use mock checkers, 8 tests use real invocation patterns

4. **Neighbor Regression Baseline Comparison**
   - Plan spec: Regression = violation in consequence not in reproducing (pre-existing failures excluded)
   - Reasoning: Prevents false positives on unrelated neighbor failures
   - Implementation: Run gate 3 against both models, compare results

5. **Append-Only Verdict Persistence**
   - Plan spec: Never overwrite; always append
   - Reasoning: Preserves full history; enables future iteration analysis
   - Implementation: Load array, push new verdict, write full array (never seek/truncate)
