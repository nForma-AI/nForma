# Phase v0.39-03 Plan 01: Fix Intent Normalization and Consequence Model Generation

Fix intent normalization and consequence model generation pipeline for solution simulation.

## Summary

This plan implemented **SIM-01** (Fix Intent Normalization) and **SIM-02** (Consequence Model Generation), the first two modules in the Cycle 2 solution simulation pipeline.

The implementation provides:

1. **Intent Normalizer Module** (`intent-normalizer.cjs`)
   - Parses fix ideas from three input channels:
     - Channel 1: Constraint syntax (if/then, when/must, unless patterns)
     - Channel 2: Variable syntax (add/track/set variable declarations)
     - Channel 3: Code snippets (```code blocks```)
   - Normalizes to structured mutation format: `{ type, target, content, reasoning }`
   - Detects ambiguities (hedging language: maybe/might/could; multiple options: or/or-alternatively)
   - Computes confidence score based on mutation count (max 3 channels → confidence 1.0)
   - Supports optional context parameter for bug description inclusion in reasoning

2. **Consequence Model Generator Module** (`consequence-model-generator.cjs`)
   - Applies normalized mutations to reproducing models
   - Supports TLA+ and Alloy formalisms with automatic extension-based detection
   - Creates session directories under `.planning/formal/cycle2-simulations/{sessionId}/`
   - Writes consequence model file (`consequence-model.tla` or `.als`) with mutations applied
   - Writes `normalized-mutations.json` with complete mutation traceability (type, target, content, applied status)
   - Fail-open behavior: invalid mutations recorded with `applied=false` but don't crash
   - Session IDs use 8 bytes from `crypto.randomBytes` for collision resistance (96-bit entropy)
