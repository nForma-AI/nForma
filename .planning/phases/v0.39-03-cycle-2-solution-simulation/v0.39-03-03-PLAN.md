**simulateSolutionLoop(input, deps)**

Parameters:
- input: { fixIdea (string), bugDescription (string), reproducingModelPath (string), neighborModelPaths (array), bugTracePath (string), maxIterations (number, default from .planning/config.json or 3), formalism: 'tla'|'alloy' }
- deps (optional): { normalizer, generator, gateRunner } — dependency injection for testability. Defaults to requiring the real modules.

Returns: { converged: boolean, iterations: Array, finalVerdict: Object, escalationReason: string|null, sessionId: string }

Implementation:

1. **Initialize:** Generate sessionId via crypto.randomBytes(8).toString('hex'). Read maxIterations from .planning/config.json if not provided (default 3). Display banner:
```
=== Solution Simulation (Phase 4.5) ===
Max iterations: {maxIterations}
Fix idea: {first 80 chars of fixIdea}...
```

2. **Iteration loop (1 to maxIterations):**

   a. **Step 1 — Normalize fix intent:**
   Call normalizeFixIntent(fixIdea, { bugDescription, reproducingModelPath }). On first iteration, normalize from scratch. On subsequent iterations, reuse prior mutations and add iteration-specific refinements if gate feedback suggests changes.

   b. **Step 2 — Generate consequence model:**
   Call generateConsequenceModel(reproducingModelPath, mutations, { sessionId, formalism }). Get consequenceModelPath.

   c. **Step 3 — Run convergence gates:**
   Call runConvergenceGates({ consequenceModelPath, reproducingModelPath, neighborModelPaths }, { bugTrace: bugTracePath, sessionId, formalism }). Get verdict.

   d. **Step 4 — Display iteration row:**
   Format: `Iteration {N}/{max}: Invariants ({pass/fail/--}) | Bug Resolved ({pass/fail/--}) | Neighbors ({pass/fail/--}) | {CONVERGED/FAILED/UNAVAILABLE}`

   e. **Step 5 — Check outcome:**
   - If verdict.converged === true: break loop, set converged=true
   - If verdict.unavailable === true: break loop immediately (HaikuUnavailableNoCorruption — do NOT continue iterating when dependency is unavailable). Set escalationReason = "Dependency unavailable: {details}. Simulation paused. All state preserved."
   - If not converged and more iterations remain: log which gates failed, continue
   - If not converged and last iteration: set escalationReason = "Max iterations ({maxIterations}) exhausted. Gates still failing: {list failed gates with details}"

3. **Display summary table:**
```
=== Simulation Results ===
| Attempt | Invariants | Bug Resolved | Neighbors | Status |
|---------|-----------|-------------|-----------|--------|
| 1       | PASS      | FAIL        | PASS      | FAILED |
| 2       | PASS      | PASS        | PASS      | CONVERGED |
```

4. **Display final verdict:**
   - If converged: "Fix CONVERGED after {N} iteration(s). Consequence model verified at: {path}"
   - If escalated (max iterations): "ESCALATION: {escalationReason}\n\nSuggestions:\n- Review gate failure details above\n- Refine fix idea with more specific constraints\n- Check model assumptions against bug trace"
   - If escalated (dependency failure): "PAUSED: {escalationReason}\n\nAll accumulated state preserved. Resume when dependency recovers."

5. **Write iteration history** to .planning/formal/cycle2-simulations/{sessionId}/iteration-history.json

Export: { simulateSolutionLoop }

```markdown
## Phase 4.5 — Solution Simulation

Simulate the proposed fix in model space before touching code.

**Skip if** `$REPRODUCING_MODEL` is empty or inconclusive.

Phase 4.5 — Solution Simulation

**Step 2: Run simulation loop**

  --fix-idea="$FIX_IDEA" \
  --bug-description="$BUG_DESC" \
  --reproducing-model="$REPRODUCING_MODEL" \
  --bug-trace="$BUG_TRACE_PATH" \
  --formalism="$FORMALISM" \
  ${VERBOSE:+--verbose}

Parse result:
- If converged:
  Display: "Fix CONVERGED in model space. Proceeding to code fix (Phase 5)."
  Set `$SIMULATION_CONVERGED` = true
  Proceed to Phase 5 with high confidence.

- If not converged (max iterations exhausted):
  Display escalation reason and suggestions.
  Prompt user: "Continue to code fix anyway? (y/n)"
  If yes: Proceed to Phase 5 with caveat.
  If no: Exit workflow.

- If paused (dependency unavailable):
  Display pause message.
  Exit workflow. User can resume later.

**Fail-CLOSED error handling (model-space gate is mandatory):**
The phase goal is "fix ideas iterate entirely in model space before any code is touched." Therefore:
- If solution-simulation-loop.cjs **is not found** (module missing): Display warning: "Phase 4.5 module not installed. Simulation cannot run." Prompt user: "Skip simulation and proceed to Phase 5? (y/n)". Only proceed if user explicitly confirms. Do NOT auto-continue.
- If solution-simulation-loop.cjs **errors during execution** (runtime error, crash): Display the error message and stack trace. Exit workflow — do NOT auto-continue to Phase 5. The user must investigate the error or explicitly restart with --skip-simulation flag.
- Rationale: A fail-open that silently skips simulation and proceeds directly to code changes contradicts the core invariant that fixes must be verified in model space first. The simulation gate must be mandatory when the module exists and is configured.
</step>
```
