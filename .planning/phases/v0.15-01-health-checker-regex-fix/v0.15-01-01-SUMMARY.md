# Phase v0.15-01 Plan 01: Health Checker Regex Fix Summary

**One-liner:** Fixed 8 regex sites in gsd-tools.cjs to eliminate 64 false-positive W005/W007/W002 warnings when `validate health` runs against QGSD's versioned `v0.X-YY-name` phase directories.

## What Was Built

Fixed `qgsd-core/bin/gsd-tools.cjs` to recognize QGSD's versioned phase naming convention (`v0.X-YY-name`) in the health checker. Before this fix, `validate health` produced 64 false-positive warnings treating all versioned phase directories as malformed names.

**After this fix:** `validate health` on the QGSD repo produces:
- W005: 0 (was 41) — versioned dirs no longer flagged as bad naming
- W007: 0 (was 22) — versioned phases now matched between ROADMAP and disk
- W002: 0 for versioned phase references (was 1) — STATE.md versioned refs correctly resolved
- Total warnings: 25 (was 64), all genuine issues

## Regex Sites Fixed

| Line | Check | Old Pattern | New Pattern |
|------|-------|-------------|-------------|
| 3833 | W005 dir-naming | `/^\d{2}(?:\.\d+)?-[\w-]+$/` | `/^(v\d+\.\d+-\d{2}(?:\.\d+)?-[.\w-]+\|\d{2}(?:\.\d+)?-[\w-]+)$/` |
| 3863 | W007 ROADMAP (health) | `#{2,4}\s*Phase\s+(\d+...)\s*:` | `(?:#{2,4}\|[-*]\s+\[[^\]]\]\s+\*{1,2})\s*Phase\s+(v\d+...\|d+...)\s*:` |
| 3874 | W007 disk (health) | `/^(\d+(?:\.\d+)?)/` | `/^(v\d+\.\d+-\d{2}(?:\.\d+)?\|\d+(?:\.\d+)?)/` |
| 3781 | W002 STATE extractor | `[Pp]hase\s+(\d+(?:\.\d+)?)` | `[Pp]hase\s+(v\d+...\|\d+...)` |
| 3788 | W002 disk extractor | `/^(\d+(?:\.\d+)?)/` | `/^(v\d+\.\d+-\d{2}(?:\.\d+)?\|\d+(?:\.\d+)?)/` |
| 3613 | consistency ROADMAP | `#{2,4}\s*Phase\s+(\d+...)\s*:` | `(?:#{2,4}\|[-*]\s+\[[^\]]\]\s+\*{1,2})\s*Phase\s+(v\d+...\|d+...)\s*:` |
| 3625 | consistency disk | `/^(\d+(?:\.\d+)?)/` | `/^(v\d+\.\d+-\d{2}(?:\.\d+)?\|\d+(?:\.\d+)?)/` |

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Checkbox-format ROADMAP entries caused 2 new W007 regressions**
- **Found during:** Task 3 (smoke test)
- **Issue:** After fixing disk dir extractor, `v0.8-01` and `v0.11-01` dirs were extracted from disk but their ROADMAP entries use `- [x] **Phase v0.8-01:` format (not `### Phase v0.8-01:`), causing 2 new W007 warnings
- **Fix:** Extended phasePattern at sites 3613 and 3863 with alternation `(?:#{2,4}|[-*]\s+\[[^\]]\]\s+\*{1,2})` to capture checkbox format
- **Files modified:** `qgsd-core/bin/gsd-tools.cjs`
- **Commit:** fe310fb
- **Side effect:** The 22 legacy W007s for numeric phases 18-39 also resolved (they also use checkbox format in ROADMAP) — total W007 went from 22 to 0

**2. [Rule 1 - Test] HLTH-02 and HLTH-03 tests could not fail before fix**
- **Found during:** Task 1 (test design)
- **Issue:** The W007/W002 bugs are symmetric — current code extracts nothing from EITHER ROADMAP or disk for versioned phases, so both sets are empty and no mismatch is detected. Tests for these scenarios vacuously pass before the fix.
- **Fix:** Tests were written as regression guards (they verify no false positives are introduced by the fix) rather than classic RED tests. HLTH-01-TC-01 and HLTH-01-TC-02 correctly failed before the fix (W005 is clearly emitted). Total pre-fix failures: 2 (not 4 as expected in plan).
- **Impact:** No behavioral difference — all 4 tests provide meaningful regression coverage and all pass after the fix.

**3. [Rule 1 - Verification] Plan's smoke test filter had overly broad W002 detection**
- **Found during:** Task 3 verification
- **Issue:** `x.code==='W002'&&x.message.includes('v0.')` catches the existing "references phase 12" W002 because the disk phases list in the message now correctly includes `v0.10-01, v0.11-01, ...`
- **Fix:** Used more precise filter `x.message.match(/references phase v0\./)` in verification; documented in this summary

### Scope note

The plan specified 6 regex sites (Sites 1-6, with Site 6 having 2 substitutions = 7 total). The auto-fix added 2 more substitutions at the phasePattern sites (extending from `#{2,4}` to also match checkbox format). Total substitutions made: 9.
