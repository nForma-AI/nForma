## Critical Gap: Orchestrator Mode A Missing INT-04 Propagation

### Root Cause

The INT-04 fix (Phase v0.7-01) correctly updated `commands/qgsd/quorum.md` Mode B (line 450) to use `--slot + --model-id` for claude-mcp servers instead of `--model <derived-key>`. However, this fix was never propagated to `agents/qgsd-quorum-orchestrator.md` Mode A (lines 232-243).

**Current state in `agents/qgsd-quorum-orchestrator.md` (lines 233-243):**
```bash
node "$HOME/.claude/qgsd-bin/update-scoreboard.cjs" \
  --model <model_name> \     ← writes to data.models{}, not data.slots{}
  ...
```
> "`--model`: ... claude-mcp servers use the `model` field from `health_check` to derive the key (e.g., `deepseek-ai/DeepSeek-V3` → `deepseek`)"

**Target state (matching `quorum.md` Mode B line 450):**
```bash
# For claude-mcp servers:
node "$HOME/.claude/qgsd-bin/update-scoreboard.cjs" \
  --slot <slot_name> \       ← e.g., claude-1, claude-2
  --model-id <full_model_id> \ ← from health_check response
  --result <vote_code> \
  ...

# For native CLI agents (unchanged):
node "$HOME/.claude/qgsd-bin/update-scoreboard.cjs" \
  --model <model_name> \     ← claude, gemini, opencode, copilot, codex
  ...
```

### Impact

All quorum rounds executed through the orchestrator (the normal path for planning decisions under CLAUDE.md R3) write to `data.models{}` instead of `data.slots{}`. SCBD-01/02/03's slot-tracking invariants are unreachable for the primary execution path.

### Fix

Single-file update: `agents/qgsd-quorum-orchestrator.md` Mode A scoreboard block (lines 232-244) — add the `--slot`/`--model-id` instruction for claude-mcp servers, same pattern as `quorum.md` Mode B line 450.
