- `ARTIFACT_PATTERNS` (7 regex entries, lines 142–150)

- `DECISION_MARKER` constant (line 178)
- `hasDecisionMarker()` function (lines 182–198)
- GUARD 5 block in `main()` (lines 251–257)

`hooks/dist/qgsd-prompt.js` (55 lines) also missing Phase 4 step 5. Source is 56 lines.

**Impact (revised):** `npm publish` is NOT affected — `package.json` `prepublishOnly: "npm run build:hooks"` auto-rebuilds dist before any publish invocation. Gap affects local source installs: anyone running `node bin/install.js` directly from the cloned repo gets the Phase 2-era hooks without GUARD 5. `hooks/dist/` is gitignored, so there is no tracked copy — the build must be run locally.

**Fix:** Run `node scripts/build-hooks.js` from repo root.


**Verification:** After fix, `wc -l hooks/dist/qgsd-stop.js` should return 320, `wc -l hooks/dist/qgsd-prompt.js` should return 56. Run `grep -c "GUARD 5\|hasArtifactCommit\|hasDecisionMarker\|DECISION_MARKER" hooks/dist/qgsd-stop.js` should return > 0.

### HIGH — GAP-02: `buildQuorumInstructions()` missing step 5 (marker path disabled for all installer users)

**What:** `bin/install.js` function `buildQuorumInstructions()` (lines 194–213) generates the `quorum_instructions` string written to `~/.claude/qgsd.json` during install. This 4-step string does NOT include step 5 (`Include the token <!-- GSD_DECISION --> somewhere in your FINAL output...`).

**Impact (upgraded from MEDIUM to HIGH):** `DEFAULT_QUORUM_INSTRUCTIONS_FALLBACK` in `qgsd-prompt.js` includes step 5 — but it is only used when `qgsd.json` has no `quorum_instructions` key. Installer-generated `qgsd.json` always includes `quorum_instructions` (4-step), so the fallback is never reached. All users installing via `npx qgsd@latest` or `node bin/install.js` receive a config that permanently disables `hasDecisionMarker()`.

GUARD 5 logic: `isDecisionTurn = hasArtifactCommit(...) || hasDecisionMarker(...)`. For non-artifact decision turns (verify-work text results, discuss-phase filtered outputs — TC19 test case), both signals are false → GUARD 5 exits 0 → quorum NOT enforced. The entire marker-based detection path is silently dead for all installer users.

The artifact-commit path (`hasArtifactCommit()`) continues to work correctly for most GSD planning turns. This is not a complete quorum bypass — but one of the two GUARD 5 signals is fully inert for all users.

**Fix:** Add step 5 to the `buildQuorumInstructions()` return string in `bin/install.js`, matching `DEFAULT_QUORUM_INSTRUCTIONS_FALLBACK` step 5 exactly.

**Also update:** `templates/qgsd.json` `quorum_instructions` field for consistency.

---

### LOW — GAP-03: Log messages hardcode `~/.claude/qgsd.json` path

**What:** Both the first-install success message (line 1734) and the reinstall summary block (lines 1743, 1746) in `bin/install.js` hardcode `~/.claude/qgsd.json` regardless of install type or `targetDir`. Per Codex review, this affects both paths, not just reinstall.

**Impact:** Cosmetic only. Config path is determined by `qgsdConfigPath = path.join(targetDir, 'qgsd.json')` which is correct at runtime. The log message is misleading for non-global installs.

**Fix:** Replace hardcoded `~/.claude/qgsd.json` string in log messages with interpolated `qgsdConfigPath` or a tilde-shortened version.
